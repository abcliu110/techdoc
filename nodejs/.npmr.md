## npmr

当你运行这条命令时，配置会被写入到一个叫做 **`.npmrc`** 的文本文件中。

默认情况下（没有加 `-g` 参数），它位于你的 **用户文件夹** 中。

### 1. 具体文件的位置
在 Windows 系统中，这个文件的完整路径通常是：
> **`C:\Users\你的用户名\.npmrc`**

*   **文件名**：`.npmrc` (注意前面有个点)
*   **如何查看**：你可以直接用记事本或 VS Code 打开它。
*   **文件内容**：打开后你会看到里面多了一行：
    ```ini
    registry=https://registry.npmmirror.com/
    ```

### 2. 快速找到它的方法
你不需要手动去翻文件夹，可以在终端（CMD 或 PowerShell）里输入以下命令，直接显示配置文件的具体路径：

```powershell
npm config get userconfig
```
系统会直接打印出那个 `.npmrc` 文件的路径。

### 3. 配置的优先级（补充知识）
`npm` 读取配置有三个级别，如果你发现设置了没生效，可能是被覆盖了：

1.  **项目级（最高优先级）**：
    *   在具体的代码项目文件夹里的 `.npmrc` 文件。
    *   *作用*：很多开源项目会在项目根目录放一个 `.npmrc`，强制该项目使用特定的源或依赖版本，这会覆盖你电脑上的全局设置。
2.  **用户级（你刚才设置的）**：
    *   位置：`C:\Users\用户名\.npmrc`
    *   *作用*：影响你当前用户下的所有项目（除非被项目级覆盖）。
3.  **全局级（Global）**：
    *   位置：通常在 Node.js 的安装目录下。
    *   *作用*：所有用户共享的设置（一般很少改这个）。

**总结**：你刚才敲的命令，修改的是 **第2层（用户级）** 的 `.npmrc` 文件。这样设置的好处是，以后你用 NVM 切换 Node.js 版本时，这个镜像源配置**依然有效**，不需要重新设置。
