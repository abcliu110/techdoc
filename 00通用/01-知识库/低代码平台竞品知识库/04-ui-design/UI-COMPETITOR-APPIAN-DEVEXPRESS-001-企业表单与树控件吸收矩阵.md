# Appian 与 DevExpress：企业表单和复杂树控件吸收矩阵

## 定位差异

| 维度 | Appian Designer | DevExpress | 自研设计器吸收原则 |
|---|---|---|---|
| 产品层级 | 业务对象、界面、流程、记录与安全协同平台 | 跨技术栈企业 UI 组件套件 | 业务元模型采用 Appian 类思路，运行时组件协议吸收 DevExpress 成熟度 |
| 表单定义 | SAIL 声明式界面、表达式、规则输入、保存目标 | 数据对象 + items + editorOptions | Schema 同时描述业务绑定、交互规则、布局和组件专有配置 |
| 布局 | Form/Section/Columns/Side-by-Side/Card 等语义容器 | Group/Tabbed/Simple/Empty/Button + 约束网格 | 语义容器之下使用响应式约束网格，不采用自由像素定位 |
| 状态规则 | 用户、记录、流程、数据条件驱动显示/只读/验证 | editing/selection/focus/validation 等组件状态 | 业务上下文状态与组件瞬时状态分层 |
| 层级浏览 | 树式和逐级分栏式浏览，强调业务选择任务 | TreeList，强调高密度列、编辑、查询和大数据 | 同一层级数据源提供 Tree Browser 和 Tree Grid 两种控件 |
| 数据规模 | 通过记录查询与规则控制数据集合 | 远程操作、虚拟滚动、分页、按需加载 | Schema 明示 local/remote 能力，服务端查询协议统一 |
| 视觉方向 | 任务引导、语义分组、响应式阅读顺序 | 高密度、状态明确、数据操作完整 | 基础资料表单偏 Appian，复杂单据分录偏 DevExpress |

## 树控件必须拆分的状态

| 状态 | 说明 | 设计器属性 | 运行时要求 |
|---|---|---|---|
| `focusedKey` | 键盘导航或当前焦点 | 是否启用焦点、默认焦点 | 不等同于业务选择 |
| `selectedKeys` | 行或节点选择集 | 单选/多选、范围、跨页 | 支持受控与非受控模式 |
| `checkedKeys` | 复选业务选择集 | 父子联动、半选、递归范围 | 需要明确远程未加载节点语义 |
| `expandedKeys` | 已展开节点 | 默认展开层级、记忆方式 | 按需加载失败可重试 |
| `currentValue` | 表单字段最终值 | 值类型、值/标签映射 | 与导航焦点、勾选集合解耦 |
| `editingKey` | 正在编辑的节点/行 | 单元格/行/弹窗/表单模式 | 提交、取消、并发冲突反馈 |
| `loadingKeys` | 正在加载子节点 | 加载策略、超时、重试 | 节点级骨架和错误状态 |
| `dragState` | 拖拽源、目标和位置 | 同级/跨级/外部拖拽权限 | 防循环、类型、权限和并发校验 |

## 合法结构建议

| 父容器 | 允许子节点 | 禁止或需包装 |
|---|---|---|
| Form | Section、Columns、Tabs、字段、命令区、验证摘要 | 裸 TreeList 需明确高度或放入 Section |
| Section | Columns、字段、Card、数据组件 | 页面级发布/流程配置不是子控件 |
| Columns | Column | 字段不能绕过 Column 直接成为 Columns 子节点 |
| Column | 字段、Section、Card、树、表格 | 超深嵌套需深度告警 |
| Editable Grid/Tree Grid | Column、行命令、编辑表单模板 | 普通页面容器不能直接作为列 |
| Tree Browser | 节点模板、搜索区、空态、命令区 | 不承载二维列汇总 |
| Popup/Drawer Editor | Form 或 Section | 不能直接绑定未定义保存目标的散乱字段 |

## 企业设计器实现优先级

1. 先实现类型化 Schema、业务上下文预览、语义容器和响应式约束网格。
2. 再实现普通 Grid 与 Tree Grid 的共享列协议，以及各自的数据语义。
3. 树控件第一版必须具备稳定键、父子适配、懒加载、搜索、焦点、选择、展开、半选和拖拽合法性校验。
4. 第二阶段补远程跨页选择、视图状态持久化、批量编辑、汇总、导出和键盘无障碍。
5. 设计器必须用大纲、面包屑、合法落点和失败原因处理复杂嵌套，不能只依赖画布命中。

## 证据等级

- Appian：官方公开资料确认，未登录产品实测。
- DevExpress：官方稳定能力模型确认，本轮在线文档/演示因网络超时未完成浏览器实测。
