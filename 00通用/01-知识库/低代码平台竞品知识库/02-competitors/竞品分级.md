# 竞品分级

采集日期：2026-07-06

## 分级原则

```text
S 级：直接影响自研平台核心模型，需源码级阅读 + 可见 UI + 后续实测。
A 级：重要参考对象，需公开资料深挖或关键源码/机制拆解。
B 级：方向级参考，先做能力矩阵和 License/商业边界。
C 级：趋势观察，暂不进入深拆。
```

## 不能升级的条件

- 未完成源码级证据的开源项目，不能升 S。
- 未完成本地运行或试用验证的对象，不能把能力标为“已验证”。
- 金蝶等闭源平台没有公开源码，内部机制不得标为“直接事实”。
