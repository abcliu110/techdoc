# 研究范围

采集日期：2026-07-06

## 本轮研究目标

按《01A-工作流-知识库建设与持续完善》执行存量知识库增强。本轮不推倒重建，重点补齐以下缺口：

1. 固定知识库结构：补 `00-domain/`、`02-competitors/`、`gaps/` 入口。
2. 金蝶公开资料深挖：从单一产品页扩展到开发服务云、流程服务云、集成服务云、KDDM 深度资料。
3. GitHub 开源项目深化：在 Frappe/NocoBase 之外，补 Appsmith 与 Directus 的文档、源码或安全公告级机制证据。
4. 更新能力矩阵、成熟度声明、缺口清单和下一轮补证计划。

## 本轮重点专题

```text
专题：低代码平台的业务元模型、流程/自动化、权限与扩展机制
金蝶侧：KDDM、流程服务云、集成服务云、企业基础架构能力
GitHub 侧：Appsmith widgets/datasources/queries/JS binding，Directus fields/permissions/policies/items service
```

## 证据强度边界

- 金蝶公开页面、官网文章、公开报道：可作为“直接事实”证明其公开定位和能力表述；对内部实现只能形成“高可信推断”或“假设”。
- GitHub 源码页面、安全公告、官方文档：可作为“源码初证”或“直接事实”；没有本地运行验证前不升级为“已验证”。
- 社区讨论、第三方文章：只作为侧面证据，不单独支撑高风险 ADR。

## 本轮不做

- 不启动 Docker 实测；此前缺口清单已记录 Docker daemon 阻塞。
- 不把 ADR 从 `proposed` 升级为 `accepted`。
- 不把 Appsmith/Directus 提升为 S 级；本轮只完成初始源码/机制补证。
