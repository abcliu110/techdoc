# 术语表

采集日期：2026-07-06

| 术语 | 含义 | 本知识库使用边界 |
|---|---|---|
| 业务元模型 | 用元数据描述业务对象、字段、关系、规则、流程、权限和页面的模型体系 | 不等同于数据库表结构 |
| KDDM | 金蝶动态领域模型 | 闭源竞品公开概念，只能基于公开资料分析 |
| UI Builder | 通过组件、画布、属性面板构建界面的能力 | 不能替代业务建模能力 |
| Datasource | 外部数据库、API 或服务连接 | Appsmith/ToolJet/Directus 等均有不同抽象 |
| Query | 对数据源的读写动作或 API 调用 | 在内部工具平台中常与 UI 组件绑定 |
| Binding | 将查询、JS 或状态表达式绑定到组件属性 | Appsmith 典型形式是 mustache 表达式 |
| Collection | 数据集合或业务集合 | NocoBase/Directus/NocoDB 中语义不同，需按项目证据解释 |
| DocType | Frappe 的元数据对象 | 同时影响数据、表单、列表、权限和工作流 |
| Policy | 访问策略 | Directus 11 中与角色、权限、用户关联 |
| Accountability | Directus 服务执行时的身份/权限上下文 | 省略或错误传递会改变权限语义 |
| Flow | 自动化流程 | Directus/NocoBase/金蝶等均有流程能力，但机制不同 |
| 源码初证 | 已阅读公开源码或源码页面，但尚未本地运行验证 | 不能直接支撑 accepted ADR |
