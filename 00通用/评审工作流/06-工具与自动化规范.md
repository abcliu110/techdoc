# 评审工具与自动化规范

> 版本：v2.1
> 日期：2026-07-12
> 定位：工具配置标准、质量门禁规则、集成规范
> 适用范围：评审工具引入、CI 门禁配置、工具链管理

任何工具成为 CI 门禁前，必须先在目标项目完成安装、配置、基线校准和失败处理验证。

## 1. 工具状态分级

| 状态 | 含义 | 是否可作为门禁 |
|---|---|---|
| 已验证可用 | 当前项目已配置并能稳定运行 | 可以 |
| 模板示例 | 可参考，但需要按项目适配 | 不可以直接作为门禁 |
| 概念工具 | 表达能力方向，尚未验证或需自研 | 不可以作为门禁 |

## 2. 质量门禁规则

### SonarQube 门禁阈值

- 覆盖率 ≥80%，重复率 ≤3%
- Maintainability Rating = A
- Reliability Rating = A
- Security Rating = A
- Quality Gate 超时 300s

### JaCoCo 覆盖率规则

| 代码层级 | 覆盖率要求 |
|---|---|
| L1/L2 关键类（SqlAssembler、AccessDecider、TenantDataService 等） | 100% 行覆盖 |
| 整体 Bundle | ≥80% 行覆盖 |

### SpotBugs 重点缺陷模式

必须检查以下模式：
- `SQL_INJECTION_SPRING_JDBC`、`SQL_INJECTION_JDBC`
- `XSS_REQUEST_PARAMETER_TO_JSP_WRITER`
- `PATH_TRAVERSAL_IN`

### Checkstyle 规则

- 文件编码必须 UTF-8
- 圈复杂度（CyclomaticComplexity）max=10
- 方法长度 max=50 行
- 公共接口必须有 Javadoc

## 3. 自定义安全扫描规则

### Semgrep 必检规则

```yaml
# SQL注入检测
- id: sql-injection-risk
  pattern: String $SQL = "... " + $USER_INPUT + "..."
  message: SQL injection risk. Use parameterized queries. Ref: Rule 17-001, Trap H1
  severity: ERROR

# 租户隔离缺失检测
- id: tenant-isolation-missing
  pattern: $MAPPER.selectList($WRAPPER)
  message: Missing tenant_id check. Ref: Rule 17-010, Trap B1
  severity: ERROR

# 密码明文检测
- id: hardcoded-password
  pattern: String $PASSWORD = "..."
  message: Hardcoded password detected. Ref: Rule 17-050
  severity: ERROR

# 事务中的外部调用
- id: external-call-in-transaction
  pattern: @Transactional ... $HTTP_CLIENT.$CALL(...)
  message: External call inside transaction. Use outbox pattern.
  severity: WARNING
```

## 4. 架构约束规则（ArchUnit）

必须强制执行的规则：

```java
// 禁止循环依赖
@ArchTest
public static final ArchRule modules_should_not_have_cycles =
    slices().matching("com.lowcode.(*)..").should().beFreeOfCycles();

// Service 不依赖 Controller
@ArchTest
public static final ArchRule service_should_not_depend_on_controller =
    noClasses().that().resideInPackage("..service..")
        .should().dependOnClassesThat().resideInPackage("..controller..");

// 只有 SqlAssembler 可以拼接 SQL（Rule 17-002）
@ArchTest
public static final ArchRule only_sql_assembler_can_concat_sql =
    noClasses().that().areNotAssignableTo(SqlAssembler.class)
        .should().callMethod(String.class, "concat", String.class)
        .orShould().callMethod(StringBuilder.class, "append", String.class)
        .where(methodNameContainsSqlKeyword())
        .because("SQL must be assembled via SqlAssembler (Rule 17-002)");

// 只有 TenantAware 可以访问 Mapper（Rule 17-010）
@ArchTest
public static final ArchRule mapper_only_accessed_by_tenant_aware =
    classes().that().resideInPackage("..mapper..")
        .should().onlyBeAccessed().byClassesThat()
        .implement(TenantAware.class)
        .orShould().beAnnotatedWith(BypassTenantCheck.class)
        .because("All mapper calls must check tenant_id (Rule 17-010)");
```

## 5. CI 门禁职责

| 门禁类型 | 工具 | 阻断级别 |
|---|---|---|
| 代码规范 | Checkstyle、Spotless | ERROR 阻断 |
| 静态分析 | SonarQube Quality Gate | 门禁失败阻断 |
| 安全扫描 | SpotBugs + FindSecBugs、OWASP Dependency-Check、Semgrep | HIGH+ 阻断 |
| 架构约束 | ArchUnit | 失败阻断 |
| 测试覆盖 | JaCoCo | L1/L2 覆盖率不达标阻断 |
| 漏洞扫描 | Snyk | HIGH+ 阻断 |

禁止将 `概念工具` 或 `模板示例` 状态下的工具直接作为 CI 阻断门禁。

## 6. code-review-graph 使用规范

code-review-graph 属于概念工具。使用前提：
1. 当前项目已安装并完成索引构建
2. 已在小范围变更中验证输出可信

可用命令：
```bash
code-review-graph status
code-review-graph detect-changes --repo <path> --brief
```

详见 CLAUDE.md 中的工具使用说明。
