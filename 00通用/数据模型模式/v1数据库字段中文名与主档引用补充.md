# v1 数据库字段中文名与主档引用补充

## 1. 结论

按“世界级数据库建模”的口径，当前 v1 文档已经有领域逻辑模型，但多数只到实体、关系和不变量层，还没有完全达到数据库实施级建模。缺口主要是：

- 字段没有统一写出“英文字段名 + 中文字段名 + 含义”。
- 子表和业务单据引用主档的方式没有逐表说明。
- 主档被引用时，哪些字段只保存主键、哪些字段需要历史快照，没有统一边界。
- 唯一键、幂等键、留量字段和状态机约束有描述，但缺少可落库字段字典。

本补充文档作为所有 `*-系统化拆解-v1.md` 的数据库字段字典补丁。以后单篇 v1 若继续深化，应把本文件中的表级字段字典内嵌回对应专篇。

## 2. 建模达标标准

每个 v1 专篇要达到数据库建模级别，至少满足以下标准：

| 检查项 | 必须内容 |
|---|---|
| 表级建模 | 明确主档表、子表、流水表、快照表、关系表 |
| 字段中文名 | 每个字段必须有英文列名和中文字段名 |
| 主档关系 | 明确谁是主档，子表通过哪个字段引用主档 |
| 主档引用方式 | 执行表引用主档 ID；对外展示、法务、财务、订单等历史事实另存快照 |
| 物理约束 | 主键、唯一键、幂等键、状态机字段、留量字段必须落到字段 |
| 历史可追溯 | 规则、价格、税率、组织、供应商、会员、权限等会变化的主档必须有版本或快照 |

## 3. 全局字段规范

| 字段名 | 中文字段名 | 说明 | 主档关系 |
|---|---|---|---|
| id | 主键ID | 表内技术主键 | 被本表子表以 `{domain}_id` 引用 |
| tenant_id | 租户ID | SaaS 隔离字段，业务表必带 | 引用 `sys_tenant.id` |
| created_at | 创建时间 | 记录首次创建时间 | 无 |
| created_by | 创建人ID | 创建操作人 | 引用用户/操作员主档 |
| updated_at | 更新时间 | 最近更新时间 | 无 |
| updated_by | 更新人ID | 最近更新操作人 | 引用用户/操作员主档 |
| deleted_at | 删除时间 | 软删时间，非所有表必须有 | 无 |
| status | 状态 | 业务状态机字段 | 状态码必须由枚举/字典定义 |
| version | 版本号 | 乐观锁或业务版本 | 规则/价格/配置主档被执行表快照引用 |
| idempotent_key | 幂等键 | 防重复请求/重复消费 | 业务来源单号或外部请求号派生 |
| biz_type | 业务类型 | 多来源业务分类 | 由业务字典定义 |
| biz_id | 业务ID | 来源业务主键 | 引用对应业务主档 |
| biz_order_no | 业务单号 | 来源业务单号 | 引用对应业务单据，跨域更稳定 |
| source_type | 来源类型 | 区分人工、系统、订单、工单等来源 | 由来源域定义 |
| source_id | 来源ID | 来源记录主键 | 引用来源主档或单据 |
| snapshot_json | 快照JSON | 固化当时主档显示/计算口径 | 不反向更新主档 |

## 4. 主档引用原则

1. **主档内部用 ID 引用**：子表、关系表、流水表优先引用主档 `id`。
2. **跨域优先用稳定业务编码**：订单号、支付单号、运单号、凭证号等跨服务引用优先保存业务单号，必要时同时保存内部 ID。
3. **历史事实必须快照**：订单、发票、账务、审计、审批、履约等已经出具或流转的单据，必须保存当时的商品、价格、税率、供应商、会员、组织、地址等快照。
4. **快照不能当主档改回去**：快照是历史证据，不是主档最新值。
5. **主档变更不改历史单据**：主档名称、税号、组织路径、供应商账户、会员等级变化后，历史单据仍按当时快照解释。

## 5. 各 v1 专篇字段与主档关系

### 5.1 商品模型

主档：`prd_spu`、`prd_sku`、`prd_category`、`prd_attr`。

| 表 | 字段名 | 中文字段名 | 主档引用方式 |
|---|---|---|---|
| prd_spu | spu_id | 标准产品ID | 主档主键，被 SKU、版本、前台类目引用 |
| prd_spu | spu_code | 标准产品编码 | 对外稳定编码，唯一 |
| prd_spu | title | 商品标题 | 主档当前展示名；订单保存快照 |
| prd_spu | category_id | 后台类目ID | 引用 `prd_category.category_id` |
| prd_spu | brand_id | 品牌ID | 引用品牌主档 |
| prd_spu | shop_id | 店铺ID | 引用店铺/商户主档 |
| prd_spu | current_version | 当前版本号 | 引用 `prd_spu_version.version_no` |
| prd_sku | sku_id | SKU ID | 主档主键，被订单、库存、价格引用 |
| prd_sku | sku_code | SKU 编码 | 对外稳定编码，唯一 |
| prd_sku | spu_id | 标准产品ID | 引用 `prd_spu.spu_id` |
| prd_sku | spec_json | 规格JSON | 保存规格组合 |
| prd_sku | spec_fingerprint | 规格指纹 | 同 SPU 下防重复 SKU |
| prd_category | category_id | 后台类目ID | 类目主档主键 |
| prd_category | parent_id | 上级类目ID | 自引用 `prd_category.category_id` |
| prd_attr | attr_id | 属性ID | 属性主档主键 |
| prd_attr_value | spu_id | 标准产品ID | 引用 `prd_spu.spu_id` |
| prd_attr_value | sku_id | SKU ID | 可选引用 `prd_sku.sku_id` |
| prd_attr_value | attr_id | 属性ID | 引用 `prd_attr.attr_id` |

主档如何被引用：订单行、库存项、价格规则引用 `sku_id`；订单、发票、履约等历史事实必须保存商品标题、规格、单位、税收分类等快照，不能只依赖商品主档当前值。

## 6. 覆盖矩阵

| v1 专篇 | 当前结论 | 补充方式 |
|---|---|---|
| 商品、订单、库存、资金账务、支付、价格、营销、会员、客户会员 | 有逻辑模型，缺字段字典 | 本文件补字段中文名和主档引用 |
| 采购供应商、采购与供应链 | 采购供应商有逻辑模型；采购与供应链 v1 原缺逻辑模型 | 已补采购与供应链逻辑模型；本文件补字段引用 |
