# 报表指标模型设计专篇

> 本篇聚焦指标、维度、事实表、口径、数据血缘和钻取。报表指标模型决定企业能否把业务事实转化为管理决策。

## 0. 一句话总纲

> 报表指标模型的核心，是让每一个数字都能说清楚：怎么算、从哪来、按什么口径、能追溯到哪些业务事实。

## 1. 指标和维度分离

指标回答“多少”，维度回答“按什么看”。

示例：

```text
指标：销售额、实收额、退款额、毛利、库存周转、客单价
维度：时间、门店、商品、类目、会员、渠道、支付方式、活动
```

指标必须有公式，维度必须有来源。

## 2. 事实表和维度表

报表不应直接从大量交易表临时拼接。标准做法：

- 交易事实表。
- 支付事实表。
- 退款事实表。
- 库存流水事实表。
- 会员权益事实表。
- 商品、门店、会员、渠道等维度表。

## 3. 指标口径

每个指标要定义：

- 指标名称。
- 业务含义。
- 计算公式。
- 数据来源。
- 时间口径。
- 过滤条件。
- 是否含税。
- 是否扣退款。
- 是否含取消单。
- 更新频率。

## 4. 数据血缘和钻取

报表数字必须能钻取：

```text
总销售额 -> 门店销售额 -> 商品销售额 -> 订单明细 -> 原始订单
```

如果报表不能追溯到原始事实，出现争议时无法解释。

## 5. 实时和离线分层

不同报表需要不同实时性：

- 实时看板：分钟级，允许短暂误差。
- 经营日报：日结后准确。
- 财务报表：审核后冻结。
- 审计报表：可追溯、不可随意重算覆盖。

## 6. 评审清单

```text
1. 指标是否有明确公式和业务含义？
2. 维度是否有稳定来源？
3. 报表是否基于事实表，而不是临时多表拼接？
4. 指标是否说明时间口径和过滤条件？
5. 汇总数字是否能钻取到原始单据？
6. 实时、日结、财务、审计报表是否分层？
7. 指标变更是否版本化？
```

## 7. 总结

报表指标建模五条纪律：

1. **指标有公式**：没有公式的数字不是指标。
2. **维度有来源**：维度必须在业务发生时可追溯。
3. **事实表承载分析**：不要让业务表直接承担所有报表。
4. **口径要版本化**：指标变化必须留痕。
5. **数字可钻取**：每个汇总都要能回到业务事实。
