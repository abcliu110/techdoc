# 单组件 SOP 模板说明

309 份文档由 `generate-component-sops.mjs` 从结构化 catalog 生成。每份必须包含以下八节：

1. 身份与认证状态。
2. 原型直接事实。
3. 生产风险基线。
4. 生产交互契约。
5. 组件专属验证。
6. 七道 Gate 执行卡。
7. 证据包与发布条件。
8. 未冻结实施决策。

每份顶部必须有 `component-sop-metadata` JSON 注释块，供零依赖校验器读取。

禁止：

- 把未来 React API、Owner 或最终风险伪装成已确认事实。
- 只替换组件名称而复用空泛测试文字。
- 把旧状态字符串变化作为唯一断言。
- 把统一异常按钮当成组件真实异常。
- 使用 TBD、TODO、FIXME 或“后续补充”。
