## 2. 原型直接事实

### 2.1 定义与模型

- 摘要：组织架构树把结构化数据、运行状态与用户操作组织为可连续理解和操作的界面。
- 模型：`OrgUnit → ChildUnit[]`
- 原型状态：`expandedIds, selectedUnit`
- 不变量：对象身份、层级或字段路径必须稳定；筛选、切换与编辑不能静默丢失用户状态。
- 适用边界：适用于树与层级中的复杂任务；简单展示或单字段输入应使用更轻量的基础组件。

### 2.2 业务上下文

- 角色：赵峰｜系统管理员
- 任务：使用组织架构树核验组织、产品、仓位或权限层级关系
- 对象：组织架构树；组织与资源层级 HC-INDUSTRY
- 规则：父子关系不得形成循环，移动节点后范围、路径和继承关系必须可解释
- 原型异常：组织架构树：目标父节点无权访问、尚未加载或移动后形成循环依赖
- 预期影响：组织架构树：更新层级路径并保留原位置用于撤销和审计

职责：

- 赵峰｜系统管理员：完成当前任务并核验结果。

事件/操作时间线：

- 载入并核对 组织架构树、组织与资源层级 HC-INDUSTRY 的身份、版本和当前状态；此动作属于组件业务前置，不得用统一“开始”按钮代替。
- 通过组件自身控件完成核心操作：组织架构树把结构化数据、运行状态与用户操作组织为可连续理解和操作的界面。
- 按“父子关系不得形成循环，移动节点后范围、路径和继承关系必须可解释”校验；校验通过后才产生“组织架构树：更新层级路径并保留原位置用于撤销和审计”。

## 3. 生产风险基线

- 暂定等级：**R2**。
- 判定依据：组件包含持久化、复杂草稿、异步数据、协作冲突或可补偿的业务状态影响。
- 类别升级条件：若树结构直接修改组织、菜单权限、资源继承或生产依赖，至少 R2；越权或跨租户影响时为 R3。
- 最高后果复核：Gate 1 必须检查权限、多租户、敏感数据、金额、库存、订单、支付、不可逆操作和跨系统一致性；命中任一 R3 条件即升级，禁止降级绕过门禁。
- 冻结规则：最终等级、理由和批准人写入契约卡与机器索引后，才能进入 Gate 2。

## 4. 生产交互契约

### 4.1 核心状态

类别状态模型：展开集、选中节点、焦点节点、加载状态、父子路径、继承与拖拽目标

本组件至少覆盖：

- 初始：对象身份、权限、版本和当前 expandedIds, selectedUnit 可解释。
- 编辑/操作中：每个用户动作产生可观察但未误报成功的状态。
- 校验/提交中：防重复触发，可取消的操作提供取消能力。
- 成功：结果关联原业务对象并说明下游影响。
- 异常：真实触发组件自身失败，不调用统一壳层伪造。
- 已恢复：撤销层级变更并恢复原展开路径与焦点
- 只读、禁用、无权限、数据过期和冲突：适用性在 Gate 2 逐项冻结，不适用项必须说明理由。

### 4.2 主路径

1. 载入并核对 组织架构树、组织与资源层级 HC-INDUSTRY 的身份、版本和当前状态；此动作属于组件业务前置，不得用统一“开始”按钮代替。
2. 通过组件自身控件完成核心操作：组织架构树把结构化数据、运行状态与用户操作组织为可连续理解和操作的界面。
3. 按“父子关系不得形成循环，移动节点后范围、路径和继承关系必须可解释”校验；校验通过后才产生“组织架构树：更新层级路径并保留原位置用于撤销和审计”。

### 4.3 异常与恢复路径

1. 使用可重复夹具真实触发：组织架构树：目标父节点无权访问、尚未加载或移动后形成循环依赖
2. 组件显示原因、影响对象和可执行恢复动作；不得清空未提交输入或把失败写成成功。
3. 执行恢复：撤销层级变更并恢复原展开路径与焦点
4. 恢复后验证焦点、对象版本、草稿和重复副作用；补偿语义为：撤销层级变更并恢复原展开路径与焦点

### 4.4 键盘与辅助技术

- 原型要求：Tab 进入树，方向键导航，Enter 展开或执行，Esc 取消上下文操作
- React 契约必须在 Gate 2 细化每个键、焦点入口、焦点返回和读屏播报文本。
- 主路径、异常路径和恢复路径均须只用键盘完成；远端状态更新不得抢夺用户焦点。

## 5. 组件专属验证

### 5.1 必须断言

- 主路径必须实际完成“使用组织架构树核验组织、产品、仓位或权限层级关系”，不能只改变状态文案或调用统一壳层动作。
- 不变量断言：对象身份、层级或字段路径必须稳定；筛选、切换与编辑不能静默丢失用户状态。
- 异常断言：真实触发“组织架构树：目标父节点无权访问、尚未加载或移动后形成循环依赖”，并验证业务动作被正确阻断。
- 恢复断言：执行“撤销层级变更并恢复原展开路径与焦点”后，输入、对象身份、焦点和可继续操作性符合契约。
- 结果断言：主体区域明确呈现“组织架构树：更新层级路径并保留原位置用于撤销和审计”，并能关联到 组织架构树、组织与资源层级 HC-INDUSTRY。
- 边界断言：适用于树与层级中的复杂任务；简单展示或单字段输入应使用更轻量的基础组件。
- 键盘断言：Tab 进入树，方向键导航，Enter 展开或执行，Esc 取消上下文操作

### 5.2 类别附加测试

- 验证 aria tree 模式、方向键、Home/End、展开和焦点语义
- 验证循环、无权父节点、懒加载失败与撤销
- 验证过滤、移动和重载前后的稳定节点身份与路径

### 5.3 失败模式

- 懒加载失败或目标父节点不可访问
- 移动后形成自身或间接循环
- 过滤、虚拟化或重载后焦点和路径丢失

### 5.4 性能基线

以 10,000 节点、10 层深度和 200 个展开节点为基准；展开、定位和键盘移动的可见反馈 p95 不高于 100ms。

Gate 2 必须记录设备档位、数据规模、运行次数、p95 口径和公共子路径 gzip 增量预算；未登记预算不得进入 PR Gate。

### 5.5 旧原型特征化

- 旧观察点：`#prototypeState`
- 旧变化指纹：`not:initial`
- 旧重置指纹：`initial`
- 使用规则：这些值只锁定旧原型行为。React 测试必须转为用户可见结果、公开事件和业务对象断言，不能继续把状态字符串变化作为唯一通过条件。

## 7. 证据包与发布条件

证据目录：`quality/evidence/03-org-tree/<component-version>/`

必须包含：

- `manifest.json`：组件键、契约版本、源码修订、产物版本、完整性和 SOP 版本。
- `test-results/`：RED 原始失败、GREEN、回归、浏览器和失败注入结果。
- `accessibility/`：自动扫描，以及主/异常/恢复路径的键盘和读屏人工记录。
- `visual/`：桌面、移动、相邻断点、默认/暗色/密度和关键状态矩阵。
- `performance/`：登记场景的原始测量与 p95 结果。
- `package/`：ESM、类型、CSS、SSR、子路径、Tree Shaking 和隔离安装结果。
- `approvals.json`：与最终风险等级匹配的独立批准。
- `rollback.md`：撤回、降级、消费方通知和数据/草稿兼容说明。

晋级条件：所有 Gate 通过、零绝对红线、候选产物与证据绑定；R2 还需代表性消费项目完整验收，R3 还需受控灰度和回滚演练。
