## 2. 原型直接事实

### 2.1 定义与模型

- 摘要：列拖拽排序表格把结构化数据、运行状态与用户操作组织为可连续理解和操作的界面。
- 模型：`Column[] → OrderedColumn[]`
- 原型状态：`dragKey, order`
- 不变量：对象身份、层级或字段路径必须稳定；筛选、切换与编辑不能静默丢失用户状态。
- 适用边界：适用于表格中的复杂任务；简单展示或单字段输入应使用更轻量的基础组件。

### 2.2 业务上下文

- 角色：王蕾｜仓库主管
- 任务：使用列拖拽排序表格处理订单、库存与应收明细并核对结果
- 对象：列拖拽排序表格；销售订单与库存批次 DATA-20260714
- 规则：批量、排序、过滤和编辑必须保持业务主键、版本与汇总口径一致
- 原型异常：列拖拽排序表格：数据加载失败、结果为空或记录版本冲突时禁止静默覆盖
- 预期影响：列拖拽排序表格：更新可追溯的数据视图并保留批次与版本审计

职责：

- 王蕾｜仓库主管：完成当前任务并核验结果。

事件/操作时间线：

- 载入并核对 列拖拽排序表格、销售订单与库存批次 DATA-20260714 的身份、版本和当前状态；此动作属于组件业务前置，不得用统一“开始”按钮代替。
- 通过组件自身控件完成核心操作：列拖拽排序表格把结构化数据、运行状态与用户操作组织为可连续理解和操作的界面。

## 3. 生产风险基线

- 暂定等级：**R2**。
- 判定依据：组件包含持久化、复杂草稿、异步数据、协作冲突或可补偿的业务状态影响。
- 类别升级条件：若提交金额、库存、订单、权限或不可逆批量操作，升级为 R3。
- 最高后果复核：Gate 1 必须检查权限、多租户、敏感数据、金额、库存、订单、支付、不可逆操作和跨系统一致性；命中任一 R3 条件即升级，禁止降级绕过门禁。
- 冻结规则：最终等级、理由和批准人写入契约卡与机器索引后，才能进入 Gate 2。

## 4. 生产交互契约

### 4.1 核心状态

类别状态模型：查询条件、加载窗口、行主键、选择集、编辑草稿、版本、排序与汇总

本组件至少覆盖：

- 初始：对象身份、权限、版本和当前 dragKey, order 可解释。
- 编辑/操作中：每个用户动作产生可观察但未误报成功的状态。
- 校验/提交中：防重复触发，可取消的操作提供取消能力。
- 成功：结果关联原业务对象并说明下游影响。
- 异常：真实触发组件自身失败，不调用统一壳层伪造。
- 已恢复：保留当前条件并重新加载最新数据版本
- 只读、禁用、无权限、数据过期和冲突：适用性在 Gate 2 逐项冻结，不适用项必须说明理由。

### 4.2 主路径

2. 通过组件自身控件完成核心操作：列拖拽排序表格把结构化数据、运行状态与用户操作组织为可连续理解和操作的界面。

- 原型要求：Tab 进入表格工具，方向键移动，Enter 确认，Esc 取消未提交操作

- 主路径、异常路径和恢复路径均须只用键盘完成；远端状态更新不得抢夺用户焦点。

- 边界断言：适用于表格中的复杂任务；简单展示或单字段输入应使用更轻量的基础组件。
- 键盘断言：Tab 进入表格工具，方向键移动，Enter 确认，Esc 取消未提交操作

### 5.2 类别附加测试

- 以稳定业务主键验证排序、过滤、分页和虚拟窗口后的行身份
- 验证批量选择、部分成功、重复提交和版本冲突
- 验证网格键盘导航、编辑模式、焦点保持和屏幕阅读语义

### 5.3 失败模式

- 加载失败、空结果或分页游标失效
- 排序过滤后选择集错位或行身份漂移
- 并发版本冲突、部分提交或批量操作重复执行

### 5.4 性能基线

普通表格以 1,000 行基准，虚拟/无限/服务端表格以 100,000 行逻辑数据和当前可视窗口基准；滚动、选择和编辑反馈 p95 不高于 100ms。

Gate 2 必须记录设备档位、数据规模、运行次数、p95 口径和公共子路径 gzip 增量预算；未登记预算不得进入 PR Gate。

- 旧重置指纹：`initial`
