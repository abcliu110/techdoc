# 方法论与流程目录说明

本目录用于存放跨项目、跨领域复用的方法论和工作流。所有文档由《00-总纲-知识库与产品交付双循环研发流程.md》统一组织为"双循环 + 横切质量体系"：

```text
循环 1：知识供给循环（知识库建设与持续完善）
循环 2：产品交付循环（V 模型：需求 → 设计 → 实现 → 验证 → 运营）
横切：产出质量保障工作流 + 统一设计决策工作流 + 追溯矩阵 + 变更控制

接口 1（供给）：知识库 → 最佳设计决策输入包 → 产品交付各阶段
接口 2（回流）：产品复盘 → 证据回写 → 知识库
```

## 阅读与执行顺序

```text
新加入者（人或 AI）：00 → 01（理论）→ 01A（怎么建知识库）→ 02（怎么做产品）
日常建知识库：直接执行 01A，遇不明处回查 01
日常做产品：直接执行 02，遇不明处回查 00
做架构与设计决策：02 的统一设计决策工作流 + 03/03A/04/04A 失败模式对照
进入工程执行：以 02 的 P0-P7 定位阶段，再在 P2-P5 内应用 05 的 11 个工程活动
大型任务由 AI 执行：先按 09 对齐目标、调查事实并形成受控切片，再把每个切片映射回 05 的工程活动和门禁
涉及 UI/交互：在 05 的对应活动中嵌入 06 专项门禁；06 可以加严，不得放宽 00/02/05 的门禁
检查复杂产品是否少功能、假实现或原型覆盖不足：执行 08；领域能力基线来自知识库，项目范围合同来自具体产品目录；再由 06 检查已实现 UI 的视觉质量
```

AI 执行者注意：本目录工作流文档的第一读者是 AI。执行任何工作流前必须先读《00-总纲》第 6 章"AI 自动执行契约"，遵守三级动作等级（自动执行 / 执行后送审 / 必须人工决策）、通用禁令和断点续跑规则。

## AI 阅读路由

AI 先读 00 第 6 章，再按任务只加载必要文档，避免把全部长文一次性塞入上下文：

| 任务意图 | 必读 | 按需读取 | 执行出口 |
|---|---|---|---|
| 建设/完善知识库 | 01A 的 0 节 | 01 对应主题、03/03A/04/04A | 最佳设计决策输入包 |
| 创建/迭代产品 | 02 的 0 节 | 01/03/04、05、06 | P0-P7 阶段产物 |
| 执行研发活动 | 05 的 0 节 | 02 当前阶段、03/04、06 | 阶段记录与验证证据 |
| AI 执行大型、复杂或高风险研发任务 | 09 的 0-3 节 | 05 当前活动；复杂产品叠加 08；UI 叠加 06 | 项目章程、切片合同、依赖 DAG、分层验收和证据包 |
| 评审系统风险 | 03/03A 的 0 节 | 04；平台类再读 04A | 领域失败模式清单 |
| 生成或审核 UI | 06 的 0/0.5 节 | 05 当前活动、批准的设计基线 | 证据包与 PASS/FAIL/EVIDENCE_INSUFFICIENT |
| 检查产品能力完整性、实现真实性或原型数量 | 08 的 0-2 节和第 20 节 | 领域能力基线、项目范围合同、05 当前活动、06 | 能力差集、原型覆盖图和能力门禁报告 |

## AI 可执行规范门禁

本目录每份规范必须满足下列检查，新增或修改后逐项复核：

| 检查项 | 判定标准 |
|---|---|
| 入口 | AI 能在文档前部找到用途、最小输入和启动位置 |
| 边界 | 写明适用/不适用、上位规则和冲突优先级 |
| 动作 | 能区分自动执行、执行后送审、必须人工决策 |
| 步骤 | 输入、动作、输出、完成判据可逐项执行 |
| 状态 | 使用总纲统一状态，支持合法迁移、断点和幂等重跑 |
| 证据 | 事实、推断、决策分离；引用可追溯，不用模型记忆冒充事实 |
| 失败 | 有阻断、反例、证据不足、升级和恢复条件 |
| 完成 | 有新鲜验证证据、残余风险和未覆盖项，不靠主观“看起来完成” |
| 可维护 | 版本唯一、链接有效、术语一致；长文采用渐进式披露 |
| 战略完整性 | 关键决策已检查反馈闭环、社会技术责任、可逆性、不变量、保证论证、证据保鲜和认知债务 |

任一项不满足时，文档状态只能是 `draft`，不得作为 AI 自动放行依据。

## 使用规则

1. 通用方法论放在这里。
2. 某个具体领域的证据和知识沉淀放入 `../01-知识库/`。
3. 某个具体产品的 PRD、架构和详细设计放入 `../02-产品方案/`。
4. 方法论更新后，应检查相关产品方案是否需要同步调整流程或模板。
5. 下游文档与《00-总纲》冲突时，先回到总纲确认职责边界，再修订下游文档。
6. 新增文档必须按"编号-类型-主题"命名，并登记到本 README 的文档表。
7. 新增或修改规范必须通过“AI 可执行规范门禁”；不得只增加原则性文字而没有执行协议。
