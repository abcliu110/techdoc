# SaaS / 私有化模式边界

## 目标

把 `E4`、`F12`、`28-113`、`28-114` 涉及的部署模式边界写清楚，避免把同一份代码仓库误认为已经同时具备完整 SaaS 与私有化商用能力。

## 当前真实边界

### SaaS 模式

- 允许作为主开发模式和默认验证模式。
- 当前更多能力停留在：
  - 正式实现
  - 内存 / 演示实现
  - 模块级最小内核
- 未接入正式 OpenAPI diff、统一 License 扫描和统一 SBOM 产物。

### 私有化模式

- 当前只补齐了文档、门禁和插件 / License 生命周期的最小设计内核。
- 私有化上线前仍需额外确认：
  - 离线授权
  - 断网运行
  - 备份恢复演练
  - 观测关闭或替代方案
  - 插件升级回滚演练

## 离线授权底线

1. 私有化环境必须支持离线授权。
2. 在线授权失败不能直接删除业务数据。
3. License 过期默认进入只读或受控降级，不得清空元数据、审计和业务记录。

## 数据与功能降级边界

- 可以限制：
  - 新建
  - 发布
  - 安装
  - 外部访问
  - 高级能力
- 不可以做：
  - 删除业务数据
  - 删除元数据快照
  - 删除审计日志
  - 静默关闭错误追踪

## 网络与外部依赖边界

- 私有化环境不得把运行时核心能力绑定到必须访问公网的服务。
- Redis 不可用时，按“只读降级 + 本地只读缓存 + DB 版本轮询”处理，不允许各模块自行决定放开写入。
- 插件、应用包和通知外发若需要访问外网，必须在交付侧做白名单审批。

## 当前仓库的结论

- README 和 runbook 已经把“正式能力 / 演示能力 / 未上线边界”区分开。
- 插件、License 和私有化验收仍以模块内核和文档门禁为主，不等于已完成生产交付。
- 若要声明“私有化可上线”，必须额外补：
  - 真实离线授权测试
  - 备份恢复演练记录
  - 指标上报关闭验收
  - 插件升级失败回滚演练
