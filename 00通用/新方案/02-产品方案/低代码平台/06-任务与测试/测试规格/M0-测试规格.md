# M0 测试规格

> 版本：v0.1
> 范围：T-001~T-005
> 目标：定义 M0 完成前必须通过的测试矩阵，防止只测单租户、单实例、主路径。
> 测试依据：PRD、07 平台级陷阱清单、`../../../../../工程规范/低代码平台规范/06-陷阱覆盖矩阵.md`、`../../../../../工程规范/通用规范/03-测试与门禁规范.md`。
> ADR 门禁：PRD 的产品不变量必须测试；来自 proposed ADR 的精确 DTO、字段、枚举和校验器仅在 ADR/等价决策记录 accepted 后进入阻断性测试。未通过门禁的候选项不得用于声明 M0 完成。

---

## 1. 测试分层

| 层级 | 工具 | 范围 |
|---|---|---|
| 单元测试 | JUnit 5 / Vitest | 纯逻辑、DTO、转换、校验 |
| 集成测试 | SpringBootTest + Testcontainers | MySQL、Redis、Flyway、Mapper、Schema Sync |
| 架构测试 | ArchUnit | 模块边界、分层、禁用依赖 |
| 契约测试 | JUnit + JSON fixture | DTO JSON、snapshot 兼容、API 错误结构 |
| 安全测试 | JUnit | SQL 白名单、tenant fail-fast、危险 DDL 阻断 |
| 契约资产 | JSON/CSV fixture | field_type 转换、表达式、权限矩阵、恶意输入集、元模型 JSON 并发、API 兼容 |

## 2. 环境要求

必须使用：

- Java 21
- MySQL Testcontainers 8.0.x
- Redis Testcontainers 7.x
- Node 20 + pnpm 9

禁止：

- 用 H2 替代 MySQL 测动态 DDL。
- 用 mock 替代 Redis 测版本收敛。

失败注入：

- 第 3 个 DDL step 抛错，断言前 2 个成功日志、第 3 个失败日志、返回 partial report。
- 发布中断于每个状态，断言 `lc_meta_publish_task` 保留当前状态、step、traceId、错误码和恢复建议。
- `resume(taskId)` 前必须运行 Reconciler；登记表与 information_schema 不一致时不得直接继续执行。

## 8.1 契约资产要求

M0 必须创建契约资产目录骨架：

```text
test-contracts/
  field-type-conversion/
  expression-eval/
  permission-matrix/
  malicious-inputs/
  metamodel-json-concurrency/
  api-compatibility/
```

M0 至少落地 `field-type-conversion` 中 22 种字段类型的 DDL/转换候选向量、`malicious-inputs` 中动态 SQL 字段名/op/sort 的最小恶意集，以及 `metamodel-json-concurrency` 中字段集合、引用集合、revision 冲突和 `lc_meta_ref` 重建一致性的最小向量。表达式、权限矩阵与 API 兼容资产可在 M1 补齐，但目录和读取工具接口必须在 T-001/T-003 预留。字段类型契约资产必须由 `FieldTypeHandler` 测试读取，元模型并发契约必须由 T-003 服务测试读取，不能只服务于文档验收。

## 9. 贯穿样例

M0 最小样例：`客户-订单-审批` 的元数据，不要求运行态 CRUD。

对应 PRD §6 的 M0 子集：本阶段只验收元数据保存、全图校验、DDL Plan、动态表创建、snapshot 与 MetaGraph；权限运行态、审批执行、前端页面和 XSS 属 M1/M2 后续验收。

对象：

```text
customer:
  name text required
  level select [normal, vip]

order:
  order_no autonumber
  customer link -> customer
  amount currency
  items table -> order_item
  state: draft/submitted/approved/rejected

order_item:
  product_name text
  qty decimal
  price currency
```

M0 验收：

- 能保存对象、字段、关系、状态、动作、规则草稿。
- 能全图校验。
- 能生成 DDL Plan。
- 能创建动态表。
- 能发布 snapshot 并加载 MetaGraph。
- 能保存 REQ-070~REQ-078 的补充 DTO，并在发布前阻断任何 M0 不支持的运行时执行。
