# T-206 版本发布与前端集成详细设计

> 版本：v0.1
> 里程碑：M2
> 适用任务：T-206
> 依据：PRD REQ-050~053、REQ-088、REQ-090；ADR-LOWCODE-PUBLISH-001
> 5A 状态：baseline-approved。`../04-架构决策/UI-and-Component-Decision-Matrix.md` 已人工接受；本设计作为 M2 UI 详细设计基线，禁止提前在 M0/M1 实现。

---

## 1. 目标

把 M0 发布管线扩展到前端页面、权限、规则、状态机和运行态 renderer，完成设计态到运行态的可审计发布闭环。

## 2. 发布快照内容

Version.snapshot 必须包含：

```text
objects
fields
relations
states
actions
rules
roles
permissions
pages
views
components
menus
i18n
fieldTypeCapabilities
expressionVersion
pageSchemaVersion
```

## 3. 发布校验

发布前必须执行：

```text
元模型全图校验
权限矩阵校验
状态机校验
规则/表达式校验
页面 schema 校验
菜单/i18n 校验
DDL Plan 校验
兼容性校验
```

## 4. 版本收敛

发布成功后：

1. 写 `lc_meta_version`。
2. 更新 Redis 版本键。
3. 健康实例 60 秒内收敛。
4. 请求入口钉住进入请求时的 metaHash。
5. Redis 失效时通过 DB polling 降级收敛。

## 5. 回滚边界

回滚只承诺：

```text
元数据快照回滚
页面 schema 回滚
权限/规则/状态机定义回滚
```

不承诺：

```text
已经执行 DDL 的物理回滚
已经写入新格式业务数据的自动逆变换
已经投递外部副作用的撤回
```

UI 必须把边界展示给发布者。

## 6. 前端集成

renderer 每次加载页面携带：

```text
metaHash
pageSchemaVersion
requestId
```

写接口如果返回 `META_VERSION_STALE`，前端必须：

```text
阻止继续提交
提示刷新
保留用户未提交输入
重新拉取 meta/page schema 后进行字段兼容检查
```

## 7. 验收

1. v1 发布后修改设计态字段，运行态仍使用 v1。
2. v2 发布后 renderer 加载 v2。
3. 发布中请求继续钉住旧 metaHash。
4. 双实例 60 秒内收敛。
5. 回滚 UI 明确提示不可物理回滚项。
6. 旧页面 schema 有回放测试。

### 8.3 UI 设计系统对照

发布界面使用 AntD Steps、Alert、Progress、Table 和 Modal，但必须通过平台发布状态组件封装；危险项、不可回滚项和人工确认项要以文本和结构化列表展示，不只使用颜色。

### 8.4 可访问性规则

| 规则 | 验收方式 |
|---|---|
| 发布步骤可感知 | 当前步骤、成功、失败、阻断原因通过文本和 aria-live 提示 |
| 危险确认可键盘完成 | Modal 初始焦点落在说明区域，确认按钮需显式键盘操作 |
| 回滚边界可读 | 不可物理回滚项以列表呈现，屏幕阅读器可逐项读取 |
| 冲突恢复不丢输入 | META_VERSION_STALE 后保留未提交输入并提示刷新 |

### 8.5 状态组合

| 权限状态 | 数据状态 | 交互状态 | 发布集成行为 |
|---|---|---|---|
| design read | published | idle | 可查看版本和发布日志，不可发布 |
| design publish | draft/dirty | submitting | 可发起发布，必须完成全图校验和 DDL Plan |
| design publish | failed/partial | conflict | 停在可恢复节点，提供 resume/abandon 前先跑 Reconciler |
| runtime user | stale/conflict | conflict | 阻止继续提交，保留输入，刷新 meta/page schema 后兼容检查 |

### 8.6 失败模式

| 失败模式 | 处置 |
|---|---|
| MetaGraph 与 PageSchema 版本不匹配 | 发布前阻断；运行态返回 META_VERSION_STALE |
| DDL 已执行但元数据未切换 | Reconciler 报告差异，停留可恢复状态 |
| 双实例版本未收敛 | 60 秒内通过 DB polling 收敛，否则告警并阻断新发布 |
| 回滚误导用户 | UI 明确列出元数据可回滚、物理 DDL/新格式数据/外部副作用不可自动回滚 |
| 旧 schema 无升级器 | 兼容等级不足时阻断发布或要求补升级器 |
