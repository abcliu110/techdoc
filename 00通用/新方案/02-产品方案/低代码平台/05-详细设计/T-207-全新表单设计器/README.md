## 1. 重新定位

T-207 不是普通表单拖拽器，也不是报表设计器。

它应重新定义为：

```text
面向企业 PC Web 单据、复杂表单、主数据、审批页和业务工作台的
IDE 级可见即可得页面设计器。
```

设计目标不是“看起来炫”，而是能让实施人员、业务顾问和开发人员共同设计真实可运行的复杂企业 Web 页面。

## 2. 核心参考产品

| 分层 | 主要参考 | 吸收重点 |
|---|---|---|
| 企业单据语义 | 金蝶动态表单 / 单据设计 / BOS 思路 | 业务对象、字段、分录、权限、流程、规则、发布 |
| 设计器工作台 | DevExpress Web Designer、Visual Studio Form Designer | 工具箱、设计画布、属性窗、大纲树、吸附线、错误列表 |

| 流程表单 | Pega、Appian、Creatio、Power Apps、Salesforce Dynamic Forms | 表单和流程/状态/可见性规则一体化 |
| Web 布局能力 | Webflow、Plasmic、Builder.io | Grid、Flex、容器、组件属性、设计系统约束 |
| 强表格能力 | AG Grid、DevExtreme、Kendo、Syncfusion、DevExpress VCL Grid | Master-Detail、嵌套表格、虚拟滚动、列配置、编辑体验 |

## 3. 必须超过金蝶的方向

1. 强 Web 布局能力：Grid、Flex、Dock、Split、Tabs、Collapse、Scroll、Sticky、Portal。
2. 强表格能力：母子嵌套、Detail Grid、冻结列、列组、虚拟滚动、批量编辑、键盘录入。
3. 设计态与运行态一致：同一套 RuntimeRenderer 渲染设计、预览、运行。
4. 元数据治理：保存 Schema，不保存自由 DOM。
5. 发布前质量门禁：布局、表格、规则、权限、性能、数据绑定都必须可校验。
6. 企业级视觉密度：灰白工具界面、信息密集、低装饰、清晰边界。

## 4. 禁止项

| 禁止项 | 原因 |
|---|---|
| 做成报表设计器 | 当前目标是 PC Web 表单/单据/页面设计器 |
| 做成彩色组件墙 | 不符合企业工具规范 |
| 主体依赖绝对定位 | 运行态难以响应式和维护 |
| 只支持简单两列表单 | 无法承载真实复杂单据 |
| 表格只是普通 HTML table | 无法支撑企业分录和嵌套明细 |
| 属性面板硬编码 | 无法扩展组件库和插件 |
| 规则散落在 JS 脚本里 | 无法做影响分析、权限治理和发布校验 |
