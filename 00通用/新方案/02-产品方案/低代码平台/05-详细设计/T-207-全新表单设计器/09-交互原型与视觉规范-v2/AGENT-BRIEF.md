# T-207 V2 单页原型快速实现契约

## 输入边界

- 每个 agent 只实现一个 `PT-XXX.html`，不得读取 V1、流程规范、其他 PT 或聊天历史。
- 任务提示会直接给出 PT 编号和中文场景名；不得扩大范围。
- 只允许修改被分配的单个 HTML 文件。

## 视觉基线

- 企业 PC Web 设计器，灰白 IDE 风格，克制、紧凑、工作导向。
- 画布尺寸按 `1440x900` 设计；`body` 不允许横向滚动。
- 字体：`Microsoft YaHei UI, Segoe UI, sans-serif`；正文 `12-13px`；标题 `14-17px`；`letter-spacing: 0`。
- 颜色：页面 `#eef2f6`，面板 `#ffffff`，正文 `#1f2937`，次要文字 `#667085`，边框 `#d7dee8`，主色 `#2468c9`，成功 `#19733f`，危险 `#c73535`。
- 区块圆角不超过 `4px`；禁止渐变、装饰球、超大标题、营销卡片、卡片嵌套。
- 顶部命令栏高度约 `48px`；左活动栏约 `48px`；资源面板 `220-260px`；右属性面板 `280-320px`；中间画布占剩余空间。
- 图标优先使用 Lucide；按钮选中态不得改变尺寸；短中文不得逐字换行。

## 页面要求

- 必须是完整、可直接打开的独立 HTML，内联 CSS/JS。
- 页面必须清楚表达本 PT 的唯一业务场景，不得做成说明文档或静态报表。
- 至少提供 12 个稳定 `data-testid`；根节点包含 `data-prototype="PT-XXX"` 和一个可变化的 `data-state`。
- 至少一个真实交互：点击后改变可见内容与 `data-state`，不能只弹 `alert`。
- 所有按钮可点击，禁用按钮必须有真实 `disabled` 状态；说明文字放 tooltip 或次级区域。

## 快速验收

只做以下四项，完成后立即退出：

1. 文件存在且页面非空。
2. `1440x900` 下 `document.body.scrollWidth <= document.body.clientWidth`。
3. 浏览器控制台无 error。
4. 执行一个主交互，并断言根节点或目标节点的 `data-state` 已变化。

不做 `1920x1080`、竞品研究、全量视觉报告或其他 PT。最终只返回三行：文件路径、截图路径、`PASS/FAIL + 交互断言`。
