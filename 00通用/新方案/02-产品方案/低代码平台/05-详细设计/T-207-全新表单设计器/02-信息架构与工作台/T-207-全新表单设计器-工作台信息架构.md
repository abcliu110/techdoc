## 1. 设计目标

工作台信息架构要回答一个问题：

```text
用户如何在一个 IDE 级 Web 工作台中，
从业务对象出发，设计、校验、预览并发布一个复杂 PC Web 单据页面。
```

T-207 工作台不是控件展示墙，也不是报表画布。它必须像企业级设计 IDE：

1. 左侧有资源和结构。
2. 中央是真实运行态 Web 页面画布。
3. 右侧是面向当前选中对象的 Inspector。
4. 底部是 Analyzer 和发布质量门禁。
5. 顶部是文件、版本、预览、校验、发布等全局命令。

```text
+----------------------+----------------------------------------+----------------+
```

### 2.1 五区职责

| 区域 | 职责 | 参考吸收 |

## 3. 信息架构原则

### 3.1 业务对象优先

用户不应从空白控件开始，而应从业务对象开始。

```text
选择业务对象
  -> 生成默认页面结构
  -> 拖入字段和分录
  -> 调整布局
  -> 配置规则权限
  -> 校验发布
```

### 3.2 选中对象驱动右侧 Inspector

右侧不是固定表单，而是根据当前选中对象切换：

| 选中对象 | Inspector 重点 |
|---|---|
| Page | 页面元信息、业务对象、断点、权限、发布 |
| LayoutContainer | display、columns、gap、padding、overflow、sticky |
| Section | 标题、折叠、权限、校验、子布局 |
| Field | 字段绑定、编辑器、校验、权限、联动 |
| EntryTable | 数据源、列、行操作、冻结、分页、明细 |
| TableColumn | 字段绑定、列宽、编辑器、格式、权限、规则 |
| ActionButton | 动作、状态机、权限、二次确认 |
| Rule | 触发条件、目标对象、动作、影响分析 |

### 3.3 画布不是 DOM 编辑器

画布显示真实页面，但用户编辑的是 Schema。

| 层 | 职责 |
|---|---|
| RuntimeRenderer | 按 Schema 渲染真实页面 |
| DesignerOverlay | 绘制选中框、拖放点、吸附线、尺寸标注 |
| SchemaStore | 保存 PageSchema / LayoutSchema / ComponentSchema |
| CommandStack | 撤销、重做、单节点命令；批量命令归 P1 |

### 3.4 Analyzer 是工作台一等区域

Analyzer 不只是状态栏。它是企业级发布门禁：

1. 点击问题能定位到画布对象。
2. 问题能映射到 Schema 路径。
3. 问题能区分错误、警告、建议。
4. 发布必须经过 Analyzer。

## 4. Top Command Bar

### 4.1 区域结构

```text
+--------------------------------------------------------------------------------+
| Logo / 页面名 | 保存 | 撤销 | 重做 | 校验 | 预览 | 发布 | 角色 | 断点 | 更多 |
+--------------------------------------------------------------------------------+
```

### 4.2 命令分组

| 分组 | 命令 | P0 |
|---|---|---|
| 文件 | 保存草稿、另存为模板 | 保存草稿 |
| 历史 | 撤销、重做、命令历史 | 撤销、重做 |
| 质量 | 校验、发布检查、定位问题 | 校验 |
| 运行 | 预览、角色预览、断点预览 | 预览、断点；角色预览为 P1 |
| 发布 | 发布、回滚、版本历史 | 发布、版本历史 |
| 视图 | 缩放、显示吸附线、显示大纲 | 缩放、吸附线 |

### 4.3 状态信息

Top Command Bar 右侧显示：

| 状态 | 示例 |
|---|---|
| Schema 状态 | draft / published / archived |
| 当前版本 | v12 draft |
| 保存状态 | 已保存 / 有未保存修改 |
| 当前断点 | 1440 |
| 当前角色 | 设计者 / 销售员 / 审批人 |

P0 中"当前角色"只显示设计者或运行态默认角色，角色切换入口可占位但不可用。按角色查看权限差异属于 P1 的角色预览能力。

Schema 状态在 PageSchema 中必须使用小写常量；工作台 UI 可翻译展示为"草稿 / 已发布 / 已归档"，但不得把显示文案写回 Schema。

### 4.4 禁止项

| 禁止 | 原因 |
|---|---|
| 顶部放大量业务说明文字 | 工具栏必须高效、克制 |
| 将复杂属性放到顶部 | 属性归右侧 Inspector |
| 发布按钮无校验 | 发布必须经过 Analyzer |

## 5. Left Activity Bar 与 Dock Panel

左侧不是单一资源树，而是 `Activity Bar + Dock Panel` 的组合。Activity Bar 负责切换工作模式，Dock Panel 根据模式展示组件资产、BO 字段资产、页面大纲、问题过滤、版本列表或数据源。任何原型不得把组件面板、字段树、大纲树混成一个无语义区分的列表。

### 5.1 顶层导航

P0 采用垂直图标 Activity Bar + 面板内容：

| Mode | 图标 | Dock Panel | 切换后显示内容 | data-testid |
|---|---|---|---|---|
| 组件 | Components | Component Palette | 搜索、布局容器、基础表单、展示与状态、操作命令、业务套件、表格明细、模板片段；只显示 UI 组件和模板 | `activity-item-components` |
| 字段 | Fields | BO Field Tree | 当前 BO、单据头字段、分录字段、系统字段、计算字段、关联字段；拖入画布后生成 FieldNode 或 TableColumn | `activity-item-fields` |
| 结构 | Outline | Page Outline Tree | PageRoot、Section、Grid、Tabs、EntryTable、Column、DetailPanel 层级；支持选中同步、展开折叠、结构移动和非法移动反馈 | `activity-item-outline` |
| 问题 | Issues | Issue Filter | Analyzer Error、Warning、Suggestion、布局、绑定、权限、性能问题；点击问题定位画布节点和 Inspector | `activity-item-issues` |
| 版本 | Versions | Version List | draft、published、archived、schemaHash、发布时间、回滚入口、新建草稿入口 | `activity-item-versions` |
| 数据 | Data | Data Source Panel | 当前 BO、entries、样例数据、字段绑定摘要；P0 多数据源入口禁用并提示 | `activity-item-data` |

### 5.2 Component Palette 组件资产面板

Component Palette 是左侧默认面板，也是设计动作的第一入口。它只展示可拖拽 UI 组件，不展示字段资产。它必须以组件分组和组件卡片形式呈现，不允许只用普通树行表达；默认桌面态采用紧凑 IDE 工具箱形态，Activity Bar 约 48-56px，组件 Dock Panel 约 304px，组件卡片区域采用三列网格，每行显示 3 个组件卡片，优先保证中文名称完整可读；说明文字必须进入 tooltip 或详情区，不得常驻挤占卡片。

#### 5.2.1 顶部效率区

| 区域 | 功能 | data-testid |
|---|---|---|
| 搜索 | 按组件名称、编码、标签搜索 | `palette-search` |

Component Palette 顶部效率区 P0 只保留搜索框，不提供收藏、最近、业务域筛选 chip。常用组件通过分组顺序和默认展开策略解决，避免左侧工具箱被筛选器占用。

#### 5.2.2 组件分组

| 分组 | P0 可见组件名称 | data-testid |
|---|---|---|
| 布局容器 | 网格布局、弹性布局、分组面板、页签容器、分栏容器、滚动区域、固定栏、折叠面板 | `palette-group-layout` |
| 基础表单 | 单行文本、多行文本、数值输入、金额输入、日期选择、日期时间、下拉选择、单选组、复选框、开关、基础资料选择、附件上传 | `palette-group-input` |
| 展示与状态 | 文本显示、只读字段、状态标签、提示说明、图片、分隔线、统计摘要、进度状态 | `palette-group-display` |
| 操作命令 | 普通按钮、主按钮、按钮组、更多菜单、工具栏、行操作、提交操作、返回操作 | `palette-group-action` |
| 业务套件 | 单据标题、单据头容器、汇总栏、单据操作栏、审批信息面板、附件面板、关联单据、右侧辅助面板 | `palette-group-bill` |
| 表格明细 | 分录表格、汇总行、行操作、明细面板、明细表格、分页栏、表格工具栏 | `palette-group-table` |
| 模板片段 | 销售订单头模板、分录区模板、审批布局模板、左右分栏单据模板、主从表单模板、常用查询区模板 | `palette-group-template` |

字段资产不属于 Component Palette 分组。单据头字段、分录字段、系统字段、计算字段和关联字段只能在 `activity-item-fields` 对应的 BO Field Tree 中展示。表格列不作为独立 Palette 组件；用户从字段入口把分录字段拖入 `EntryTable.columns` 后生成或更新 TableColumn。

#### 5.2.3 组件卡片

| 卡片区域 | 要求 |
|---|---|
| 图标 | 表示组件类型，支持布局、录入、展示、命令、单据、表格、模板 |
| 名称 | 显示中文业务可理解名称；不得把 GridLayout、ReferencePicker、EntryTable 等技术类型作为用户主名称 |
| 用途 | 不常驻占用卡片空间，hover/focus 时以 tooltip 显示，或在右侧详情区显示 |
| 标签 | 不作为卡片默认必显内容；需要时进入 tooltip、选中详情或 Inspector，技术类型只进入 Schema 元数据和测试合同 |
| 投放约束 | 显示允许投放目标或不可投放原因 |
| 抓手 | 卡片内必须有明确拖拽抓手 `drag-source-palette-item-{assetId}`；卡片点击负责选中/预览，抓手负责拖拽 |

组件卡片必须覆盖 normal、hover、selected、dragging、disabled、unavailable、no-permission 七种状态。

### 5.3 BO Field Tree 字段资产面板

BO Field Tree 是字段资产视图，不能替代 Component Palette。

```text
业务对象：销售订单
├─ 单据头 Header
│  ├─ 客户 customerId
│  ├─ 销售组织 orgId
│  ├─ 单据日期 billDate
│  └─ 币别 currency
├─ 分录 Entries
│  ├─ 物料 materialId
│  ├─ 数量 qty
│  ├─ 单价 price
│  └─ 金额 amount
└─ 状态/动作
   ├─ 草稿
   ├─ 提交
   └─ 审核
```

| 数字字段 | NumberInput |
| 金额字段 | MoneyInput |
| 日期字段 | DatePicker |
| 日期时间字段 | DateTimePicker |
| 枚举字段 | Select |
| 布尔字段 | Switch |
| 引用字段 | ReferencePicker |
| 子表/分录 | EntryTable |

#### 字段搜索

| 功能 | 说明 |
|------|------|
| 搜索框 | 支持按字段名、编码、类型搜索 |
| 分组树 | 按单据头字段、分录字段、系统字段、计算字段、关联字段分组展示 |

## 6. Center Design Surface

中央画布是 T-207 的核心，必须表现真实 PC Web 页面。

### 6.1 画布结构

```text
CanvasViewport
├─ CanvasToolbar：缩放 / 断点 / 标尺 / 吸附线
├─ RuntimeFrame
│  └─ RuntimeRenderer(PageSchema)
└─ DesignerOverlay
   ├─ SelectionBox
   ├─ ResizeHandle
   ├─ DropIndicator
   ├─ Snapline
   └─ SizeHint
```

### 6.2 RuntimeRenderer 要求

| 要求 | 说明 |
|---|---|
| 同源渲染 | 设计态、预览态、运行态使用同一套组件渲染 |
| 非自由 DOM | 用户编辑 Schema，不直接编辑 DOM |
| 真实布局 | 使用真实 Grid/Flex/Scroll/Sticky |
| 可测量 | Overlay 从真实 DOM 读取位置和尺寸 |

### 6.3 DesignerOverlay 要求

| Overlay | 用途 |
|---|---|
| SelectionBox | 显示当前选中对象 |
| ParentSelector | 快速选中父容器 |
| ResizeHandle | 调整字段 span、容器高度、表格列宽 |
| DropIndicator | 显示合法投放点 |
| Snapline | 对齐线、列线、边距线 |
| SizeHint | 显示 span、px、百分比 |
| GhostPreview | 拖放时的预览效果 |

### 6.4 画布模式

| 模式 | 说明 | P0 |
|---|---|---|
| 设计态 | 可选中、拖拽、编辑属性 | 是 |
| 预览态 | 隐藏设计辅助，查看运行效果 | 是 |
| 角色预览 | 按角色查看权限差异 | P1 |
| 断点预览 | 1280 / 1440 / 1600 / 1920 | 是 |
| Schema 调试 | 查看选中节点 Schema | P1 |

### 6.5 选中规则

| 用户操作 | 结果 |
|---|---|
| 单击字段 | 选中 FieldNode，右侧显示字段 Inspector |
| 单击 Section 标题 | 选中 Section |
| 单击表格单元格 | 默认选中 TableColumn |
| 单击表格外框 | 选中 EntryTable |
| 双击字段 | 进入快速编辑标签/默认值 |
| Esc | 向上选中父节点 |
| Ctrl+点击 | P1：多选（批量操作） |
| Shift+点击 | P1：范围选择 |

P0 只要求稳定支持单选、父级选择和当前节点编辑。多选、范围选择、跨节点批量编辑归 P1，避免第一期把设计器复杂度扩展到 Excel-like 编辑器。

### 6.6 拖放规则

| 拖入来源 | 合法目标 | 结果 |
|---|---|---|
| BO 字段 | GridLayout / Section / HeaderForm | 生成 FieldNode |
| BO 分录 | Section / Tabs / PageMain | 生成 EntryTable |
| LayoutContainer | Page / Section / Tabs.tabContent / SplitPane 子区 / ScrollContainer | 生成容器 |
| EntryFieldAsset | EntryTable.columns | 新建或更新 TableColumn |
| DetailPanel/DetailTable | EntryTable.detail | 添加一层受控明细；不得写入 `children` 或 `detailTables[]` |

#### 拖放交互细节

| 阶段 | 行为 |
|------|------|
| 拖动开始 | 显示 Ghost 预览元素，跟随鼠标 |
| 拖动中 | 合法目标高亮，非法目标灰显，显示投放占位符 |
| 拖动结束 | 投放占位符确认位置 |
| 投放成功 | 命令入栈，Schema 更新，画布刷新 |
| 投放失败 | Ghost 回退原位，显示错误提示（2秒） |

非法投放显示原因：

```text
不能将 EntryTable 投放到 TextInput 中。
DetailTable P0 只允许一层明细。
StickyBar 必须位于 PageRoot 或 ScrollContainer 下。
SplitPane 只能承载两个子区，不能直接追加第 3 个子节点。
```

P0 拖放必须通过稳定抓手和投放点完成，便于自动化测试平台模拟真实用户操作：

| 交互点 | 测试定位要求 |
|---|---|
| Activity 模式 | `data-testid="activity-item-{mode}"` |

| 拖拽抓手 | `data-testid="designer-handle-{nodeId}-drag"` |
| Grid 投放点 | `data-testid="designer-grid-cell-{gridId}-{row}-{column}"` |

### 7.7 校验、状态与规则 Tab 边界

P0 必须把字段级基础能力和完整业务规则分开：

| Tab | P0 职责 | P1 扩展 |
|---|---|---|
| 校验 | `FieldNode.validation`：required、长度、数值范围、格式、错误文案 | 远程校验、跨字段校验 |
| 状态 | 静态 `visible`、`required`、`readonly`、`disabled`、`hidden` | 由角色、流程、数据状态驱动的动态状态 |
| 规则 | 规则摘要、受控静态计算、后续规则文档入口、查看字段被规则影响的占位 | `RuleSchema`：visibleWhen、requiredWhen、readonlyWhen、跨字段 computeValue、validate、影响分析 |

P0 不开放完整规则编辑器，也不开放任意 JS。进入 P1 后，规则编辑器必须显示：

1. 触发对象。
2. 条件。
3. 动作。
4. 目标对象。
5. 错误提示。

P0 的"静态权限/状态"和 P1 的"动态规则"必须分工清晰：权限 Tab 处理对象级、角色无关或当前角色默认值的静态 visible/editable/readonly/mask；规则 Tab 处理数据值、流程状态、角色和跨字段条件驱动的动态显隐、必填、只读和计算。

EntryTable P0 一层明细的可见、可编辑和行操作只使用静态权限属性表达。基于行数据、流程状态或角色的动态行级规则归 P1。

| Field | visible、editable、required |

| EntryTable | visible、editable |

| 来源 | 阶段 | 说明 |
|---|---|---|
| 静态权限 | P0 | 直接配置对象 visible/editable/readonly/mask |
| 角色权限 | P1 | 按角色预览和运行态求值 |

| 数据状态 | P1 | 根据字段值或业务状态动态授权 |

### 7.9 性能 Tab

| 指标 | 阈值 | 适用对象 |

## 8. Bottom Analyzer

Analyzer 是底部发布质量门禁。

### 8.1 分组

| Tab | 问题类型 |
|---|---|
| 全部 | 所有问题 |

| 发布 | 阻断发布的问题 |

### 8.2 问题结构

```json
{
  "severity": "error",
  "code": "TABLE_DETAIL_DEPTH_EXCEEDED",
  "message": "P0 只允许一层明细表格。",
```

### 8.3 严重级别

| 级别 | 是否阻断发布 | 示例 |
|---|---|---|
| Error | 是 | 字段未绑定、规则循环、Schema 超限 |
| Warning | 否 | 表格列过多、布局接近溢出 |
| Suggestion | 否 | 建议调整字段顺序、建议使用模板 |

### 8.4 与画布联动

| 操作 | 结果 |
|---|---|
| 点击问题 | 定位并选中画布节点 |
| 双击问题 | 打开右侧对应 Inspector Tab |
| Hover 问题 | 画布临时高亮节点 |
| Quick Fix | 执行受控修复，进入命令历史 |

### 8.5 批量操作边界

| 功能 | 说明 |
|------|------|
| 单条 Quick Fix | P0：对当前问题执行受控修复，进入命令历史 |
| 问题聚合 | P0：相同节点多个问题聚合展示 |
| 忽略 Warning/Suggestion | P0：仅允许忽略非阻断项，并记录忽略原因 |
| 批量忽略 | P1：同一类非阻断问题可批量标记忽略 |

Error 级问题永远不可忽略，必须修复后才能发布。Analyzer 的忽略能力只能用于 Warning 和 Suggestion，且不能绕过发布门禁。

## 9. 三向同步模型

T-207 工作台必须保持四个视图一致：

```text
Canvas Selection
  <-> Outline Selection
  <-> Inspector Target
  <-> Analyzer Node
```

"三向"指 Canvas / Outline / Inspector 三者之间的双向同步。"四视图"指上述三者加上 Analyzer 作为同步终点。

### 9.1 状态同步

| 状态 | 来源 | 同步目标 |
|---|---|---|
| selectedNodeId | 画布 / 大纲 / Analyzer | Inspector、大纲、画布 |
| hoverNodeId | 画布 / Analyzer | 画布高亮 |
| activeInspectorTab | Inspector / Analyzer | Inspector |
| currentBreakpoint | Top Bar | Canvas |
| currentRole | Top Bar | Runtime Preview |

### 9.2 命令历史

所有修改进入 CommandStack：

| 命令 | 示例 |
|---|---|
| AddNode | 拖入字段 |
| MoveNode | 调整容器位置 |
| UpdateProp | 修改字段标题 |
| ResizeColumn | 调整表格列宽 |
| AddRule | 添加显隐规则 |
| SetPermission | 设置字段只读 |
| DeleteNode | 删除节点 |
| BatchOperation | P1：批量操作（原子性） |

命令必须支持撤销/重做。P0 的用户操作以单节点命令为主；P1 批量操作具有原子性：要么全部成功，要么全部回退。

## 10. 三向同步实现机制

### 10.1 事件总线设计

```
SchemaStore (单例)
  ↓ dispatch
EventBus (发布-订阅)
  ├── canvas:select(nodeId)
  ├── canvas:hover(nodeId)
  ├── outline:select(nodeId)
  ├── outline:scrollTo(nodeId)
  ├── inspector:update(prop, value)
  ├── inspector:tabChange(tabId)
  ├── analyzer:itemClick(nodeId)
  ├── analyzer:quickFix(issueId)
  ├── analyzer:batchFix(code)        // P1
  ├── command:executed(cmd)          // 命令执行后通知刷新
  ├── schema:published(pageId, version)
  └── schema:updated(pageId, version)
```

> **说明**：`command:executed` 由 CommandStack 发出，通知 SchemaStore 更新后触发 RuntimeRenderer 重渲染和 Analyzer 重新校验。不在 EventBus 中执行命令，只传递命令执行完毕的事件。
> `schema:published` / `schema:updated` 是工作台内的发布通知；运行态多实例热更新仍通过 07 定义的运行态事件通道和全量 Schema 重新拉取完成。

### 10.2 状态管理

| 状态 | 存储位置 | 作用域 | 同步方式 |
|------|---------|--------|---------|
| selectedNodeId | GlobalState | 工作台全局 | EventBus 广播 `canvas:select` / `outline:select` |
| hoverNodeId | GlobalState | 工作台全局 | EventBus 广播 `canvas:hover` |
| inspectorTab | LocalState (Inspector 组件) | Inspector 自身 | 组件内部管理，不广播 |
| commandStack | CommandStack 单例 | 全局撤销/重做 | 内部管理，通过 EventBus 通知刷新 |
| analyzerFilter | LocalState (Analyzer 组件) | Analyzer 自身 | 组件内部管理 |
| currentBreakpoint | GlobalState | Canvas | EventBus 广播 `canvas:breakpointChange` |
| currentRole | GlobalState | Canvas / Preview | EventBus 广播 `preview:roleChange` |

> **说明**：CommandStack 负责记录和回放命令序列，不直接参与 EventBus 广播。每次命令执行后触发 `command:executed` 事件，通知 SchemaStore 更新 → RuntimeRenderer 重渲染。
