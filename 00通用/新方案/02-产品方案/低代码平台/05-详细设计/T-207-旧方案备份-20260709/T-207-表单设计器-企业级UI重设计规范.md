# T-207 表单设计器 · 企业级 UI 重设计规范

> 适用范围：T-207 表单设计器、单据设计器、表单元素组件库  
> 修订原因：现有 Figma 原型视觉表达过于展示化，颜色、卡片、标签过多，不符合企业级后台工具规范。  
> 目标：吸收金蝶设计器的密度、三栏结构和运行态表单感，但在 PC Web 布局能力、复杂单据页面、治理能力、权限闭环、发布影响和测试契约上超过金蝶。

---

## 1. 结论

当前原型需要重画。

问题不在功能方向，而在视觉表达方式：

| 问题 | 当前表现 | 企业级修正 |
|---|---|---|
| 色彩过多 | 蓝、绿、黄、红大面积同时出现 | 主色只用于选中、焦点、主操作；风险色只用于真实异常 |
| 卡片过多 | 每个能力都做成彩色卡片 | 工具界面以表格、属性行、树、分组、抽屉为主 |
| 标签过多 | chip/pill 密集铺满页面 | 标签只用于状态、权限、风险等级，不用于装饰 |
| 展示感太强 | 像汇报型方案页 | 像可长期使用的企业设计器 |
| 信息层级太散 | 治理能力直接暴露在画布上 | 主流程保持安静，治理能力进入右侧面板、底部校验栏、发布抽屉 |

重画原则：**视觉比金蝶更克制，布局和治理能力比金蝶更完整。**

---

## 2. 企业级视觉定位

### 2.1 产品气质

表单设计器是高频生产工具，不是营销页、概念页或功能展示墙。

它的准确定位是：**PC Web 可视化业务页面设计器**。表单和单据是核心场景，但画布必须能设计真实 PC 端 Web 页面，包括多区块、分栏、子表、局部滚动、固定操作区和复杂业务组件组合。

应呈现：

| 维度 | 要求 |
|---|---|
| 密度 | 高密度但不拥挤，适合长时间配置 |
| 色彩 | 低饱和、弱装饰、强状态语义 |
| 布局 | 稳定三栏，中心画布优先 |
| 文案 | 少说明，多字段名、属性名、状态值 |
| 交互 | 可扫描、可定位、可批量编辑 |
| 风险提示 | 精准出现，不常驻刷屏 |

### 2.2 金蝶参考边界

保留：

| 金蝶特征 | 吸收方式 |
|---|---|
| 顶部命令栏 | 保留 48-56px 高度，提供保存、预览、发布、撤销重做 |
| 左侧控件/字段面板 | 保留树 + 搜索 + 分类，宽度约 260-300px |
| 中央白色画布 | 保留运行态表单质感，不做花哨画布 |
| 右侧属性面板 | 保留密集属性行和页签 |
| 蓝色选中框 | 保留 1px/2px 蓝色描边、8px 控制点 |
| 下划线字段风格 | 表单字段默认采用企业系统常见紧凑样式 |

不照搬：

| 金蝶不足 | 我方增强 |
|---|---|
| 权限来源不够显式 | 字段/操作显示后端权限来源和拒绝原因 |
| 发布影响弱 | 发布抽屉显示页面、角色、API、数据迁移影响 |
| 规则依赖弱 | 右侧规则页展示依赖、循环、影响范围 |
| 设计运行一致性弱 | 保存/发布前生成一致性校验报告 |
| AI 能力缺失 | AI 只作为 Patch 建议，必须 diff、校验、审批 |
| Web 布局能力弱 | 我方必须支持 Grid、Flex、嵌套容器、滚动、Sticky、SplitPane 和复杂单据区块 |

## 3. Figma 重画目标

### 3.1 必须重画的画板

| 画板 | 当前问题 | 重画要求 |
|---|---|---|
| `01 Main / 表单设计器主界面` | 基本方向可保留，但右侧治理信息偏展示 | 改为真实工具界面：左树、中画布、右属性、底部状态栏 |
| `02 Elements / 表单元素设计与状态矩阵` | 彩色卡片太多 | 改为组件库规格表 + 少量控件样例 |

| `04 Prototype Flow / 关键交互流程` | 展示感较强 | 保留流程，但改成低饱和灰白流程图 |
| `05 Bill Designer / 单据设计器主界面` | 右侧面板被截断，颜色偏多 | 改为企业单据设计器：分录表格、状态流、操作流、影响抽屉 |
| `06-08 Form Elements` | 尚未写入 Figma | 按本规范新增，避免彩色展示墙 |

### 3.2 重画后的第一屏结构

```text
--------------------------------------------------------------------------------+
| 顶部命令栏 48-52px：返回 / 名称 / 保存 / 预览 / 发布 / 更多 / 状态             |
+----------------------+----------------------------------+----------------------+
| 左侧 280px            | 中央画布                         | 右侧 320px            |
| - BO 字段树           | - PC Web 运行态页面画布          | - 属性页签            |
| - 控件库              | - 选中框/吸附线/拖放/尺寸标注    | - 属性行              |
| - 大纲                | - 子表/分组/页签                 | - 规则/权限/操作       |
| - 搜索                |                                  | - 校验摘要            |
+----------------------+----------------------------------+----------------------+
| 底部 28-32px：面包屑 / 当前选中节点 / 校验状态 / 缩放 / 协作状态              |
+--------------------------------------------------------------------------------+
```

### 3.3 禁止项

| 禁止项 | 原因 |
|---|---|
| 大面积彩色背景卡片 | 降低企业系统专业感 |
| 大量圆角彩色 pill | 像展示页，不像工具 |
| 每个功能都做成卡片 | 信息过散，扫描效率低 |
| 渐变、装饰图形、视觉噱头 | 与企业后台工具定位冲突 |
| 过多解释性正文 | 设计器用户需要操作入口，不需要说明书 |
| 将风险/权限常驻大红大黄 | 会造成告警疲劳 |
| 只展示简单两列表单 | 无法证明 PC Web 布局能力 |
| 文本换行错乱或底部裁切 | 原型质量不达标 |

## 4. 视觉系统

### 4.1 色彩

| 用途 | 色值建议 | 使用规则 |
|---|---|---|
| 页面背景 | `#F3F5F8` | 仅作为工作区底色 |
| 面板背景 | `#FFFFFF` | 左、中、右主容器 |
| 次级背景 | `#F7F9FC` | 属性行、表头、工具条 |
| 边框 | `#D8DEE8` | 面板、输入框、表格线 |
| 主文本 | `#111827` | 标题、字段值 |
| 次文本 | `#64748B` | 说明、辅助信息 |
| 主色 | `#2563EB` | 选中、焦点、主按钮 |
| 成功 | `#047857` | 通过状态，小面积 |
| 警告 | `#B45309` | 风险提示，小面积 |
| 错误 | `#B91C1C` | 阻断/错误，小面积 |

主界面中同时出现的语义色不得超过 3 种。默认只出现蓝色和灰色；绿色、黄色、红色只在状态真实存在时出现。

### 4.2 字体

| 用途 | 字号 | 字重 | 行高 |
|---|---:|---|---:|
| 顶部产品名 | 14-16 | Medium/Bold | 20 |
| 面板标题 | 13-14 | Medium/Bold | 18 |
| 属性名 | 12 | Regular/Medium | 18 |
| 属性值 | 12 | Regular | 18 |
| 表格内容 | 12 | Regular | 18 |
| 辅助说明 | 11 | Regular | 16 |
| 状态标签 | 10-11 | Medium | 14 |

不使用超大标题。设计器内标题不得超过 22px，除封面页外不使用宣传型 H1。

### 4.3 尺寸

| 区域 | 尺寸 |
|---|---|
| 顶部命令栏 | 48-52px |
| 左侧面板 | 280px |
| 右侧属性面板 | 320px |
| 底部状态栏 | 28-32px |
| 属性行高度 | 32-36px |
| 输入框高度 | 28-32px |
| 树节点高度 | 28px |
| 表格行高 | 34-38px |
| 面板圆角 | 4-6px |
| 控件圆角 | 4px |

## 5. 主界面规范

### 5.1 顶部命令栏

命令栏只放高频命令：

| 区域 | 内容 |
|---|---|
| 左侧 | 返回、表单/单据名称、版本状态 |
| 中部 | 撤销、重做、保存、预览 |
| 右侧 | 校验、发布、更多、协作/锁定状态 |

发布影响、权限冲突、AI 建议不在顶部铺开，只显示一个状态入口。

### 5.2 左侧面板

左侧面板采用三页签：

| 页签 | 内容 | 风格 |
|---|---|---|
| 字段 | BO 字段树、子表、关系字段 | 树表结构 |
| 控件 | 基础控件、选择控件、布局控件、操作控件 | 分组列表，不使用大彩卡 |
| 大纲 | 当前页面节点树 | 层级树 |

控件项标准高度 28-32px，使用小图标 + 名称 + 类型说明。禁止大块彩色卡片。

### 5.3 中央画布

中央画布以运行态 PC Web 页面为主体，必须能呈现复杂业务单据，而不是抽象示意图：

| 元素 | 规范 |
|---|---|
| 画布背景 | 浅灰工作区 |
| 表单纸张 | 白底，轻边框 |
| 字段 | 紧凑下划线或轻边框 |
| 选中 | 蓝色 1-2px 描边 + 控制点 |
| 拖放 | 只高亮目标区域，显示列线、吸附线和尺寸标注，不全屏变色 |
| 非法投放 | 红色细线 + 小 tooltip，不使用大告警卡 |
| 子表 | 表格样式优先，行高 34-38px |
| 复杂布局 | 支持 Toolbar、StatusBar、FormGrid、Subtable、SummaryBar、SidePanel、BottomActionBar |
| 滚动固定 | 支持局部滚动、横向滚动、Sticky 表头、Sticky 底部操作 |

### 5.4 右侧属性面板

属性面板采用页签 + 分组 + 属性行：

| 页签 | 内容 |
|---|---|
| 属性 | 字段绑定、控件类型、默认值、提交策略 |
| 布局 | display、columns、span、gap、标签位置、对齐、宽度、高度、overflow、sticky |
| 规则 | 显隐、必填、只读、计算、联动 |
| 权限 | 角色状态、后端权限来源、脱敏策略 |
| 操作 | 按钮、状态机、二次确认、事务 |
| 扩展 | 插件事件点、AI Patch |

规则和权限只在需要时显示风险徽标；默认保持灰白属性表。

### 5.5 底部状态栏

底部状态栏显示：

| 信息 | 示例 |
|---|---|
| 面包屑 | 销售订单 / 基本信息 / 客户 |
| 当前节点 | FieldNode: customerId |
| 校验状态 | 2 warning / 0 blocker |
| 缩放 | 100% |
| 协作状态 | locked by 李工 |

## 6. 表单元素视觉规范

### 6.1 元素列表项

左侧控件库中的元素使用列表项，不使用大卡片：

```text
[icon] 文本输入        string / 单行
[icon] 金额输入        decimal / currency
[icon] 日期选择        date
[icon] 参照选择        reference / remote
```

列表项规范：

| 属性 | 值 |
|---|---|
| 高度 | 30px |
| 图标 | 16px |
| 主文本 | 12px |
| 辅助文本 | 10px 灰色 |
| hover | `#F3F7FF` |
| selected | 左侧 2px 蓝线 + 浅蓝底 |

### 6.2 画布中的字段

默认字段样式：

```text
客户名称        杭州未来科技有限公司
              --------------------
```

字段状态：

| 状态 | 表现 |
|---|---|
| 默认 | 灰色下划线 |
| hover | 下划线加深 |
| focus | 蓝色下划线 |
| selected | 外层蓝色描边 |
| required | label 后小红星 |
| error | 红色下划线 + 错误文案 |
| readonly | 值文本灰蓝，禁止编辑 |
| disabled | 低透明度 |
| masked | 显示掩码和小锁图标 |
| permission denied | 设计态显示占位，运行态不渲染 |

### 6.3 属性面板中的元素契约

每个元素必须有统一属性组：

| 属性组 | 内容 |
|---|---|
| 数据 | fieldRef、dataType、defaultValue、formatter |
| 展示 | label、placeholder、helpText、density |
| 布局 | span、labelPosition、width、align |
| 校验 | required、min、max、pattern、asyncValidator |
| 规则 | visibleRule、readonlyRule、computedRule、linkageRule |
| 权限 | readPolicy、writePolicy、maskPolicy、denyReason |
| 测试 | testId、snapshotState、coverageTag |

## 7. 单据设计器特别规范

单据设计器不能只是“表单设计器换个标题”。

必须突出：

| 区域 | 规范 |
|---|---|
| 单据头 | 编号、状态、组织、客户、日期等主信息 |
| 分录 | 表格结构，支持列配置、行级权限、批量录入 |
| 尾部 | 合计、税额、折扣、审批摘要 |
| Web 布局 | Grid/Flex/SplitPane/局部滚动/横向滚动/Sticky 操作区 |
| 状态流 | 草稿、提交、审批中、生效、关闭 |
| 操作流 | 保存、提交、审批、驳回、关闭、打印、转换 |
| 业务影响 | 库存、财务、报表、接口、打印模板 |

视觉上仍保持灰白企业工具风格，不使用大面积彩色卡。

复杂单据原型必须至少展示：顶部命令栏、单据状态条、主表 FormGrid、分录 Subtable、汇总 SummaryBar、右侧审批/附件/关联单据面板、底部固定操作栏。

## 8. PC Web 布局能力视觉规范

强 Web 布局能力必须在界面上可见、可配、可校验。

| 能力 | UI 表达 |
|---|---|
| Grid | 右侧布局页显示 columns、span、gap；画布显示列线和 span 标注 |
| Flex | 右侧布局页显示 direction、align、justify、wrap、grow/shrink |
| SplitPane | 画布显示左右/上下分栏拖拽手柄，属性区显示固定宽度或填充 |
| Tabs/Collapse | 作为真实容器承载字段、子表和面板，不做静态装饰 |
| Scroll | 子表和右侧面板显示局部滚动边界，Sticky 区域有轻阴影 |
| Sticky | 顶部、表头、汇总、底部操作栏显示固定层级和滚动容器 |
| Size | resize 时显示 px、span、百分比、min/max |
| DropTarget | 合法投放显示蓝色线，不合法投放显示红色细线和原因 |

页面主体不得靠绝对定位堆字段。绝对定位只用于选中框、吸附线、浮层、下拉、Tooltip 和套打模板。
