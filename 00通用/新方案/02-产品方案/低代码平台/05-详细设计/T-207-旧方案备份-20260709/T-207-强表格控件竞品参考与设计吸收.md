# T-207 强表格控件竞品参考与设计吸收

> 适用范围：T-207 TableEngine、单据分录、嵌套表格、列表页、参照选择  
> 日期：2026-07-08  
> 结论：Web 嵌套表格以 AG Grid Enterprise 为第一参考；Delphi/VCL 传统商业控件以 DevExpress ExpressQuantumGrid/cxGrid 为历史和交互参考。

---

## 1. 结论

如果只选一个 Web 商业表格作为嵌套表格参考，选择 **AG Grid Enterprise**。

原因：

| 维度 | AG Grid Enterprise 表现 | 对 T-207 的价值 |
|---|---|---|
| Web 原生 | 面向现代 JavaScript/React/Vue/Angular | 与 T-207 PC Web 设计器目标一致 |
| Master/Detail | 官方明确支持 Master Grid 展开 Detail Grid | 适合单据头 + 分录、列表 + 明细 |
| Nesting | 官方明确支持 Detail Grid 再作为 Master Grid | 适合母子孙多层嵌套表格 |
| 性能 | 企业级大数据、服务端模型、虚拟滚动能力强 | 适合宽表和大数据量 |
| 设计启发 | 配置化列定义、detail renderer、grid options | 适合抽象为 TableSchema/ColumnSchema |

如果参考 Delphi 高版本第三方商业控件，选择 **DevExpress VCL ExpressQuantumGrid/cxGrid**。

原因：

| 维度 | DevExpress VCL Grid 表现 | 对 T-207 的价值 |
|---|---|---|
| Delphi 生态影响力 | 经典商业 VCL Grid 控件 | 符合用户提到的 Delphi 商业控件记忆 |
| Master/Detail | 支持任意复杂度 master-detail | 适合多层业务数据展示 |
| 设计期体验 | Level/View 结构、Object Inspector 配置 | 适合借鉴到设计器大纲和属性面板 |
| 大数据模式 | Grid Mode / Server Mode 思路成熟 | 适合 T-207 的表格性能设计 |

T-207 不直接复制任何产品 UI 或实现，而吸收其架构思想和交互模式。

## 4. T-207 应吸收的 AG Grid 设计

| AG Grid 思想 | T-207 转化 |
|---|---|
| Master Grid / Detail Grid | TableNode 支持 detailTemplates，展开行渲染子 TableNode |
| Detail Grid 继续作为 Master Grid | 支持母子孙多层嵌套表格，schema 递归但深度可控 |
| detailCellRendererParams | 转化为 detailRenderer 配置，引用子表、页签、卡片或表单片段 |
| Grid Options | 转化为 tableOptions，控制滚动、选择、编辑、虚拟化、聚合 |
| Column Definitions | 转化为 ColumnSchema，统一字段绑定、控件、权限、规则 |
| Server-Side Row Model | 转化为 remoteDataSource、pagination、lazyLoad、virtualRows |
| Detail Height | 转化为 detailHeightMode：fixed、auto、maxHeight |

## 6. T-207 嵌套表格目标形态

销售订单复杂嵌套表格示例：

```text
销售订单分录 EditableTable
├─ 行 1：物料 A / 数量 / 单价 / 金额
│  └─ 展开：批次明细 DetailTable
│      ├─ 批次号 / 库位 / 数量 / 生产日期
│      └─ 展开：序列号 DetailTable
├─ 行 2：物料 B / 数量 / 单价 / 金额
│  └─ 展开：促销政策 DetailPanel
└─ 汇总：数量合计 / 金额合计 / 税额 / 折扣
```

设计要求：

| 能力 | 设计要求 |
|---|---|
| 母子嵌套 | 行展开后可显示子表格、页签、表单片段或卡片 |
| 多层嵌套 | 默认允许 2 层，管理员可放宽到 3 层，超过需发布风险确认 |
| 懒加载 | 子表默认展开时加载，避免一次性加载所有 detail |
| 高度控制 | detail 支持 fixed、auto、maxHeight，防止页面撑爆 |
| 列宽同步 | 可选择父子表列宽独立或按关键列对齐 |
| 权限继承 | 子表权限继承父行上下文，再叠加自身行列权限 |
| 错误定位 | 错误路径包含 parentRowKey、childTableId、rowKey、columnId |
| 汇总传播 | 子表汇总可回填父行，父表汇总可回填主表 |

## 7. T-207 表格设计器界面方案

表格设计器采用 Visual Studio / 金蝶式工具布局：

```text
+--------------------------------------------------------------------------------+
| 顶部命令栏：保存 / 预览 / 发布 / 复制列 / 冻结列 / 表格设置 / 校验              |
+----------------------+----------------------------------+----------------------+
| 左侧                 | 中央设计画布                     | 右侧属性面板          |
| - 字段树             | - 可编辑表格                     | - Table 属性          |
| - 表格控件           | - 表头列线                       | - Column 属性         |
| - 子表数据源         | - 冻结分割线                     | - Cell Editor         |
| - 表格大纲           | - 展开行 detail 区域             | - 权限/规则/性能       |
+----------------------+----------------------------------+----------------------+
| 底部状态栏：Table / Row / Column / Cell / 校验问题 / 缩放 / 数据模式            |
+--------------------------------------------------------------------------------+
```

右侧属性分层：

| 页签 | 内容 |
|---|---|
| Table | dataSource、rowKey、density、height、scroll、selection、virtualization |
| Columns | 列顺序、宽度、冻结、显隐、分组、排序、过滤 |
| Cell | editor、formatter、validation、required、readonly、mask |
| Detail | detail type、child table、lazy load、height、tabs、max depth |
| Rule | 行内计算、跨行汇总、父子联动、错误定位 |
| Permission | 表格、行、列、单元格、操作权限 |
| Performance | virtualRows、serverMode、pageSize、cache、expand strategy |

## 8. 不照搬项

| 不照搬 | 原因 |
|---|---|
| 不直接复制 AG Grid UI | T-207 是设计器，不是直接嵌入第三方表格产品 |
| 不直接复制 DevExpress VCL 外观 | Delphi/VCL 视觉不适合现代 PC Web，但结构思想有价值 |
| 不把表格做成报表设计器 | 首版聚焦业务录入和分录，不做 BI 透视报表 |
| 不无限嵌套 | 嵌套过深会带来性能、权限和可用性风险 |
| 不允许自由 JS 单元格脚本 | 单元格逻辑必须进入规则引擎和插件白名单 |

---

## 9. 设计结论

T-207 的强表格控件应采用：

1. **AG Grid Enterprise 的 Web Master/Detail/Nesting 思想**作为主设计参考。
2. **DevExpress ExpressQuantumGrid/cxGrid 的 Level/View 和企业级 master-detail 经验**作为结构参考。
3. **Visual Studio Form Designer 的 Toolbox + Canvas + Properties + Outline + Snaplines**作为设计器交互骨架。
4. **金蝶表单设计器的企业密度和属性面板风格**作为视觉密度参考。

最终目标不是做一个表格组件，而是做一个能设计、校验、发布、运行复杂企业表格的 **TableEngine + Table Designer**。
