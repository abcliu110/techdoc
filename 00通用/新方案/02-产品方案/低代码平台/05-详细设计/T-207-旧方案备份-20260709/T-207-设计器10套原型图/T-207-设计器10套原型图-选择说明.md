# T-207 Web 页面设计器 10 套候选原型图选择说明

> 日期：2026-07-09  
> 范围：T-207 PC Web 可视化页面 / 表单 / 单据设计器  
> 重要修正：本版不是报表设计器。DevExpress 只作为设计器工作台布局参考，中央画布必须是复杂 PC Web 业务页面。

## 1. 修正原则

上一版候选图过于接近报表设计器表达，本版已按以下原则重做：

1. **画布对象是 Web 页面，不是报表页**  
   中央画布必须呈现真实 PC Web 单据页面，包括顶部业务操作区、页签、导航、表单区、分录表格、右侧信息区、底部动作区。

2. **核心能力是 Web 布局容器**  
   支持 Grid、Flex、Dock、Split、Scroll、Sticky、Portal、Responsive，而不是报表 band 和分页布局。

3. **强表格是企业单据核心**  
   表格应支持分录、主从明细、批次、序列号、冻结列、虚拟滚动、键盘编辑、复制粘贴。

4. **DevExpress 只吸收工作台骨架**  
   顶部命令、左侧资源、中央画布、右侧属性、底部诊断可以参考 DevExpress，但不能把产品做成报表设计器。

   这里的“DevExpress 五区”指设计器工作台壳层，不是业务页面内部区域：

   | 区域 | 英文口径 | 在 T-207 中的含义 |
   |---|---|---|
   | 01 顶部命令区 | Top Command / Ribbon | 菜单、撤销重做、保存、预览、校验、发布 |
   | 02 左侧工具箱区 | Left Toolbox / Field List | 布局控件、表单字段、表格控件、页面大纲 |
   | 03 中央设计画布区 | Center Designer Surface | PC Web 可见即可得画布，显示真实业务页面 |
   | 04 右侧属性区 | Right Property Grid | 属性、布局、数据绑定、规则、权限、事件 |
   | 05 底部错误列表区 | Bottom Error List / Analyzer | 校验错误、发布阻断、布局诊断、Schema 摘要 |

5. **金蝶提供企业密度和业务语义**  
   视觉保持克制、高密度、业务导向，重点是单据、字段、分录、规则、权限、审批。

6. **Visual Studio 提供设计器交互**  
   工具箱、Document Outline、Property Grid、Snapline、错误列表、文档页签是可吸收的交互骨架。

## 3. 推荐组合

建议主方案采用：

```text
01 企业单据页面设计器
+ 04 强 Web 布局容器型
+ 05 强表格 / 母子嵌套型
+ 02 Visual Studio Web Form Designer 型的属性窗/大纲/错误列表
+ 03 金蝶单据高密度型的企业视觉和业务语义
```

## 4. 默认工作流建议

默认设计器不应该先让用户“画报表”，而应该让用户设计一个真实 Web 页面：

1. 从单据模板进入：销售订单、采购申请、费用报销、客户经营工作台。
2. 左侧拖入布局容器、业务字段、表格组件、复合业务组件。
3. 中央画布渲染真实 Web 页面结构，不保存自由 DOM，只保存 PageSchema / LayoutSchema / ComponentSchema / TableSchema。
4. 选中页面元素后，右侧属性面板切换为布局、数据、规则、权限、事件、性能。
5. 底部 Analyzer 持续提示布局断点、字段绑定、表格性能、权限缺口、发布风险。

## 5. 哪些方案不适合默认入口

| 方案 | 原因 |
|---|---|
| 06 业务对象字段驱动型 | 更适合作为字段绑定视图，默认打开会显得偏建模 |
| 07 PC 响应式布局预览型 | 更适合作为布局检查模式，不适合长期编辑 |
| 08 规则 / 审批联动型 | 信息复杂，适合作为规则视图或发布检查 |
| 09 组件库 / 设计系统型 | 属于平台治理模块，不是普通表单设计主流程 |
| 10 设计态 / 运行态双视图型 | 适合高级调试和发布门禁，默认会显得过重 |

## 6. 下一步深化

选定方向后，建议继续输出：

1. 默认工作台高清单图。
2. 五个关键状态：空白页面、选中布局容器、选中字段、选中表格、发布校验。
3. 真实控件规格：字段控件、布局容器、表格列、明细模板、权限标记。
4. LayoutEngine / ComponentRegistry / TableEngine 的交互契约。
5. 可交互 HTML 原型，而不是静态示意图。
