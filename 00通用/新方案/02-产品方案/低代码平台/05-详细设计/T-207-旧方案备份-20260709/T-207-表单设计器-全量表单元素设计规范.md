# T-207 表单设计器 · 全量表单元素设计规范

> 适用范围：表单设计器控件库、单据设计器、运行态 Renderer、测试规格  
> UI 风格：企业级、克制、紧凑、低饱和、可扫描  
> 设计原则：元素不是“控件外观”，而是数据、展示、交互、规则、权限、发布、测试的统一契约。

---

## 1. 元素分类总览

| 大类 | 子类 | 首版优先级 | 用途 |
|---|---|---|---|
| 基础输入 | 文本、数字、金额、日期、长文本、富文本 | P0 | 普通字段录入 |
| 选择引用 | 下拉、多选、单选、复选、开关、参照、树选择、人员、组织 | P0 | 枚举、主数据、组织权限 |
| 上传展示 | 附件、图片、二维码、条码、只读文本、公式展示 | P1 | 文件、图片、展示型内容 |
| 业务语义 | 单据编号、单据状态、审计字段、审批意见、流程节点 | P0/P1 | 企业业务对象 |
| 布局容器 | 分组、字段组、页签、栅格、子表、折叠面板、提示 | P0 | 页面信息结构 |
| 操作命令 | 保存、提交、审批、驳回、删除、打印、导入、导出、自定义动作 | P0/P1 | 状态机与业务动作 |

---

## 2. 通用元素契约

每个元素必须提供以下契约。

| 契约 | 字段 | 说明 |
|---|---|---|
| 数据契约 | `fieldRef`、`dataType`、`defaultValue`、`formatter` | 与 BO 字段或虚拟字段绑定 |
| 展示契约 | `label`、`placeholder`、`helpText`、`density` | 决定画布和运行态显示 |
| 布局契约 | `span`、`labelPosition`、`width`、`align` | 决定栅格和响应式 |
| 校验契约 | `required`、`min/max`、`pattern`、`asyncValidator` | 保存、提交、发布可校验 |
| 规则契约 | `visibleRule`、`readonlyRule`、`computedRule`、`linkageRule` | 进入依赖图 |
| 权限契约 | `readPolicy`、`writePolicy`、`maskPolicy`、`denyReason` | 不得放宽后端权限 |
| 运行契约 | `renderMode`、`loadingState`、`emptyState`、`errorState` | 设计态和运行态一致 |
| 测试契约 | `testId`、`snapshotState`、`coverageTag` | 自动生成测试用例 |

## 3. 通用状态

所有元素必须支持统一状态。

| 状态 | 视觉表现 | 行为 | 测试要求 |
|---|---|---|---|
| default | 灰边/灰下划线 | 正常展示 | 截图基线 |
| hover | 背景轻微变蓝或边框加深 | 可点击/可编辑提示 | hover 截图 |
| focus | 蓝色边框或下划线 | 键盘焦点 | tab 顺序测试 |
| selected | 设计态蓝色外框 | 属性面板联动 | 选中同步测试 |
| dragging | 半透明浮层 | 拖拽中 | 拖拽测试 |
| drop-target | 目标区域蓝色细线 | 可投放 | drop 成功测试 |
| invalid-drop | 红色细线 + tooltip | 阻止投放 | 阻断原因测试 |
| dirty | 小圆点或浅色标记 | 未保存 | 保存后消失 |
| readonly | 值可见，不可编辑 | 禁止输入 | 输入阻断测试 |
| disabled | 低透明度 | 不响应 | 点击无效测试 |
| hidden | 设计态占位，运行态隐藏 | 不参与布局或占位策略 | 运行态不可见 |
| masked | 掩码 + 锁图标 | 原值不出前端 | API/导出脱敏测试 |
| required | label 小红星 | 空值阻断 | 必填测试 |
| error | 红色下划线 + 错误文案 | 阻断提交 | 错误定位 |
| loading | 骨架/加载标记 | 等待异步数据 | 超时测试 |
| permission-denied | 设计态显示策略来源 | 运行态拒绝 | 越权测试 |
| publish-conflict | 属性面板/发布抽屉提示 | 阻断发布 | 不生成版本 |

## 4. 基础输入元素

| 元素 | 数据类型 | 默认样式 | 关键属性 | 权限表现 | 测试重点 |
|---|---|---|---|---|---|
| 文本 Text | string | 下划线输入 | maxLength、trim、pattern | readonly、masked、hidden | 输入、粘贴、超长、正则 |
| 密码 Password | string | 密码框 | encrypt、visibilityToggle | 默认 masked | 可见切换、保存加密 |
| 数字 Number | number | 右对齐输入 | precision、min、max、step | readonly、hidden | 边界、步进、非法字符 |
| 金额 Money | decimal | 右对齐 + 千分位 | currency、precision、rounding | 财务字段可脱敏 | 精度、四舍五入、币种 |
| 百分比 Percent | decimal | 数字 + `%` | precision、range | readonly | 0-100、比例换算 |
| 日期 Date | date | 日期选择 | disabledDate、quickRange | readonly | 禁选、清空、快捷 |
| 时间 Time | time | 时间选择 | step、format | readonly | 步长、格式 |
| 日期时间 DateTime | datetime | 日期 + 时间 | timezone、format | readonly | 时区、秒级 |
| 日期范围 DateRange | date[] | 起止日期 | startField、endField | readonly | 起止关系、禁选 |
| 长文本 Textarea | string | 多行输入 | rows、maxLength、autoSize | readonly | 字数、换行、高度 |
| 富文本 RichText | html | 富文本编辑 | toolbar、sanitizePolicy | 可禁用图片/链接 | XSS、白名单 |
| 代码 Code | string | 等宽编辑 | language、lintRule | readonly | 语法提示、注入防护 |

## 10. 左侧控件库呈现规范

控件库不使用彩色卡片，采用企业工具列表：

```text
基础输入
  [T] 文本输入       string / 单行
  [#] 数字输入       number / precision
  [¥] 金额输入       decimal / currency

选择引用
  [v] 下拉选择       enum / remote
  [↗] 参照选择       master data
  [人] 人员选择       org permission

布局容器
  [▦] 分组           section
  [≡] 子表           line table
```

| 交互 | 表现 |
---|---|
| hover | 浅蓝背景 |
| dragging | 轻透明拖拽影子 |
| disabled | 灰色文字，不可拖 |
| recommended | 小型灰蓝徽标，不用大色块 |

## 11. 右侧属性面板呈现规范

属性面板按表单行呈现：

```text
属性
字段绑定        Contract.amount
控件类型        金额 Money
显示名称        合同金额
默认值          空

校验
必填            是
最小值          0
精度            2

权限
读取策略        P-FIN-READ
写入策略        P-FIN-WRITE
脱敏策略        AuditMasked
```

| 规则 | 要求 |
|---|---|
| 属性行高度 | 32-36px |
| label 宽度 | 80-96px |
| 输入控件 | 28-32px 高 |
| 风险提示 | 行内 icon + tooltip |
| 高级属性 | 默认折叠 |
| AI 建议 | 单独抽屉，不常驻占面板 |
