# T-202 默认页面详细设计

> 版本：v0.1
> 里程碑：M2
> 适用任务：T-202
> 依据：PRD REQ-020~023、REQ-090；`03-设计器与前端设计.md`
> 5A 状态：baseline-approved。`../04-架构决策/UI-and-Component-Decision-Matrix.md` 已人工接受；本设计作为 M2 UI 详细设计基线，禁止提前在 M0/M1 实现。

---

## 1. 目标

业务对象发布后自动获得列表、表单、详情三类默认页面。默认页面由模型生成，能直接完成 CRUD 和状态动作，不要求设计者先拖拽页面。

## 2. 默认页面生成规则

| 页面 | 生成依据 |
|---|---|
| 列表 | in_list 字段、title_field、默认排序、允许动作 |
| 表单 | required 字段、字段分组、字段顺序、table 子表 |
| 详情 | title_field、普通字段、附件、状态流转日志入口 |

生成结果写入 `PageSnapshot`，随版本发布冻结。

## 3. 列表页

能力：

```text
服务端分页
服务端筛选
服务端排序
字段权限裁剪
行级动作
批量动作
导出入口
```

列表默认只展示 `in_list=true` 且有 read 权限字段。

## 4. 表单页

能力：

```text
新增/编辑
字段必填
字段类型校验
fetch_from 前端预览
depends_on 显隐
子表编辑
附件上传
状态动作按钮
```

提交必须走后端复验；前端校验只用于体验。

## 5. 详情页

能力：

```text
字段分组展示
脱敏展示
附件预览
状态流转记录
审计记录入口
下游/上游引用入口预留
```

## 6. 页面生成率指标

默认页面被直接使用或只做布局覆盖时，计入页面生成率。完全自定义页面不计入。

## 7. 验收

1. 建 `customer` 对象后无需页面配置即可 list/add/get/update。
2. 建 `order` 对象后默认页面能显示 customer link、order_item 子表和 amount。
3. 状态动作按钮按 AccessView 和状态机显示。
4. 修改模型后未发布前运行态仍使用旧页面。
5. 页面生成率指标能统计默认页面使用情况。

### 8.3 UI 设计系统对照

默认列表使用 AntD Table/ProTable 风格但通过平台 renderer 封装；默认表单使用 AntD Form 语义但字段组件必须来自 FieldRegistry；默认详情使用描述列表/分组布局，但不得直接展示未经 `/meta` 裁剪的字段。

### 8.4 可访问性规则

| 规则 | 验收方式 |
|---|---|
| 列表表头、排序、筛选可键盘操作 | 表格列、筛选控件和分页控件可通过 Tab/Enter 操作 |
| 表单错误可定位 | 提交失败聚焦首个错误字段，错误文本与控件关联 |
| 状态动作按钮可解释 | 禁用按钮提供可访问说明，不只依赖颜色 |
| 详情页结构可感知 | 分组标题使用语义 heading 或 aria-labelledby |

### 8.5 状态组合

| 权限状态 | 数据状态 | 交互状态 | 默认页面行为 |
|---|---|---|---|
| NONE | any | any | 默认页面生成时排除字段和无权限动作 |
| MASKED | clean | idle | 列表/详情显示脱敏值，表单编辑能力以 meta 为准 |
| READ | clean/stale | loading/conflict | 可展示不可编辑，版本冲突提示刷新但保留上下文 |
| WRITE | empty/dirty/error | submitting | 可编辑，提交失败按字段 path 展示错误 |

### 8.6 失败模式

| 失败模式 | 处置 |
|---|---|
| 默认页面引用已删除字段 | 发布前阻断，运行态旧版本按 snapshot 回放 |
| 隐藏必填字段导致提交失败 | 后端仍校验，前端错误指向被隐藏字段所在分组并要求恢复显示或补默认值 |
| 状态动作可见性与 API 不一致 | 以 API/AccessView 为准，记录 drift 测试失败 |
| 页面生成率指标误导 | 只统计默认页面或布局覆盖，完全自定义页面不计入 |
