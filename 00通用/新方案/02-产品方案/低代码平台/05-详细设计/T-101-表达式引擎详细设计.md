# T-101 表达式引擎详细设计

> 版本：v0.1
> 里程碑：M1
> 适用任务：T-101 表达式引擎封装
> 依据：`../03-需求/PRD-产品需求规格说明书.md` REQ-011、REQ-040、REQ-087、REQ-088；`../04-架构决策/02-运行时引擎设计.md` §4；ADR-LOWCODE-EXPR-001 待补终审；知识库 `BR-LOWCODE-001`、`MT-LOWCODE-001`

---

## 1. 目标

T-101 建立统一表达式门面，支撑默认值、字段显隐、字段只读、校验规则、数据范围、状态机条件、规则条件和公式字段的同一套语法版本。

M1 只实现安全表达式求值与依赖提取，不实现跨对象聚合公式、不实现持久化公式重算队列、不开放自由 JS。

## 2. 模块与边界

| 模块 | 职责 |
|---|---|
| `lowcode-expression-api` | 门面接口、上下文模型、错误码、AST/依赖描述 |
| `lowcode-expression-engine` | Aviator 适配、安全配置、函数注册、超时控制 |
| `lowcode-expression-contract` | 契约资产加载、恶意表达式集、兼容测试 |
| `lowcode-expression-web` | 表达式语法预检 API，不暴露运行态业务数据 |

业务模块只能依赖门面接口，禁止直接 import Aviator。

## 3. 核心接口

```java
public interface ExpressionService {
  ExpressionCompileResult compile(ExpressionCompileCommand command);
  ExpressionEvalResult eval(ExpressionEvalCommand command);
  ExpressionDependencyGraph analyzeDependencies(ExpressionCompileCommand command);
  ExpressionCompatibilityReport checkCompatibility(ExpressionVersion from, ExpressionVersion to);
}
```

`ExpressionCompileCommand` 必含：

```text
tenantId
appCode
objectCode
usage: DEFAULT / VISIBLE / READONLY / VALIDATE / DATA_SCOPE / TRANSITION / RULE / FORMULA
expressionText
expressionVersion
allowedVariables
allowedFunctions
```

`ExpressionEvalCommand` 额外包含：

```text
metaHash
traceId
timeoutMs
record
oldRecord
userContext
envContext
```

## 4. 变量模型

| 变量 | 可用场景 | 说明 |
|---|---|---|
| `$record` | 全部运行态场景 | 当前记录，按字段权限裁剪后进入表达式 |
| `$old` | update、transition、rule | 变更前记录 |
| `$user` | 权限、规则、状态机 | 当前用户、角色、部门、组织路径 |
| `$env` | 全部运行态场景 | tenant、workspace、app、metaHash、requestTime |
| `$input` | action、rule | 动作参数，必须先经参数 schema 校验 |

字段无读权限时不得进入 `$record`。表达式不能通过 link 展开读取当前用户无权读取的目标对象字段。

## 5. 函数白名单

M1 内置函数只包含：

```text
字符串：len, contains, startsWith, endsWith
数值：abs, round, min, max
日期：today, now, dateAdd, dateDiff
集合：in, isEmpty, size
逻辑：ifElse, coalesce
```

禁止：

```text
反射、类加载、文件 IO、网络 IO、线程、系统环境变量、任意 Bean 调用、SQL 调用
```

新增函数必须声明：

```text
functionCode
inputTypes
outputType
deterministic
sideEffectFree
allowedUsages
versionIntroduced
```

## 6. 依赖图与循环检测

表达式保存和发布前必须分析依赖：

```text
fieldDependencies: 字段 code 集合
linkDependencies: link 字段的一层目标字段集合
systemDependencies: user/env/requestTime 等系统变量
```

规则：

1. `formula` 字段不得依赖自己。
2. `formula` 字段之间不得形成环。
3. `assign` 规则修改字段后最多触发一轮 fetch_from 和公式重算。
4. M1 禁止跨对象聚合依赖，例如 `sum(order_items.amount)`。
5. 依赖无读权限字段时，发布阻断；运行时权限变化导致依赖不可读时，表达式返回权限错误而不是空值。

## 7. 运行安全

| 风险 | 设计 |
|---|---|
| 死循环/耗时 | 每次 eval 默认 100ms 超时，超时返回 `EXPR_TIMEOUT` |
| 大对象输入 | record/input 大小限制，超过返回 `EXPR_CONTEXT_TOO_LARGE` |
| 类型漂移 | 编译期基于 FieldTypeHandler 做类型检查，运行期二次校验 |
| 权限泄漏 | 上下文由 AccessView 裁剪，不允许表达式自行查库 |
| 双端漂移 | 前端只实现 visible/readonly 的安全子集，校验以服务端为准 |

## 8. 错误码

```text
EXPR_SYNTAX_ERROR
EXPR_UNSUPPORTED_FUNCTION
EXPR_FORBIDDEN_VARIABLE
EXPR_TIMEOUT
EXPR_CONTEXT_TOO_LARGE
EXPR_TYPE_MISMATCH
EXPR_PERMISSION_DENIED
EXPR_DEPENDENCY_CYCLE
EXPR_VERSION_UNSUPPORTED
```

错误响应必须包含 `path`、`expressionUsage`、`traceId`、`message`，不得包含业务字段值。

## 9. 版本与兼容

表达式语法版本首版为 `expr-v1`。所有表达式配置必须保存：

```text
expressionText
expressionVersion
compiledDigest
dependencyDigest
lastValidatedMetaHash
```

升级表达式语法时必须提供：

1. 旧表达式回放测试。
2. 升级器或兼容解释器。
3. 不兼容表达式影响报告。

## 10. 验收

1. 恶意表达式不能访问 Java 类、文件、网络、系统环境变量。
2. 死循环或超时表达式在 100ms 量级被阻断。
3. `amount > 10000 && isEmpty(approval_comment)` 能用于校验规则。
4. `customer.level == "vip"` 能用于 visible 条件。
5. 公式字段循环依赖发布阻断。
6. 字段无读权限时，表达式不能通过 `$record` 或 link 展开泄漏该字段。
7. ArchUnit 阻断业务模块直接依赖 Aviator。
