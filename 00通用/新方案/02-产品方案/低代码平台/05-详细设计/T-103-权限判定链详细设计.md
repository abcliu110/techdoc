# T-103 权限判定链详细设计

> 版本：v0.1
> 里程碑：M1
> 适用任务：T-103
> 依据：`../03-需求/PRD-产品需求规格说明书.md` REQ-030~034、REQ-084；`../04-架构决策/02-运行时引擎设计.md` §3；`../04-架构决策/ADR/ADR-LOWCODE-PERM-001-access-view-permission-core.md`；知识库 `PM-LOWCODE-001`
> 决策门禁：权限 ADR 当前为 proposed。租户隔离、服务端统一判权、字段/记录权限一致和 `OWNER_SNAPSHOT` 产品语义是强制不变量；`AccessView` 类型、七层判定、角色/字段枚举、缓存键和 explain 契约是候选，ADR accepted 或形成等价已批准决策记录前不得据此开工。

---

## 1. 目标

建立 AccessView 权限内核，统一服务 `/meta/list/get/add/update/del/action/suggest/export/import/级联写/link 展开`。

权限不能只做菜单或应用访问，必须覆盖租户成员、应用访问、对象权限、数据范围、字段权限、动作权限和 break-glass。

## 2. 判定链

```text
L1 租户/工作区成员资格
L2 应用访问权限：design/runtime
L3 对象权限：read/create/update/delete/export/import
L4 数据范围：self/dept/dept_tree/all/custom/snapshot
L5 字段权限：NONE/READ/WRITE/MASKED
L6 动作权限：Action + StateTransition.allowedRoles
L7 运行态特殊边界：break-glass、密钥不可见、附件下载、link suggest
```

任一层拒绝即终止，并写 `PERMISSION_DENY` 事件。

## 3. 权限元数据

`PermissionDef` 存储在 `lc_meta_role.permissions` JSON：

```text
objectCode
operations
dataScope
scopeCondition
fieldPermissions
actionCodes
exportAllowed
importAllowed
attachmentAllowed
breakGlassAllowed
```

角色分为：

```text
platform_admin
tenant_admin
app_designer
runtime_user
audit_operator
plugin_operator
```

设计态权限和运行态权限必须分离。

## 4. AccessView

```text
AccessView:
  objectCode
  operationSet
  fieldView: fieldCode -> NONE/READ/WRITE/MASKED
  dataScopeAst
  actionSet
  exportableFields
  suggestableFields
  attachmentPolicy
  permVersion
  metaHash
```

AccessView 是不可变对象。运行时所有消费者只能读取 AccessView，不能重新组合权限。

## 5. 数据范围语义

| scope | 语义 |
|---|---|
| self | owner_user_lid = 当前用户 |
| dept | owner_dept_lid = 当前部门快照 |
| dept_tree | owner_org_path 前缀匹配当前部门树 |
| all | 租户内全部，仍强制 tenant_id |
| custom | 受限表达式转 AST |
| owner_snapshot | 按记录创建或归属快照判断 |
| current_org | 按当前组织树判断 |

REQ-033 要求对象级明确使用历史快照或当前组织语义，默认 `OWNER_SNAPSHOT`。

## 6. 字段权限

字段权限消费规则：

1. `/meta` 不返回 NONE 字段，MASKED 字段只返回脱敏策略。
2. `list/get/export` 对 NONE 字段裁剪，对 MASKED 字段脱敏。
3. `add/update/import` 对无 WRITE 字段拒绝。
4. `suggest/link expand` 对目标对象再次计算 AccessView。
5. 前端隐藏字段不能替代后端字段权限。

## 7. break-glass

运维代客操作必须：

```text
申请原因
审批人
有效期
目标租户
目标应用/对象范围
允许操作
审计不可篡改
```

break-glass 不得显示密钥明文，不得绕过字段脱敏，不得关闭审计。

## 8. 缓存与失效

AccessView 缓存键：

```text
tenantId:userLid:appCode:objectCode:metaHash:permVersion:orgVersion
```

失效来源：

```text
角色变更
组织树变更
字段权限变更
对象发布
用户禁用
break-glass 到期
```

缓存失效失败时走保守拒绝，而不是放权。

## 9. 权限解释接口

```text
POST /api/permission/explain
```

输入：

```text
userLid
appCode
objectCode
operation
fieldCode?
recordLid?
```

输出六层判定结果、命中的角色、数据范围 AST、字段裁剪原因、permVersion、metaHash。解释接口本身仅平台管理员、租户管理员和审计人员可用。

## 10. 验收

1. 无租户成员资格直接 403。
2. 设计者能设计应用但不能读取运行数据源密钥明文。
3. self 数据范围看不到他人记录。
4. 字段无 read 权限时，在 meta/list/get/export 中都不可见。
5. 字段只读时 update/import 被拒绝。
6. suggest 不泄露目标对象无权字段。
7. 组织调岗按对象配置的 OWNER_SNAPSHOT 或 current_org 得到可解释结果。
8. break-glass 全链路有审批、有效期和审计。
