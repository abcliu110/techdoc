# ADR-LOWCODE-APP-PACKAGE-001: 应用包、市场与 License 生命周期

> 状态：proposed
> 日期：2026-07-06

## 背景

商用低代码平台不仅要让用户建应用，还要支持行业模板、应用包、插件、报表打印模板和私有化授权。若这些能力靠手工导入 JSON、复制 SQL 或在线 License 服务临时控制，会导致升级不可追踪、客户定制被覆盖、私有化不可用和合规风险。

## 决策

应用包与 License 生命周期必须独立治理：

- 包 manifest 记录来源、供应商、版本、归属层、依赖、资产清单、License 和升级策略。
- 包升级必须保护 customer 扩展层，冲突进入升级报告。
- 私有化 License 支持离线授权，在线校验只能作为增量能力。
- License 过期或降级不能删除业务数据，只能限制新增、发布、安装或高级能力。

## 理由

应用包是商业交付和生态扩展的载体，License 是商业边界。二者如果不成为治理对象，会污染元数据升级、模板分发、私有化部署和审计。

## 否决方案

- 手工导入 JSON/SQL：短期可演示，但不可升级、不可审计。
- 强依赖在线授权服务：SaaS 可用，私有化和离线客户不可接受。
- 允许第三方后端代码直接运行：生态灵活，但共享 SaaS 安全和资源隔离风险过高。

## 后果

- M0 只预留 package/source_kind/version 等字段和 License 配置边界。
- M3 前必须补包管理、市场、离线授权、升级回放和安全隔离详细设计。
- 第三方后端代码默认不进入共享 SaaS 运行时，除非另补插件沙箱 ADR。

## 验证

- 包安装、升级、回滚、卸载均有审计状态。
- License 过期降级不破坏已有业务数据。
- vendor 包升级不覆盖 customer 扩展层。
