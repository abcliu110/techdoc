## 1. 追溯口径

```text
金蝶公开证据 / 自研产品决策
→ 用户角色和用户故事
→ 功能需求 / 非功能需求
→ 验收断言
→ 功能设计矩阵
→ 测试矩阵
→ 开发任务
```

状态口径：

| 状态 | 含义 |
|---|---|
| baseline | 已进入需求基线 |
| strict-required | L3 加严门槛下，后续阶段必须矩阵化承接 |
| review-needed | 需要人工确认范围或风险 |
| planned | 后续里程碑，不阻断首版 P0 闭环 |
