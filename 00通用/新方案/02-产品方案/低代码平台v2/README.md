# 低代码平台 V2 设计器

本目录是依据《AI 竞品证据驱动的新产品原型统一工程方法》生成的独立交付包，目标是验证企业级低代码设计器的产品模型、工作台信息架构和关键交互。

## 当前交付

- `prototype-method/`：运行清单、竞品证据、任务模型、能力树、优势命题、需求、决策和追踪矩阵。
- `design/`：产品设计、信息架构、流程、状态模型、视觉合同和设计 Token。
- `prototype/`：无外部运行依赖的可执行 Web 原型。
- `tests/`：状态逻辑与静态原型合同测试。
- `reports/`：浏览器验证、截图、对标结论和独立评审。

## 运行原型

```powershell
cd prototype
python -m http.server 4173 --bind 127.0.0.1
```

浏览器打开 `http://127.0.0.1:4173/`。

## 验证

```powershell
node --test tests/*.test.mjs
```

## 真实性边界

- 原型展示的是可验证交互和产品合同，不连接生产数据库、权限中心或发布服务。
- 金蝶云·苍穹是主标杆；Mendix、OutSystems、Power Apps、Retool、Appian 是重点研究对象。
- 没有官方运行证据支持的能力保持 `candidate` 或 `UNKNOWN`，不声明全面领先。
- 当前结果不能替代真实用户研究、生产性能测试、安全评审或正式发布审批。

## 验证状态

- 项目级验证：Node 测试 9/9；1280x800、1440x900、1920x1080 几何检查通过；核心浏览器演示路径通过；控制台和页面错误为 0。
- 方法论正式运行：`BLOCKED`。阶段 7 缺认证几何矩阵和有效独立评审，阶段 9 缺六产品同条件任务数据，阶段 10 缺五席独立委员会和签名派生器。
- 这不影响原型本地试用，但禁止据此声明全面领先、生产可用或 `AUTONOMOUS_ACCEPTED`。
