# 低代码平台 V3 企业级表单设计器

本目录是依据《AI 竞品证据驱动的新产品原型统一工程方法》生成的独立交付包，目标是验证企业级低代码设计器的产品模型、工作台信息架构和关键交互。

## 当前交付

- `prototype-method/`：运行清单、竞品证据、任务模型、能力树、优势命题、需求、决策和追踪矩阵。
- `design/`：产品设计、信息架构、流程、状态模型、视觉合同和设计 Token。
- `prototype/`：无外部运行依赖的可执行 Web 原型。
- `tests/`：状态逻辑、静态原型合同、递归几何审计和浏览器故障夹具。
- `reports/`：浏览器验证、截图、对标结论和独立评审。

## 已实现的核心闭环

- 106 个组件协议清单、10 个分类、搜索、别名、设备与成熟度门禁。
- 组件拖入 Schema，节点同级排序和跨容器移动，失败事务不修改 Schema。
- 鼠标 `before / inside / after` 落点、键盘添加及上移/下移/移入/移出。
- 子单据体、树形子单据体和子卡片分录的父实体、子实体、主外键关系约束。
- Canvas、大纲、属性和面包屑共享选择状态；标题修改进入撤销/重做历史。
- 桌面、平板、手机画布和新增/修改/查看/审核业务状态。
- 版本化本地草稿恢复、运行预览、规则/权限检查和发布阻断演示。

## 企业 UI 质量门禁

本项目不再用“页面外框没有溢出”代替完整布局验收。质量门禁同时使用两棵树：Schema 树负责组件身份和业务归属，渲染 DOM 树负责真实矩形、逐层裁剪、滚动、文本行数与命中测试。检查会递归到任意嵌套深度；任何后代出现 P0/P1，所属组件及其祖先都不能通过。

- 可执行合同：`design/visual-contract.json`。
- 人类可读合同：`design/visual-contract.yaml`。
- 几何策略：`prototype-method/tests/geometry-policy.yaml`。
- 纯审计引擎：`prototype/geometry-audit.mjs`。
- 浏览器监听器：`prototype/quality-gate-runtime.mjs`，在 DOM、滚动和视口变化后的稳定帧重新审计。
- 故障夹具：`tests/fixtures/geometry.html` 和 `tests/fixtures/geometry-snapshots.mjs`。

门禁覆盖七个基准视口，并自动从 `prototype/styles.css` 提取每个宽度断点，增加断点前 1px、断点本身和断点后 1px。必须覆盖设计、预览、桌面/平板/手机、预览对话框、发布对话框，以及高风险滚动容器的起点/中点/终点。

最终状态按证据失败关闭：

- `FAIL`：任一机械、交互或视觉证据失败，或仍存在 P0/P1 finding。
- `BLOCKED`：没有已知失败，但证据缺失、报告过期、遍历覆盖不完整，或报告视口与请求视口不一致。
- `PASS`：机械、交互和视觉三类必需证据均为 PASS。

浏览器运行时把最新机械结果写入隐藏的 `[data-testid="quality-gate-json"]` 输出。自动化读取其 DOM `value` 属性前，必须等待报告中的 `viewport` 与目标视口完全一致；旧报告和旧截图不得作为当前通过证据。

## 真实性边界

- 原型展示的是可验证交互和产品合同，不连接生产数据库、权限中心或发布服务。
- 106 表示组件协议目录覆盖；当前多数通用组件使用协议驱动占位渲染，不等于 106 个生产级专用 Renderer。规划中组件保持禁用。
- 子单据体自动生成的实体 ID 与 `id / parentId` 主外键用于验证关系合同，正式产品必须由业务对象建模器配置。
- 金蝶云·苍穹是主标杆；Mendix、OutSystems、Power Apps、Retool、Appian 是重点研究对象。
- 没有官方运行证据支持的能力保持 `candidate` 或 `UNKNOWN`，不声明全面领先。
- 当前结果不能替代真实用户研究、生产性能测试、安全评审或正式发布审批。

## 验证状态

- 历史的 3 个桌面截图和单个手机截图只作为回归参考，不再等同于当前门禁通过。
- 当前代码的正式结论必须读取本轮生成的 `reports/quality-gate-report.json` 和 `reports/quality-gate-report.md`；报告缺失或未覆盖完整矩阵时状态为 `BLOCKED`。
- 方法论证据仍不足以支持“全面领先”或“生产可用”结论；本交付定位为高保真、可运行的产品原型。
