# 储值套餐-需求分析文档（完整版V2）

## 术语表

| 术语 | 含义 | 来源 |
|------|------|------|
| 储值套餐 | 商户为用户设定的储值充值方案，包含金额档位、赠送规则、适用范围等 | 📸 |
| 档位 | 储值金额的固定等级，每个档位有独立的储值金额、赠送金额、积分、礼品等配置 | 📸 |
| 可用周期 | 储值套餐生效的时间周期规则（按日/按周/按月） | 📸 |
| 可用时段 | 储值套餐在一天中可使用的时间段 | 📸 |
| 排除日期 | 储值套餐不可使用的特定日期 | 📸 |
| 储值协议 | 约定商家与用户之间储值权益和义务的协议文本 | 📸 |
| 发放规则 | 赠送礼品的发放方式（一次性/周期性） | 📸 |
| 可用金额限制 | 储值后金额的使用限制（不限制/当日限制/当餐限制） | 📸 |
| 券包 | 包含多张优惠券的组合包 | 📸 |
| 等级延期 | 储值后对用户已有会员等级有效期的延长 | 📸 |
| 单人购买总次数 | 单个用户累计可购买该套餐的总次数限制 | 📸 |
| 单人周期购买次数 | 单个用户在每日/每周/每月周期内可购买该套餐的次数限制 | 📸 |
| 售卖总量 | 该储值套餐的总售卖份数限制 | 📸 |

# 一、业务概述

## 1.1 系统定位

- **系统**：全来店商户中心 📸
- **子系统**：会员中心 > 储值活动 📸
- **页面**：新增储值套餐 📸
- **操作者**：商户管理员 🤖 AI推断
- **受益者**：商户（营销工具）、用户（储值优惠） 🤖 AI推断

## 1.2 表单形态

- **交互模式**：单页长表单，左侧步骤锚点导航（基础信息 / 储值规则 / 购买规则） 📸
- **保存机制**：顶部「保存」按钮一次性提交所有区块 👤 用户确认
- **新增 vs 编辑**：当前为新增状态，面包屑显示「新增储值套餐」 📸
- **导航按钮**：「取消」「上一步」「保存」 📸

# 二、页面结构

## 2.1 区块分区

| 区块编号 | 区块名称 | 核心功能 | UI位置 |
|----------|----------|----------|--------|
| A | 步骤导航 | 锚点导航：基础信息、储值规则、购买规则 | 左侧 📸 |
| B | 活动基础信息 | 品牌、分类、名称、日期、周期、时段、排除日期、协议、说明 | 主区域上部 📸 |
| C | 储值推荐规则 | 储值规则（固定金额档位）、档位配置（金额/赠送/礼品/等级）、金额限制 | 主区域中部 📸 |
| D | 购买规则 | 适用渠道、小程序、组织、用户、推荐、购买次数限制、售卖总量 | 主区域下部 📸 |
| E | 底部操作 | 法律声明确认 + 操作按钮 | 底部 📸 |

## 2.2 信息流向

```
品牌选择(B) → 活动分类过滤(B)
适用渠道(D) → 适用小程序显隐(D)
会员方案(C-表格) → 会员等级过滤(C-表格)
单人周期购买次数=限制(D) → 周期Tab联动提示文案默认值(D)
```

## 2.3 负空间

- 未见「活动状态」字段 → 新增时默认为草稿/未发布 [🤖 AI推断]
- 未见「创建人」「创建时间」字段 → 系统自动记录 [🤖 AI推断]
- 未见「排序」字段 → 可能在列表页排序 [🤖 AI推断]
- 未见「编辑」状态的差异 → 未截图编辑页面 [⚠️ 假设]
- 「推荐」切换后无额外配置 → 纯布尔标记 📸

# 三、数据模型

## 3.1 数据容器识别

```
[M] 主实体：「新增储值套餐」→ stored_value_package
  [J] 关联1：「品牌」多选标签 → package_brand（N:N）
  [D] 明细1：「可用周期」日期选择 → package_cycle_day（1:N）
  [D] 明细2：「可用时段」时间范围 → package_time_slot（1:N）
  [D] 明细3：「排除日期」日期标签 → package_exclude_date（1:N）
  [D] 明细4：「档位」可重复卡片（最多5档）→ package_tier（1:N）
    [P] 子实体4.1：「加入礼品」弹窗 → tier_gift（1:N）
    [P] 子实体4.2：「加入券包」弹窗 → tier_coupon_pack（1:N）
      [G] 嵌套表格4.2.1：「礼品核销时计入实收」→ tier_pack_gift_revenue（1:N）
    [G] 可编辑表格4.3：「升级到指定会员等级」→ tier_upgrade_level（1:N）
  [J] 关联2：「适用渠道」多选Checkbox → package_channel（N:N）
  [J] 关联3：「适用组织-指定门店」→ package_store（N:N） 📸
  [J] 关联4：「适用用户-会员方案」→ package_member_plan（N:N） 📸
  [J] 关联5：「适用用户-会员等级」→ package_member_level（N:N） 📸

[REF] 引用实体（不建模，仅存外键ID）：
  - activity_category（活动分类）← 「活动分类」下拉框
  - stored_value_agreement（储值协议）← 「请选择协议」下拉框
  - mini_program（小程序）← 「适用小程序」下拉框
  - gift（礼品/优惠券）← 「礼品名称」下拉框
  - coupon_pack（券包）← 「券包名称」下拉框
  - member_plan（会员方案）← 「会员方案」下拉框 / 「选择会员方案」弹窗
  - member_level（会员等级）← 「升级到的会员等级」下拉框 / 「选择会员等级」弹窗
  - store（门店）← 「选择组织」弹窗

[REF-内联创建] 活动分类（activity_category）：
  - 可通过「+ 新增分类」弹窗内联创建
  - 字段：分类名称、品牌（多选）、备注
    [J] 关联：category_brand（N:N）

[REF-跨模块] 储值协议（stored_value_agreement）：
  - 可通过「+ 添加储值协议」跳转独立页面创建
  - 字段：协议名称、协议类型（枚举）、协议内容（富文本）
```

## 3.2 实体层级

```
stored_value_package [M] (PK: id)
├── package_brand [J] (PK: id, FK: package_id, brand_id) — N:N — UI:「品牌」标签
├── package_cycle_day [D] (PK: id, FK: package_id) — 1:N — UI:「可用周期」按钮
├── package_time_slot [D] (PK: id, FK: package_id) — 1:N — UI:「可用时段」时间范围
├── package_exclude_date [D] (PK: id, FK: package_id) — 1:N — UI:「排除日期」标签
├── package_tier [D] (PK: id, FK: package_id) — 1:N(max5) — UI:「档位」卡片
│   ├── tier_gift [P] (PK: id, FK: tier_id) — 1:N — UI:「+ 加入礼品」弹窗
│   ├── tier_coupon_pack [P] (PK: id, FK: tier_id) — 1:N — UI:「+ 加入券包」弹窗
│   │   └── tier_pack_gift_revenue [G] (PK: id, FK: pack_id) — 1:N — UI: 弹窗内表格
│   └── tier_upgrade_level [G] (PK: id, FK: tier_id) — 1:N — UI:「升级到指定会员等级」表格
├── package_channel [J] (PK: id, FK: package_id, channel_type) — N:N — UI:「适用渠道」Checkbox
├── package_store [J] (PK: id, FK: package_id, store_id) — N:N — UI:「选择组织」弹窗 📸
├── package_member_plan [J] (PK: id, FK: package_id, member_plan_id) — N:N — UI:「选择会员方案」弹窗 📸
└── package_member_level [J] (PK: id, FK: package_id, member_level_id) — N:N — UI:「选择会员等级」弹窗 📸
```

## 4.1 RULE穷举

| 编号 | 条件 | 效果 | 默认 | 来源 |
|------|------|------|------|------|
| RULE-1 | WHEN date_type=「永久有效」 | THEN 隐藏日期范围选择器 | ELSE 显示开始日期+结束日期 | 📸 |
| RULE-2 | WHEN cycle_type=「按日」 | THEN 不显示子选项（每天可用） | — | 📸 |
| RULE-2 | WHEN cycle_type=「按周」 | THEN 显示周一~周日多选按钮 | — | 📸 |
| RULE-2 | WHEN cycle_type=「按月」 | THEN 显示1号~31号多选按钮 | — | 📸 |
| RULE-3 | WHEN time_slot_type=「全天」 | THEN 隐藏时间段列表 | ELSE 显示可重复时间范围（+添加/-删除） | 📸 |
| RULE-4 | WHEN agreement_enabled=开启 | THEN 显示协议下拉框+添加储值协议按钮 | ELSE 隐藏 | 📸 |
| RULE-5 | WHEN「赠送礼品」勾选「优惠券」 | THEN 显示「+ 加入礼品」按钮+「发放规则」 | — | 📸 |
| RULE-6 | WHEN validity_type=「相对有效期」 | THEN 显示「有效天数」+「生效类型」+「何时生效」 | — | 📸 |
| RULE-6 | WHEN validity_type=「固定有效期」 | THEN 显示「有效期」日期范围，隐藏生效类型/何时生效 | — | 📸 |

# 七、风险

## 7.1 并发

| 风险 | 说明 | 建议 |
|------|------|------|
