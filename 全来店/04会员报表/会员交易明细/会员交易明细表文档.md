# 会员交易明细表

## 1. 报表概览

**会员交易明细表**是记录会员所有交易行为的详细清单，包括储值、消费、调账等各类交易记录，位于全来店商户中心的会员报表模块中。

## 2. 功能区域

### 2.1 筛选条件
- **储值范围**：下拉选择框，可选择批量储值等选项
- **营业日期**：日期范围选择器，可选择交易发生的时间范围
- **查询**：绿色按钮，执行查询操作
- **高级筛选**：灰色按钮，打开高级筛选选项
- **重置**：灰色按钮，重置所有筛选条件
- **说明**：右上角按钮，查看报表说明
- **自定义列表**：右上角按钮，自定义显示字段
- **全屏**：右上角按钮，全屏显示报表

## 3. 字段说明

### 3.1 基本信息字段
| 字段 | 描述 |
|------|------|
| **序号** | 交易记录的序号 |
| **会员姓名** | 会员的姓名 |
| **会员卡号** | 会员的卡号 |
| **手机号** | 会员的手机号码（部分隐藏） |
| **会员方案** | 会员所属的会员卡类型 |
| **会员等级** | 会员的等级 |
| **交易类型** | 交易的类型（储值、消费、调账、退款等） |
| **收款方式** | 交易的收款方式（人民币、微信扫码等） |
| **交易渠道** | 交易发生的渠道（门店pos、商户中心、微信小程序等） |
| **交易时间** | 交易发生的具体时间 |

### 3.2 财务信息字段
| 字段 | 描述 |
|------|------|
| **交易后本金** | 交易后会员卡内的本金余额 |
| **交易后赠金** | 交易后会员卡内的赠金余额 |
| **交易后积分** | 交易后会员的积分余额 |
| **操作人** | 执行该交易的操作人员 |
| **备注** | 交易的备注信息 |
| **开票状态** | 交易的开票状态（未开票等） |
| **开票备注** | 开票相关的备注信息 |
| **手续费** | 交易产生的手续费 |

### 5.5 金额字段关联分析

#### 5.5.1 完整金额字段定义

| 字段 | 定义 | 数据类型 |
|------|------|---------|
| **交易前本金** | 交易开始前会员卡内的本金余额 | 数值型（元） |
| **交易前赠金** | 交易开始前会员卡内的赠金余额 | 数值型（元） |
| **交易前积分** | 交易开始前会员的积分余额 | 数值型（分） |
| **交易金额** | 交易的总金额 | 数值型（元） |
| **交易本金** | 交易中使用或增加的本金金额 | 数值型（元） |
| **交易赠金** | 交易中使用或增加的赠金金额 | 数值型（元） |
| **交易积分** | 交易中产生的积分变动 | 数值型（分） |
| **退款扣除金额** | 退款时扣除的金额 | 数值型（元） |
| **交易后本金** | 交易完成后会员卡内的本金余额 | 数值型（元） |
| **交易后赠金** | 交易完成后会员卡内的赠金余额 | 数值型（元） |
| **交易后积分** | 交易完成后会员的积分余额 | 数值型（分） |
| **手续费** | 交易产生的手续费 | 数值型（元） |

#### 5.5.2 核心金额字段关联关系

1. **交易前后本金关系**：
   ```
   交易后本金 = 交易前本金 + 交易本金
   ```
   - 储值交易：交易本金为正数（增加）
   - 消费交易：交易本金为负数（减少）
   - 调账交易：交易本金可正可负

2. **交易前后赠金关系**：
   ```
   交易后赠金 = 交易前赠金 + 交易赠金
   ```
   - 储值交易：交易赠金为正数（增加）
   - 消费交易：交易赠金为负数（减少）
   - 调账交易：交易赠金可正可负

3. **交易前后积分关系**：
   ```
   交易后积分 = 交易前积分 + 交易积分
   ```
   - 储值/消费：交易积分为正数（增加）
   - 退款：交易积分为负数（减少）

4. **交易金额与交易本金/赠金关系**：
   ```
   交易金额 = |交易本金| + |交易赠金|
   ```
   - 对于储值交易，交易金额等于交易本金（通常赠金单独计算）
   - 对于消费交易，交易金额等于交易本金和交易赠金的总和

#### 5.5.3 不同交易类型的金额关联

| 交易类型 | 交易本金 | 交易赠金 | 交易积分 | 交易金额 |
|---------|---------|---------|---------|---------|
| **储值** | 正数（储值金额） | 正数（赠送金额） | 正数（获得积分） | 储值金额 + 赠送金额 |
| **消费** | 负数（使用本金） | 负数（使用赠金） | 正数（获得积分） | 消费总金额 |
| **储值调账** | 可正可负 | 可正可负 | 可正可负 | 0 |
| **会员余额退款** | 负数（退款本金） | 负数（退款赠金） | 负数（扣除积分） | 退款总金额 |
| **消费赠送积分** | 0 | 0 | 正数（赠送积分） | 0 |

## 6. 金额字段公式

### 6.1 核心金额字段公式

#### 6.1.1 本金余额公式
```
交易后本金 = 交易前本金 + 交易本金
```
- **说明**：交易本金为正时表示增加（如储值），为负时表示减少（如消费）

#### 6.1.2 赠金余额公式
```
交易后赠金 = 交易前赠金 + 交易赠金
```
- **说明**：交易赠金为正时表示增加（如储值赠送），为负时表示减少（如消费使用）

#### 6.1.3 积分余额公式
```
交易后积分 = 交易前积分 + 交易积分
```
- **说明**：交易积分为正时表示增加（如消费获得），为负时表示减少（如退款扣除）

#### 6.1.4 交易金额公式
```
交易金额 = |交易本金| + |交易赠金|
```
- **说明**：交易金额为交易中涉及的总金额，取交易本金和交易赠金的绝对值之和

#### 6.1.5 退款扣除金额公式
```
退款扣除金额 = 交易金额（当交易类型为退款时）
```
- **说明**：仅在退款交易中使用，等于退款的总金额

### 6.2 辅助公式

#### 6.2.1 卡内总余额公式
```
卡内总余额 = 交易后本金 + 交易后赠金
```

#### 6.2.2 交易前总余额公式
```
交易前总余额 = 交易前本金 + 交易前赠金
```

#### 6.2.3 余额变动公式
```
余额变动 = 交易后总余额 - 交易前总余额 = 交易本金 + 交易赠金
```

#### 6.2.4 手续费公式（如有）
```
手续费 = 交易金额 × 手续费率（根据业务规则）
```

### 6.3 公式验证示例

#### 6.3.1 示例1：储值交易
- 交易前本金 = 0.00
- 交易前赠金 = 0.00
- 储值金额 = 10000.00
- 赠送金额 = 10000.00
- 储值获得积分 = 1000.00

**计算**：
- 交易本金 = 10000.00
- 交易赠金 = 10000.00
- 交易积分 = 1000.00
- 交易金额 = 10000.00 + 10000.00 = 20000.00
- 交易后本金 = 0.00 + 10000.00 = 10000.00
- 交易后赠金 = 0.00 + 10000.00 = 10000.00
- 交易后积分 = 0.00 + 1000.00 = 1000.00

#### 6.3.2 示例2：消费交易
- 交易前本金 = 107.70
- 交易前赠金 = 0.97
- 消费金额 = 10.00

**计算**：
- 交易本金 = -9.91（优先扣除本金）
- 交易赠金 = -0.09（本金不足部分）
- 交易积分 = 0
- 交易金额 = 9.91 + 0.09 = 10.00
- 交易后本金 = 107.70 - 9.91 = 97.79
- 交易后赠金 = 0.97 - 0.09 = 0.88
- 交易后积分 = 1.00 + 0 = 1.00

#### 6.3.3 示例3：储值调账
- 交易前本金 = 100.00
- 交易前赠金 = 10.00
- 交易前积分 = 32.00
- 调账积分 = 10.00

**计算**：
- 交易本金 = 0
- 交易赠金 = 0
- 交易积分 = 10.00
- 交易金额 = 0
- 交易后本金 = 100.00 + 0 = 100.00
- 交易后赠金 = 10.00 + 0 = 10.00
- 交易后积分 = 32.00 + 10.00 = 42.00

## 7. 报表用途

### 7.1 交易明细查询
- 快速查询会员的所有交易记录
- 按时间、交易类型、会员等维度筛选交易
- 查看每笔交易的详细信息和状态

### 7.2 财务对账
- 核对储值金额与实际收款
- 监控赠金和积分的发放情况
- 检查手续费的收取是否正确

### 7.3 会员行为分析
- 分析会员的消费习惯和偏好
- 识别高价值会员和活跃会员
- 评估会员等级体系的合理性

### 7.4 异常交易监控
- 监控大额交易和异常调账
- 检查交易渠道的异常活动
- 识别潜在的欺诈行为

### 7.5 运营决策支持
- 基于交易数据制定促销策略
- 优化会员等级和权益设置
- 改进交易流程和用户体验

## 8. 操作指南

### 8.1 筛选交易记录
1. 选择储值范围（如批量储值）
2. 选择营业日期范围
3. 点击【查询】按钮执行筛选
4. 如需更详细的筛选条件，点击【高级筛选】

### 8.2 自定义显示字段
1. 点击【自定义列表】按钮
2. 选择需要显示的字段
3. 调整字段的显示顺序
4. 保存设置

### 8.3 导出交易数据
1. 筛选出需要导出的交易记录
2. 点击【导出】按钮（如果有）
3. 选择导出格式（如Excel、CSV等）
4. 保存导出的文件

### 8.4 查看交易详情
1. 找到需要查看的交易记录
2. 点击交易记录的相关链接（如果有）
3. 查看交易的详细信息和关联数据

## 9. 注意事项

1. **数据时效性**：报表数据为实时数据，反映当前系统中的交易记录
2. **数据安全**：手机号等敏感信息已做隐藏处理，保护会员隐私
3. **字段说明**：部分字段可能为空，如手机号、备注等，属于正常情况
4. **开票状态**：目前所有交易的开票状态均为"未开票"，后续可根据实际情况更新
5. **手续费**：目前所有交易的手续费均为0.00，后续可根据业务需求设置
6. **数据导出**：导出的数据可用于离线分析和存档，但请注意数据安全
7. **高级筛选**：使用高级筛选功能可以设置更复杂的筛选条件，提高查询效率
8. **性能优化**：查询大量交易记录时，可能需要一定时间加载，请耐心等待

## 10. 总结

会员交易明细表是会员管理系统中的重要报表，通过详细记录每笔交易的信息，为商家提供了全面的交易数据视图。通过分析各个字段之间的关联关系，商家可以深入了解会员的消费行为、财务状况和运营效果，为会员营销决策提供科学依据。

合理利用会员交易明细表，可以帮助商家：
- 精准掌握会员交易动态
- 有效管理会员财务信息
- 科学制定会员运营策略
- 及时发现和解决交易异常
- 持续优化会员服务体验

最终实现会员满意度和忠诚度的提升，促进业务的持续增长。
