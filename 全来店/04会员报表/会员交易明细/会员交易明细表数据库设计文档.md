# 会员交易明细表数据库设计文档

## 1. 数据库表结构

### 1.1 会员交易明细表 (`member_transaction_detail`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY AUTO_INCREMENT` | 交易记录ID |
| `serial_number` | `INT` | `NOT NULL` | 交易记录序号 |
| `member_name` | `VARCHAR(50)` | `DEFAULT NULL` | 会员姓名 |
| `member_card_number` | `VARCHAR(50)` | `NOT NULL` | 会员卡号 |
| `phone_number` | `VARCHAR(20)` | `DEFAULT NULL` | 手机号（部分隐藏） |
| `member_plan` | `VARCHAR(50)` | `NOT NULL` | 会员方案 |
| `member_level` | `VARCHAR(50)` | `NOT NULL` | 会员等级 |
| `transaction_type` | `VARCHAR(50)` | `NOT NULL` | 交易类型（储值、消费、调账、退款等） |
| `payment_method` | `VARCHAR(50)` | `DEFAULT NULL` | 收款方式（人民币、微信扫码等） |
| `transaction_channel` | `VARCHAR(50)` | `DEFAULT NULL` | 交易渠道（门店pos、商户中心、微信小程序等） |
| `transaction_time` | `DATETIME` | `NOT NULL` | 交易发生的具体时间 |
| `before_transaction_principal` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易前本金 |
| `before_transaction_bonus` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易前赠金 |
| `before_transaction_points` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易前积分 |
| `transaction_amount` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易金额 |
| `transaction_principal` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易本金 |
| `transaction_bonus` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易赠金 |
| `transaction_points` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易积分 |
| `refund_deduction_amount` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 退款扣除金额 |
| `after_transaction_principal` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易后本金 |
| `after_transaction_bonus` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易后赠金 |
| `after_transaction_points` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 交易后积分 |
| `operator` | `VARCHAR(50)` | `DEFAULT NULL` | 操作人 |
| `remark` | `VARCHAR(255)` | `DEFAULT NULL` | 交易备注 |
| `invoice_status` | `VARCHAR(50)` | `DEFAULT NULL` | 开票状态（未开票等） |
| `invoice_remark` | `VARCHAR(255)` | `DEFAULT NULL` | 开票备注 |
| `service_fee` | `DECIMAL(18,2)` | `NOT NULL DEFAULT 0.00` | 手续费 |
| `created_at` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 记录创建时间 |
| `updated_at` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 记录更新时间 |

## 2. 索引设计

### 2.1 主键索引
- `PRIMARY KEY` (`id`): 唯一标识每条交易记录

### 2.2 普通索引
- `INDEX` (`member_card_number`): 加速按会员卡号查询
- `INDEX` (`transaction_time`): 加速按交易时间查询
- `INDEX` (`transaction_type`): 加速按交易类型查询
- `INDEX` (`member_level`): 加速按会员等级查询
- `INDEX` (`transaction_channel`): 加速按交易渠道查询

### 2.3 联合索引
- `INDEX` (`member_card_number`, `transaction_time`): 加速按会员卡号和交易时间范围查询
- `INDEX` (`transaction_type`, `transaction_time`): 加速按交易类型和交易时间范围查询

## 3. 表结构创建SQL

```sql
CREATE TABLE `member_transaction_detail` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `serial_number` INT NOT NULL,
  `member_name` VARCHAR(50) DEFAULT NULL,
  `member_card_number` VARCHAR(50) NOT NULL,
  `phone_number` VARCHAR(20) DEFAULT NULL,
  `member_plan` VARCHAR(50) NOT NULL,
  `member_level` VARCHAR(50) NOT NULL,
  `transaction_type` VARCHAR(50) NOT NULL,
  `payment_method` VARCHAR(50) DEFAULT NULL,
  `transaction_channel` VARCHAR(50) DEFAULT NULL,
  `transaction_time` DATETIME NOT NULL,
  `before_transaction_principal` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `before_transaction_bonus` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `before_transaction_points` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `transaction_amount` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `transaction_principal` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `transaction_bonus` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `transaction_points` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `refund_deduction_amount` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `after_transaction_principal` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `after_transaction_bonus` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `after_transaction_points` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `operator` VARCHAR(50) DEFAULT NULL,
  `remark` VARCHAR(255) DEFAULT NULL,
  `invoice_status` VARCHAR(50) DEFAULT NULL,
  `invoice_remark` VARCHAR(255) DEFAULT NULL,
  `service_fee` DECIMAL(18,2) NOT NULL DEFAULT 0.00,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  INDEX `idx_member_card_number` (`member_card_number`),
  INDEX `idx_transaction_time` (`transaction_time`),
  INDEX `idx_transaction_type` (`transaction_type`),
  INDEX `idx_member_level` (`member_level`),
  INDEX `idx_transaction_channel` (`transaction_channel`),
  INDEX `idx_member_card_time` (`member_card_number`, `transaction_time`),
  INDEX `idx_type_time` (`transaction_type`, `transaction_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 4. 数据类型说明

### 4.1 数值类型
- `BIGINT`: 用于主键ID，支持更大的数值范围
- `INT`: 用于序号等整数类型
- `DECIMAL(18,2)`: 用于金额和积分字段，确保精度为2位小数

### 4.2 字符串类型
- `VARCHAR(50)`: 用于一般文本字段，如会员姓名、会员卡号等
- `VARCHAR(20)`: 用于手机号等固定长度的字段
- `VARCHAR(255)`: 用于备注等可能较长的文本字段

### 4.3 日期时间类型
- `DATETIME`: 用于交易时间、创建时间和更新时间

## 5. 数据完整性约束

### 5.1 非空约束
- 确保核心字段（如会员卡号、交易类型、交易时间等）不为空

### 5.2 默认值约束
- 为数值型字段设置默认值0.00，确保计算的一致性
- 为日期时间字段设置默认值，自动记录创建和更新时间

### 5.3 索引约束
- 通过索引提高查询性能，特别是针对常用的查询条件

## 6. 表关系

### 6.1 与会员表的关系
- 通过`member_card_number`字段与会员表关联
- 建议会员表中也应有对应的`member_card_number`字段作为唯一标识

### 6.2 与其他表的关系
- 可根据业务需求与会员方案表、会员等级表等关联
- 可通过`transaction_type`、`payment_method`等字段与相关配置表关联

## 7. 性能优化建议

### 7.1 分区策略
- 对于交易记录较多的系统，建议按`transaction_time`字段进行分区
- 可采用按月或按年分区，提高查询性能

### 7.2 数据归档
- 对于历史交易数据，建议定期归档到历史表
- 可设置归档策略，如将超过1年的交易数据归档

### 7.3 查询优化
- 合理使用索引，避免全表扫描
- 对于复杂查询，建议使用视图或存储过程

## 8. 安全考虑

### 8.1 数据加密
- 对于敏感信息（如手机号），建议进行加密存储
- 可使用MySQL的加密函数或应用层加密

### 8.2 访问控制
- 为不同角色设置不同的访问权限
- 限制对交易数据的修改权限，确保数据完整性

### 8.3 备份策略
- 定期备份交易数据，确保数据安全
- 可采用增量备份和全量备份相结合的方式

## 9. 总结

本数据库设计文档基于会员交易明细表的业务需求，设计了一个结构合理、性能优化的MySQL数据库表结构。通过合理的数据类型选择、索引设计和约束设置，确保了数据的完整性、一致性和查询性能。

该设计方案适用于全来店系统的会员交易记录管理，能够满足日常的交易明细查询、财务对账、会员行为分析等业务需求。同时，通过分区、归档等策略，为系统的长期稳定运行提供了保障。
