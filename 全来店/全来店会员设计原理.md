# 全来店会员设计原理

## 一、核心架构关系

### 整体逻辑
```
会员方案（容器）
    ↓ 包含
会员等级（权益层）
    ↓ 决定
会员积分（计算规则）
```

### 一句话总结
**会员方案定卡，等级定权益，积分按等级算，升降看成长值。**

---

## 二、三层架构详解

### 第一层：会员方案（顶层容器）

#### 定义
会员方案 = 一套完整的会员体系

#### 后台路径
会员中心 → 会员方案 → 新增会员方案

#### 包含内容
- 会员卡名称（会员身份标识）
- 卡样式设计
- 电子卡 / 实体卡类型
- 会员等级规则
- 会员积分规则
- 会员折扣规则
- 升级 / 降级规则

#### 关系说明
- 1 个会员方案 = 1 种会员卡类型
- 1 个会员方案下可设置多个会员等级
- 会员方案是容器，等级和积分都在它里面

---

### 第二层：会员等级（中层权益）

#### 定义
在会员方案内部设置的多个等级层次

#### 等级示例
- V1 普通会员
- V2 银卡会员
- V3 金卡会员
- V4 钻石会员

#### 每个等级独立配置
1. **等级名称**：自定义等级名称
2. **升级所需成长值门槛**：达到多少成长值可升级
3. **降级条件**：周期内成长值不足时降级
4. **该等级享受的折扣**：消费折扣比例
5. **该等级享受的积分倍率**：积分计算倍数

#### 升降级逻辑

##### 升级流程
```
会员消费
    ↓
增加成长值
    ↓
成长值达标
    ↓
自动升级到下一等级
```

##### 降级流程
```
到达降级周期（如自然年/滚动周年）
    ↓
系统重新核算会员成长值
    ↓
成长值不满足当前等级门槛
    ↓
自动降级
```

---

### 第三层：会员积分（底层计算）

#### 核心计算公式
```
实付金额 × 等级积分倍率 = 获得积分
```

#### 等级积分倍率示例
| 会员等级 | 积分倍率 | 消费100元获得积分 |
|---------|---------|------------------|
| V1 普通 | 1.0倍 | 100积分 |
| V2 银卡 | 1.2倍 | 120积分 |
| V3 金卡 | 1.5倍 | 150积分 |
| V4 钻石 | 2.0倍 | 200积分 |

#### 积分来源
1. **消费支付**（主要来源）
   - 按支付方式实收积分
   - 按商品实收积分
2. **签到奖励**（可选）
3. **活动奖励**（可选）
4. **开卡送礼**（可选）

#### 积分用途
1. **积分抵现**：按设定比例抵扣订单金额
2. **积分兑换**：兑换礼品 / 优惠券
3. **积分清零**：按规则定期清零

## 三、完整业务流程

### 商家端配置流程

#### 步骤1：创建会员方案
```
进入：会员中心 → 会员方案 → 新增会员方案
配置：
  - 会员卡名称
  - 卡样式
  - 电子/实体卡类型
```

#### 步骤2：设置会员等级
```
在会员方案内 → 新增/编辑会员等级
配置每个等级：
  - 等级名称（普通/银卡/金卡/钻石）
  - 升级所需成长值门槛
  - 降级周期与规则
  - 该等级会员折扣
  - 该等级积分倍率
```

#### 步骤3：配置积分规则
```
设置积分权益：
  - 积分获取规则（按支付方式/按商品）
  - 积分抵扣规则（抵扣比例、限制条件）
  - 积分清零规则（有效期、清零方式）
```

#### 步骤4：保存并生效
```
保存会员方案 → 会员体系生效
```

---

### 用户端消费流程

#### 完整流程图
```
1. 顾客开卡成为会员
    ↓
2. 进入默认等级（如V1普通会员）
    ↓
3. 顾客到店消费
    ↓
4. 收银端识别会员身份
    ↓
5. 系统读取会员当前等级
    ↓
6. 系统读取该等级对应的积分倍率
    ↓
7. 按公式计算积分：实付金额 × 等级积分倍率 = 本次获得积分
    ↓
8. 同时增加成长值
    ↓
9. 成长值达到下一等级门槛 → 自动升级
    ↓
10. 升级后，下次消费自动按新高等级计算积分与折扣
```

#### 详细说明

**第1步：会员开卡**
- 用户通过小程序、门店POS等渠道开卡
- 自动进入默认等级（通常为V1普通会员）

**第2步：消费识别**
- 收银端扫码或输入手机号识别会员
- 系统调取会员档案信息

**第3步：等级判定**
- 系统读取会员当前等级（V1/V2/V3/V4）
- 确定该等级对应的权益配置

**第4步：积分计算**
- 获取该等级的积分倍率
- 计算公式：实付金额 × 积分倍率 = 本次获得积分
- 积分实时到账

**第5步：成长值累计**
- 消费同时累计成长值
- 成长值用于等级升降判定

**第6步：自动升级**
- 成长值达到下一等级门槛
- 系统自动升级会员等级
- 发送升级通知

**第7步：享受新权益**
- 升级后立即享受新等级权益
- 更高的积分倍率
- 更大的消费折扣

## 四、核心数据流转

### 数据总览（会员中心首页）

#### 客户数据
- **客户总数**：系统中所有注册客户
- **会员数量**：已激活的会员数量
- **会员占比**：会员数量 / 客户总数

#### 储值数据
- **有储值会员数**：账户有储值余额的会员数量
- **累计储值总额**：历史累计充值金额
- **累计储值额**：当前所有会员储值余额总和
- **累计储值抵扣金额**：累计使用储值抵扣的金额
- **累计本年度储值金额**：本年度充值金额统计
- **沉默储值金额**：长期未使用的储值余额

#### 消费数据
- **有消费会员数**：有过消费记录的会员数量
- **消费会员占比**：有消费会员数 / 会员总数

## 五、会员权益体系

### 1. 积分权益

#### 积分获取规则
- **按支付方式实收积分**：根据实付金额计算积分
- **按商品实收积分**：根据商品维度配置积分规则
- **可用周期**：按日/按周/按月设置
- **可用时段**：全天/指定时段
- **订单场景限制**：堂食/外卖/自提/外带
- **订单渠道限制**：门店POS/小程序/自助大屏等

#### 积分抵扣规则
- **积分抵扣比例**：多少积分抵扣1元
- **起扣积分**：最低使用积分要求
- **抵扣上限**：单笔订单抵扣上限
- **商品限制**：指定商品参与/不参与
- **场景渠道限制**：限制使用场景和渠道

#### 积分清零规则
- **自发放日期清零**：从积分发放日期开始计算
- **年度统一清零**：当年发放的积分在指定日期统一清零
- **消息提醒**：清零前提醒用户

---

### 2. 储值权益

#### 储值套餐
- **套餐名称**：自定义套餐名称
- **储值金额**：用户需支付的金额
- **赠送金额**：额外赠送的储值金额
- **赠送积分**：额外赠送的积分
- **赠送礼品**：优惠券/券包
- **升级选项**：升级到指定会员等级、等级延期
- **有效期**：永久有效/固定日期/自创建日起限制

#### 储值分期送礼
三种储值规则：

**规则1：限定金额次日解冻**
- 储值后次日解冻全部金额
- 适合短期促销，资金快速回笼
- 当天金额可用比例可设置

**规则2：限定金额固定天数解冻**
- 分批次解冻，每次解冻固定金额
- 适合长期绑定用户，提高复购率
- 设置赠送次数和间隔天数

**规则3：首次消费后解冻剩余金额**
- 首次消费后解冻剩余全部金额
- 适合引导首次消费，提升转化率
- 首次可消费比例可设置

#### 智能储值
- **订单金额**：消费金额档位
- **储值金额**：对应储值金额
- **赠送比例**：自动计算赠送金额
- **赠送积分**：额外赠送积分
- **支持多档位**：可设置多个金额档位

---

### 3. 会员卡券
- **优惠券发放**：开卡送券、消费送券
- **券包管理**：组合多张优惠券
- **使用限制**：时间、场景、渠道限制

---

### 4. 会员营销
- **会员任务**：签到、消费、分享等任务
- **签到中奖**：每日签到抽奖活动
- **会员群体**：会员分群精准营销
- **会员消息**：消息推送通知

### 配置流程
```
创建【会员方案】
    ↓（1对1）

    ↓（1对多）

```

    ↓

## 八、设计原理总结

### 核心设计思想
1. **分层设计**：方案 → 等级 → 积分，层次清晰
2. **灵活配置**：每个等级独立配置权益
3. **自动化管理**：升降级自动判定，无需人工干预
4. **精准营销**：基于等级和标签的精准营销
5. **数据驱动**：完整的数据统计和分析体系

### 业务价值
1. **提升用户粘性**：通过等级体系绑定用户
2. **促进消费增长**：高等级享受更多权益，激励消费
3. **精细化运营**：基于会员数据的精准营销
4. **资金沉淀**：储值体系带来资金沉淀
5. **品牌忠诚度**：完整的会员体系提升品牌忠诚度

### 技术特点
1. **实时计算**：积分和成长值实时计算
2. **自动触发**：升降级自动触发
3. **多渠道支持**：支持门店POS、小程序、自助大屏等多渠道
4. **灵活配置**：丰富的配置项满足不同业务需求
5. **数据统计**：完整的数据统计和分析功能

## 九、最佳实践建议

### 等级设置建议
- 建议设置3-5个等级，不宜过多
- 等级门槛要合理，既有挑战性又可达成
- 等级权益要有明显差异，激励用户升级

### 积分倍率建议
- 普通会员：1.0倍（基准）
- 银卡会员：1.2-1.5倍
- 金卡会员：1.5-2.0倍
- 钻石会员：2.0-3.0倍

### 储值活动建议
- 新店开业：使用"限定金额次日解冻"快速回笼资金
- 成熟品牌：使用"限定金额固定天数解冻"长期绑定用户
- 线上平台：使用"首次消费后解冻剩余金额"引导首次消费

### 营销策略建议
- 定期推出储值活动，保持用户活跃
- 利用会员群体功能进行精准营销
- 设置会员任务，增加用户互动
- 关注沉默储值预警，激活沉默用户
