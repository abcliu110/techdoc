# 24-配置与特性开关细则

> 版本：v0.1（初稿）
> 地位：约束环境配置、租户配置和灰度开关，防止配置成为架构暗门。

## 1. 配置分层

规则 24-001：配置优先级固定为：系统默认 < 环境 < 租户 < 应用 < 对象 < 用户。
强制方式：配置解析单测。

规则 24-002：每个配置项必须声明 key、类型、默认值、单位、范围、敏感标记、是否可租户覆盖。
强制方式：配置 schema 校验。

规则 24-003：敏感配置不得进入普通日志、前端响应和导出包。
强制方式：敏感词扫描 + API 测试。

## 2. 特性开关生命周期

规则 24-010：特性开关必须有状态：`EXPERIMENT`、`GRAY`、`ON`、`REMOVING`、`REMOVED`。
强制方式：开关注册表校验。

规则 24-011：开关必须有 owner、创建原因、默认值、过期时间、移除任务卡。
强制方式：扫描无 owner/过期开关。

规则 24-012：禁止用特性开关实现永久租户特判；超过两个版本未移除必须转元模型/插件能力或删除。
强制方式：过期开关 CI 报警。

## 3. 灰度与回滚

规则 24-020：灰度维度只能是租户、应用、角色、用户，不得使用任意 SQL 条件。
强制方式：灰度规则 schema 校验。

规则 24-021：灰度必须绑定观测指标和回滚条件。
强制方式：发布 checklist。

规则 24-022：多实例配置刷新必须带版本号，实例不得静默使用未知版本。
强制方式：多实例集成测试。

## 4. 安全配置

规则 24-030：表达式函数、插件权限、数据源连接均必须默认最小权限。
强制方式：安全配置测试。

规则 24-031：密钥配置必须支持加密、脱敏展示和轮换。
强制方式：配置服务集成测试。

规则 24-032：break-glass 配置必须声明默认关闭、审批方式、最大有效期、授权范围上限和审计保留期；不得用配置关闭审计或字段脱敏。
强制方式：安全配置测试，对应 B5。

## 5. 部署模式与权限刷新

规则 24-040：SaaS 与私有化差异必须集中在 `deployment_mode`、配置 schema 和适配层，业务服务不得散落模式判断。
强制方式：代码扫描 + 双模式回归，对应 E4。

规则 24-041：权限缓存、组织树缓存、字段权限缓存必须有版本配置和最大刷新时限；首版最大刷新时限固定为 60 秒，默认 TTL 不超过 300 秒；用户禁用、调岗、角色撤销必须主动提升 `permVersion` 并广播失效，不得依赖自然过期。
强制方式：权限变更集成测试，对应 F7。

## 6. Redis 降级与分布式锁配置

规则 24-042：Redis 不可用时首版采用“只读降级 + 本地只读缓存 + DB 版本号轮询”的单一策略；写接口、发布、导入和权限变更必须拒绝，不得各模块自行选择回源或继续写。
强制方式：Redis 闪断集成测试，对应 A6。

规则 24-043：MetaGraph 版本号写 DB 成功但 Redis 更新失败时，必须由 DB 版本号轮询在最大 60 秒内收敛；收敛前写接口必须校验 metaVersion，避免旧视图提交。
强制方式：DB 已提交/Redis 未更新故障注入测试，对应 A6/C3。

规则 24-044：发布锁必须配置 watchdog 续期或 fencing token 二选一；本项目默认使用 fencing token，所有发布状态推进和 DDL 步骤都必须校验 token 单调递增。
强制方式：锁过期双持有者并发测试，对应 A6。

## 7. 统一调度配置

规则 24-050：平台调度统一通过调度注册表声明任务，禁止业务模块散落 `@Scheduled` 直接执行。注册项必须包含 taskCode、租户范围、时区语义、幂等键、补跑策略和资源配额。
强制方式：扫描业务包中裸 `@Scheduled` + 调度注册表单测，对应 C6。

规则 24-051：多实例部署时首版采用 DB 抢锁调度；调度锁必须带租户、taskCode、计划触发时间和 fencing token，避免重复执行。
强制方式：双实例调度防重测试，对应 C6。

规则 24-052：调度补跑语义必须按任务分类声明：outbox 投递、归档、清理必须补跑；缓存预热可跳过。所有补跑任务必须具备幂等键。
强制方式：实例宕机后补跑测试，对应 C6/F1。

规则 24-053：按日任务必须声明使用平台时区或租户时区；涉及业务编号、工作流超时和通知计划的任务默认使用租户时区。
强制方式：跨时区租户调度测试，对应 C6/D1。
