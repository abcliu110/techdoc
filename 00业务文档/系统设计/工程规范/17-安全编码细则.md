## 1. 动态 SQL

规则 17-001：表名、列名、别名、op、sort、join path、函数、分页、group by、having 必须逐项白名单。

规则 17-002：所有动态 SQL 只能通过统一 QueryBuilder/SqlAssembler。

规则 17-003：用户输入只能作为参数值，不得拼接进 SQL 结构。

规则 17-004：data_scope 必须在 filters AST 层合并为根 AND 子节点。

规则 17-005：分页 size、ids 数量、in 参数数量必须有限制。

## 2. 租户隔离

规则 17-010：动态数据访问层 tenantId 为空直接抛异常。

规则 17-011：异步、定时、MQ、规则任务必须显式携带 tenantId，不依赖裸 ThreadLocal。

规则 17-012：平台管理跨租户入口必须单独授权、审计、记录原因。

规则 17-013：双租户混测是核心链路必测项。

规则 17-014：运行态 link/user/org 写入禁止引用其他租户记录；校验必须走统一写入口和目标对象 AccessView，不得由前端传入已展开对象绕过。
强制方式：跨租户 link 写入安全测试，对应 B4。

## 3. 权限

规则 17-020：对象权限、字段权限、动作权限、数据范围使用唯一判定器。

规则 17-021：/meta 裁剪、list/get 返回裁剪、update 写校验必须调用同一判定器。

规则 17-022：权限缓存 key 必须包含用户、角色、组织树、数据范围、应用权限、字段权限版本。

规则 17-023：用户禁用、移出租户、调岗、角色撤销必须触发权限缓存失效。

规则 17-024：link 展开和 suggest 查询必须对目标对象重新执行 AccessView，不得复用源对象权限；无目标对象 read 权限时只能返回空摘要或 tombstone，不得泄露 title_field。
强制方式：目标对象无权限展开测试，对应 B3/B7/B10。

## 4. 表达式沙箱

规则 17-030：表达式禁反射、类加载、IO、线程、网络、系统属性。

规则 17-031：白名单函数不得查库或访问外部系统；需要数据由引擎预取注入。

规则 17-032：`$user/$record/$old/$env` 使用显式 DTO，字段白名单。

规则 17-033：表达式必须有超时和复杂度限制。

## 5. 导入包、Connector、附件

规则 17-040：导入包禁止携带密钥明文、token、运行时业务数据；必须校验版本、清单、大小、引用、表达式。

规则 17-041：Connector 禁止访问 localhost、私网段、云 metadata、保留地址；需防 DNS rebinding。

规则 17-042：Connector 必须有超时、限流、最大响应体、最大结果集、租户配额、脱敏日志。

规则 17-043：附件上传必须扩展名和 MIME 双校验，大小限制，存储 key 含租户隔离且不可由用户输入拼接。

规则 17-044：附件下载必须重新走权限判定，禁止暴露内部 bucket 或签名 URL 明文日志。

## 6. 密钥与日志

规则 17-050：DataSource 密钥加密存储，运行态最小范围解密。

规则 17-051：日志禁止输出密码、token、密钥、完整业务数据、SQL 参数明文。

规则 17-052：错误响应禁止泄露 SQL、堆栈、路径、内部类名。

## 7. 超级管理员、个人信息与渲染安全

规则 17-060：超级管理员不得默认绕过 tenant_id 隔离、字段脱敏、密钥不可见和审计写入；确需代客操作必须进入 break-glass 流程。
强制方式：普通管理员跨租户失败、break-glass 到期失效、密钥不可见安全测试，对应 B5。

规则 17-061：break-glass 必须记录审批人、操作者、被代理用户、租户、授权范围、理由、工单号、有效期、traceId、前后数据摘要；审计记录不得被同一权限主体修改或删除。
强制方式：审计不可篡改集成测试，对应 B5。

正例：

```java
breakGlassService.open(command.withReason("INC-1001").expiresAt(expiresAt));
```

反例：

```java
TenantContext.runAs("targetTenant", () -> dataService.listAll());
```

规则 17-062：个人信息必须分级标注，手机号、身份证、邮箱、地址、token、密钥、附件、审计 diff、导出包至少进入敏感字段注册表。
强制方式：敏感字段 schema 扫描 + API 脱敏测试，对应 B6。

规则 17-063：日志、审计、导出、前端错误上报不得保存超出用途的明文个人信息；保留期到期必须删除、匿名化或进入只读归档，并写明产品边界。
强制方式：日志敏感词扫描 + 保留期清理任务测试，对应 B6。

规则 17-064：富文本、页面 schema、错误消息、附件预览必须按 13 前端安全规范执行，后端不得把未净化 HTML 或可执行 schema 原样返回给 renderer。
强制方式：XSS 安全测试，对应 G4。

## 8. 认证、导入导出与通知出域

规则 17-070：浏览器会话接口必须启用 CSRF 防护，首版采用 SameSite=Lax/Strict + 自定义请求头双重校验；所有非 GET 写接口必须经过统一 CSRF filter。
强制方式：CSRF 攻击安全测试，对应 F9。

规则 17-071：token/会话生命周期必须声明签发、过期、续期和强制失效时限；用户禁用、移出租户、改密后旧 token 最大 60 秒内失效，并与 F7 权限缓存时限一致。
强制方式：禁用用户旧 token 测试，对应 F9/F7。

规则 17-072：登录防护必须包含失败次数锁定、验证码触发、会话固定防护和登录审计；复用既有身份系统时必须记录集成契约。
强制方式：认证安全测试，对应 F9。

规则 17-073：开放 API token 首版不支持；若后续支持，必须新增 token scope、限流、审计和密钥轮换设计，禁止复用浏览器 cookie 会话。
强制方式：API 边界扫描 + 产品边界评审，对应 F9。

规则 17-080：业务数据导入必须复用统一写入口，禁止通过 mapper/批量 SQL 直写动态表绕过类型转换、权限、租户注入、审计和规则。
强制方式：导入安全测试 + mapper 入口扫描，对应 F10。

规则 17-081：导出必须走统一权限判定器和 data_scope 注入；导出文件不得包含无权限字段、密钥明文、token 或未脱敏个人信息。
强制方式：越权字段导出裁剪测试 + 敏感字段扫描，对应 F10/B6。

规则 17-090：通知模板可引用字段必须通过白名单，敏感等级高于普通业务字段的内容默认禁止进入通知；确需进入必须声明脱敏策略。
强制方式：敏感字段模板测试，对应 F11/B6。

规则 17-091：通知发送前必须校验收件人对记录的 read 权限；若产品选择“通知视为授权披露”，必须写入产品边界并记录审计，本规范默认不采用该豁免。
强制方式：无权限收件人测试，对应 F11/B3。

规则 17-092：通知通道必须复用 F2 outbox/重试/死信机制，不得另建不可追踪重试；每次发送必须关联业务记录、模板版本、traceId 和收件人摘要。
强制方式：通知重试幂等测试，对应 F11/F2。

规则 17-093：租户级通知频率、并发和失败重试必须纳入 C5 资源配额，超额时合并、延迟或拒绝，并给出错误码。
强制方式：通知轰炸配额测试，对应 F11/C5。

## 8. 正反例代码骨架

### 8.1 动态 SQL 白名单

反例 17-BAD-001：用户输入参与 SQL 结构拼接。

```java
String sql = "select * from " + request.tableName()
    + " where " + request.field() + " = #{value}"
    + " order by " + request.sort();
```

正例 17-GOOD-001：表、列、op、sort 都来自元数据白名单，用户值只走参数。

```java
public SqlSegment buildListSql(QueryRequest request, ObjectDef objectDef) {
  TableRef table = tableRegistry.requireTable(objectDef.code());
  ColumnRef sortColumn = objectDef.requireReadableField(request.sort().field()).column();
  FilterNode scoped = dataScopeMerger.mergeAsRootAnd(
      permissionService.requireScope(request.user(), objectDef.code()),
      request.filters()
  );
  return sqlAssembler.select(table)
      .where(filterCompiler.compile(scoped, objectDef))
      .orderBy(sortColumn, request.sort().order())
      .limit(request.page().offset(), request.page().size());
}
```

强制方式：
- ArchUnit：只有 `SqlAssembler` 包可以引用动态 SQL 片段类型。
- 安全测试：字段名、op、sort、link path、分页、批量 ids 注入全部覆盖。

### 8.2 data_scope AST 合并

反例 17-BAD-002：字符串层拼接权限条件。

```java
String where = scopeSql + " and " + userFilterSql;
```

正例 17-GOOD-002：scope 是根 AND 的强制子节点。

```java
public FilterNode mergeAsRootAnd(FilterNode scope, FilterNode userFilter) {
  if (scope == null) {
    throw new BizException(ErrorCode.TENANT_SCOPE_REQUIRED);
  }
  return FilterNode.and(List.of(scope, userFilter == null ? FilterNode.alwaysTrue() : userFilter));
}
```

必须测试的恶意 filters：

```json
{
  "or": [
    {"field": "ownerId", "op": "eq", "value": "attacker"},
    {"field": "amount", "op": "gt", "value": "0"}
  ]
}
```

断言：最终 SQL 语义必须等价于 `scope AND (ownerId = ? OR amount > ?)`，不得变成 `(scope AND ownerId = ?) OR amount > ?`。

### 8.3 TenantContext 传播

反例 17-BAD-003：异步任务里直接读取 ThreadLocal。

```java
CompletableFuture.runAsync(() -> ruleExecutor.execute(recordId));
```

正例 17-GOOD-003：提交任务时冻结租户上下文。

```java
TenantSnapshot snapshot = TenantContext.captureRequired();
CompletableFuture.runAsync(() -> {
  try (TenantScope ignored = TenantContext.open(snapshot)) {
    ruleExecutor.execute(recordId);
  }
}, tenantAwareExecutor);
```

强制方式：
- 禁止业务代码直接使用默认 `ForkJoinPool.commonPool()`。
- 双租户异步规则测试必须证明 tenantId 不串位。

### 8.4 表达式函数白名单

反例 17-BAD-004：表达式函数内查库。

```java
public Object call(Map<String, Object> env, String recordId) {
  return recordMapper.selectById(recordId);
}
```

正例 17-GOOD-004：运行时预取数据，函数只读上下文 DTO。

```java
public Object call(ExpressionContext ctx, String fieldCode) {
  return ctx.record().readAllowedValue(fieldCode);
}
```

强制方式：表达式函数实现包禁止依赖 Mapper、DataSource、RestTemplate、WebClient。

### 8.5 Connector SSRF 防线

反例 17-BAD-005：直接访问用户配置 URL。

```java
return httpClient.get(dataSourceConfig.url());
```

正例 17-GOOD-005：解析、DNS 校验、IP 段阻断、重定向复检。

```java
URI uri = connectorUrlGuard.requireAllowed(dataSourceConfig.url(), tenantId);
HttpResponse response = outboundClient.get(uri, ConnectorLimits.standard());
```

必须阻断：
- `localhost` / `127.0.0.1` / `0.0.0.0`
- RFC1918 私网段
- 云 metadata 地址
- DNS 解析后落入保留地址
- 重定向到禁止地址

### 8.6 附件对象 Key

反例 17-BAD-006：用户文件名参与存储路径。

```java
String key = tenantId + "/" + request.fileName();
```

正例 17-GOOD-006：服务端生成不可猜测 key，原始文件名只做展示元数据。

```java
String key = attachmentKeyGenerator.newKey(tenantId, appCode, objectCode, recordLid, detectedExtension);
attachmentStore.save(key, content, metadata.withOriginalName(request.fileName()));
```

强制方式：上传、预览、下载都必须重新走权限判定；日志不得输出完整 key 或签名 URL。

## 9. 安全测试最低清单

| 测试类 | 必测内容 |
|---|---|
| `DynamicSqlInjectionSecurityTest` | 字段名、op、sort、join path、分页、ids、导出、统计注入 |
| `TenantIsolationSecurityTest` | 双租户 add/list/get/update/del/action/meta 全链路隔离 |
| `DataScopeAstSecurityTest` | OR 短路、嵌套 AND/OR、link path 过滤 |
| `FieldPermissionSecurityTest` | /meta/list/get/update 字段读写一致 |
| `ExpressionSandboxSecurityTest` | 反射、IO、死循环、内存、白名单函数查库禁止 |
| `ImportPackageSecurityTest` | 恶意表达式、跨租户 id、重复 code、超大 JSON、未知字段 |
| `ConnectorSecurityTest` | SSRF、DNS rebinding、超时、限流、脱敏日志 |
| `AttachmentSecurityTest` | MIME/扩展名、跨租户下载、软删除、超大文件、路径穿越 |
| `BreakGlassSecurityTest` | break-glass 审批、到期失效、授权范围、审计不可篡改、密钥不可见 |
| `PersonalDataProtectionTest` | API/日志/审计/导出脱敏、保留期清理、租户注销匿名化 |
| `RendererXssSecurityTest` | 富文本、字段 label、错误消息、附件文件名、导入 schema XSS |
| `CsrfAndSessionSecurityTest` | CSRF、禁用用户旧 token、跨租户 token 重放、会话固定 |
| `BusinessImportExportSecurityTest` | 导入绕过权限、导出字段裁剪、CSV 公式注入、导出文件下载鉴权 |
| `NotificationSecurityTest` | 无权限收件人、敏感字段模板、通知重试幂等、租户频率配额 |
| `ErrorDisclosureSecurityTest` | 不存在/无权限响应归一、SQL/列名/堆栈/路径/连接串响应扫描 |
