# 25-错误处理与恢复细则

> 版本：v0.1（初稿）
> 地位：定义失败时如何报错、补偿、恢复、告警，覆盖正常路径之外的商用可靠性。

## 陷阱覆盖表

| 07 条目 | 本文覆盖点 | 强制/检测方式 |
|---|---|---|
| A1 DDL 半应用 | 失败状态、对账、人工恢复 | 失败注入测试 |
| A2 回滚边界 | 不可回滚变更阻断 | 回滚预检测试 |
| B1 租户上下文丢失 | fail-fast 与异步恢复 | 双租户测试 |
| C2 审计丢失 | outbox 与重试 | 集成测试 |
| C3 版本不一致 | 元数据版本过旧错误处理 | 契约测试 |
| E3 环境漂移 | 导入预检失败恢复 | 导入测试 |
| G7 错误信息泄露与探测面 | 外部错误归一、内部 traceId、敏感信息扫描 | 安全测试 |

## 1. 错误分类

规则 25-001：错误必须归类为用户可修复、配置错误、数据不一致、权限拒绝、并发冲突、外部依赖失败、系统缺陷。
强制方式：ErrorCode 枚举字段校验。

规则 25-002：错误响应必须包含 code、message、traceId，可定位字段时包含 fieldErrors。
强制方式：API 契约测试。

规则 25-003：错误响应不得包含 SQL、堆栈、文件路径、密钥、token、内部类名。
强制方式：异常响应扫描测试。

## 2. 恢复策略

规则 25-010：自动重试只允许用于幂等操作，必须有幂等键或去重记录。
强制方式：异步任务测试。

规则 25-011：动态 DDL 失败后禁止盲目重试，必须先运行 Reconciler 分类。
强制方式：发布失败测试。

规则 25-012：租户上下文为空时，动态数据访问层必须 fail-fast。
强制方式：安全单测。

## 3. 异步失败

规则 25-020：审计、规则日志、通知等异步任务必须定义可靠性等级：同事务、outbox 可靠异步、可丢弃。
强制方式：任务注册表校验。

规则 25-021：outbox 消息必须记录 tenantId、traceId、eventId、重试次数、最后错误。
强制方式：集成测试。

规则 25-022：死信必须可查询、可重放、可标记放弃，并写审计。
强制方式：运维接口测试。

## 4. 前端错误处理

规则 25-030：字段级校验错误必须绑定到具体控件，禁止只弹全局错误。
强制方式：组件测试。

规则 25-031：元数据版本过旧时，前端必须提示刷新 meta 后重试，不得继续提交旧 schema。
强制方式：前端契约测试。

## 5. 错误信息泄露防线

规则 25-040：对外错误必须采用双层口径：外部响应只给安全分类、用户可处理信息和 traceId；内部日志按 traceId 记录完整诊断信息。
强制方式：错误响应契约测试，对应 G7。

规则 25-041：对象不存在、记录不存在、无权限访问在查询和详情场景必须对外归一为同一类不可访问响应；表单校验、字段格式错误等用户可修复场景可返回精确错误。
强制方式：越权探测归一化测试，对应 G7/B2。

规则 25-042：错误响应体禁止包含 SQL、物理表名、物理列名、堆栈、内部类名、文件路径、连接串、bucket key、签名 URL。
强制方式：响应体敏感信息扫描测试，对应 G7/B6。

规则 25-043：状态机、发布、规则、调度和 outbox 错误必须落库错误码、traceId、可恢复建议和脱敏 message；禁止只在日志中记录。
强制方式：失败状态落库测试，对应 H2/G7。
