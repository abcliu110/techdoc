# 09-统一命名规范

> 版本：v0.1（初稿）
> 地位：所有后端、前端、数据库、API、文档、任务卡命名的源头规范。其他细则不得自行发明同义词。

## 陷阱覆盖表

| 07 条目 | 本文覆盖点 | 强制/检测方式 |
|---|---|---|
| D3 对象/字段 rename 语义 | code 创建后稳定，显示名可改 | 元数据校验测试 |
| E1 DTO 兼容纪律 | 字段名、枚举 code、API 字段名稳定 | OpenAPI diff + 快照测试 |
| E2 临时逃逸腐蚀 | 禁止租户/对象 code 特判命名 | 扫描脚本 |
| F3 导入导出重映射 | id/code/name 语义边界固定 | 导入包校验 |
| G1 配置引用完整性 | 页面/规则/权限引用统一 code | lc_meta_ref 校验 |

## 1. 业务词表

规则 09-001：同一中文业务概念只能有一个英文词，不得混用。

| 中文 | 统一英文 | 禁用同义词 |
|---|---|---|
| 租户 | tenant | customer/account |
| 工作区 | workspace | space |
| 应用 | app | application（类名可用 Application 但业务字段不用） |
| 对象 | object | entity/model/doc |
| 字段 | field | column/property/attr |
| 关系 | relation | association/ref |
| 视图 | view | screen |
| 页面 | page | ui/pageSchema 混写 |
| 块 | block | widget/card |
| 动作 | action | operation/command（Command 只用于后端入参类型） |
| 状态 | state | status（status 只表示启停状态） |
| 流程 | workflow | flow/process |
| 权限 | permission | privilege/authz |
| 规则 | rule | policy（policy 只用于权限策略） |
| 表达式 | expression | formula（formula 是字段类型） |
| 插件 | plugin | extension/addon |
| 迁移 | migration | transfer |
| 审计 | audit | log（log 是通用日志） |

强制方式：业务词表人工评审 + 代码扫描禁用词。

## 2. 通用命名规则

规则 09-010：标识命名必须区分 `id/lid/code`：数据库内部主键用 `id`，运行时业务记录跨环境稳定标识用 `lid`，元模型设计态稳定编码用 `code`。
正例：`record.lid`、`objectCode`、`fieldCode`、`actionCode`。
反例：导出包里引用 `fieldId=123` 或业务记录引用 `customer_id` 作为长期契约。
强制方式：导入导出包 schema 校验。

规则 09-011：显示名用 `name`，说明用 `description`，不得用 `label/title/displayName` 混写；`titleField` 只表示业务对象的标题字段配置。
强制方式：DTO 字段扫描。

规则 09-012：布尔命名只允许 `is/has/can/enable/allow/readonly/required/disabled` 语义前缀。
正例：`isTree`、`hasWorkflow`、`canUpdate`、`enableState`。
反例：`treeFlag`、`workflowYn`、`updateable`。
强制方式：Java/TS 命名扫描 + 人工评审。

规则 09-013：缩写白名单：`id/api/url/uri/ddl/sql/dto/vo/acl/spi/cas/json/xml/http/db/ui`；其他缩写必须入词表。

规则 09-014：集合命名用复数：Java/TS 变量 `fields`，API 字段 `fieldCodes`；数据库表名不因集合加复数，统一业务名 snake_case。

## 3. 后端命名

规则 09-020：包名固定 `com.lowcode.{module}.{layer}`，模块名白名单：`common/metamodel/runtime/designer/expression/plugin/workflow/app`。

规则 09-021：类后缀边界：

| 后缀 | 用途 |
|---|---|
| Controller | REST 入口 |
| Service / ServiceImpl | 业务用例服务 |
| Mapper | MyBatis mapper |
| Entity | 静态表持久化对象 |
| DTO | API 请求/响应传输对象 |
| VO | 复杂响应视图对象 |
| Command | service 写命令 |
| Query | 查询条件 |
| Def | 元模型 JSON 定义对象 |
| Registry | 注册表 |
| Resolver | 解析器 |
| Validator | 校验器 |
| Assembler | 对象转换器 |

规则 09-022：对外接口方法固定 `add/update/get/list/del/action/meta`，不得用 `create/delete/getDetail/remove`。
强制方式：Controller 方法名扫描。

规则 09-023：领域方法必须使用业务动词，禁止 `handle/process/doSomething`.
正例：`publishVersion`、`mergeDataScope`、`reconcilePhysicalSchema`。
强制方式：扫描 + 人工评审。

规则 09-024：枚举类以业务名结尾 `Enum`，枚举项用大写下划线，DB/API 传稳定 code。
正例：`FieldTypeEnum.TEXT.code()`。

## 4. 前端命名

规则 09-030：目录名小写 kebab 或单词，不使用拼音：`builder/renderer/shared/api/hooks/stores/components/pages/routes`。

规则 09-031：React 组件文件 PascalCase：`FieldEditor.tsx`；hook 文件 `useMetaGraph.ts`；store 文件 `designerStore.ts`；API 文件 `objectApi.ts`；类型文件 `objectTypes.ts`。

规则 09-032：事件 props 使用 `onXxx`，渲染函数使用 `renderXxx`，受控组件固定 `value/onChange`。

规则 09-033：TanStack Query key 固定数组格式：`['tenant', tenantId, 'app', appCode, 'object', objectCode, 'list', paramsHash]`。

规则 09-034：Zustand action 使用动词：`setSelectedObject`、`resetDraft`；state 不使用 `data/info/temp` 泛名。

## 5. 数据库命名

规则 09-040：库、表、列全部小写 snake_case，禁止拼音、大小写混用。

规则 09-041：表名前缀固定：
- 元数据表：`lc_meta_*`
- 运行时系统表：`lc_rt_*`
- 动态业务表：`lc_{app_code}_{object_code}`
- 插件表：`lc_plg_{plugin_code}_*`

规则 09-042：系统列顺序固定：`id, tenant_id, app_id, object_id, lid, state_code, revision, deleted, deleted_at, delete_token, create_time, create_by, update_time, update_by`，不适用列可省略但相对顺序不变。

规则 09-043：索引命名：主键 `pk_{table}`，唯一 `uk_{table}_{cols}`，普通 `idx_{table}_{cols}`，全文 `ft_{table}_{cols}`，长度超限按稳定 hash 截断。

规则 09-044：外部引用列：物理主键引用用 `xxx_id`，业务记录稳定引用用 `xxx_lid`，元模型编码引用用 `xxx_code`；不得混用。动态业务 link 默认使用 `xxx_lid`。

## 6. API 与文档命名

规则 09-050：API path 固定：
- 设计态：`/api/designer/{resource}/{action}`
- 运行态：`/api/data/{appCode}/{objectCode}/{action}`
- 平台管理：`/api/admin/{resource}/{action}`

规则 09-051：API 字段 camelCase，数据库字段 snake_case，由映射层转换，禁止直接把 DB 字段暴露给前端。

规则 09-052：ADR 文件名：`ADR-LOWCODE-{DOMAIN}-{NNN}-{slug}.md`；任务卡：`T-001-project-skeleton`；分支：`feature/T-001-project-skeleton`。

## 7. 验收清单

- [ ] 禁用词扫描无新增混用。
- [ ] Java/TS/SQL/API 命名扫描通过。
- [ ] 导入导出包不依赖数据库自增 id。
- [ ] 页面/规则/权限引用使用稳定 code。

## 8. 模块码与错误码命名

规则 09-060：错误码模块段固定如下：

| 模块 | 错误码前缀 | 范围 |
|---|---|---|
| common | LC-COMM | 通用参数、认证、限流、系统错误 |
| metamodel | LC-META | 元模型、版本、发布、MetaGraph |
| runtime data | LC-DATA | 动态数据、类型转换、乐观锁、状态 |
| permission | LC-PERM | 租户、角色、数据范围、字段权限 |
| expression | LC-EXPR | 表达式语法、沙箱、超时 |
| designer | LC-DESN | 设计器页面、草稿、校验 |
| plugin | LC-PLUG | 插件安装、升级、扩展点 |
| workflow | LC-WF | 流程定义、任务、流转 |
| connector | LC-CONN | 外部连接、SSRF、超时、凭据 |
| attachment | LC-FILE | 上传、下载、预览、存储 |

规则 09-061：错误码格式 `LC-{MODULE}-{四位数字}`，同一错误码语义永久不变。

## 9. 事件、缓存、锁命名

规则 09-070：领域事件命名使用过去式业务事实：
- `app.version.published`
- `object.schema.reconciled`
- `record.transitioned`
- `permission.changed`

反例：`doPublish`、`updateEvent`。

规则 09-071：Redis key 模板必须带环境、租户和版本维度：

```text
lc:{env}:tenant:{tenantId}:app:{appCode}:meta:{metaVersion}
lc:{env}:tenant:{tenantId}:idem:{api}:{idempotencyKey}
lc:{env}:tenant:{tenantId}:lock:publish:{appCode}
```

规则 09-072：分布式锁 key 必须表达资源和作用域，不得只用 `lock:{id}`。

## 10. 文件与包命名

规则 09-080：Flyway 文件禁止中文和拼音：

正例：`V202607051030__create_meta_tables.sql`
反例：`V202607051030__创建元数据表.sql`

规则 09-081：导入导出包文件名：

```text
lowcode-app-{appCode}-{versionNo}-{yyyyMMddHHmm}.zip
```

规则 09-082：插件包文件名：

```text
lowcode-plugin-{pluginCode}-{pluginVersion}.zip
```

## 11. 禁止命名清单

以下名称除局部极短生命周期变量外禁止使用：

```text
data, info, temp, obj, manager, handler2, newService, oldService,
process, handle, execute2, commonUtil, platformUtil
```

例外：`ActionHandler`、`EventHandler` 这类明确模式名允许，但必须有具体前缀。
