# 工程规范体系索引

> 版本：v0.2
> 用途：通用工程规范入口。本文只做索引，具体规则以 09~28 各细则为准。
> 来源：本规范最初从低代码平台方案中沉淀，因此部分条款仍包含低代码、多租户、动态元数据和插件体系的具体约束；其他项目复用时应保留通用安全底线，并按自身技术栈补充项目专属细则。

## 适用边界

- 通用适用：命名、Java、数据库、API、前端、测试、门禁、注释、安全、Git、模块边界、兼容演进、发布回滚、配置开关、错误恢复、可观测性、依赖合规等工程规则。
- 项目专属：`低代码`、`元模型`、`动态表`、`页面 schema`、`插件 SPI`、`builder/renderer` 等条款是低代码平台沉淀出的强约束；其他项目引用时需要替换为本项目的领域对象、技术栈和门禁实现。
- 冲突处理：项目级规范比本目录更具体时，优先项目级规范；但安全、兼容、验证和可回滚底线不得降低。

1. 当前项目的 PRD、架构设计、领域模型、接口契约和项目级 AGENTS/CLAUDE 规则。

## 使用规则

- 写代码前必须先定位任务涉及哪些细则，不得只读项目骨架规范。
- 任何涉及高风险陷阱的代码，必须找到至少一条机器门禁或测试证据。
- 细则冲突时，先以当前项目设计文档和风险清单为准，停止实现并上报冲突。
- 发现新陷阱时，先登记到当前项目风险清单，再补对应细则和门禁。
