# 13-前端编码细则

> 版本：v0.1（初稿）
> 地位：React 18 + TypeScript + Ant Design 5 前端编码细则，覆盖 builder、renderer、shared。

## 陷阱覆盖表

| 07 条目 | 本文覆盖点 | 强制/检测方式 |
|---|---|---|
| B3 meta/执行一致性 | 前端只按 /meta 渲染，不自行放权 | 组件测试 |
| C1 展开爆炸 | 列表不自动加载子表/深展开 | API 契约 |
| F1 重复点击 | action 幂等键与按钮 loading | 组件测试 |
| G1 保存不等于可发布 | 草稿校验与发布全图校验展示 | E2E |
| G2 表达式漂移 | 前后端表达式对照测试 | 契约测试 |
| G3 大对象性能 | 虚拟列表、搜索、编辑锁 | 性能测试 |
| G4 渲染器 XSS/Schema 注入 | Sanitizer、props 白名单、附件预览隔离 | ESLint + XSS 组件测试 |
| G5 OpenAPI 与动态对象类型边界 | 动态对象只按 runtime schema 校验 | TypeScript 约束 + 契约测试 |
| G6 设计态预览边界 | 预览只读、草稿隔离、预览审计标识 | E2E + 契约测试 |
| G7 错误信息展示 | 模糊错误统一文案、traceId 展示 | 组件测试 |
| G8 报表/打印/菜单/门户/移动端/i18n 边界 | 独立元数据、权限裁剪、模板版本、端适配和 fallback | 组件/E2E/契约测试 |

## 1. 目录结构

规则 13-001：前端 workspace 固定：

```text
lowcode-web/
  packages/builder/
  packages/renderer/
  packages/shared/
```

规则 13-002：`shared` 只能放 API client、类型、表达式子集、字段组件接口、通用工具；禁止放具体页面业务。

规则 13-003：builder 与 renderer 共享类型必须从 OpenAPI 或 shared schema 生成，不得手写两份。

## 2. TypeScript

规则 13-010：`strict: true`，禁止 `any`；确需使用必须单行 eslint-disable 并写原因。

规则 13-011：Props 必须用 interface 声明，字段注释说明业务含义和是否受控。

规则 13-012：API 响应类型不得手写，必须从 OpenAPI 生成。

规则 13-013：枚举 code 使用后端生成类型或集中常量，不得散落字符串。

## 3. 组件

规则 13-020：一个组件文件只导出一个主组件；超过 300 行必须拆分。

规则 13-021：受控组件统一 `value/onChange`，不得同时维护内部不可控状态。

规则 13-022：字段组件必须实现 `FieldComponent { view, edit, validate }` 并注册到 FieldRegistry。

规则 13-023：字段组件不得直接调用业务 API；数据读取通过上层注入或统一 hook。

规则 13-024：action 按钮提交期间必须 disabled/loading，并使用 idempotencyKey。

## 4. 状态管理

规则 13-030：服务端状态用 TanStack Query；客户端跨组件 UI 状态用 zustand；表单局部状态留在组件内。

规则 13-031：Query key 使用 09 规范数组格式，必须包含 tenant/app/object/metaVersion 关键维度。

规则 13-032：权限、字段可见性、动作可见性只消费 /meta 结果，不在前端自行推导放权。

规则 13-033：权限拒绝时隐藏或禁用操作，但不能作为安全边界；后端拒绝必须正确展示。

## 5. 表达式与 Schema

规则 13-040：前端表达式只支持 depends_on/visible_condition 子集；校验类表达式以后端为准。

规则 13-041：前端表达式求值器必须与后端共用测试样例，覆盖 null、空串、数字、日期、in、contains。

规则 13-042：页面 schema 保存前做局部校验，发布前展示服务端全图校验报告。

规则 13-043：Schema 节点引用 field/action/state/rule 必须通过选择器选择，不允许自由输入未校验 code。

## 6. 大对象性能

规则 13-050：字段数超过 100、权限项超过 500、页面节点超过 300 时必须启用搜索、分组或虚拟列表。

规则 13-051：字段表格、权限矩阵、页面节点树不得因 hover/loading 改变布局尺寸。

规则 13-052：设计器保存必须携带 revision；冲突时展示差异，不得静默覆盖。

## 7. 样式

规则 13-060：优先 AntD token 和 CSS module；禁止大范围全局 CSS 污染。

规则 13-061：按钮文本不得溢出，固定工具按钮优先 lucide/AntD 图标并加 tooltip。

规则 13-062：设计器属于操作型工具，界面应高密度、可扫描，禁止营销式 hero/card 堆叠。

## 8. FieldComponent 接口模板

规则 13-080：所有字段组件必须实现统一接口，禁止在表单渲染器中按 field_type 写 switch-case。

```ts
export interface FieldComponent<TValue = unknown, TOptions = unknown> {
  type: FieldTypeCode;
  View: React.ComponentType<FieldViewProps<TValue, TOptions>>;
  Edit: React.ComponentType<FieldEditProps<TValue, TOptions>>;
  validate?: (value: TValue, context: FieldValidateContext<TOptions>) => FieldError[];
  normalize?: (value: unknown, context: FieldNormalizeContext<TOptions>) => TValue | null;
}
```

注册：

```ts
fieldRegistry.register(textFieldComponent);
fieldRegistry.register(decimalFieldComponent);
fieldRegistry.register(linkFieldComponent);
```

反例：

```tsx
if (field.type === 'text') return <Input />;
if (field.type === 'decimal') return <InputNumber />;
```

## 9. link 字段组件要求

规则 13-090：link 组件查询选项必须走后端授权 API，不得前端缓存全量目标对象。

规则 13-091：link 显示值由目标对象 title_field 决定，但字段权限不足时显示 tombstone 或空，不得显示敏感字段。

规则 13-092：link 搜索必须 debounce，默认 300ms；结果数量默认不超过 20。

## 10. 大对象设计器交互门槛

| 对象规模 | 必须启用能力 |
|---|---|
| 字段数 > 50 | 字段搜索、类型筛选、分组折叠 |
| 字段数 > 100 | 虚拟列表、批量编辑抽屉 |
| 权限项 > 300 | 权限矩阵分页或按对象分组 |
| 页面节点 > 200 | 节点树搜索、折叠、懒渲染 |
| 规则数 > 50 | 按 trigger/action 分组 |

规则 13-100：设计器保存冲突必须展示冲突字段、对方修改时间和可选处理动作；禁止“后保存覆盖先保存”。

## 11. 前端错误处理模板

规则 13-110：字段级错误进入表单项，非字段错误进入页面级错误条，系统错误展示 traceId。

```ts
function applyApiError(error: ApiError, form: FormInstance) {
  if (error.fieldErrors?.length) {
    form.setFields(error.fieldErrors.map(item => ({
      name: item.field.split('.'),
      errors: [item.message],
    })));
    return;
  }
  showGlobalError(error.message, error.traceId);
}
```

规则 13-111：`LC-META-4091` 元数据版本过旧时，前端必须刷新 /meta 后提示用户重试，不得继续提交旧 schema。

## 12. 前端安全

规则 13-120：richtext 渲染必须使用统一 Sanitizer，禁止 `dangerouslySetInnerHTML` 直接渲染后端内容。

规则 13-121：附件预览/下载必须通过后端授权 URL，不得拼接对象存储地址。

规则 13-122：DataSource 密钥字段只显示脱敏值，前端不得持有明文。

规则 13-123：页面 schema props 必须经过白名单映射，禁止透传 `dangerouslySetInnerHTML`、事件 handler、`iframe/srcdoc`、字符串形式的 `style` 表达式。
强制方式：ESLint 禁止危险 prop + schema renderer 单测，对应 G4。

正例：

```ts
const safeProps = pickSchemaProps(node.props, allowedPropsByComponent[node.type]);
return renderRegisteredComponent(node.type, safeProps);
```

反例：

```tsx
return <DynamicComponent {...node.props} />;
```

规则 13-124：富文本、错误消息、字段 label、附件文件名和导入 schema 必须进入 XSS 测试集；测试样例至少包含 script、事件属性、javascript URL、svg onload、HTML 实体绕过。
强制方式：Vitest/Playwright XSS 测试，对应 G4。

## 13. OpenAPI 与动态对象类型边界

规则 13-130：平台管理 API、元模型 DTO、错误响应、分页结构可以从 OpenAPI 生成静态类型；`/api/data/*` 动态对象响应只能声明为 `Record<string, unknown>` + runtime schema，不得伪装成固定 DTO。
强制方式：TypeScript 类型扫描 + OpenAPI lint，对应 G5。

正例：

```ts
type RuntimeRecord = Record<string, unknown>;
const parsed = runtimeSchema.parse(record);
```

反例：

```ts
interface OrderRecord {
  customerLevel: string;
  amount: number;
}
```

规则 13-131：动态对象提交前必须校验 metaVersion、schemaVersion、必填、字段权限裁剪和 field_type 真值表；编译期类型不得替代运行时校验。
强制方式：旧 meta/新前端、新 meta/旧前端、权限裁剪字段缺失的契约测试，对应 G5。

## 14. 设计态预览边界

规则 13-140：设计器预览首版采用“只读真实数据 + 禁止写操作”的产品语义；预览态按钮/action/transition/import/export 等写入口必须禁用或由后端拒绝。
强制方式：预览态写操作阻断 E2E 测试，对应 G6。

规则 13-141：预览请求必须携带 `preview=true`、draftVersion 和 traceId；前端不得把草稿元数据写入已发布 MetaGraph 缓存 key。
强制方式：请求契约测试 + Query key 测试，对应 G6/C3。

规则 13-142：预览使用草稿元数据渲染，但所有数据读取仍由后端按当前用户权限裁剪；前端不得为了预览绕过 /meta 或权限字段隐藏。
强制方式：预览权限裁剪测试，对应 G6/B3。

规则 13-143：预览操作的错误、审计和日志展示必须带“预览”标识，避免与真实业务操作混淆。
强制方式：预览审计展示测试，对应 G6。

规则 13-150：不存在、无权限、已删除等安全模糊错误统一展示为不可访问类文案，并显示 traceId；前端不得根据内部错误码自行提示对象或记录是否存在。
强制方式：错误展示组件测试，对应 G7。

## 16. 报表、打印、菜单、门户、移动端与多语言边界

规则 13-160：报表、打印模板、菜单、门户、移动端布局和多语言资源必须作为独立元数据渲染，不得塞入 PageSchema 的自由 props 或普通页面节点。
强制方式：schema lint + 元数据引用测试，对应 G8。

规则 13-161：报表前端只能请求后端授权后的报表数据集和字段清单；不得复用列表页导出接口自行拼报表，也不得在前端隐藏无权限列后当作权限裁剪。
强制方式：报表字段权限组件测试 + API 契约测试，对应 G8/B3/F10。

规则 13-162：打印模板必须绑定模板版本、数据快照口径和权限策略；预览、打印、下载三种入口展示同一授权结果，不得因前端预览缓存泄露旧权限字段。
强制方式：打印预览/下载 E2E 测试，对应 G8/B3/C3。

规则 13-163：菜单和门户入口必须消费后端裁剪后的导航树；前端不得通过本地路由表、隐藏菜单或 URL 猜测开放页面访问。
强制方式：菜单权限和深链访问 E2E 测试，对应 G8/B3。

规则 13-164：移动端布局不得直接缩放桌面 PageSchema；必须有独立的端能力声明和组件白名单。桌面独有组件在移动端必须发布前阻断或降级为明确的只读展示。
强制方式：移动端 schema 能力检查和视觉回归测试，对应 G8/G1。

规则 13-165：多语言文本不得散落在页面 schema、组件 props 或硬编码字符串中；业务对象、字段、动作、状态、菜单、错误消息和报表标题必须通过 `I18nResource` key 解析。
强制方式：i18n key 扫描 + fallback 组件测试，对应 G8。

规则 13-166：多语言 fallback 顺序必须固定为 `用户语言 -> 租户默认语言 -> 系统默认语言 -> code`；缺失翻译不得阻断运行时，但必须在设计态和发布报告中列出。
强制方式：i18n fallback 契约测试，对应 G8。
