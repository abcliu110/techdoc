# 27-可观测性与运维细则

> 版本：v0.1（初稿）
> 地位：定义商用平台上线后的日志、指标、追踪、告警、容量、备份和运维动作。

## 1. Trace

规则 27-001：traceId 必须贯穿 REST、异步任务、规则执行、DDL 发布、插件调用、前端错误上报。
强制方式：集成测试检查日志/响应 traceId。

规则 27-002：异步任务必须继承或显式传入 traceId 和 tenantId。
强制方式：异步测试。

## 2. 日志

规则 27-010：日志分为应用日志、审计日志、规则日志、DDL 日志、权限拒绝日志。
每类日志必须定义字段、级别、脱敏、保留期。
强制方式：日志 schema 校验。

规则 27-011：权限拒绝、跨租户阻断、表达式沙箱拒绝必须记录安全日志。
强制方式：安全测试断言日志。

规则 27-012：日志不得输出完整业务数据行、token、密钥、密码、SQL 参数明文。
强制方式：敏感词扫描。

## 3. 指标

规则 27-020：必须采集动态查询耗时、SQL 次数、展开对象数、返回行数、慢查询数量。
强制方式：指标测试。

规则 27-021：必须采集 MetaGraph 加载耗时、版本号、缓存命中率、多实例收敛延迟。
强制方式：多实例测试。

规则 27-022：必须采集 DDL 计划数量、执行耗时、失败数、等待锁时间。
强制方式：发布集成测试。

## 4. 告警与容量

规则 27-030：动态表数量、单表行数、索引数量、元数据大小、日志表增长必须有预警线和硬上限，并与 11-060 的数值一致：单租户动态表 300/500，单实例动态表 20000/50000，单表行数 500 万/1000 万，单表普通索引 8/12，单对象元数据 JSON 1MB/4MB。
强制方式：监控配置审查。

规则 27-031：出现 Reconciler 差异、租户上下文为空、权限条件短路测试失败必须高优先级告警。
强制方式：告警规则测试。

规则 27-032：必须采集租户级资源隔离指标：并发查询、导出行数、Connector 调用、规则执行时长、附件容量、异步队列积压、DDL 发布频率。
强制方式：租户级指标测试 + 噪声租户演练，对应 C5。

规则 27-033：break-glass 使用、密钥查看拒绝、个人信息脱敏命中、保留期清理、租户注销匿名化必须形成可审计事件。
强制方式：安全审计测试，对应 B5/B6。

规则 27-034：必须采集 DDL MDL 风险指标：预检阻断次数、锁等待耗时、`lock_wait_timeout` 次数、进入 `PARTIAL_APPLIED` 次数，并按 app/object/tenant 维度可追踪。
强制方式：发布集成测试 + 告警规则测试，对应 A5。

规则 27-035：必须采集 Redis 降级和发布锁指标：Redis 不可用状态、DB 版本轮询延迟、fencing token 拒绝次数、发布锁续期/过期次数。
强制方式：Redis 故障演练，对应 A6。

规则 27-036：必须采集调度指标：任务计划触发数、实际执行数、跳过数、补跑数、重复抢锁拒绝数、失败数、租户时区和积压量。
强制方式：双实例调度演练，对应 C6。

规则 27-037：导入导出必须有任务级进度、成功/失败行数、错误报告下载审计、导出文件下载次数、过期清理结果和租户配额命中指标。
强制方式：导入导出演练，对应 F10。

规则 27-038：通知必须记录模板版本、通道、收件人摘要、权限判定结果、重试次数、死信状态和租户频率配额命中；不得记录通知正文敏感明文。
强制方式：通知审计测试，对应 F11。

规则 27-039：错误响应敏感信息扫描命中、越权探测归一失败、对外暴露内部错误码必须触发安全告警。
强制方式：错误响应扫描告警测试，对应 G7。

规则 27-039a：autonumber 必须采集取号等待耗时、CAS 冲突次数、跳号原因计数和单对象取号 TPS；超过 DB 行锁基线时触发容量评审，不得直接改为号段缓存。
强制方式：指标测试 + 压测报告，对应 B3/C4。

规则 27-039b：元数据 JSON 保存必须采集 revision 冲突次数、JsonUpgrader 失败次数、`lc_meta_ref` 重建失败次数；任一失败必须能通过 traceId 关联对象 code 和 metaVersion。
强制方式：指标测试 + 告警规则测试，对应 A4/B5/B12。

## 5. 运维动作

规则 27-040：必须提供只读模式、插件禁用、Reconciler 手工运行、outbox 重放、租户隔离排查的操作手册。
强制方式：运维演练。

规则 27-041：备份恢复必须覆盖元数据表、动态业务表、插件表、审计日志和导出包。
强制方式：恢复演练。

## 6. 指标命名与标签

规则 27-060：指标命名采用 `lowcode_{domain}_{metric}`，标签必须控制基数，禁止把 recordId、userId 放入指标标签。

| 指标 | 类型 | 标签 | 说明 |
|---|---|---|---|
| `lowcode_api_request_duration_ms` | histogram | tenantTier, appCode, api, result | API 耗时 |
| `lowcode_dynamic_query_duration_ms` | histogram | appCode, objectCode, queryType | 动态查询耗时 |
| `lowcode_dynamic_query_sql_count` | histogram | appCode, objectCode | 单请求 SQL 次数 |
| `lowcode_metagraph_load_duration_ms` | histogram | appCode | MetaGraph 加载耗时 |
| `lowcode_metagraph_version_lag` | gauge | appCode, instance | 实例版本落后 |
| `lowcode_schema_sync_duration_ms` | histogram | appCode, ddlType | Schema Sync 耗时 |
| `lowcode_schema_sync_lock_wait_ms` | histogram | appCode, objectCode, ddlType | DDL MDL 锁等待 |
| `lowcode_reconciler_diff_count` | gauge | appCode, diffType | 对账差异数 |
| `lowcode_outbox_backlog_count` | gauge | tenantTier, eventType | outbox 积压 |
| `lowcode_scheduler_backlog_count` | gauge | tenantTier, taskCode | 调度积压 |
| `lowcode_import_export_job_count` | counter | tenantTier, jobType, result | 导入导出任务结果 |
| `lowcode_notification_send_count` | counter | tenantTier, channel, result | 通知发送结果 |
| `lowcode_expression_timeout_count` | counter | appCode, expressionUse | 表达式超时 |
| `lowcode_permission_denied_count` | counter | appCode, objectCode, accessType | 权限拒绝 |
| `lowcode_autonumber_wait_ms` | histogram | appCode, objectCode, fieldCode | 取号等待耗时 |
| `lowcode_meta_json_conflict_count` | counter | appCode, objectCode, conflictType | 元数据 JSON 保存冲突 |

规则 27-061：tenantId 可进入日志和 trace，但默认不进入高基数指标标签；如需租户级指标，按租户等级或采样输出。

## 7. 日志字段标准

规则 27-070：结构化日志至少包含：

```json
{
  "time": "2026-07-05T10:00:00.000Z",
  "level": "INFO",
  "traceId": "...",
  "tenantId": "t001",
  "appCode": "sales",
  "module": "runtime",
  "event": "record.transitioned",
  "result": "success",
  "durationMs": 32
}
```

规则 27-071：安全日志必须额外包含 `securityEventType`、`decision`、`reasonCode`，但不得包含敏感值。

## 8. 告警规则初稿

| 告警 | 触发条件 | 等级 |
|---|---|---|
| 租户上下文缺失 | 动态数据访问 tenantId 为空 > 0 | P0 |
| Reconciler 差异 | `lowcode_reconciler_diff_count > 0` 持续 5 分钟 | P0/P1 |
| DDL 发布失败 | 发布进入 PARTIAL_APPLIED | P0 |
| MetaGraph 版本落后 | 任一实例落后当前版本超过 60 秒 | P1 |
| outbox 积压 | 积压超过阈值 10 分钟 | P1 |
| 表达式超时突增 | 5 分钟内超过基线 3 倍 | P2 |
| 动态表数量逼近上限 | 超过上限 80% | P2 |
| 噪声租户资源占用 | 单租户资源超过配额并影响队列/连接池 | P1 |
| DDL 锁等待超时 | 任一发布触发 `lock_wait_timeout` 或锁等待超过阈值 | P1/P0 |
| Redis 降级 | Redis 不可用导致只读降级或版本轮询延迟超阈值 | P1 |
| 调度补跑积压 | 必须补跑任务积压超过阈值 | P1 |
| 导出敏感扫描失败 | 导出文件或错误报告命中敏感明文 | P0 |
| 通知出域权限拒绝 | 通知模板引用敏感字段或收件人无 read 权限 | P1/P0 |
| 错误信息泄露 | 响应体扫描命中 SQL/堆栈/物理列名等 | P0 |
| break-glass 使用 | 任意 break-glass 会话开启 | P1 |
| 敏感数据脱敏失败 | 脱敏扫描或测试命中明文泄露 | P0 |
