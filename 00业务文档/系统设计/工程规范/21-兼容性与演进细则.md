# 21-兼容性与演进细则

> 版本：v0.1（初稿）
> 地位：正式商用产品的长期兼容契约，约束 API、元模型 JSON、页面 schema、插件 SPI 和导入导出包。

## 1. 兼容等级

| 对象 | 兼容等级 | 承诺 |
|---|---|---|
| 对外 REST API | 稳定 | 小版本只增不破坏 |
| 插件 SPI | 稳定/实验分级 | 稳定 SPI 至少保留两个小版本废弃期 |
| 元模型 JSON DTO | 稳定 | 字段只增不删，不复用语义 |
| 页面 schema | 稳定 | 旧 schema 必须可渲染 |
| 导入导出包 | 稳定 | 至少兼容最近两个主版本 |
| 内部 service 接口 | 内部 | 不承诺外部兼容，但需模块内测试 |

规则 21-001：稳定契约字段允许新增可选字段，禁止删除字段、改名、改类型、改变语义。
强制方式：OpenAPI diff + JSON schema diff。

规则 21-002：枚举 code 只能新增，禁止删除、复用、改变含义。
强制方式：枚举快照测试。

## 2. JSON DTO 演进

规则 21-010：元模型 JSON 必须带 `_v`，读取时按版本升级到当前内存模型。
强制方式：JsonUpgrader 单测。

规则 21-011：未知字段默认忽略但记录 WARN，禁止反序列化失败导致历史快照不可读。
强制方式：历史快照回放测试。

规则 21-012：废弃字段必须保留读取能力，至少两个小版本后才能移除升级器。
强制方式：兼容矩阵检查。

## 3. API 演进

规则 21-020：分页结构、错误响应结构、日期格式、金额格式属于高稳定契约，变更必须新版本 API。
强制方式：OpenAPI breaking change 检查。

规则 21-021：响应新增字段必须是可忽略字段，不得要求旧前端必须消费。
强制方式：旧前端契约回放。

规则 21-022：错误码只能新增，禁止改变既有错误码语义。
强制方式：错误码注册表快照测试。

## 4. 前端 Schema 与插件兼容

规则 21-030：组件注册 key 永久不可复用；废弃组件必须保留兼容渲染器。
强制方式：注册表快照测试。

规则 21-031：页面 schema 升级器必须可重复执行，且同输入稳定输出。
强制方式：schema upgrader 单测。

规则 21-032：插件 SPI 废弃必须标注 `@DeprecatedSince`、替代 API、移除版本。
强制方式：Java annotation 扫描。

规则 21-033：`/api/data/*` 动态对象不得承诺固定 DTO 兼容；兼容承诺落在 metaVersion、schemaVersion、field_type 真值表和 runtime schema 上。
强制方式：旧 meta/新前端、新 meta/旧前端契约测试，对应 G5。

规则 21-034：动态 record 容器结构属于稳定 API，`id/lid/revision/metaVersion/record/errors` 等容器字段受 OpenAPI diff 保护；record 内业务字段属于元模型契约，字段新增、删除、类型变化必须通过发布流程、兼容报告和 metaVersion 处理。
强制方式：OpenAPI diff + 元模型兼容报告测试，对应 B11/G5。

## 5. 工作流与部署模式兼容

规则 21-040：工作流实例创建时必须记录 workflowVersion、节点版本、表单 schema 版本、角色解析版本；运行时不得无条件读取最新流程定义。
强制方式：旧流程实例回放测试，对应 F8。

规则 21-041：流程定义升级必须选择继续旧版、人工迁移、自动映射或终止重开之一，并输出影响报告；节点/角色/字段删除不得让在途实例进入无主状态。
强制方式：流程升级兼容测试，对应 F8。

规则 21-042：SaaS shared 与私有化单租户差异只能通过 `deployment_mode`、配置、适配层和 ADR 表达，禁止业务代码散落 `if private` 分支。
强制方式：代码扫描 + 双模式最小回归，对应 E4。

## 6. 兼容测试

- 历史 Version.snapshot 回放。
- 旧导出包导入。
- 旧页面 schema 渲染。
- 旧插件安装和禁用。
- OpenAPI diff。
- 旧工作流实例回放。
- SaaS shared 与私有化单租户双模式回归。
- 旧 meta/新前端、新 meta/旧前端动态对象契约测试。

## 7. 标准对象扩展与包升级兼容

规则 21-050：标准对象升级不得覆盖客户扩展。升级必须以 `base object + extension layer` 生成合并报告，报告必须包含冲突字段、冲突规则、冲突页面、废弃项和人工处理建议。
强制方式：标准对象升级回放测试。

规则 21-051：扩展层兼容等级必须标注来源：system、industry_template、app_template、customer、plugin。不同来源的优先级和冲突策略必须固定，禁止按加载顺序偶然决定。
强制方式：扩展优先级快照测试。

规则 21-052：对象继承首版只允许一层受控扩展，不允许多层继承链。后续若开放父子单据继承，必须新建兼容矩阵，覆盖字段、状态、权限、规则、页面、转换规则和动态表结构。
强制方式：多层继承拒绝测试。

规则 21-060：应用包、行业模板包和插件包升级必须声明可从哪些历史版本升级；缺失迁移路径时阻断升级，不得跳版本猜测。
强制方式：包版本兼容矩阵测试。

规则 21-061：License 降级策略必须向后兼容已有数据读取和审计查看；不得因授权过期让用户无法导出自身数据或查看历史审计。
强制方式：License 降级回放测试。
