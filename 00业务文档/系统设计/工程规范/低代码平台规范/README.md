# 低代码平台规范

> 版本：v1.2.0　更新日期：2026-07-10　维护：工程规范组
>
> 定位：适用于基于低代码平台架构交付的项目。仅在低代码平台项目中强制生效，并叠加 `../通用规范/` 的工程底线。

## 0. 方法论声明

本目录所有契约遵循以下核心方法论原则，作为规则制定和执行的基本前提：

1. **契约即代码**：所有契约可通过 CI 机器校验。每条 `MUST` 级别规则必须声明可执行的验证方法（运行命令、检查文件或 CI 脚本路径）；暂无自动门禁时必须登记门禁缺口并跟踪偿还。
2. **版本兼容即测试**：每个 `metaVersion` 变更附带兼容性测试向量。新增/变更字段类型或元模型 DTO 时，必须同步提交契约向量测试，覆盖 DDL、API 输入、空串、NULL、组件能力和导入导出语义。
3. **元模型即 Schema**：元模型与 Schema 等价。后端 `FieldTypeHandler` 注册表导出的版本化 JSON Schema 或 `.d.ts` 声明文件是前后端共享的唯一事实源；前端不得维护独立的字段类型真值表。
4. **门禁缺口即债务**：`MUST` 级别规则尚无自动门禁时，必须阻断验收并登记门禁缺口（记录缺口编号、关联规则编号、责任人、计划偿还时间）；缺口不消除规则效力，只标记自动化程度。
5. **扩展点经济性**：每新增一个 SPI 意味着永久兼容承诺、安全暴露面扩大和测试矩阵膨胀。只有当至少一个消费方在当前里程碑有可验证的业务需求，且已有接口无法安全承载时，才引入新 SPI。引入时同步声明废弃条件和迁移窗口。
6. **信任边界不可越权**：平台对运行代码按信任级别隔离（T0 内核 > T1 内置插件 > T2 一方扩展 > T3 第三方应用包 > T4 不可信输入）。低信任级别代码不得通过任何路径（反射、序列化、ThreadLocal）访问高信任级别资源。
7. **风险即债务**：陷阱矩阵中每个未决风险条目等同于技术债务，必须登记 owner 和偿还计划；无 owner 的风险条目不得新增。

## 0.1 规则编号规范

本目录所有 `MUST`/`SHOULD`/`条件规则` 按以下格式编号：

```text
L{文件号}-{节号}-{条号}
示例：L01-03-02 = 01-元模型与物理存储契约.md 第3节第2条
     L04-01-01 = 04-扩展插件与应用包契约.md 第1节第1条
     L06-A-01  = 06-陷阱覆盖矩阵.md A类第1条
```

AI Agent 在 PR 描述或任务卡中引用规则时必须使用编号格式。

## 3. 加载规则

1. 先从本页定位任务，只加载相关 `通用规范/`、一份主契约和 `06-陷阱覆盖矩阵.md` 的相关行；跨域变更或全局评审才通读本目录。
2. 涉及发布、权限、动态 SQL、字段类型、插件或跨版本数据时，必须同时查矩阵中的相关风险、主规范和最低证据。
3. `MUST` 在与规则匹配的控制点阻断，例如设计评审、保存、发布、安装、构建或运行时请求；尚无自动门禁时，由评审阻断验收并登记门禁缺口。`SHOULD` 偏离需在 ADR 或任务卡记录理由。
4. 文档、接口骨架和候选测试只能证明“契约已定义”，不能证明“门禁已实现”；交付必须给出仓库中的测试、配置或运行记录。
5. 数值阈值必须写明适用部署画像、测量口径和校准方式；没有这些证据的数字只能作为保守默认护栏，不得表述为通用容量结论。

## 5. AI Agent 加载协议

当 AI Agent 在本规范体系下工作时，遵循以下加载协议：

1. **按需加载**：先读取本 README 定位任务类型，按任务路由表加载 1 份主契约 + 相关通用规范 + `./06-陷阱覆盖矩阵.md` 的相关行；禁止跳过定位直接编辑代码或契约。
2. **区分约束层级**：`MUST` 条款在对应控制点阻断，AI Agent 不得自动跳过或降级；`SHOULD` 偏离需在输出中说明理由和替代控制；`条件规则`仅在前提成立时生效，AI Agent 不得为满足条件而引入项目不需要的框架或抽象。
3. **规则编号引用**：AI Agent 在 PR 描述或任务卡中必须使用 `L{文件号}-{节号}-{条号}` 格式引用对应的契约条目编号。
4. **事实来源优先**：规范中的示例数值、技术选型和场景假设不是强制事实。AI Agent 输出中引用的固定值必须标注来源（PRD、accepted ADR 或实测基线），推断和假设必须显式标注，不得写成既定事实。
5. **门禁 vs 实现**：文档中的"门禁已要求"不等于"门禁已实现"。AI Agent 不得以文档或代码骨架作为通过证据，交付必须引用仓库中的测试、配置或运行记录。
6. **安全底线不可降级**：AI 生成或辅助生成的代码、插件和配置，必须经过与人工编写代码等价的安全审查。安全、兼容、验证和可回滚底线不得以"AI 辅助"为由降级。
7. **输出结构**：AI Agent 的设计输出应包含事实来源、推断与假设、适用范围和未覆盖风险四个部分。

### 5.1 AI Agent Do-Not 清单

```text
禁止凭训练语料推断字段类型、物理命名、软删除语义或导入导出边界
禁止跳过陷阱覆盖矩阵直接编码
禁止自行定义新的字段类型或元模型 DTO
禁止在代码、注释或日志中包含密钥、令牌或 PII 数据
禁止以"AI 生成"为由跳过供应链检查或安全审查
禁止将条件规则标记为 N/A 而不附不命中理由
禁止将"门禁已要求"当作"门禁已实现"的证据
禁止以检查清单通过代替人类评审——清单是自检工具不是放行工具
禁止硬编码本规范中的占位变量（如 <上游方案根>）为实际路径
禁止在规则冲突时自行选择——必须标记"待裁决"并提交人类决策
```
