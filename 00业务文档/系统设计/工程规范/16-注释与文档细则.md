# 16-注释与文档细则

> 版本：v0.1（初稿）
> 地位：约束代码注释、Javadoc、README、ADR、KB-REF 和交付文档。

## 1. 注释原则

规则 16-001：注释解释为什么、边界、风险、兼容承诺，不复述代码。

规则 16-002：安全关键代码必须有 `SECURITY:` 注释，说明防御对象和失败后果。

规则 16-003：竞品借鉴实现必须有 `KB-REF:` 指向知识库卡片，不得直接粘贴竞品代码。

规则 16-004：暂定架构决策影响代码必须有 `ADR-PENDING:`。

规则 16-005：TODO 必须写 `TODO(T-xxx):`，禁止裸 TODO。

## 2. 必写 Javadoc

规则 16-010：Controller 方法 Javadoc 包含业务目的、入参、返回、错误码。

规则 16-011：Service 公开方法 Javadoc 包含业务语义、事务边界、幂等性、租户语义。

规则 16-012：SPI 接口 Javadoc 必须声明稳定等级、兼容承诺、调用上下文、禁止事项。

规则 16-013：动态 DDL 分支必须说明为什么允许或拒绝该变更。

规则 16-014：权限判定每层必须说明判定内容和失败错误码。

## 3. DTO 与枚举

规则 16-020：DTO/VO/Command/Query 类必须有中文类注释，字段必须说明含义、单位、格式、是否必填、默认值。

规则 16-021：枚举类和枚举项必须说明业务含义，禁止只写 code。

规则 16-022：废弃字段必须写替代字段、废弃版本、计划移除版本。

## 4. README 与 ADR

规则 16-030：每个 Maven 模块和前端 package 必须有 README，包含职责、公开接口、依赖边界、测试命令。

规则 16-031：ADR 模板必须包含背景、约束、备选方案、决策、后果、撤销成本、验证方式。

规则 16-032：单向门决策必须有 ADR，不允许只留在聊天记录或 PR 评论。

## 5. 反例

规则 16-040：禁止复述型注释：`// 遍历字段列表`。

规则 16-041：禁止注释掉的死代码。

规则 16-042：禁止空 Javadoc：`@param x x`。

规则 16-043：禁止用注释解释本应由类型/命名表达的简单含义。
