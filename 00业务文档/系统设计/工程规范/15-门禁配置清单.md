# 15-门禁配置清单

> 版本：v0.1（初稿）
> 地位：把 09~28 中可机器化的规则落到 CI、Linter、ArchUnit、Checkstyle、扫描脚本和契约测试。

## 陷阱覆盖表

| 07 条目 | 本文覆盖点 | 强制/检测方式 |
|---|---|---|
| B1/B2/B3/B7 权限租户 | 安全测试集强制执行，含组织变更历史范围 | CI |
| B4/H1 沙箱与 SQL | import/SQL 拼接扫描 | ArchUnit + 扫描 |
| F9/F10/F11/G7 认证、导入导出、通知、错误泄露 | 安全测试集与响应扫描 | CI |

| H3 测试过浅 | 必测矩阵检查 | CI checklist |
| H4 抽象膨胀 | 人工评审 checklist | PR |

## 1. CI 阶段

规则 15-001：CI 阶段固定：

```text
validate-env -> backend-lint -> frontend-lint -> archunit -> unit-test
-> integration-test -> security-test -> contract-test -> coverage
-> license-check -> build
```

规则 15-002：任一阶段失败不得合并，禁止用人工口头豁免。

## 2. 后端门禁

规则 15-010：Maven Enforcer 校验 JDK 21、Maven 版本、依赖收敛、禁止快照依赖进入 main。

规则 15-011：Spotless 校验 UTF-8、import 顺序、格式。

规则 15-012：Checkstyle 覆盖方法长度、参数数量、嵌套深度、catch、线程池、魔法值、BigDecimal。

规则 15-013：ArchUnit 覆盖：
- 模块依赖方向
- api/service/dao 分层
- Controller 禁 Mapper/SQL
- Entity 禁入 API
- 禁直接 import Aviator
- 禁插件/业务绕过 QueryBuilder
- 权限判定门面唯一入口
- `@Transactional` 只在 service 层

## 3. 前端门禁

规则 15-020：ESLint `max-warnings=0`，TS strict，禁止 any。

规则 15-021：Prettier/stylelint 必跑。

规则 15-022：文件命名扫描：组件、hook、store、api、types。

规则 15-023：OpenAPI 生成类型 diff 检查，禁止手写重复 API 类型。

## 4. SQL 与迁移扫描

规则 15-030：Flyway SQL 扫描表/列 COMMENT、命名、禁止外键/触发器/存储过程/ENUM。

规则 15-031：动态 DDL 生成器测试覆盖 22 类型和危险变更。

规则 15-032：扫描 Mapper XML 和 Java 字符串，禁止动态 SQL 绕过统一组装器。

## 5. 安全与契约

规则 15-040：安全测试集包含 SQL 注入、表达式沙箱、租户隔离、权限短路、组织变更历史范围、SSRF、附件、导入包、CSRF、禁用用户旧 token、业务数据导入导出、通知出域、错误响应敏感信息扫描。

规则 15-041：OpenAPI diff 阻断 breaking change。

规则 15-042：license-check、dependency-check、pnpm audit、SBOM 必须产出。

## 6. 人工评审清单

以下无法完全机器化，必须进入 PR checklist：

- 是否新增未证明抽象。
- 是否把业务语义放进 common。
- 是否用配置/特性开关做永久租户特判。
- 是否有兼容性单向门但缺 ADR。
- 是否有仅靠文档约束的安全红线。

## 7. ArchUnit 规则骨架

规则 15-050：T-001 必须落地以下架构测试类。

```java
@AnalyzeClasses(packages = "com.lowcode")
class ArchitectureRulesTest {

  @ArchTest
  static final ArchRule controllers_should_not_access_mappers =
      noClasses().that().resideInAPackage("..api..")
          .should().accessClassesThat().resideInAPackage("..dao..");

  @ArchTest
  static final ArchRule api_should_not_expose_entities =
      noMethods().that().areDeclaredInClassesThat().resideInAPackage("..api..")
          .should().haveRawReturnType(resideInAnyPackage("..dao.."));

  @ArchTest
  static final ArchRule business_should_not_import_aviator =
      noClasses().that().resideOutsideOfPackage("..expression..")
          .should().dependOnClassesThat().resideInAnyPackage("com.googlecode.aviator..");

  @ArchTest
  static final ArchRule dynamic_sql_only_in_assembler =
      noClasses().that().resideOutsideOfPackage("..runtime.sql..")
          .should().accessClassesThat().resideInAnyPackage("..runtime.sql.segment..");
}
```

说明：上面代码是门禁意图骨架，T-001 落地时可按 ArchUnit 实际 API 调整写法，但规则语义不得删减。

## 8. Checkstyle / 扫描规则清单

| 规则编号 | 检测对象 | 失败条件 |
|---|---|---|
| CS-001 | 方法复杂度 | 方法超过 80 行、嵌套超过 3、参数超过 5 |
| CS-002 | 异常处理 | 空 catch、catch Exception 后不 rethrow/不转 BizException/不落状态 |
| CS-003 | 线程池 | 使用 `Executors.newFixedThreadPool/newCachedThreadPool` |
| CS-004 | 精度 | 使用 float/double 表示金额、decimal、percent |
| CS-005 | Optional | Optional 作为字段或参数 |
| SCAN-001 | SQL | 出现 `"select * from " +`、`${table}`、`${column}` 等动态拼接模式 |
| SCAN-002 | 租户特判 | 出现 `tenantId.equals("...")`、`objectCode.equals("...")` |
| SCAN-003 | TODO | 裸 `TODO`，未写 `TODO(T-xxx)` |
| SCAN-004 | 敏感日志 | log 语句包含 password、token、secret、credential、jdbcUrl |

## 9. CI YAML 骨架

```yaml
name: lowcode-ci

on:
  pull_request:
  push:
    branches: [ main ]

jobs:
  backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '21'
      - name: Backend verify
        run: mvn -B clean verify

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Enable pnpm
        run: corepack enable
      - name: Frontend verify
        run: |
          cd lowcode-web
          pnpm install --frozen-lockfile
          pnpm lint
          pnpm test
          pnpm build

  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Dependency and license check
        run: mvn -B -Psecurity verify
      - name: Frontend audit
        run: |
          cd lowcode-web
          pnpm audit --audit-level high
```

## 10. 不能机器化条款登记

```text
规则编号：
为什么不能机器化：
评审问题：
通过标准：
失败示例：
对应 07 陷阱：
```

禁止只写“人工评审”四个字而不给检查问题。
