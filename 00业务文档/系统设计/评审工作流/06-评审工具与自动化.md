# 评审工具与自动化

> 版本：v2.0  
> 日期：2026-07-07  
> 目标：通过工具实现评审自动化，提高评审效率和质量  
> 注意：本文是工具配置模板。任何工具成为 CI 门禁前，必须先在目标项目完成安装、配置、基线校准和失败处理验证。

## 工具状态分级

| 状态 | 含义 | 是否可作为门禁 |
|---|---|---|
| 已验证可用 | 当前项目已配置并能稳定运行 | 可以 |
| 模板示例 | 可参考，但需要按项目适配 | 不可以直接作为门禁 |
| 概念工具 | 表达能力方向，尚未验证或需自研 | 不可以作为门禁 |

下文示例默认属于“模板示例”；只有目标项目已实际跑通并记录基线后，才能升级为“已验证可用”。

```java
// 自定义规则：检测租户隔离缺失
public class TenantIsolationRule extends BaseTreeVisitor implements JavaFileScanner {
    @Override
    public void visitMethodInvocation(MethodInvocationTree tree) {
        String methodName = tree.symbol().name();
        if (methodName.equals("selectList") || methodName.equals("selectOne")) {
            // 检查是否有tenant_id过滤

        }
```

### 4. ArchUnit（架构约束）

**测试类**：

```java
@AnalyzeClasses(packages = "com.lowcode")
public class ArchitectureTest {
    
    // 禁止循环依赖
    @ArchTest
    public static final ArchRule modules_should_not_have_cycles =
        slices().matching("com.lowcode.(*)..").should().beFreeOfCycles();
    
    // Service不依赖Controller
    @ArchTest
    public static final ArchRule service_should_not_depend_on_controller =
        noClasses().that().resideInPackage("..service..")
            .should().dependOnClassesThat().resideInPackage("..controller..");
    
    // 只有SqlAssembler可以拼接SQL
    @ArchTest
    public static final ArchRule only_sql_assembler_can_concat_sql =
        noClasses().that().areNotAssignableTo(SqlAssembler.class)
            .should().callMethod(String.class, "concat", String.class)
            .andShould().callMethod(StringBuilder.class, "append", String.class)
            .where(methodNameContainsSqlKeyword())
            .because("SQL must be assembled via SqlAssembler (Rule 17-002)");
    
    // 只有TenantAware可以访问Mapper
    @ArchTest
    public static final ArchRule mapper_only_accessed_by_tenant_aware =
        classes().that().resideInPackage("..mapper..")
            .should().onlyBeAccessed().byClassesThat()
            .implement(TenantAware.class)
            .orShould().beAnnotatedWith(BypassTenantCheck.class)
            .because("All mapper calls must check tenant_id (Rule 17-010)");
}
```

### 5. Semgrep（自定义规则扫描）

**配置文件**：

```yaml
# .semgrep.yml
rules:
  # SQL注入检测
  - id: sql-injection-risk
    pattern: |
      String $SQL = "... " + $USER_INPUT + "..."
    message: |
      SQL injection risk detected.
      Use parameterized queries instead.
      Ref: Rule 17-001, Trap H1
    severity: ERROR
    languages: [java]
    
  # 租户隔离检测
  - id: tenant-isolation-missing
    pattern: |
      $MAPPER.selectList($WRAPPER)
    message: |
      Missing tenant_id check.
      Must call TenantContext.getTenantId() first.
      Ref: Rule 17-010, Trap B1
    severity: ERROR
    languages: [java]
    
  # 密码明文检测
  - id: hardcoded-password
    pattern: |
      String $PASSWORD = "..."
    message: |
      Hardcoded password detected.
      Use environment variable or secret manager.
      Ref: Rule 17-050
    severity: ERROR
    languages: [java]
    
  # 事务中的外部调用
  - id: external-call-in-transaction
    pattern: |
      @Transactional
      public void $METHOD(...) {
        ...
        $HTTP_CLIENT.$CALL(...)
        ...
      }
    message: |
      External HTTP call inside transaction.
      Move it outside transaction or use outbox pattern.
    severity: WARNING
    languages: [java]
```

**运行扫描**：

```bash
# 本地扫描
semgrep --config=.semgrep.yml --error

# CI中扫描
semgrep --config=.semgrep.yml --error --json > semgrep-report.json
```

```xml
<plugin>

  <executions>

      </goals>

      <id>report</id>

      <goals>
        <goal>report</goal>
      </goals>
    </execution>
    <execution>
      <id>check</id>
      <goals>
        <goal>check</goal>
      </goals>
      <configuration>
        <rules>
          <rule>
            <element>CLASS</element>
            <limits>
              <!-- L1/L2关键类要求100%覆盖 -->
              <limit>
                <counter>LINE</counter>
                <value>COVEREDRATIO</value>
                <minimum>1.00</minimum>
              </limit>
            </limits>
            <includes>
              <include>**/SqlAssembler.class</include>
              <include>**/AccessDecider.class</include>
              <include>**/TenantDataService.class</include>
            </includes>
          </rule>
          <rule>
            <element>BUNDLE</element>
            <limits>
              <!-- 整体覆盖率≥80% -->
              <limit>
                <counter>LINE</counter>
                <value>COVEREDRATIO</value>
                <minimum>0.80</minimum>
              </limit>
            </limits>
          </rule>
        </rules>
      </configuration>
    </execution>
  </executions>
```

## 六、CI/CD评审门禁

### GitHub Actions配置模板

```yaml
# .github/workflows/code-review.yml
name: Code Review Pipeline

on:
  pull_request:
    branches: [main, develop]

jobs:
  # Job 1: 代码规范检查
  code-style:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK 21
        uses: actions/setup-java@v3
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven
      
      - name: Checkstyle
        run: mvn checkstyle:check
      
      - name: Spotless Check
        run: mvn spotless:check
  
  # Job 2: 静态分析
  static-analysis:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0  # SonarQube需要完整历史
      
      - name: Set up JDK 21
        uses: actions/setup-java@v3
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven
      
      - name: SonarQube Scan
        run: |
          mvn clean verify sonar:sonar \
            -Dsonar.projectKey=lowcode-platform \
            -Dsonar.host.url=${{ secrets.SONAR_HOST_URL }} \
            -Dsonar.login=${{ secrets.SONAR_TOKEN }}
      
      - name: SonarQube Quality Gate
        uses: sonarsource/sonarqube-quality-gate-action@master
        timeout-minutes: 5
        env:
          SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
  
  # Job 3: 安全扫描
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK 21
        uses: actions/setup-java@v3
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven
      
      - name: SpotBugs
        run: mvn spotbugs:check
      
      - name: Dependency Check
        run: mvn dependency-check:check
      
      - name: Semgrep Scan
        uses: returntocorp/semgrep-action@v1
        with:
          config: .semgrep.yml
      
      - name: Upload Security Report
        uses: actions/upload-artifact@v3
        if: always()
        with:
          name: security-report
          path: target/dependency-check-report.html
  
  # Job 4: 架构约束检查
  architecture-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK 21
        uses: actions/setup-java@v3
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven
      
      - name: ArchUnit Tests
        run: mvn test -Dtest=ArchitectureTest
  
  # Job 5: 测试覆盖率
  test-coverage:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK 21
        uses: actions/setup-java@v3
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven
      
      - name: Run Tests with Coverage
        run: mvn clean test jacoco:report
      
      - name: JaCoCo Coverage Check
        run: mvn jacoco:check
      
      - name: Upload Coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          files: target/site/jacoco/jacoco.xml
  
  # Job 6: 影响分析
  impact-analysis:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      
      - name: Code Review Graph Analysis
        run: |
          # 概念工具：仅在当前项目已安装、已索引、已验证时启用
          code-review-graph detect-changes \
            --repo . \
            --base origin/${{ github.base_ref }} \
            --brief > impact-report.md
      
      - name: Comment PR with Impact
        uses: actions/github-script@v6
        with:
          script: |
            const fs = require('fs');
            const report = fs.readFileSync('impact-report.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: report
            });
  
  # Job 7: 综合报告
  review-report:
    runs-on: ubuntu-latest
    needs: [code-style, static-analysis, security-scan, architecture-test, test-coverage]
    if: always()
    steps:
      - name: Generate Review Report
        run: |
          echo "# Code Review Report" > review-report.md
          echo "" >> review-report.md
          echo "## Status" >> review-report.md
          echo "- Code Style: ${{ needs.code-style.result }}" >> review-report.md
          echo "- Static Analysis: ${{ needs.static-analysis.result }}" >> review-report.md
          echo "- Security Scan: ${{ needs.security-scan.result }}" >> review-report.md
          echo "- Architecture Test: ${{ needs.architecture-test.result }}" >> review-report.md
          echo "- Test Coverage: ${{ needs.test-coverage.result }}" >> review-report.md
      
      - name: Comment PR
        uses: actions/github-script@v6
        with:
          script: |
            const fs = require('fs');
            const report = fs.readFileSync('review-report.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: report
            });
```
