# 总索引与导读 - 系统化拆解 v1

## 1. 本质模型

总索引与导读的本质，是整套模型文档体系的导航和元模型。它不是某个业务域的设计方案，而是告诉读者：有哪些模型、它们如何分组、共享哪些底层纪律、遇到具体问题应该读哪篇。

核心抽象：

```text
总览方法论 -> 领域专篇 -> 共享纪律 -> 阅读路径 -> 问题索引 -> 评审清单 -> 维护约定
```

## 2. 核心张力

最大张力是：

```text
文档体系完整性 vs 使用者快速定位
```

文档越来越多，如果没有索引，知识会变成堆积；但索引过细又会变成另一本难读的大文档。

## 3. 不变量

- 总索引必须反映当前文档体系。
- 每篇专篇必须有一句话定位。
- 共享底层纪律必须跨域可迁移。
- 阅读路径要按角色和问题组织。
- 新增专篇必须有门槛，不能无限膨胀。
- 评审清单要能指向具体专篇。

## 4. 为什么这样设计

模型知识不是孤立文章，而是一套体系。总览负责方法论，专篇负责领域落地，总索引负责把二者组织成可检索、可学习、可执行的知识地图。

## 5. 失败模式

- 文档很多但没有地图，用户不知道先读哪篇。
- 同一底层纪律在不同篇重复出现但没有抽象，无法迁移。
- 新增文档没有门槛，体系膨胀失控。
- 评审时不知道用哪份清单。
- 索引不更新，反而误导读者。

## 6. 适用边界

适用：

- 模型设计知识库、架构文档体系、企业级研发规范、AI 知识检索入口。

不适用：

- 单篇独立方案，不需要体系化导航的临时文档。

## 7. 设计取舍

优先保证：

- 可导航。
- 可迁移。
- 可评审。
- 可维护。

牺牲：

- 索引需要持续更新。
- 文档结构需要统一约束。
- 新增专篇不能完全自由。
