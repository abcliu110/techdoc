# 库存模型设计专篇 - 系统化拆解 v1

## 1. 本质模型

库存模型的本质，是数量维度的账务系统，加上对物理世界差异的纠偏机制。它既要保证系统账面数量守恒，也要承认真实仓库、门店、盘点、损耗会让账实出现差异。

核心抽象：

```text
库存项 -> 库存余额 -> 锁定记录 -> 库存流水 -> 库存单据 -> 盘点差异
```

## 2. 逻辑模型

### 2.1 实体清单

| 实体 | 关键字段 | 职责 |
|---|---|---|
| inv_stock 库存项余额 | sku_id, warehouse_id, stock_status(可售/残次/待检/冻结), batch_no, owner_id, on_hand, locked, in_transit, version | 粒度由维度组合定义；可用=on_hand-locked 派生不落库 |
| inv_flow 库存流水 | stock_id, biz_type, biz_order_no, biz_line_id, idempotent_key(唯一), change_qty(带符号), qty_after, unit_cost 成本快照 | 数量守恒的账本，只增不改，qty_after 链式对账 |
| inv_lock 锁定记录 | 订单号, sku_id, warehouse_id, qty, 过期时间, status(锁定/已扣/已释放) | 回答"锁定被谁占用"，支撑超时释放 |
| inv_doc 库存单据 | 单据号(唯一), doc_type(采购入/销售出/调拨/报损报溢), warehouse_id, status, 审批人 | 一切库存变动的驱动源与凭证 |
| inv_doc_line 单据行 | doc_id, sku_id, batch_no, 计划 qty, 实际 qty, unit_cost | 行级关联流水，最小追溯颗粒 |
| inv_check 盘点单 | warehouse_id, 盘点范围(全盘/循环), 快照时间, status | 账实纠偏的正规入口 |
| inv_check_line 盘点明细 | check_id, sku_id, batch_no, 账面数(快照), 实盘数, 差异数, 差异原因, 复盘标记 | 差异被测量、被解释的证据 |
| inv_serial 序列号 | serial_no(唯一), stock_id, status(在库/锁定/已售/返修), 绑定单据行 | 一物一码追踪，数量流水之外的单件状态机 |
| inv_channel_quota 渠道配额 | sku_id, warehouse_id, channel_id, 配额数, 已用数 | 共享池的虚拟分配规则，不是新库存数 |

### 2.2 实体关系

```text
inv_stock (1) ──< inv_flow (N)          每次变动一条流水
inv_stock (1) ──< inv_lock (N)          locked 字段 = Σ有效锁定记录
inv_doc (1) ──< inv_doc_line (N)        单据-行一对多
inv_doc_line (1) ──< inv_flow (N)       流水经 biz_order_no+biz_line_id 回指单据行
inv_check (1) ──< inv_check_line (N)    盘点单-明细
inv_check_line (1) ──> inv_doc (报损/报溢单)  差异确认后生成调整单据
inv_stock (1) ──< inv_serial (N)        序列号挂库存项
inv_stock (1) ──< inv_channel_quota (N) 配额建在实物层之上
外部引用：sku_id → 商品域；warehouse_id → 主数据域；订单号 → 订单域；
         owner_id → 货主/租户域（引用不落外键，应用层校验）
```

### 2.3 关键约束与不变量落点

```text
唯一键：inv_stock (sku_id, warehouse_id, stock_status, batch_no, owner_id)；
       inv_flow idempotent_key 唯一（同一单据行不重复记账）；serial_no 全局唯一
留量校验：锁定 UPDATE ... SET locked = locked + n
         WHERE on_hand - locked >= n（原子防超卖，先查再扣即事故）；
         CHECK (on_hand >= 0 AND locked <= on_hand) 双保险
恒等式：任一 SKU，Σ入库流水 - Σ出库流水 = Σ各仓各状态 on_hand + in_transit；
       qty_after 链条连续即对账通过
状态机约束：状态间移动记一减一增两条流水，不直接改 stock_status；
           盘点差异必经报损/报溢单，不允许无单据 ADJUST
分片键：sku_id + warehouse_id（扣减热点路径）；热点 SKU 分桶拆行
```

## 3. 核心张力

库存最大的张力是：

```text
销售承诺速度 vs 实物准确性
```

销售希望快速知道能不能卖，仓库关心实物在哪里，财务关心库存价值，运营关心补货和周转。一个库存字段无法同时服务这些目标。

## 4. 不变量

- 可用库存不是存储值，而是计算值。
- 所有库存变化必须由单据驱动。
- 每次变化必须有库存流水。
- 库存扣减必须原子校验。
- 库存项粒度必须明确。
- 系统库存和实物库存差异必须通过盘点调整。
- 成本快照必须和出库行为绑定。

## 5. 为什么这样设计

四数分离是为了解决不同视角混用同一字段的问题：

```text
实物库存：仓库真实有多少
锁定库存：订单占用但未出库
可用库存：可以承诺销售多少
在途库存：已发出但未到达
```

单据驱动是为了让库存变化可追溯；盘点模型是为了给账实不符提供正规出口。

## 6. 失败模式

- 一个库存字段同时表示实物、可售和锁定。
- 直接修改库存数，没有单据和流水。
- 下单先查再扣，导致并发超卖。
- 没有锁定记录，无法知道库存被谁占用。
- 没有盘点差异模型，只能人工改库存。
- 成本不随出库固化，毛利报表随成本变化漂移。

## 7. 适用边界

适用：

- ERP、零售、电商、WMS、POS、供应链、生产和门店库存。

简化场景：

- 无实物履约、无库存成本的小系统可以简化，但只要有销售承诺和实物盘点，就不能省略流水和锁定。

## 8. 设计取舍

优先保证：

- 防超卖。
- 可追溯。
- 账实纠偏。
- 成本可核算。

牺牲：

- 实现简单性。
- 实时查询直观性。
- 仓内作业复杂度。

## 9. 同一模型的多业务形态建模

库存要同时支撑多种业务形态，核心方法是：**统一账本骨架 + 库存项粒度扩展 + 单据类型吸收差异**。业态变化落在"库存项加什么维度、增加什么单据类型"，而不是另建一套库存数。

### 9.1 统一骨架（所有业态不变的部分）

```text
库存项（维度组合定义粒度）→ 库存余额 → 锁定记录 → 库存流水 → 库存单据 → 盘点差异
不变量全集：四数分离、单据驱动、流水守恒、扣减原子校验、账实差异走盘点
```

骨架决不为某个业态特化——任何业态的"有多少、能卖多少、被谁占用、为什么变"都用同一套流水和单据回答。

### 9.2 各业态差异建模

| 业务形态 | 差异点 | 建模方式 |
|---|---|---|
| 电商仓库存 | 下单锁定、支付确认、超时释放 | 标准形态（骨架原型）：锁定记录挂订单号 + 超时释放任务，四数分离直接支撑 |
| 门店零售库存 | 当场售出无锁定、门店多且盘差频繁 | POS 销售单直接驱动扣减流水（跳过锁定段，不发明新数）；库存项加门店维度；高频循环盘点单 |
| 批次效期库存（食品医药） | 先进先出/近效期先出、批次召回 | 库存项粒度下钻到批次（item+仓+批次一行）；出库单按效期规则分配批次；召回按批次反查流水 |
| 序列号库存（高值） | 一物一码、每件独立追踪 | 序列号表挂库存项，状态机（在库/锁定/已售/返修）；数量流水不变，序列号状态变更与单据行绑定 |
| 多货主 3PL | 同仓不同货主库存必须隔离 | 库存项加货主维度（owner_id 进粒度组合）；所有单据带货主，跨货主调整走货权转移单而非改数 |
| 渠道共享库存 | 多渠道共卖一池，需防超卖又防饥饿 | 物理库存一份；渠道配额表做虚拟分配（配额是派生规则不是新库存数）；扣减仍在物理层原子校验 |
| 虚拟库存（卡券名额） | 无实物、无仓、无在途 | 库存项退化为"名额池"（仓维度取虚拟仓）；保留锁定/扣减/流水防超卖；盘点和在途裁剪掉 |

### 9.3 建模决策规则

- **业态差异优先落在库存项粒度**：批次、序列号、货主都是往维度组合里加列，不是新建库存表——流水和单据机制原样复用。
- **新业务动作 = 新单据类型，不是直接改数**：门店报损、货权转移、渠道调配额，都注册为单据类型走同一驱动管线。
- **允许裁剪扣减时序但不允许绕过原子校验**：POS 可跳过锁定段，但扣减仍必须原子校验，不得先查再扣。
- **拆独立域的信号**：当"数量守恒"不再是核心不变量（如内容资产管理、无限供给的纯虚拟商品），才不是库存，而是别的域。
