# 价格促销模型设计专篇

> 本篇聚焦价格、促销、优惠、分摊和规则版本。价格促销模型连接商品、订单、会员、渠道、结算和报表，是交易系统中最容易造成资损的模型之一。

## 0. 一句话总纲

> 价格促销模型的核心，是让“为什么这个人此刻以这个价格成交”可以被计算、解释、复现和审计。

## 1. 价格不是商品字段

商品定义“是什么”，价格定义“卖多少”。价格应独立建模：

- 基础价。
- 渠道价。
- 门店价。
- 客户价。
- 会员等级价。
- 合同价。
- 阶梯价。
- 临时活动价。

每个价格都应包含：

- 适用对象。
- 生效时间。
- 失效时间。
- 币种。
- 优先级。
- 版本。
- 发布状态。

## 2. 价格计算链路

标准计算顺序示例：

```text
基础价
  -> 渠道/门店价
  -> 客户/会员价
  -> 活动价
  -> 优惠券/满减/折扣
  -> 运费/税费
  -> 应付金额
```

必须记录计算快照：

- 命中的价格规则。
- 命中的促销规则。
- 命中的会员权益。
- 优惠分摊结果。
- 尾差处理。
- 规则版本。

**为什么需要快照**：下单后规则可能变化，但订单必须能解释当时价格。

## 3. 促销规则模型

促销规则至少包含：

- 活动类型。
- 适用商品。
- 适用门店/渠道。
- 适用会员。
- 使用门槛。
- 优惠方式。
- 叠加规则。
- 排斥规则。
- 预算。
- 生效时间。
- 规则版本。

常见类型：

- 满减。
- 满折。
- 直降。
- 买赠。
- 第 N 件优惠。
- 套餐价。
- 优惠券。
- 会员价。

## 4. 优惠叠加和排斥

促销最复杂的不是单条规则，而是多条规则如何组合。

标准做法：

```text
优惠分组：商品级、订单级、支付级、运费级
优先级：先商品价，再订单优惠，再支付优惠
排斥组：同组不可叠加
叠加组：允许叠加但有上限
预算控制：平台、商家、供应商分别承担
```

## 5. 优惠分摊

订单级优惠必须分摊到行，否则退款、结算、开票、毛利都无法准确计算。

分摊记录应包含：

- 订单行。
- 优惠类型。
- 活动或券 ID。
- 出资方。
- 分摊金额。
- 规则版本。
- 尾差归属。

## 6. 防资损设计

价格促销必须有保护：

- 金额恒等式校验。
- 优惠不得超过商品金额。
- 预算不得超发。
- 券不得重复核销。
- 叠加规则必须确定。
- 计算结果必须可重放。
- 高风险活动灰度发布。

## 7. 评审清单

```text
1. 价格是否独立于商品？
2. 价格是否有生效区间、适用范围、优先级和版本？
3. 价格计算链路是否可解释？
4. 促销规则是否支持叠加、排斥和预算？
5. 优惠是否分摊到订单行？
6. 出资方是否记录？
7. 退款、结算、开票是否能按分摊结果处理？
8. 规则变更后历史订单是否可复现？
9. 是否有防重复核销、防超预算、防负价校验？
```

## 8. 总结

价格促销建模五条纪律：

1. **价格独立**：商品不是价格容器。
2. **规则版本化**：历史成交必须可复现。
3. **优惠可分摊**：订单级优惠必须落到行。
4. **出资方明确**：优惠是谁承担，决定结算和报表。
5. **防资损优先**：促销系统宁可少灵活，也不能算不清。
