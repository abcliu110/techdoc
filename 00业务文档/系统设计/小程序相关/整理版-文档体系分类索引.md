# 整理版：文档体系分类索引

## 1. 整理目的

本目录历史上以“小程序相关”为目录名，但内容已经扩展到：

```text
小程序开发与测试。
PC 后端管理界面设计。
ERP / 金蝶类单据设计。
跨端通用页面契约。
自动化测试和证据规范。
经典界面结构知识库。
```

为了避免继续混淆，本文把现有文档按适用范围重新分类。

原文档不修改。

后续新增规范应优先按下面四类写：

```text
1. 跨端通用原则：小程序和 PC 后台都适用。
2. PC 后端管理界面专用：ERP、SaaS 后台、业务中台、金蝶类单据。
3. 小程序/移动端专用：微信小程序、Taro、uni-app、移动端交互和平台限制。
4. 自动化测试与证据：跨端可复用，但实现细节按端区分。
```

## 2. 新整理文档

| 新文档 | 适用范围 | 用途 |
|---|---|---|
| `整理版-跨端通用设计原则.md` | 小程序 + PC 后台通用 | 抽出任何业务页面都必须遵守的原则 |
| `整理版-PC后端管理界面设计规范.md` | PC 后台 / ERP / SaaS 管理端 | 抽出左树右表、主从表、金蝶单据、表格、报表、审批等 PC 后台专用规则 |
| `整理版-小程序移动端设计规范.md` | 小程序 / 移动端 | 抽出 tabBar、页面栈、分包、移动端布局、扫码、触摸交互等专用规则 |
| `整理版-文档体系分类索引.md` | 文档治理 | 说明哪些旧文档应归入哪类 |


```text

```

## 4. 适用范围判断规则

### 4.1 同时适用于小程序和 PC 的原则

只要原则关注的是业务对象、状态、流程、接口、证据，而不是具体终端能力，就属于跨端通用。

典型关键词：

```text
业务对象。
业务编号。
页面身份。
入口来源。
参数契约。
状态机。
业务上下文。
操作反馈。
错误恢复。
权限。
审计。
后端证据。
自动化等待。
测试契约。
```

### 4.2 主要适用于 PC 后端管理界面的原则

只要原则依赖大屏、鼠标、键盘、高密度表格、多栏结构、批量作业、复杂筛选，就属于 PC 后台专用。

典型关键词：

```text
左树右表。
主从表。
树表一体。
矩阵表。
三栏工作台。
宽表格。
复杂筛选。
批量导入导出。
套打。
报表钻取。
ERP 单据。
凭证。
反写。
单据转换。
```

### 4.3 主要适用于小程序 / 移动端的原则

只要原则依赖微信平台、小屏、触摸、页面栈、分包、授权、扫码、移动端交互，就属于小程序专用。

典型关键词：

```text
微信小程序。
Taro。
uni-app。
tabBar。
页面栈。
分包。
包体。
授权弹窗。
扫码。
触摸手势。
移动端卡片列表。
底部操作栏。
下拉刷新。
上拉加载。
```

## 5. 后续重写建议

后续不要再写“企业级小程序页面契约”这种混合标题。

建议命名：

```text
跨端页面契约规范。
PC 后端管理界面页面契约规范。
小程序移动端页面契约规范。
ERP 单据页面设计规范。
企业后台经典界面模式知识库。
跨端自动化测试证据规范。
```

## 6. 最终结论

这套文档现在的主要问题不是内容不足，而是边界混在一起。

整理后的边界应该是：

```text
跨端通用：业务对象、状态机、契约、测试、证据。
PC 后台专用：表格、多栏、ERP 单据、批量、报表、权限审计。
小程序专用：页面栈、分包、移动端布局、微信能力、触摸交互。
```
