# 证据：金蝶设计器与页面类型元模型

## 原始观察

“设计器整体介绍”说明金蝶开发平台设计器位于开发平台中，页面创建完成后进入设计器。设计器界面分为设计视图、功能按钮、控件栏、设计区域、属性栏五大块。

以单据页面为例，设计视图包括表单设计、列表设计、移动表单设计、移动列表设计和 XML 设计。XML 设计视图展示页面所有元素的 XML 设计，并区分页面元数据和实体元数据。保存会把设计生成的 XML 元数据保存到数据库，预览会即时生成运行态页面。

同一组 PC 页面开发文档将页面类型区分为动态表单、单据、基础资料、报表、参数、卡片、轻量卡片、布局等：

- 动态表单是最基础、最高灵活度的页面类型，不直接提供数据库存储操作，存储和查询需要通过插件自行完成。
- 单据基于动态表单，增加数据存储、查询、常用字段、常用操作，并自动生成表单、列表、移动表单、移动列表四类页面。
- 基础资料是特殊单据，可作为企业基础资料被其他表单引用，典型对象包括物料、币别、科目等。
- 报表用于展示动态经营数据，不用于录入；支持查询数据源、自定义数据源、多实体查询和过滤条件配置。
- 参数页面提供公共、云、应用、单据、用户、组织等不同作用域的参数配置。
- 卡片是特殊动态表单，强调复用，可用于首页、门户、业务统计和复杂单据的卡片式布局。
- 布局允许同一基础资料或单据在不同场景下使用不同界面配置，但布局只能使用对应业务实体中的字段，不能新增字段和单据体。

## 证据强度

直接事实：官方开发指南明确说明设计器组成、多视图、XML 元数据、保存与预览机制，以及各页面类型的差异。

推断边界：该证据不能证明 XML 元数据的完整字段结构，只能证明页面元模型的大类和公开行为。

## 可抽取知识

- 金蝶页面设计器不是纯 UI 画布，而是元数据编辑器。
- 动态表单是页面类型基类，单据、基础资料等在其上叠加业务语义和框架能力。
- 表单、列表、移动表单、移动列表是同一业务对象的多视图，而不是彼此独立页面。
- 布局与实体结构分离，说明数据模型和展示模型需要解耦。
- 自研低代码平台应把页面类型、实体元数据、页面元数据、视图、布局、运行预览做成统一模型。
