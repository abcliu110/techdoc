### 1.1 统一命名规范（09-统一命名规范.md）

本项目必须新增独立命名规范，先统一业务词表、技术词表和命名模式，再被 Java、前端、数据库、API、测试、Git 细则引用。命名规范不能散落在各细则里各写一套。

至少覆盖以下内容，每项都要正反例与强制方式：

**通用命名原则**
- 中文业务概念到英文词表的映射表：租户、应用、对象、字段、关系、视图、块、动作、状态、流程、权限、规则、表达式、插件、迁移、审计。
- 禁止同一概念多译：如对象只能用 `object` 或 `entity` 二选一并定死；字段只能用 `field`，不能在代码里混用 `column/property/attr`。
- 缩写规则：允许 `id`、`api`、`url`、`ddl`、`sql`、`dto`、`vo`、`acl`；其他缩写必须入词表，禁止自造缩写。
- 单复数规则：集合、列表、表名、API path、变量名分别如何使用单复数。
- 布尔命名规则：后端字段、前端 props、数据库列统一使用 `is/has/can/enable` 的边界。

**后端命名规范**
- 包名：`com.lowcode.{module}.{layer}`，模块名和层名白名单。
- 类名后缀：Controller、Service、ServiceImpl、Mapper、Entity、DTO、VO、Enum、Config、Handler、Factory、Registry、Resolver、Validator、Assembler 的使用边界。
- 方法名：对外接口固定 `add/update/get/list/del`；领域方法必须用业务动词，禁止 `handle/process/doSomething`。
- DTO/VO/Command/Query 命名边界：请求、响应、内部命令、查询条件分别用哪个后缀。
- 枚举命名：枚举类、枚举项、code 字段、数据库存储值、前端映射名的端到端规则。
- 常量命名：业务常量、配置 key、缓存 key、锁 key、MQ topic/event name 的命名模板。
- 异常码命名：`LC-{MODULE}-{四位数}` 的模块码、编号段和新增流程。
- 测试类与测试方法命名：单元测试、集成测试、架构测试、安全测试分别的命名模板。

**前端命名规范**
- 目录命名：`builder/renderer/shared/api/hooks/stores/components/pages/routes` 等目录边界。
- 文件命名：React 组件 PascalCase，hooks 用 `useXxx.ts`，store 用 `xxxStore.ts`，API 文件用 `xxxApi.ts`，类型文件用 `xxxTypes.ts`。
- 组件命名：页面组件、业务组件、字段组件、布局组件、弹窗组件、表格列组件的后缀规则。
- props 命名：事件 `onXxx`，布尔 `is/has/can/disabled/readonly`，渲染插槽 `renderXxx`，受控值 `value/onChange`。
- hook 命名：查询 hook、mutation hook、本地状态 hook、订阅 hook 的命名模板。
- TanStack Query key 命名：数组层级、租户/应用/对象维度、列表/详情/元数据缓存的统一格式。
- Zustand store 命名：store 文件、state 字段、action 方法、selector 的命名规则。
- 字段组件命名：22 种 `field_type` 到前端组件名、注册 key、后端 code 的映射规则。

**数据库命名规范**
- 库名、表名、列名全部小写 snake_case；禁止大小写混用和拼音。
- 元数据表：`lc_meta_*`；运行时系统表：`lc_rt_*`；动态业务表：`lc_{app_code}_{object_code}`。
- 系统列命名与顺序：`id/tenant_id/app_id/object_id/lid/revision/deleted/deleted_at/delete_token/create_time/create_by/update_time/update_by` 等必须定死。
- 主键、业务唯一键、普通索引、全文索引、唯一索引命名：`pk_ / uk_ / idx_ / ft_` 模板与长度限制。
- 外部引用列命名：`xxx_id`、`xxx_lid`、`xxx_code` 的使用边界。
- JSON 列命名：`*_json` 或 `*_config` 的边界，必须配 COMMENT 结构说明和 `_v` 版本字段。
- 动态字段列名生成：field_code 到 column_name 的映射、关键字转义、超长截断、冲突后缀规则。
- Flyway 迁移文件命名：`V{yyyyMMddHHmm}__{english_or_pinyin_free_description}.sql`，禁止中文文件名进入代码仓库迁移目录。

**API 与文档命名规范**
- API path 命名：固定前缀、资源名、动态接口 `/api/data/*`、设计器接口 `/api/designer/*` 的边界。
- 请求/响应字段统一 camelCase；数据库字段 snake_case 到 API 字段 camelCase 的映射规则。
- 文档、ADR、任务卡、分支、commit、PR 标题命名模板。

**强制方式**
- Java 命名：Checkstyle + ArchUnit + 自定义扫描脚本。
- 前端命名：ESLint 规则、文件命名扫描脚本、TypeScript 类型约束。
- 数据库命名：Flyway SQL 扫描脚本、无 COMMENT 列扫描、动态 DDL 生成器单测。
- API 命名：OpenAPI lint、契约测试。
- 无法机器化的业务词表一致性：列入人工评审 checklist。
