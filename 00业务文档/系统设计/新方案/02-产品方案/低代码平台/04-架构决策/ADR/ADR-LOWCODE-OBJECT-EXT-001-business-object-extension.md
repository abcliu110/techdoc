# ADR-LOWCODE-OBJECT-EXT-001: 业务对象扩展与标准单据继承

> 状态：proposed
> 日期：2026-07-06

## 背景

平台定位是商用企业业务低代码，不只是从零创建对象。行业模板、标准单据和客户定制会同时存在，如果没有受控扩展层，客户修改会在模板升级时被覆盖，或者标准对象会被客户直接改成不可升级的分叉。

## 决策

M0/M1 采用“模板复制 + 受控扩展层”的策略：

- `system/vendor/customer` 三层归属必须显式记录。
- 标准对象不可被客户直接修改，只允许通过 `ObjectExtension` 增量扩展字段、规则、页面覆盖和权限覆盖。
- 禁止无限继承和运行时多层继承展开。
- 父子单据继承、行业对象继承和复杂覆盖合并必须在 M2/M3 前另行终审。

## 理由

受控扩展层能保留标准对象可升级性，也能给客户定制留出口；无限继承虽然表达力强，但升级冲突、权限合并和运行时解释成本过高。

## 否决方案

- 允许客户直接改标准对象：短期简单，但升级时无法区分厂商变更和客户变更。
- 完整面向对象继承：概念强，但对单据、权限、页面和数据迁移的组合复杂度过高。
- 只复制模板不保留来源：首版简单，但无法做模板升级、冲突报告和 License 管控。

## 后果

- T-002/T-003 只能预留 `source_kind/base_object_code/extension_policy/ObjectExtensionDef` 和校验。
- M0 不做扩展合并运行时，只阻断非法覆盖。
- M2/M3 前必须补升级回放、冲突报告和客户扩展不被覆盖的测试。

## 验证

- 标准对象升级不覆盖客户扩展。
- 禁止客户覆盖系统字段和删除基准对象必需字段。
- 扩展冲突输出报告并阻断发布。
