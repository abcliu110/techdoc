# M2 测试规格

> 版本：v0.1
> 范围：T-201~T-214
> 目标：验证设计器、renderer、页面 schema、状态机/权限配置、版本发布闭环和 M2 竞品补全能力。

---

## 1. 测试分层

| 层级 | 范围 |
|---|---|
| 前端单元测试 | 字段组件、schema 校验、表达式安全子集 |
| 前端集成测试 | renderer 与 mock API、错误展示、metaHash 刷新 |
| E2E 测试 | 客户-订单-审批零代码配置和运行 |
| 后端集成测试 | 页面 schema 发布、版本收敛、回滚边界 |
| 安全测试 | XSS、字段越权、附件越权、schema 放权 |

## 10. 贯穿 E2E

M2 结束必须全程零代码完成：

```text
创建 customer/order/order_item
配置状态机和权限
生成默认页面
调整表单分组布局
发布 v1
运行态新增订单并审批
修改字段发布 v2
回滚到 v1 并说明边界
配置跨对象规则、定时提醒、rollup 订单总额、Excel 导入导出、保存视图、站内消息
```
