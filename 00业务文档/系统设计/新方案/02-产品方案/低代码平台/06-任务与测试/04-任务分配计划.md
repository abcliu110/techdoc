## 0. 执行 AI 通用规则（每个任务开始前必读）

1. **必读输入**：先读 `../03-需求/PRD-产品需求规格说明书.md`，再读 `../04-架构决策/` 下 00~03 设计文档 + `../05-详细设计/08-详细设计总纲.md` + `../05-详细设计/README.md` + 当前任务卡详细设计 + **`05-代码规范与AI协作约定.md`（编码/注释/测试/Git 流程的完整规范，本节 2~4 条只是速查）** + `../07-知识库同步/07-低代码平台级架构陷阱与高难度问题清单.md` + `../04-架构决策/ADR/` 中相关 ADR + 任务卡指定的知识库卡片。需求、设计、规范与本计划冲突时，先修订 PRD/设计/规范/测试规格形成一致结论，再实现；没有文档承接不得让代码反向定义需求。
2. **技术基线**：Java 21 / Spring Boot 3.4.x / Maven 多模块 / MyBatis-Plus / MySQL 8 / Redis；前端 React 18 + TS + AntD 5 + pnpm workspace。Java 源码 UTF-8 无 BOM，2 空格缩进。
3. **编码约束**（沿用团队规范）：
   - 固定业务选项集合的字段端到端建模为枚举，禁止裸 Integer 魔法值
   - 金额/精度敏感字段禁 float/double
   - 接口全 POST，方法命名 add/update/get/list/del
   - DTO/VO 中文注释（类+字段），复杂方法写 Javadoc
   - 表带 revision/deleted/deleted_at/delete_token/create_time/create_by/update_time/update_by 标准列
4. **安全红线**：动态 SQL 的表名列名必须走元数据白名单；所有查询强制 tenant_id 条件并有单测断言；DataSource 密钥加密存储，日志不得输出明文。
5. **完成定义**：每个任务必须交付 ①代码 ②单元测试（核心逻辑覆盖）③任务验收标准逐条自检结果 ④未覆盖风险清单 ⑤质量锚点与覆盖的 07 陷阱编号。测试不过不得声明完成。
6. **禁止事项**：不得引入任务卡未列出的重量级依赖（引擎/框架级）而不报告；不得跳过验收标准中的验证条件；不得把"发现设计缺陷"只写进聊天或代码注释。能按 PRD 和 ADR 明确裁决的，先补 PRD/设计/测试再实现；涉及产品取舍或单向门不明确的，停下报告给人类裁决。

## 1. 里程碑总览

```text
M0 元模型内核（先行，其他一切依赖它）
   → M1 运行时引擎（数据API/权限/状态机/规则）
      → M2 设计器与渲染器（前端）+ 版本发布
         → M3 工作流/插件/数据源/商业增强
            → M4 商用后增强（移动端、多环境、跨应用主数据、自由脚本、丰富 Connector）
贯穿验收样例：「客户-订单-审批」应用（见 §4）
```

依赖关系：T-002~T-005 依赖 T-001；T-101~T-108 依赖 M0；T-201~T-214 依赖 M1 对应 API；T-301~T-320 依赖 M2 或对应运行时底座。
M1 运行态采用两阶段接入，避免 CRUD 与权限互相阻塞：T-102a 先实现仅 tenant/deleted 注入的无权限 CRUD 骨架，T-103 接入 AccessView，T-102b 在权限约束下复验 CRUD 和导入/导出写入口。
每张任务卡开工前必须输出接口清单、契约冻结时点和下游通知对象；无下游依赖时写“无”。
任务卡引用 `proposed` ADR 时，PRD 已确认的业务不变量继续生效，但 ADR 中的精确类名、字段、表结构、枚举、状态机和算法仅是候选。候选实现进入代码、DDL、公共 DTO 或阻断性测试前，必须先把 ADR 终审为 `accepted`/`accepted for Mx`，或形成等价的已批准版本化决策记录，并同步详细设计与测试规格。

- **质量锚点**：正确性/一致性、可审计性、可测试性；覆盖 A1/A3/A5/A6/E3/H2

### T-005 MetaGraph 元数据缓存
- **依赖**：T-003
- **输入**：01 文档 §3.3

| T-302 插件生命周期 | install/enable/disable/upgrade/rollback/uninstall | 插件失败回滚、双租户隔离、禁用后菜单/路由/字段组件消失 |

| T-306 通知通道 | 模板字段白名单、收件人权限校验、outbox 重试 | 无权限收件人不发送；敏感字段模板被拒；死信可重放 |
| T-307 业务对象扩展与行业模板 | 标准对象、行业模板、客户扩展层、插件扩展层、升级合并报告 | 标准对象升级不覆盖客户字段/规则/页面；冲突必须出报告并阻断 |
| T-308 单据转换、反写与引用追踪 | LinkConfig、Conversion、WriteBack、LinkTrace 引擎骨架 | 订单转出库、出库转收款样例可追踪来源、反写数量/金额并防超额 |

| T-310 报表、打印、菜单、门户、移动端与多语言 | Report、PrintTemplate、Menu、I18nResource 元数据与权限边界 | 报表不是列表导出；打印模板有版本归档；菜单/门户/移动端入口按角色裁剪；i18n 不写死 |
| T-311 ConvertRule 单据下推 | ConvertRule、下推动作、LinkTrace、1:1 单级下推 | 订单下推发货单样例可生成目标单并支持上查/下查 |
| T-312 打印模板运行时 | PrintTemplate 服务端渲染、模板版本、字段脱敏、审计 | 送货单打印 HTML/PDF，旧模板可回放，敏感字段被裁剪 |
| T-313 Webhook 与外部事件订阅 | EventSubscription webhook、URL guard、签名、重试、死信 | webhook 不可访问内网/metadata，失败进入死信可重放 |

| T-319 回收站 | 应用级回收站、恢复、彻底删除、保留期 | 恢复校验唯一约束和引用，彻底删除写审计 |
