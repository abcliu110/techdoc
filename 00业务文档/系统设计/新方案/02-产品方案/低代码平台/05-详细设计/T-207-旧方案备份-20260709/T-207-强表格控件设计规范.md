# T-207 强表格控件设计规范

> 适用范围：表单设计器、单据设计器、Subtable 分录、列表页、参照弹窗、运行态 Renderer  
> 版本：v1.0  
> 日期：2026-07-08  
> 定位：强表格控件是 T-207 的核心能力之一，与强 Web 布局能力、丰富表单元素同级。
> 竞品参考：Web 嵌套表格以 AG Grid Enterprise 为第一参考；Delphi/VCL 传统商业控件以 DevExpress ExpressQuantumGrid/cxGrid 为结构参考。

---

## 1. 核心结论

企业低代码平台的单据能力，成败很大程度取决于表格控件。

T-207 的表格不能只是“明细行组件”，必须形成独立的 **TableEngine**：

| 核心能力 | 说明 |
|---|---|
| 表格布局 | 列宽、冻结列、横向滚动、表头分组、行高、密度 |
| 单元格编辑 | 文本、金额、日期、参照、下拉、附件、公式等单元格控件 |
| 行列操作 | 增行、删行、复制行、批量填充、拖拽排序、列显隐 |
| 数据能力 | 分页、虚拟滚动、远程加载、排序、过滤、汇总 |
| 权限能力 | 列权限、单元格权限、行级权限、操作权限、脱敏 |
| 规则能力 | 单元格校验、行内公式、跨行汇总、联动带出 |
| 企业集成 | 导入、导出、打印、复制粘贴、键盘录入、审计 |
| 设计态能力 | 可视化配置列、编辑单元格控件、约束校验、运行一致 |

判断是否达标：销售订单分录、采购入库分录、费用报销明细、报价明细、库存明细这类复杂表格必须能设计、预览、发布和运行。

竞品参考结论详见 `T-207-强表格控件竞品参考与设计吸收.md`。

## 2. 适用场景

| 场景 | 表格形态 |
|---|---|
| 单据分录 | 可编辑明细行，支持批量录入、公式、汇总 |
| 列表页 | 只读或半可编辑列表，支持排序、过滤、批量操作 |
| 主数据子集 | 组织联系人、收货地址、银行账号等子对象 |
| 参照选择 | 弹窗表格，支持搜索、分页、多选、权限过滤 |
| 审批/日志 | 只读表格，支持时间线和状态 |
| 导入校验 | 错误行、错误列、批量修复 |

## 3. 表格结构模型

### 3.1 TableNode

```json
{
  "nodeId": "entry_sales_order_lines",
  "type": "table",
  "component": "EditableTable",
  "dataSource": {
    "kind": "boChild",
    "childRef": "salesOrder.lines",
    "rowKey": "lineId"
  },
  "layout": {
    "heightMode": "auto",
    "maxHeight": 420,
    "density": "compact",
    "stickyHeader": true,
    "horizontalScroll": true,
    "frozenLeft": 2,
    "frozenRight": 1
  },
  "columns": []
}
```

### 3.2 ColumnSchema

```json
{
  "columnId": "col_material",
  "fieldRef": "materialId",
  "title": "物料",
  "component": "ReferenceSelect",
  "width": 220,
  "minWidth": 180,
  "fixed": "left",
  "editable": true,
  "required": true,
  "permissionRef": "perm_material",
  "rules": {
    "onChange": ["bringMaterialSpec", "bringUnit"]
  }
}
```

### 3.3 CellState

| 状态 | 说明 |
|---|---|
| default | 正常展示 |
| editing | 单元格正在编辑 |
| dirty | 单元格值未保存 |
| error | 单元格校验失败 |
| readonly | 可见不可编辑 |
| disabled | 当前状态不可操作 |
| masked | 脱敏展示 |
| selected | 设计态或运行态选择 |
| copied | 复制范围 |
| formula | 公式计算值 |

## 4. P0 表格能力

| 能力 | P0 要求 |
|---|---|
| 列配置 | 标题、字段绑定、控件类型、宽度、最小宽度、显隐、排序 |
| 编辑控件 | Text、Number、Money、Date、Select、Reference、Attachment、DisplayText |
| 冻结列 | 左侧至少 2 列、右侧操作列可冻结 |
| 横向滚动 | 列超宽时横向滚动，表头和单元格对齐 |
| Sticky 表头 | 局部滚动时表头固定 |
| 行操作 | 新增、删除、复制、上移、下移、批量删除 |
| 键盘录入 | Tab/Enter 移动，Esc 取消，复制粘贴区域 |
| 校验定位 | 单元格错误定位到行号、列名、字段 |
| 行级权限 | 行可编辑、只读、隐藏操作 |
| 列权限 | 列可见、只读、脱敏 |
| 汇总 | 金额列求和，汇总区与字段联动 |
| 性能 | 1000 行 x 30 列可滚动，编辑单元格不全表重渲染 |
| 母子嵌套 | 行展开后可显示子表格、页签、表单片段或卡片 |
| 多层嵌套 | 默认允许 2 层，管理员可配置到 3 层，超过需发布风险确认 |

## 5. P1/P2 增强能力

| 能力 | 优先级 | 说明 |
|---|---|---|
| 表头分组 | P1 | 多级表头，适合财务、库存明细 |
| 列拖拽排序 | P1 | 设计态和运行态可选 |
| 列宽拖拽 | P1 | 支持 px 和自适应策略 |
| 高级过滤 | P1 | 列过滤、条件组合、快捷过滤 |
| 单元格批量填充 | P1 | 向下填充、批量替换 |
| 行虚拟化 | P1 | 大数据量渲染 |
| 单元格虚拟化 | P2 | 超宽表格性能优化 |
| 树表 | P2 | 分层明细、BOM、组织结构 |
| 透视汇总 | P2 | 非首版，避免变成报表设计器 |

## 6. 设计态交互

| 交互 | 要求 |
|---|---|
| 选择表格 | 外框蓝色，右侧属性进入 Table 页 |
| 选择列 | 表头列高亮，右侧属性进入 Column 页 |
| 选择单元格控件 | 可配置控件类型、字段绑定、校验、权限 |
| 拖入字段到表格 | 自动新增列，生成默认列宽和控件 |
| 拖入控件到列 | 替换单元格编辑控件，保持 fieldRef |
| 调整列宽 | 显示 px/minWidth，不能小于最小宽度 |
| 冻结列配置 | 表头显示冻结分割线，右侧属性显示 fixed |
| 校验失败 | 定位到表格、行、列、单元格 |

---

## 7. 运行态交互

| 交互 | 要求 |
|---|---|
| 单击 | 选中单元格 |
| 双击/Enter | 进入编辑 |
| Tab | 下一个可编辑单元格 |
| Shift+Tab | 上一个可编辑单元格 |
| Ctrl+C/Ctrl+V | 支持区域复制粘贴，按权限过滤 |
| 行新增 | 默认值、带出规则、行号更新 |
| 行删除 | 二次确认或撤销，触发汇总重算 |
| 参照选择 | 支持弹窗、分页、权限过滤、带出字段 |
| 批量导入 | 错误行可定位、可下载错误报告 |
| 展开明细 | 行展开后懒加载子表，支持一行多个 detail 页签 |
| 嵌套编辑 | 子表单元格可编辑，错误路径能回到父行和子行 |

## 8. 权限与规则

| 类型 | 要求 |
|---|---|
| 表格权限 | 整个表格 hidden/readonly/editable |
| 列权限 | 列 hidden/readonly/masked/editable |
| 行权限 | 按行数据、状态、角色决定编辑和操作 |
| 单元格权限 | 行权限和列权限交叉后取最严格状态 |
| 操作权限 | 新增、删除、复制、导入、导出按角色和状态控制 |
| 校验规则 | 单元格、行、表格、汇总四级校验 |
| 计算规则 | 行内计算、跨行汇总、主表回填 |

---

## 9. 性能要求

| 指标 | P0 目标 |
|---|---|
| 表格规模 | 1000 行 x 30 列可滚动 |
| 首屏渲染 | 复杂单据页面 2 秒内可交互，最终指标由实现阶段压测固化 |
| 单元格编辑 | 不触发全表重渲染 |
| 横向滚动 | 表头和单元格不错位 |
| 冻结列 | 滚动时冻结列不抖动 |
| 规则计算 | 单行变化只重算相关行、汇总和依赖字段 |
| 嵌套展开 | 子表默认懒加载，展开行不导致父表滚动明显卡顿 |

## 10. Figma 原型验收

后续 Figma 必须新增或重画 `10 Table Engine / 强表格控件` 画板，并满足：

| 编号 | 标准 |
|---|---|
| TABLE-001 | 展示企业单据分录表格，而不是简单静态表 |
| TABLE-002 | 展示冻结列、横向滚动、Sticky 表头、右侧操作列 |
| TABLE-003 | 展示单元格编辑态、错误态、只读态、脱敏态 |
| TABLE-004 | 展示列属性配置：fieldRef、component、width、fixed、required、permission |
| TABLE-005 | 展示行操作：新增、删除、复制、批量操作 |
| TABLE-006 | 展示汇总区：金额合计、税额、折扣或总金额 |
| TABLE-007 | 视觉保持灰白企业工具风格，不做彩色卡片墙 |
| TABLE-008 | 文本不换行错乱、不重叠、不裁切，列宽与行高稳定 |
| TABLE-009 | 展示母子嵌套表格：父分录行展开后出现子表或 detail tabs |
| TABLE-010 | 展示嵌套表格的懒加载、高度控制、错误路径和权限继承 |

## 11. 不通过判定

出现以下任一情况，表格能力不得验收：

| 不通过项 | 原因 |
|---|---|
| 表格只是普通 HTML 表格展示 | 不能支撑企业单据分录 |
| 不支持冻结列和横向滚动 | 无法承载宽表 |
| 不支持单元格级编辑和校验 | 无法支撑复杂录入 |
| 不支持行/列/单元格权限 | 企业权限模型不完整 |
| 不支持母子嵌套 detail grid | 无法对标成熟商业 Web 表格 |
| 编辑单元格触发全表重渲染 | 大数据量不可用 |
| 表头与单元格滚动错位 | 基础可用性失败 |
| 原型中文本裁切或重叠 | 视觉验收失败 |
