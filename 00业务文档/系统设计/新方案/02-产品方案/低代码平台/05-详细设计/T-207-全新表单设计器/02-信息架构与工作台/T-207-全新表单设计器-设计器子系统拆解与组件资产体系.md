# T-207 全新表单设计器：设计器子系统拆解与组件资产体系

> 阶段：02 信息架构与工作台回退修复
> 状态：draft
> 日期：2026-07-09
> 触发原因：09 原型人工预评审发现左侧工具栏/组件面板未形成稳定组件资产体系，说明上游工作台信息架构和功能规格颗粒度不足。

---

## 1. 回退结论

表单设计器不是单一页面，而是多个复杂设计子系统组成的 IDE。每个子系统单独拿出来都具备复杂产品形态，不能只在原型中以“某个区域”或“某个能力名称”带过。

本次回退将“设计器子系统”作为阶段 2 的强制建模对象。任何后续原型、交互图、测试规格和技术设计，必须先追溯到本文件定义的子系统职责、对象模型、状态机和自动化抓手。

## 2. 设计器子系统矩阵

| 子系统 | 主要用户动作 | 核心对象 | 必须定义的状态 | 自动化抓手前缀 | 原型门禁 |
|---|---|---|---|---|---|
| Top Command Bar | 保存、撤销、重做、预览、发布、版本、回滚、校验 | Command、Draft、PublishedVersion、GateResult | normal、dirty、saving、publishing、blocked、success、disabled | `top-command-*`、`publish-dialog-*`、`version-*` | 必须展示命令分组、禁用原因、发布/回滚弹窗 |
| Left Activity Bar | 切换组件、字段、结构、问题、版本、数据 | DockMode、ActivityItem | active、hover、collapsed、disabled | `activity-item-*`、`left-dock-*` | 必须展示模式切换，不允许左侧语义混用 |
| Component Palette | 搜索、浏览、拖拽组件 | ComponentAsset、PaletteGroup、ComponentCard | normal、hover、selected、dragging、disabled、unavailable、empty | `palette-*`、`drag-source-*`、`drag-ghost-*` | 必须分组展示布局容器、基础表单、展示与状态、操作命令、业务套件、表格明细、模板片段；桌面态采用三列紧凑图标卡，默认只显示图标、中文名称和拖拽抓手，说明进入 tooltip 或详情；字段资产不混入组件库 |
| BO Field Tree | 浏览业务对象字段、分录、系统字段、关联字段 | BusinessObject、FieldAsset、EntryAsset | loading、ready、empty、readonly、no-permission、dragging | `bo-field-*`、`binding-asset-*` | 必须与 Component Palette 分离，不得混成同一树 |
| Outline / Page Structure | 定位、展开折叠、重排、跨容器移动 | PageNode、ContainerNode、FieldNode、TableNode | synced、selected、dragging、invalid-move | `outline-*`、`designer-node-*` | 必须展示复杂嵌套页面结构 |
| Center Design Surface | 拖放、选中、框选、resize、snap、断点预览 | DesignNode、DropZone、Handle、Selection | empty、dragging、drop-legal、drop-illegal、selected、resizing、preview | `designer-*`、`dropzone-*` | 必须展示合法/非法投放和原因 |
| Right Inspector | 配属性、布局、数据、校验、权限、事件、性能 | InspectorTab、PropertyEditor、SchemaPath | no-selection、single-selection、multi-selection、dirty、invalid、readonly | `inspector-*` | 必须按选中对象切换属性集 |
| Bottom Analyzer | 查看错误、定位、Quick Fix、发布 Gate | AnalyzerIssue、QuickFix、GatePanel | empty、warning、error、active、fixed、blocked | `analyzer-*`、`publish-gate-*` | 必须能从问题定位到画布和 Inspector |
| Table Designer | 列设计、冻结、汇总、明细、虚拟滚动、分页 | EntryTable、Column、DetailPanel、VirtualWindow | selected、column-dragging、resizing、large-data、detail-expanded | `table-*` | 必须覆盖大表和 Level 1 明细 |
| Publish / Version | 发布、阻断、查看版本、回滚、证据包 | DraftVersion、PublishedVersion、ArchivedVersion、RollbackDraft、PublishEvidence | draft、stale、publishing、blocked、published、archived、rollback-confirm、rollback-draft-created | `publish-*`、`version-*`、`rollback-*`、`schema-hash-*` | 必须展示发布阻断、发布成功、版本历史、回滚为新 draft 和 active published 不变 |

## 3. Component Palette 信息架构

### 3.1 左侧结构

左侧不再定义为单一 `Left Resource Dock` 树列表，而是由两层组成：

```text
Left Dock
├─ Activity Bar
│  ├─ 组件 Components
│  ├─ 字段 Fields
│  ├─ 结构 Outline
│  ├─ 问题 Issues
│  ├─ 版本 Versions
│  └─ 数据 Data
└─ Dock Panel
   ├─ Component Palette
   ├─ BO Field Tree
   ├─ Outline Tree
   ├─ Issue Filter
   ├─ Version List
   └─ Data Source Panel
```

默认打开 `Component Palette`。`BO Field Tree` 是字段资产视图，不等同于组件面板。`Outline Tree` 是页面结构视图，不承担组件投放入口。

### 3.2 Component Palette 分组

| 分组 | data-testid | P0 组件 | 用途 |
|---|---|---|---|
| 布局容器 | `palette-group-layout` | 网格布局、弹性布局、分组面板、页签容器、分栏容器、滚动区域、固定栏、折叠面板 | 组织 PC Web 页面结构 |
| 基础表单 | `palette-group-input` | 单行文本、多行文本、数值输入、金额输入、日期选择、日期时间、下拉选择、单选组、复选框、开关、基础资料选择、附件上传 | 搭建基础输入和选择 |
| 展示与状态 | `palette-group-display` | 文本显示、只读字段、状态标签、提示说明、图片、分隔线、统计摘要、进度状态 | 搭建只读展示、状态和说明 |
| 操作命令 | `palette-group-action` | 普通按钮、主按钮、按钮组、更多菜单、工具栏、行操作、提交操作、返回操作 | 搭建页面和表格操作入口 |
| 业务套件 | `palette-group-bill` | 单据标题、单据头容器、汇总栏、单据操作栏、审批信息面板、附件面板、关联单据、右侧辅助面板 | 快速搭建单据语义区块；业务套件只提供容器/复合区块，字段资产仍从 BO Field Tree 拖入 |
| 表格明细 | `palette-group-table` | 分录表格、汇总行、行操作、明细面板、明细表格、分页栏、表格工具栏 | 搭建分录、行操作、汇总和一层明细；表格列由分录字段资产绑定生成，不作为独立 Palette 组件拖入 |
| 模板片段 | `palette-group-template` | 销售订单头模板、分录区模板、审批布局模板、左右分栏单据模板、主从表单模板、常用查询区模板 | 快速生成常用单据结构 |

### 3.3 组件卡片形态

每个组件资产必须呈现为可拖拽卡片，而不是普通树行。

| 区域 | 要求 |
|---|---|
| 图标 | 表示组件类型，如布局、输入、业务区块、表格、模板 |
| 名称 | 展示中文业务可理解名称，如“网格布局”“基础资料选择”“分录表格”；技术类型只作为 Schema 元数据和测试合同 |
| 用途说明 | 一行以内说明投放后生成的对象 |
| 能力标签 | 如 `layout`、`field`、`table`、`business`、`template` |
| 投放约束 | 展示可投放容器或不可投放原因 |
| 拖拽抓手 | 卡片整体可拖，显式 handle 可选 |
| 状态 | normal、hover、selected、dragging、disabled、unavailable、no-permission |

### 3.4 组件资产元模型

```text
ComponentAsset
├─ assetId
├─ groupCode
├─ displayName
├─ icon
├─ description
├─ kind: layout | basic | business | table | template
├─ defaultSchemaFactory
├─ allowedTargets
├─ blockedTargets
├─ requiredCapability
├─ permissionPolicy
├─ testId
└─ tags
```

`defaultSchemaFactory` 决定组件投放后生成的初始 `ComponentSchema`。`allowedTargets` 和 `blockedTargets` 必须参与拖拽过程中的合法/非法投放判断。字段资产不是 `ComponentAsset`，不得在 Component Palette 中作为组件展示；真实所有权属于 `FieldAsset / EntryAsset`，入口为 `activity-item-fields` 对应的 BO Field Tree。

### 3.5 字段资产动作语义

| 动作 | 触发 | 结果 | 不允许 |
|---|---|---|---|
| create-new-node | 字段资产拖到 PageRoot、Grid cell、Section body、Flex slot | 根据字段类型生成 FieldNode 或业务组件，并写入 binding | 不得生成无 binding 的空控件 |
| rebind-existing-component | 字段资产拖到已有 FieldNode 或业务组件绑定区 | 更新目标节点 binding、editor 默认值和 Inspector 数据 Tab | 不得新建兄弟节点 |
| bind-table-column | 分录字段拖到目标级列投放区 `designer-dropzone-{tableId}-columns` | 新建或更新 TableColumn.binding，并选中列 Inspector | 不得把列写入 `children` |
| reject | 字段资产拖到不兼容目标 | 显示目标级 illegal dropzone 和 reject reason | 不得写 Schema patch |

## 4. 拖拽与投放合同

### 4.1 主路径

```text
打开设计器
  -> Activity Bar 选择 Components
  -> 展开布局容器
  -> 从 palette-item-grid 拖入 dropzone-page-root
  -> 生成 designer-node-grid 并自动选中
  -> 从 binding-asset-field-salesOrder-customerId 拖入 Grid cell
  -> 生成 designer-node-business-field-customerId
  -> 从 palette-item-entry-table 拖入 Section
  -> 生成 designer-node-entry-table
  -> 从 binding-asset-field-salesOrder-entries-materialId 拖入 table columns dropzone
  -> 生成 table-column-materialId
  -> Inspector 展示对应 schemaPath
  -> Analyzer 无 P0 Error
```

### 4.2 非法投放

| 情况 | UI 反馈 | Analyzer / 状态 |
|---|---|---|
| 表格列拖到普通 Section body | dropzone 显示 illegal | `drop-reject-reason-table-column-target` |
| DetailTable 拖入 Field cell | dropzone 显示 illegal | `drop-reject-reason-detail-target` |
| Level 2 明细投放 | dropzone 显示 illegal | `TABLE_DETAIL_DEPTH_EXCEEDED` |
| P1 组件在 P0 阶段投放 | 卡片 disabled 或 unavailable | 对应 `*_NOT_ENABLED` |
| 无权限组件投放 | 卡片 no-permission | `COMPONENT_PERMISSION_DENIED` |

## 5. 自动化抓手合同

| 对象 | 抓手 |
|---|---|
| 左侧根 | `left-dock-root` |
| Activity Bar | `activity-bar` |
| Activity Item | `activity-item-{mode}` |
| Dock Panel | `left-dock-panel-{mode}` |
| Palette 根 | `palette-root` |
| Palette 搜索 | `palette-search` |
| Palette 分组 | `palette-group-{groupCode}` |
| Palette 分组开关 | `palette-group-{groupCode}-toggle` |
| Palette 分组计数 | `palette-group-{groupCode}-count` |
| Palette 组件项 | `palette-item-{assetId}` |
| Palette 组件拖拽源 | `drag-source-palette-item-{assetId}` |
| BO 字段资产 | `binding-asset-field-{fieldPath}` |
| 拖拽影子 | `drag-ghost`、`drag-ghost-kind-{kind}` |
| 合法投放 | 正式合同使用 `dropzone-state-{containerId}-{slot}-legal`；`dropzone-state-legal` 仅保留为单一投放点兼容示意 |
| 非法投放 | 正式合同使用 `dropzone-state-{containerId}-{slot}-illegal`；`dropzone-state-illegal` 仅保留为单一投放点兼容示意 |
| 非法原因 | 正式合同使用 `drop-reject-reason-{containerId}-{slot}-{reasonCode}`；`drop-reject-reason-{reasonCode}` 仅保留为单一投放点兼容示意 |
| 目标级合法投放 | `dropzone-state-{containerId}-{slot}-legal` |
| 目标级非法投放 | `dropzone-state-{containerId}-{slot}-illegal` |
| 目标级非法原因 | `drop-reject-reason-{containerId}-{slot}-{reasonCode}` |
| 投放完成节点 | `designer-node-{nodeId}` |
| 当前 schemaPath | `inspector-current-schema-path` |
| 当前 Inspector 节点 | `inspector-current-node-{nodeId}` |
| Inspector 当前 Tab | `inspector-tab-{tabId}`、`inspector-tab-{tabId}-active` |
| Analyzer 状态 | `analyzer-run`、`analyzer-status-running`、`analyzer-status-idle` |
| 表格列投放区 | `designer-dropzone-{tableId}-columns`、`dropzone-state-{tableId}-columns-{legal|illegal}` |
| 表格明细投放区 | `designer-dropzone-{tableId}-detail`、`dropzone-state-{tableId}-detail-{legal|illegal}` |

缺少上述 P0 抓手时，不允许 09 原型通过人工检查，也不允许进入测试规格生成。

## 6. 对 09 原型的重做要求

09 原型必须新增或重画以下图件：

| 图件 | 要求 |
|---|---|
| Component Palette 默认态 | 展示 Activity Bar、搜索、七个中文组件分组和企业级三列紧凑图标卡 |
| Component Palette 搜索态 | 搜索结果按组件类型分组，保留空结果态 |
| Component Palette 不可投放态 | 卡片 disabled/unavailable，说明能力未启用或目标不合法 |
| BO Field Tree 独立态 | 与组件面板分离，展示 Header、Entries、系统字段、计算字段、关联字段 |
| Outline Tree 独立态 | 展示复杂页面结构，和画布选中同步 |
| Outline Tree 状态态 | 展示选中同步、展开折叠、empty、readonly、no-permission、dragging、invalid-move |
| 合法拖拽态 | drag ghost、legal dropzone、投放后 selected node 和 Inspector schemaPath |
| 非法拖拽态 | illegal dropzone、reject reason、释放后不生成节点 |
| 端到端建模主路径 | 从 Component Palette 拖网格布局和分录表格，从 BO Field Tree 拖字段生成字段控件和表格列，再配置 Inspector 并通过 Analyzer |
| Publish / Version 态 | 展示 stale draft、发布阻断、发布成功、版本历史、回滚确认、回滚生成新 draft、active published 不变 |

## 7. 防复发门禁

后续每个设计器主区域都必须回答以下问题，未回答不得进入原型阶段：

1. 这个区域的一级对象是什么。
2. 用户可以执行哪些动作。
3. 每个动作对应哪些 UI 状态。
4. 哪些状态需要单独原型图。
5. 每个可点击、可拖拽、可调整对象的 `data-testid` 是什么。
6. 操作完成信号是什么。
7. 非法操作如何反馈。
8. 对应 Schema patch 或状态变化是什么。
9. Analyzer 是否参与校验。
10. 人工检查如何判断该区域通过。
