## 一、快速开始

### 1.1 安装

```bash
pnpm add @form-designer/core @form-designer/renderer @form-designer/components
```

### 1.2 Hello World

```tsx
import { FormModel, FormRenderer } from '@form-designer/renderer';
import { BusinessObject, FormDefinition } from '@form-designer/core';

// 1. 定义业务对象
const UserBO: BusinessObject = {
  boId: 'user',
  boName: '用户',
  fields: [
    { fieldId: 'username', fieldName: '用户名', fieldType: 'string', required: true },
    { fieldId: 'email', fieldName: '邮箱', fieldType: 'string', required: true },
  ],
};

// 2. 定义表单
const UserForm: FormDefinition = {
  formId: 'user_form',
  formName: '用户表单',
  boId: 'user',
  formType: 'edit',
  fields: [
    {
      fieldId: 'username',
      boField: 'username',
      componentType: 'input',
      label: '用户名',
    },
    {
      fieldId: 'email',
      boField: 'email',
      componentType: 'input',
      label: '邮箱',
    },
  ],
};

// 3. 创建并使用
function App() {
  const formModel = new FormModel(UserForm, UserBO);
  
  const handleSubmit = async () => {
    const valid = await formModel.validate();
    if (valid) {
      const data = formModel.getValues();
      console.log('提交数据:', data);
    }
  };
  
  return <FormRenderer formModel={formModel} onSubmit={handleSubmit} />;
}
```

## 二、核心概念

### 2.1 MVVM架构

```
View (React组件)
  ↕ (MobX响应式)
Model (Field/Form Model)
  ↕
Data (纯数据)
```

**关键点**：
- View只负责渲染，不包含业务逻辑
- Model管理状态、校验、计算
- 通过MobX实现自动更新

### 2.2 元数据驱动

```
Business Object (BO)    业务逻辑定义
    ↓
Form Definition         UI展现定义
    ↓
Form Model (Runtime)    运行时实例
```

**关键点**：
- BO定义业务规则（必填、唯一、计算等）
- Form定义UI展现（组件类型、布局）
- 一个BO可以有多个Form

### 2.3 路径系统

支持复杂路径访问：

```typescript
// 简单路径
formModel.getField('username')

// 嵌套路径
formModel.getField('user.address.city')

// 数组路径
formModel.getField('items[0].product')

// 通配符（表达式中）
'${SUM(items.*.amount)}'
```

## 三、BO定义指南

### 3.1 基本字段定义

```typescript
interface BOField {
  fieldId: string;           // 字段ID
  fieldName: string;         // 字段名称
  fieldType: 'string' | 'number' | 'date' | 'boolean' | 'reference' | 'calculated';
  dataType: string;          // 数据类型
  required?: boolean;        // 必填
  unique?: boolean;          // 唯一
  defaultValue?: any;        // 默认值
}
```

**示例**：

```typescript
const OrderBO: BusinessObject = {
  boId: 'sales_order',
  boName: '销售订单',
  fields: [
    {
      fieldId: 'order_no',
      fieldName: '订单号',
      fieldType: 'string',
      dataType: 'varchar',
      length: 50,
      required: true,
      unique: true,
    },
    {
      fieldId: 'order_date',
      fieldName: '订单日期',
      fieldType: 'date',
      dataType: 'date',
      required: true,
      defaultValue: '${TODAY()}',
    },
  ],
};
```

### 3.2 计算字段

```typescript
{
  fieldId: 'total_amount',
  fieldName: '订单总额',
  fieldType: 'calculated',
  dataType: 'decimal',
  precision: 2,
  formula: '${SUM(items.*.amount)}',  // 引用子表聚合
}
```

### 3.3 引用字段

```typescript
{
  fieldId: 'customer_id',
  fieldName: '客户',
  fieldType: 'reference',
  refBO: 'customer',         // 引用customer BO
  refField: 'customer_id',
  required: true,
}
```

### 3.4 主子表关系

```typescript
relations: [
  {
    relationId: 'order_items',
    relationType: 'one-to-many',
    sourceBO: 'sales_order',
    targetBO: 'order_item',
    sourceField: 'order_id',
    targetField: 'order_id',
    cascadeDelete: true,
  },
]
```

## 五、Field Model API

### 5.1 基本属性

```typescript
class FieldModel {
  readonly fieldId: string;     // 字段ID
  readonly boFieldId: string;   // BO字段ID
  
  value: any;                   // 当前值
  initialValue: any;            // 初始值
  errors: string[];             // 错误信息
  
  visible: boolean;             // 是否可见
  disabled: boolean;            // 是否禁用
  readonly: boolean;            // 是否只读
  touched: boolean;             // 是否被触摸
  dirty: boolean;               // 是否被修改
}
```

### 5.2 常用方法

```typescript
// 获取值
const value = fieldModel.getValue();

// 设置值
fieldModel.setValue('new value');

// 校验
const valid = await fieldModel.validate();
if (!valid) {
  console.log('错误:', fieldModel.errors);
}

// 重置
fieldModel.reset();

// 设置可见性
fieldModel.setVisible(false);

// 设置禁用
fieldModel.setDisabled(true);
```

### 5.3 监听变化

```typescript
// 监听值变化
fieldModel.onChange((newValue, oldValue) => {
  console.log('值变化:', oldValue, '->', newValue);
});

// 监听错误变化
fieldModel.onErrorsChange((errors) => {
  console.log('错误:', errors);
});
```

## 六、Form Model API

### 6.1 创建实例

```typescript
const formModel = new FormModel(formDefinition, businessObject);
```

### 6.2 字段访问

```typescript
// 获取字段（支持路径）
const field = formModel.getField('username');
const nestedField = formModel.getField('user.address.city');
const arrayField = formModel.getField('items[0].product');

// 设置字段值
formModel.setFieldValue('username', 'john');
formModel.setFieldValue('items[0].qty', 10);
```

### 6.3 表单数据

```typescript
// 获取所有数据
const data = formModel.getValues();

// 设置所有数据
formModel.setValues({
  username: 'john',
  email: 'john@example.com',
  items: [
    { product: 'A', qty: 10, price: 100 },
  ],
});

// 获取变化的数据
const changes = formModel.getChangedValues();
```

### 6.4 校验

```typescript
// 校验整个表单
const valid = await formModel.validate();

// 校验单个字段
const fieldValid = await formModel.validateField('username');

// 获取所有错误
const allErrors = formModel.getAllErrors();
// { username: ['用户名不能为空'], email: ['邮箱格式不正确'] }
```

### 6.5 状态管理

```typescript
// 重置表单
formModel.reset();

// 提交表单
formModel.submit();

// 检查是否被修改
if (formModel.isDirty()) {
  console.log('表单有修改');
}

// 检查是否有效
if (formModel.isValid()) {
  console.log('表单有效');
}
```

## 八、表达式引擎

### 8.1 基本语法

表达式用`${}`包裹：

```typescript
'${username}'                    // 字段引用
'${user.address.city}'          // 嵌套字段
'${qty * price}'                // 计算
'${country === "China"}'        // 条件
'${SUM(items.*.amount)}'        // 聚合函数
```

### 8.2 内置函数

**数学函数**：

```typescript
SUM(path)       // 求和: SUM(items.*.amount)
AVG(path)       // 平均: AVG(items.*.price)
MAX(path)       // 最大: MAX(items.*.qty)
MIN(path)       // 最小: MIN(items.*.qty)
COUNT(path)     // 计数: COUNT(items.*)
```

**日期函数**：

```typescript
TODAY()         // 今天
NOW()           // 现在
DATE(str)       // 解析日期
FORMAT_DATE(date, format)  // 格式化
```

**字符串函数**：

```typescript
UPPER(str)      // 大写
LOWER(str)      // 小写
TRIM(str)       // 去空格
CONCAT(...)     // 连接
```

**条件函数**：

```typescript
IF(condition, true_value, false_value)
SWITCH(value, case1, result1, case2, result2, default)
```

### 8.3 自定义函数

```typescript
expressionEngine.registerFunction('DISCOUNT', (amount, rate) => {
  return amount * (1 - rate / 100);
```

## 十、最佳实践

### 10.1 BO设计原则

1. ✅ **单一职责**：一个BO只负责一个业务实体
2. ✅ **字段语义化**：字段ID和名称要清晰
3. ✅ **计算字段集中**：所有计算逻辑放在BO层
4. ✅ **关系明确**：主子表关系要明确定义

### 10.2 Form设计原则

1. ✅ **绑定BO字段**：不要自创字段，都绑定BO
2. ✅ **适配场景**：不同场景用不同Form（新建、编辑、查询）
3. ✅ **表单级校验**：可以在Form层增加额外校验
4. ✅ **布局合理**：使用Grid等布局组件

### 10.3 性能优化

1. ✅ **使用MobX**：自动精确更新
2. ✅ **大表单分页**：超过50个字段考虑分页
3. ✅ **虚拟滚动**：子表超过100行用虚拟滚动
4. ✅ **按需加载**：组件懒加载

### 10.4 常见问题

**Q: 如何实现字段联动？**

A: 使用reactions或计算字段：

```typescript
reactions: [{
  condition: '${country === "China"}',
  target: 'province',
  action: 'set-visible',
  value: true,
}]
```

**Q: 如何实现复杂校验？**

A: 在BO层定义校验规则：

```typescript
validations: [{
  type: 'custom',
  validator: (value, formData) => {
    if (value < formData.min_value) {
      return '不能小于最小值';
    }
    return true;
  },
}]
```

**Q: 如何优化大表单性能？**

A: 
1. 使用依赖追踪（自动）
2. 分页/分步
3. 虚拟滚动
4. 组件懒加载
