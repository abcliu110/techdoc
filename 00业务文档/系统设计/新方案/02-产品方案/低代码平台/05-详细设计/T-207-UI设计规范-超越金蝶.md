## 一、金蝶BOS的问题分析

### 1.1 金蝶BOS的优点

✅ 功能强大，企业级成熟  
✅ 支持复杂业务场景  
✅ BO概念清晰  

### 1.2 金蝶BOS的痛点

❌ **学习曲线陡峭**：新手需要培训3-5天  
❌ **界面老旧**：Windows桌面风格，不够现代  
❌ **信息密度低**：大量空白，需要频繁切换  
❌ **操作繁琐**：很多操作需要多步完成  
❌ **缺少智能提示**：完全靠手工配置  
❌ **预览不方便**：设计态和运行态差异大

## 二、革新性设计方案

### 2.1 核心设计理念

```
专业性（金蝶的优点）
    +
现代化（VSCode/Figma的体验）
    +
智能化（AI辅助）
    =
超越金蝶的新一代设计器
```

### 2.2 三大创新

#### 创新1：AI智能辅助（降低学习成本）

**功能**：
- 🤖 **自然语言生成表单**："创建一个销售订单表单" → 自动生成
- 🤖 **智能字段推荐**：添加"客户"字段 → 自动推荐"客户电话"、"客户地址"
- 🤖 **布局建议**：分析字段数量和类型 → 推荐最佳布局
- 🤖 **错误预警**：实时检测配置错误，给出修复建议

**示例**：
```
用户输入："我要做一个员工入职表单"
AI自动生成：
  - 姓名（必填）
  - 身份证号（必填、校验格式）
  - 手机号（必填、校验格式）
  - 入职日期（必填、默认今天）
  - 部门（下拉选择）
  - 职位（下拉选择）
  - 附件上传（简历、学历证明）
```

#### 创新2：所见即所得（易用性）

**金蝶的问题**：
- 设计态：灰色背景、控件样式

- ✅ **响应式预览**：一键切换桌面/平板/手机预览

**布局**：

#### 画布节点操作栏可见性策略

**设计原则**：降低认知负担，无需悬停即可看到操作提示

| 节点状态 | 控制栏透明度 | 说明 |
|---|---|---|
| 未选中 | `opacity: 0.6` | 半透明可见，提示存在操作能力 |
| 选中 | `opacity: 1.0` | 完全可见 |
| 鼠标悬停控制栏区域 | `opacity: 1.0` | 悬停时完全可见 |

**为什么不使用 `opacity: 0`？**
- 新手首次使用时无法发现操作能力，学习成本高
- 交互元素（input、textarea、select、button）的点击需要正常响应，不触发节点选中
- 半透明策略兼顾了"可见性提示"和"不干扰预览"两个目标

**交互元素与选择区域的区分**：
- 点击交互元素（input/textarea/select/button）：事件正常传播，不触发节点选中
- 点击节点内容区（非交互元素）：触发节点选中，高亮边框
- 点击画布空白区：取消当前选中
