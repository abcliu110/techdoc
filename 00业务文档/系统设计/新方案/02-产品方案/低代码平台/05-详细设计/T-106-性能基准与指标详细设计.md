# T-106 性能基准与平台指标详细设计

> 版本：v0.1
> 里程碑：M1
> 适用任务：T-106
> 依据：`../03-需求/PRD-产品需求规格说明书.md` REQ-082~084、REQ-090；`NFR-LOWCODE-001`、`MT-LOWCODE-001`；公共工程规范 14/27

---

## 1. 目标

建立可重复生成的数据集、性能脚本和产品指标采集基线。性能指标证明运行时可用，产品指标证明低代码平台确实降低复杂度。

## 2. 基准数据集

最小数据集：

```text
租户：2
应用：每租户 1 个
对象：customer/order/order_item
customer：1 万行/租户
order：1 万行/租户
order_item：每订单 3 行
角色：sales/clerk/manager/auditor
权限：self、dept、字段只读、审批动作
```

数据必须包含：

```text
link
table
currency
select
datetime
attachment metadata
state_code
owner_user_lid
owner_dept_lid
owner_org_path
deleted/delete_token
```

## 3. 性能场景

| 场景 | 目标 |
|---|---|
| MetaGraph 冷加载 | P95 < 2s |
| list 简单分页 | 1 万行 P95 < 500ms |
| list 带 data_scope | P95 < 700ms |
| list 一层 link 展开 | P95 < 900ms |
| update + 规则 + 审计 | P95 < 800ms |
| transition + postRule | P95 < 1000ms |
| 权限判定缓存命中 | P95 < 1ms |
| 发布版本收敛 | 60 秒内健康实例收敛 |

无法达到目标时必须输出数据量、索引、SQL、硬件、P95/P99 和瓶颈分析。

## 4. 平台运营指标

`PlatformMetricEvent`：

```text
metricCode
tenantId
appCode
objectCode
actorType
value
tags
traceId
occurredAt
```

M1/M2/M3 分层采集：

| 指标 | 起始里程碑 | 说明 |
|---|---|---|
| 应用数/对象数/字段数 | M0 | 元模型统计 |
| 已发布应用数 | M0 | Version 状态 |
| 发布失败率 | M0/M1 | 发布任务结果 |
| 权限拦截数 | M1 | PERMISSION_DENY |
| 流程失败率 | M1/M3 | transition/workflow 日志 |
| 页面生成率 | M2 | 默认页面 vs 手工页面 |
| 脚本逃逸率 | M2/M3 | 自定义表达式/脚本/插件逃逸 |
| 平均构建周期 | M2 | 创建应用到首次发布 |
| Connector 失败率 | M3 | Connector 调用结果 |
| License 超限次数 | M3 | 授权模块 |

## 5. 隐私与合规

1. 指标不得包含业务字段值。
2. 指标中的对象 code、字段 code 在 SaaS 汇总上报时可 hash。
3. 私有化模式指标上报默认关闭。
4. 租户级指标只能被租户管理员和平台管理员查看。
5. 全局指标必须聚合到不可反推单租户数据的粒度。

## 6. 验收

1. 基准数据集可一键重建。
2. 性能报告包含硬件、数据量、查询形态、索引、P95/P99。
3. 权限拦截、发布失败、流程失败都能生成指标事件。
4. SaaS 汇总指标不含业务字段值、密钥、附件和用户身份明文。
5. 私有化关闭指标上报后不再产生外发请求。
