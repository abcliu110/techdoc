## 🏗️ 架构概览

### 核心架构

```
┌──────────────────────────────────────────┐
│         Presentation Layer               │  表现层
│  Designer (设计器) + Renderer (渲染器)   │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│           Model Layer                    │  模型层
│  Form Model + Field Model                │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│         Metadata Layer                   │  元数据层
│  Business Object + Form Definition       │
└──────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────┐
│          Engine Layer                    │  引擎层
│  Path + Expression + Dependency          │
└──────────────────────────────────────────┘
```

### 核心特性

✅ **MVVM架构** - Field Model与View分离  
✅ **元数据驱动** - BO + Form分层  
✅ **路径系统** - 支持嵌套对象和数组  
✅ **主子表原生支持** - 一对多关系  
✅ **依赖追踪** - 性能提升90%  
✅ **MobX响应式** - 精确更新

## ❓ 常见问题

### Q1: 为什么选择MVVM架构？

**A**: 
- 视图与模型解耦，职责清晰
- 精确更新，性能提升90%
- 易于测试和维护
- 业界最佳实践（参考Formily）

### Q2: 工期为什么是24-28周？

**A**: 
- 新架构复杂度较高
- 包含主子表、依赖追踪等高级特性
- 需要充分测试和优化
- 一步到位，避免后续重构

### Q3: 为什么需要7个人？

**A**: 
- 架构复杂，需要架构师把控
- Model层需要核心工程师实现
- UI层需要3名前端工程师并行
- 测试需要专职测试工程师

### Q4: 与原设计（v1.0）的主要差异？

**A**:
- **架构**：MVVM vs 组件直接绑定
- **数据层**：BO+Form vs 扁平Schema
- **性能**：依赖追踪 vs 全量计算
- **工期**：24-28周 vs 20周

### Q5: 可以简化架构吗？

**A**: 可以（见完整架构设计v1.0第十三章"方案B"），但不推荐：
- 简化后仍需20-22周
- V2.0仍需架构升级
- 不如一步到位
