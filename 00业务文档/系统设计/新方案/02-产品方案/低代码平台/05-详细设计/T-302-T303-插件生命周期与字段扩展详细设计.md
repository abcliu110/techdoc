## 1. 目标

实现插件安装、启用、禁用、升级、回滚、卸载和字段类型扩展试点。插件是平台能力扩展点，不允许绕过租户隔离、权限、SQL 白名单、表达式沙箱和审计。

## 2. Plugin Manifest

```json
{
  "pluginCode": "sample-field",
  "version": "1.0.0",
  "platformVersionRange": "[1.0,2.0)",
  "capabilities": ["fieldType", "menu", "rendererBlock"],
  "permissions": ["metadata:read", "runtime:field-render"],
  "migrations": [],
  "frontendAssets": [],
  "backendExtensions": [],
  "licenseRequired": false
}
```

Manifest 必须签名，安装前校验。

## 3. 生命周期状态

```text
UPLOADED
VERIFIED
INSTALLED
ENABLED
DISABLED
UPGRADING
ROLLING_BACK
UNINSTALLED
FAILED
```

每次状态变化写审计。

## 4. 插件隔离

1. 插件不能直接访问动态表。
2. 插件后端扩展只能通过 SPI。
3. 插件前端只能注册允许的 Block、FieldRenderer、MenuEntry。
4. 插件禁用后，菜单、路由、字段组件、表达式函数全部不可用。
5. 插件配置按租户隔离。

## 5. 字段类型扩展 SPI

新增字段类型必须提供：

```text
FieldTypeHandler
rendererKey
optionsSchema
contractVectors
ddlMapping
converter
validator
queryCapability
```

发布前必须验证插件字段类型在当前 License 和平台版本下可用。

## 6. 升级与回滚

1. 升级前生成影响报告。
2. migration 必须幂等。
3. 失败后回滚到上一个插件版本。
4. 不可回滚的 migration 必须提前标注并要求人工确认。

## 7. 验收

1. 安装字段类型插件后可创建实验字段。
2. 插件禁用后使用该字段类型的对象发布被阻断或进入降级说明。
3. 插件升级失败能回滚。
4. 插件不能绕过 AccessView 或直接执行 SQL。
5. Manifest 签名不合法被拒绝。
