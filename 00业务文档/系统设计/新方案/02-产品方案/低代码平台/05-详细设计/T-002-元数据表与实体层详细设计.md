## 1. 目标

建立元模型持久层，包括 9 张核心 `lc_meta_*` 聚合表、`lc_meta_ref` 引用索引表、基础物理结构登记表、发布任务表、Entity/Mapper、JSON DTO、枚举与 ID 生成接口。

T-002 只做持久化和 DTO，不做完整业务服务。

## 2. 表清单

| 表 | 说明 | 聚合列 |
|---|---|---|
| `lc_meta_tenant` | 租户 | config |
| `lc_meta_workspace` | 工作区 | config |

## 3. 标准列

所有 `lc_meta_*` 表必须包含：

```text
id bigint not null primary key        -- 雪花 ID
tenant_id bigint not null
revision bigint not null default 0
deleted tinyint not null default 0
deleted_at datetime(3) null
delete_token bigint not null default 0
create_time datetime(3) not null
create_by bigint not null
update_time datetime(3) not null
update_by bigint not null
```

带 app 归属的表必须包含 `app_id bigint`。

## 4. ID 生成

接口：

```java
public interface IdGenerator {
  long nextSnowflakeId();
  String nextUlid();
}
```

规则：

- Entity 主键使用 `nextSnowflakeId()`。
- 动态业务记录 `lid` 使用 `nextUlid()`。
- 元模型对象的稳定引用主要使用 `code`，不是 `lid`。
- T-002 只提供接口和默认实现骨架，workerId 分配策略可在 T-001 配置中给默认本地值。

### 7.1 商用业务平台补充 DTO（M0 仅承载）

这些 DTO 用于承接 PRD REQ-070~REQ-078 和 08 §6 的 DEC-REQ-006~009。M0 只做 JSON 结构、枚举、序列化、快照承载和非法误执行阻断；不做单据继承展开、转换执行、反写执行、弹性域查询、报表运行、包市场和 License 运行时。

| DTO | 最小字段 | M0 用途 | 禁止事项 |
|---|---|---|---|
| `ObjectExtensionDef` | `baseObjectCode/sourceKind/packageCode/packageVersion/extensionType/conflictPolicy/items` | 标准对象与客户扩展分层承载 | 禁止直接覆盖 system/vendor 层 |
| `LinkConfigDef` | `sourceObjectCode/targetObjectCode/linkType/tracePolicy` | 单据链路和引用追踪预留 | 禁止运行时上查/下查 |
| `ConversionDef` | `sourceObjectCode/targetObjectCode/fieldMappings/lineMappings/filterCondition/idempotencyStrategy` | 单据转换模型预留 | 禁止执行转换 |
| `WriteBackDef` | `sourceObjectCode/targetObjectCode/aggregationPolicy/triggerTiming/conflictPolicy/compensationPolicy` | 反写模型预留 | 禁止触发副作用 |
| `LinkTraceDef` | `traceCode/sourceObjectCode/targetObjectCode/direction/permissionPolicy` | 上查/下查链路预留 | 禁止绕过权限展示 |
| `FlexFieldDef` | `flexCode/applyToObject/dimensions/valueType/visibilityPolicy/validationPolicy/sourceKind` | 弹性域/辅助属性预留 | 禁止作为普通 JSON 大字段运行 |
| `OrgRelationDef` | `relationCode/orgRole/sourceObjectCode/targetOrgType/timePolicy` | 多组织业务关系预留 | 禁止只用单一 `org_id` 表达 |
| `CodeRuleDef` | `ruleCode/objectCode/fieldCode/orgScope/dateScope/sequencePolicy/gapPolicy` | 复杂业务编码预留 | 禁止替代 M0 autonumber 运行时 |
| `ReportDef` | `reportCode/datasetRef/fields/permissionPolicy/exportPolicy` | 报表元数据预留 | 禁止等同列表导出 |
| `PrintTemplateDef` | `templateCode/objectCode/templateVersion/dataSnapshotPolicy/permissionPolicy` | 打印模板预留 | 禁止无版本模板 |
| `MenuDef` | `menuCode/parentCode/targetType/targetRef/permissionPolicy/deviceScope` | 菜单/门户入口预留 | 禁止把导航写入 PageSchema |
| `I18nResourceDef` | `resourceKey/locale/text/fallbackPolicy/sourceKind` | 多语言文案预留 | 禁止散落硬编码文案 |
| `PackageManifestDef` | `packageCode/version/sourceKind/dependencies/license/assets/compatibility` | 应用包/行业模板预留 | 禁止手工复制元数据替代包 |
| `LicensePolicyDef` | `licenseMode/capabilityCodes/offlineAllowed/degradePolicy/auditPolicy` | License 配置预留 | 禁止绑定在线校验假设 |

建议承载位置：

| DTO | JSON 聚合列 |
|---|---|
| ObjectExtensionDef、LinkConfigDef、ConversionDef、WriteBackDef、LinkTraceDef、FlexFieldDef、OrgRelationDef、CodeRuleDef | `lc_meta_object.options` |
| ReportDef、PrintTemplateDef、MenuDef、I18nResourceDef | `lc_meta_app.config` 或 `lc_meta_page.views` 的受控子结构，M0 实现时只能选一种并固定 |
| PackageManifestDef、LicensePolicyDef | `lc_meta_plugin.config` 或 `lc_meta_app.config` 的受控子结构，M0 只承载不执行 |

## 9. Mapper 设计

每张静态表一个 Mapper：

```text
MetaObjectMapper
MetaPageMapper
MetaRoleMapper
...
```

Mapper 只做单表 CRUD 和必要查询，不承载业务校验。

禁止：

- Mapper XML 拼接动态业务表 SQL。
- Mapper 返回 API DTO。
- Mapper 做 JSON 结构升级。

## 10. JsonUpgrader 骨架

接口：

```java
public interface JsonUpgrader<T extends VersionedJson> {
  int fromVersion();
  int toVersion();
  T upgrade(T oldValue);
}
```

T-002 只需实现注册表和 v1 空升级器，T-003/T-005 使用它加载 DTO。

严格模式：

- 启动时必须校验每个 JSON DTO 从最小支持版本到当前版本的升级链连续。
- 历史 snapshot 出现缺失升级器时，应用启动失败或该 app 加载失败，不得忽略。
- 未来版本（snapshot `_v` 大于当前代码支持版本）必须阻断加载并返回兼容错误。

## 12. T-002 测试

测试类：

```text
MetaTableMigrationTest
MetaJsonDtoSerializationTest
MetaEnumCodeTest
IdGeneratorTest
MetaMapperSmokeTest

```

必测：

- Flyway 能在 MySQL Testcontainers 上执行。
- 9 张核心 `lc_meta_*` 聚合表、`lc_meta_ref`、`lc_rt_physical_schema`、`lc_rt_ddl_log` 存在。
- JSON DTO 序列化包含 `_v`。
- 枚举 code 不重复。
- `id` 为 long，`lid` 为 26 位 ULID；测试不得把 `lid` 全局时间排序作为业务保证，业务排序必须使用 `create_time` 或业务时间列。
- 唯一约束不得使用 nullable `deleted_at`；必须使用 `delete_token` 墓碑列。
- `lc_meta_publish_task` 存在，状态、plan_json、fencing_token、trace_id 字段齐全。
- JsonUpgrader 升级链缺失时启动失败，未来版本 snapshot 阻断加载。
- `FieldTypeCapabilities` 和 `FieldTypeHandlerDescriptor` 可表达 22 种内置字段类型的能力。
- `lc_meta_object` 包含 `object_category/source_kind/base_object_code/extension_policy` 字段，默认值符合 01 与 08 的 M0 边界。
- ObjectExtension/Conversion/WriteBack/LinkTrace/FlexField/OrgRelation/CodeRule/Report/PrintTemplate/Menu/I18nResource/PackageManifest/LicensePolicy DTO 均可序列化、反序列化并带 `_v`。
- 商用补充能力枚举 code 稳定且不使用 ordinal。
- AppSnapshotDef 能承载上述 DTO，但测试必须断言 M0 不把这些 DTO 标记为 runtime enabled。
