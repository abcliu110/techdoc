# T-104/T-105 状态机、动作与规则详细设计

> 版本：v0.1
> 里程碑：M1
> 适用任务：T-104、T-105
> 依据：`../03-需求/PRD-产品需求规格说明书.md` REQ-040~043、REQ-083、REQ-087；`../04-架构决策/02-运行时引擎设计.md` §2、§4；知识库 `SM-LOWCODE-001`、`BR-LOWCODE-001`

---

## 1. 目标

T-104 实现对象状态机、动作分发和流转日志。T-105 实现 validate/assign 两类规则、fetch_from 带出和规则日志。

M1 不实现复杂工作流编排、等待节点、人工任务池和流程实例版本冻结，这些由 T-301 承接。

## 2. 状态机模型

`StateDef`：

```text
code
name
stateType: initial/normal/final
docEditable
allowedUpdateFields
```

`StateTransitionDef`：

```text
code
fromState
toState
actionCode
allowedRoles
conditionExpression
postRuleCodes
idempotencyRequired
```

启用状态机的对象：

1. 新增记录自动进入唯一 initial 状态。
2. 普通 update 不得修改 `state_code`。
3. final 状态默认不可编辑，除非字段在 `allowedUpdateFields` 中。

## 3. 动作模型

`ActionDef`：

```text
code
name
actionType: create/update/delete/transition/custom/import/export
scope: single/batch/object
confirmRequired
visibleCondition
enabledCondition
parameterSchema
idempotencyPolicy
```

动作执行固定顺序：

```text
Action 存在
AccessView 动作授权
参数 schema 校验
visible/enabled 条件服务端复验
状态机合法性检查
DataWriteService 执行
审计 + outbox
```

## 4. 规则模型

`RuleDef`：

```text
code
name
trigger: before_save/after_save/before_delete/after_transition/field_change
ruleType: validate/assign/fetch_from/notify/call_api
conditionExpression
config
sortNo
maxRuns
```

M1 支持：

```text
validate
assign
fetch_from
```

M1 只写 notify/call_api 的 outbox 预留事件，不实际投递外部通道。

## 5. 执行顺序

新增/更新：

```text
1. 类型转换
2. fetch_from
3. before_save validate
4. before_save assign
5. assign 后最多一轮 fetch_from/公式重算
6. 状态编辑性检查
7. 持久化
8. after_save 规则
9. 审计和 outbox
```

状态流转：

```text
1. 当前状态读取并加乐观锁
2. transition 匹配 fromState
3. Action 权限
4. conditionExpression
5. before_transition validate
6. 写 state_code
7. postRuleCodes
8. transition_log
9. outbox
```

## 6. 循环与副作用

1. 每个规则链有最大执行轮数，默认 10。
2. assign 规则不得无限触发 field_change。
3. 外部副作用只写 outbox，不在主事务中同步调用外部系统。
4. outbox 事件必须包含 idempotencyKey。

## 7. 日志

`lc_rt_transition_log`：

```text
tenant_id, app_code, object_code, record_lid,
from_state, to_state, action_code,
actor_lid, idempotency_key, meta_hash, perm_version, trace_id,
result, error_code, create_time
```

`lc_rt_rule_log`：

```text
tenant_id, app_code, object_code, record_lid,
rule_code, trigger, rule_type, duration_ms,
result, error_code, trace_id, create_time
```

## 8. 验收

1. 不能通过 update 篡改 `state_code`。
2. 非法 fromState、无角色、条件不满足都拒绝且原因明确。
3. 重复 transition 不重复执行 postRules 和 outbox。
4. 金额大于 10000 时审批意见必填。
5. 选择客户后能带出客户等级。
6. assign 循环被阻断。
7. 规则和流转日志能通过 traceId 串联。
