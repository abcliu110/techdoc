# T-301 工作流运行时详细设计

> 版本：v0.1
> 里程碑：M3
> 适用任务：T-301
> 依据：PRD REQ-040~043、REQ-083；知识库 `SM-LOWCODE-001`、Frappe/NocoBase workflow 源码证据卡

---

## 1. 目标

在 M1 对象状态机之上实现工作流运行时。对象状态机处理单记录状态合法性，工作流处理跨步骤、等待、审批、通知和异步执行。

## 2. 核心对象

```text
WorkflowDef
WorkflowNodeDef
WorkflowEdgeDef
WorkflowVersion
WorkflowInstance
WorkflowTask
WorkflowExecutionLog
```

## 3. 节点类型

M3 最小支持：

```text
start
approval
condition
action
notify
wait
end
```

不支持并行网关的复杂会签前，必须在 ADR 中说明。

## 4. 版本策略

1. 工作流定义发布生成 WorkflowVersion。
2. 新实例使用最新版本。
3. 在途实例钉住创建时版本。
4. 删除节点、角色、字段时必须生成影响报告。
5. 旧版本依赖的字段被删除或权限变化时，在途实例进入 `NEEDS_INTERVENTION`。

## 5. 实例状态

```text
CREATED
RUNNING
WAITING_APPROVAL
WAITING_TIMER
COMPLETED
FAILED
NEEDS_INTERVENTION
CANCELLED
```

## 6. 执行安全

1. 节点执行必须带 tenantId、metaHash、workflowVersion。
2. 审批任务必须复用 AccessView。
3. 外部副作用只写 outbox。
4. wait 节点由调度器恢复，恢复前重建 ExecutionContext。
5. 重试必须幂等。

## 7. 验收

1. “订单审批”工作流能创建审批任务并完成。
2. 工作流定义升级后，旧实例继续按旧版本执行。
3. 删除审批角色时，在途实例给出影响报告。
4. 节点失败有日志、重试和人工介入。
5. 工作流指标能统计失败率和平均耗时。
