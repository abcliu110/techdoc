T-001 不实现业务功能，只建立工程底座和机器门禁。完成后，后续任务必须在该骨架内增量开发，不得重新组织仓库结构。

## 3. Maven 模块职责

| 模块 | 允许职责 | T-001 产出 |
|---|---|---|
| lowcode-common | Result、ErrorCode、BizException、TenantContext、Trace 工具、ID 接口 | 空实现骨架 + 单测 |
| lowcode-metamodel | 元模型 DTO、Entity、Mapper、Schema Sync、MetaGraph | 包结构与空接口 |
| lowcode-runtime | 动态数据 API、权限、状态、规则 | 包结构与占位 README |
| lowcode-designer | 设计态 API、发布、导入导出 | 包结构与占位 README |
| lowcode-expression | 表达式门面 | 禁直接 Aviator 的 ArchUnit 规则 |
| lowcode-plugin | 插件 SPI | 包结构与占位 README |
| lowcode-workflow | 工作流扩展 | 包结构与占位 README |
| lowcode-app | Spring Boot 启动装配 | 最小启动类 |

## 4. 根 POM 设计

父 POM 固定：

```text
groupId: com.lowcode
artifactId: lowcode-platform
version: 0.1.0-SNAPSHOT
packaging: pom
java.version: 21
spring-boot.version: 3.4.x
```

dependencyManagement 首批依赖：

| 依赖 | 用途 |
|---|---|
| spring-boot-dependencies | Boot 基线 |
| mybatis-plus-boot-starter | 静态元数据表 |
| mysql-connector-j | MySQL |
| flyway-core / flyway-mysql | 静态表迁移 |
| spring-boot-starter-data-redis | Redis 版本失效/锁 |
| jackson-databind | JSON DTO |
| jakarta.validation-api | DTO 校验 |
| archunit-junit5 | 架构测试 |
| testcontainers-mysql / testcontainers-junit-jupiter | 集成测试 |

T-001 不引入 Aviator、Flowable、MinIO、OpenAPI 生成器等后续依赖，除非任务卡明确升级。

## 5. 后端包结构

每个模块遵守：

```text
com.lowcode.{module}
├── api
```

例外：

- `lowcode-common` 不设 `api/dao`。

## 8. 门禁配置

T-001 必须让以下门禁能运行，即使业务代码还很少：

| 门禁 | 命令 |
|---|---|
| 后端格式/编译/测试 | `mvn clean verify` |
| ArchUnit | 随 `mvn test` 执行 |
| 前端 lint | `pnpm lint` |
| 前端类型检查 | `pnpm typecheck` |
| 前端测试 | `pnpm test` |
| 前端构建 | `pnpm build` |

## 9. 必落 ArchUnit 规则

T-001 必须写一个故意违规样例测试或注释说明，证明规则会红。

规则：

1. Controller 禁访问 Mapper。
2. Entity 禁出现在 API 方法签名。
3. 非 `lowcode-expression` 禁直接依赖 Aviator 包。
4. 非 `runtime.sql` 禁访问动态 SQL segment。
5. 模块禁止跨模块访问 `support`。
6. `@Transactional` 只允许 service 层。
