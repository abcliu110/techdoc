## 2. 核心概念

### 2.1 架构概览

```
┌─────────────────────────────────────────┐
│         @form-designer/designer         │  设计器（可视化编辑）
│  - 组件面板                              │
│  - 画布区域                              │
│  - 属性面板                              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│         @form-designer/engine           │  核心引擎
│  - ComponentRegistry (组件注册)         │
│  - SchemaManager (Schema管理)           │
│  - ExpressionEngine (表达式)            │
│  - ValidatorEngine (校验)               │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        @form-designer/renderer          │  运行时渲染器
│  - FormRenderer (表单渲染)              │
└─────────────────────────────────────────┘
```

### 2.2 Schema结构

```typescript
interface FormSchema {
  formId: string;           // 表单ID
  formName: string;         // 表单名称
  version: string;          // 版本号
  config: FormConfig;       // 表单配置
  fields: FieldSchema[];    // 字段列表
  logic?: FormLogic;        // 业务逻辑
}

interface FieldSchema {
  id: string;               // 字段唯一ID
  type: string;             // 组件类型
  key: string;              // 字段标识（数据字段名）
  label?: string;           // 标签
  props: Record<string, any>;  // 组件属性
  rules?: ValidationRule[]; // 校验规则
  display?: DisplayConfig;  // 显示配置
  children?: FieldSchema[]; // 子字段（容器组件）
}
```

### 2.3 数据流

```
用户操作 → 设计器 → SchemaManager → Schema
                                        ↓
                         API保存 ← JSON Schema
                                        ↓
                         API加载 → JSON Schema
                                        ↓
                         渲染器 → 表单展示 → 用户填写 → 数据提交
```

## 3. 组件注册

### 3.1 注册自定义组件

```typescript
import { componentRegistry, ComponentMeta, ComponentCategory } from '@form-designer/engine';
import { MyCustomInput } from './MyCustomInput';

// 定义组件元数据
const myCustomInputMeta: ComponentMeta = {
  type: 'my-custom-input',
  name: '自定义输入框',
  category: ComponentCategory.BASIC,
  icon: <IconComponent />,
  order: 100,
  
  // 默认Schema
  defaultSchema: {
    id: '',
    type: 'my-custom-input',
    key: '',
    label: '自定义输入',
    props: {
      placeholder: '请输入',
      maxLength: 100,
      customProp: 'defaultValue',
    },
  },
  
  // 属性配置Schema
  propsSchema: [
    {
      key: 'label',
      label: '标题',
      type: 'input',
      category: 'basic',
      required: true,
    },
    {
      key: 'placeholder',
      label: '占位符',
      type: 'input',
      category: 'basic',
    },
    {
      key: 'customProp',
      label: '自定义属性',
      type: 'input',
      category: 'advanced',
    },
  ],
  
  // 支持的事件
  events: ['onChange', 'onBlur', 'onCustomEvent'],
  
  // 渲染组件
  component: MyCustomInput,
};

// 注册组件
componentRegistry.register(myCustomInputMeta);
```

## 4. Schema管理

### 4.1 创建SchemaManager

```typescript
import { SchemaManager } from '@form-designer/engine';

// 创建新的SchemaManager
const schemaManager = new SchemaManager();

// 使用初始Schema创建
const schemaManager = new SchemaManager(initialSchema);
```

### 4.2 Schema操作

```typescript
// 获取当前Schema
const schema = schemaManager.getSchema();

// 设置Schema
schemaManager.setSchema(newSchema);

// 添加字段
const newField = {
  id: 'field_001',
  type: 'input',
  key: 'userName',
  label: '用户名',
  props: {},
};
schemaManager.addField(newField);

// 添加到指定位置
schemaManager.addField(newField, { 
  parentId: 'container_001',  // 添加到容器内
  index: 2,                   // 插入到第2个位置
});

// 更新字段
schemaManager.updateField('field_001', {
  label: '新标题',
  props: { placeholder: '请输入用户名' },
});

// 删除字段
schemaManager.deleteField('field_001');

// 移动字段
schemaManager.moveField('field_001', {
  parentId: 'container_002',
  index: 0,
});

// 查找字段
const field = schemaManager.findField('field_001');
```

### 4.3 撤销/重做

```typescript
// 撤销
const prevSchema = schemaManager.undo();

// 重做
const nextSchema = schemaManager.redo();

// 判断是否可以撤销/重做
const canUndo = schemaManager.canUndo();
const canRedo = schemaManager.canRedo();
```

### 4.4 订阅Schema变化

```typescript
// 订阅变化
const unsubscribe = schemaManager.subscribe((type, payload) => {
  console.log('Schema变化:', type, payload);
  
  switch (type) {
    case 'add':
      console.log('添加字段:', payload);
      break;
    case 'update':
      console.log('更新字段:', payload);
      break;
    case 'delete':
      console.log('删除字段:', payload);
      break;
    case 'move':
      console.log('移动字段:', payload);
      break;
  }
});

// 取消订阅
unsubscribe();
```

## 5. 表达式引擎

### 5.1 基础用法

```typescript
import { ExpressionEngine } from '@form-designer/engine';

const engine = new ExpressionEngine();

// 设置上下文
engine.setContext({
  formData: {
    price: 100,
    quantity: 3,
    type: 'enterprise',
  },
  userInfo: {
    role: 'admin',
  },
});

// 执行表达式
const total = engine.execute('${formData.price * formData.quantity}');
// 结果: 300

const visible = engine.execute('${formData.type === "enterprise"}');
// 结果: true
```

### 5.2 内置函数

#### 数学函数

```typescript
// SUM - 求和
const sum = engine.execute('${SUM(10, 20, 30)}');  // 60

// AVG - 平均值
const avg = engine.execute('${AVG([10, 20, 30])}');  // 20

// MAX - 最大值
const max = engine.execute('${MAX(10, 20, 30)}');  // 30

// MIN - 最小值
const min = engine.execute('${MIN(10, 20, 30)}');  // 10
```

#### 字符串函数

```typescript
// CONCAT - 连接
const result = engine.execute('${CONCAT("Hello", " ", "World")}');  // "Hello World"

// UPPER - 转大写
const upper = engine.execute('${UPPER("hello")}');  // "HELLO"

// LOWER - 转小写
const lower = engine.execute('${LOWER("HELLO")}');  // "hello"
```

#### 逻辑函数

```typescript
// IF - 条件判断
const result = engine.execute('${IF(formData.price > 100, "贵", "便宜")}');

// AND - 与运算
const result = engine.execute('${AND(true, false, true)}');  // false

// OR - 或运算
const result = engine.execute('${OR(false, false, true)}');  // true
```

### 5.3 注册自定义函数

```typescript
// 注册自定义函数
engine.registerFunction('DISCOUNT', (price, rate) => {
  return price * (1 - rate);
});

// 使用自定义函数
const discountPrice = engine.execute('${DISCOUNT(100, 0.2)}');  // 80
```

### 5.4 在Schema中使用表达式

```typescript
const fieldSchema = {
  id: 'field_002',
  type: 'input',
  key: 'address',
  label: '地址',
  props: {},
  display: {
    visible: '${formData.type === "enterprise"}',  // 当类型为企业时显示
    disabled: '${formData.status === "approved"}',  // 审批通过后禁用
  },
};
```

## 6. 校验引擎

### 6.1 基础校验规则

```typescript
// 必填校验
{
  type: 'required',
  message: '此字段不能为空',
}

// 正则校验
{
  type: 'pattern',
  pattern: '^1[3-9]\\d{9}$',
  message: '请输入正确的手机号',
}

// 范围校验
{
  type: 'range',
  min: 0,
  max: 100,
  message: '值必须在0-100之间',
}
```

### 6.2 自定义校验

```typescript
{
  type: 'custom',
  validator: 'validateIdCard',  // 注册的校验器名称
  message: '身份证号格式不正确',
}

// 注册自定义校验器
validatorEngine.registerValidator('validateIdCard', (value, formData, field) => {
  // 返回true表示通过，false或错误消息表示失败
  if (!value) return true;
  
  const pattern = /(^\d{15}$)|(^\d{18}$)|(^\d{17}(\d|X|x)$)/;
  return pattern.test(value) || '身份证号格式不正确';
});
```

### 6.3 远程校验

```typescript
{
  type: 'remote',
  url: '/api/validate/username',
  message: '用户名已存在',
}
```

### 6.4 使用校验引擎

```typescript
import { ValidatorEngine } from '@form-designer/engine';

const validatorEngine = new ValidatorEngine();

// 校验整个表单
const result = await validatorEngine.validateForm(schema, formData);
if (!result.valid) {
  console.log('校验失败:', result.errors);
  // errors: { fieldKey: ['错误消息1', '错误消息2'] }
}

// 校验单个字段
const fieldErrors = await validatorEngine.validateField(
  fieldSchema,
  value,
  formData
);
```

## 8. 渲染器使用

### 8.1 基础渲染

```tsx
import { FormRenderer } from '@form-designer/renderer';

function MyForm() {
  const [formData, setFormData] = useState({});

  const handleSubmit = async (data) => {
    console.log('提交数据:', data);
    await api.submitForm(data);
  };

  const handleChange = (data) => {
    console.log('数据变化:', data);
    setFormData(data);
  };

  return (
    <FormRenderer
      schema={schema}
      initialData={formData}
      onSubmit={handleSubmit}
      onChange={handleChange}
      mode="edit"
    />
  );
}
```

### 8.2 只读模式

```tsx
<FormRenderer
  schema={schema}
  initialData={formData}
  mode="view"
/>
```

### 8.3 自定义渲染逻辑

```tsx
<FormRenderer
  schema={schema}
  initialData={formData}
  onSubmit={handleSubmit}
  renderField={(field, defaultRender) => {
    // 自定义某些字段的渲染
    if (field.type === 'custom-type') {
      return <MyCustomComponent {...field.props} />;
    }
    
    // 其他字段使用默认渲染
    return defaultRender(field);
  }}
/>
```

### 8.4 表单方法

```tsx
function MyForm() {
  const formRef = useRef();

  const handleReset = () => {
    formRef.current?.resetFields();
  };

  const handleValidate = async () => {
    try {
      const valid = await formRef.current?.validate();
      console.log('校验结果:', valid);
    } catch (errors) {
      console.log('校验失败:', errors);
    }
  };

  return (
    <>
      <FormRenderer
        ref={formRef}
        schema={schema}
        onSubmit={handleSubmit}
      />
      <Button onClick={handleReset}>重置</Button>
      <Button onClick={handleValidate}>校验</Button>
    </>
  );
}
```

## 10. 最佳实践

### 10.1 组件设计原则

1. **单一职责**：每个组件只负责一个功能
2. **可配置**：提供丰富的props配置
3. **可扩展**：支持自定义渲染和事件
4. **可测试**：编写单元测试

### 10.2 Schema设计建议

```typescript
// ✅ 好的做法
const goodSchema = {
  formId: 'form_001',
  formName: '用户注册表单',
  fields: [
    {
      id: 'field_001',

        maxLength: 20,

        { type: 'required', message: '用户名不能为空' },

      props: {},              // 缺少必要配置
    },
  ],
};
```

## 11. 常见问题

### Q1: 如何添加自定义组件？

**A**: 参考[第3节 组件注册](#3-组件注册)，实现组件并注册到`componentRegistry`。

### Q2: 表达式不生效怎么办？

**A**: 检查以下几点：
1. 表达式语法是否正确（必须使用`${}`包裹）
2. 引用的字段key是否正确
3. 上下文数据是否已设置
4. 浏览器控制台是否有错误信息

### Q3: 如何实现字段联动？

**A**: 使用表达式引擎：

```typescript
{
  id: 'field_002',
  type: 'input',
  key: 'companyAddress',
  label: '公司地址',
  display: {
    visible: '${formData.userType === "enterprise"}',  // 当用户类型为企业时显示
  },
}
```

### Q4: 如何自定义校验规则？

**A**: 使用custom类型校验：

```typescript
// 注册校验器
validatorEngine.registerValidator('checkAge', (value) => {
  return value >= 18 || '年龄必须大于18岁';
});

// 使用
{
  rules: [
    { type: 'custom', validator: 'checkAge' }
  ]
}
```

### Q5: 渲染器如何对接后端API？

**A**: 在组件外层处理数据：

```typescript
const handleSubmit = async (formData) => {
  try {
    const response = await fetch('/api/submit', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(formData),
    });
    const result = await response.json();
    message.success('提交成功');
  } catch (error) {
    message.error('提交失败');
  }
};

<FormRenderer schema={schema} onSubmit={handleSubmit} />
```

### Q6: 如何实现表单预填充？

**A**: 使用`initialData`属性：

```typescript
const [initialData, setInitialData] = useState({});

useEffect(() => {
  // 加载数据
  fetchFormData().then(data => {
    setInitialData(data);
  });
}, []);

<FormRenderer 
  schema={schema} 
  initialData={initialData}
  onSubmit={handleSubmit} 
/>
```

### Q7: 如何调试表达式？

**A**: 使用浏览器控制台：

```typescript
// 在表达式引擎中添加调试日志
const result = engine.execute(expression, formData);
console.log('表达式:', expression);
console.log('上下文:', formData);
console.log('结果:', result);
```

### Q8: 性能优化建议？

**A**: 
1. 使用虚拟滚动处理大量组件
2. 懒加载重型组件
3. 使用React.memo避免不必要渲染
4. 防抖/节流处理频繁操作
5. Schema序列化优化

### Q9: 如何实现多语言？

**A**: 使用i18n库：

```typescript
import { useTranslation } from 'react-i18next';

const { t } = useTranslation();

const fieldSchema = {
  label: t('form.username'),
  props: {
    placeholder: t('form.usernamePlaceholder'),
  },
};
```

### Q10: 如何处理文件上传？

**A**: 配置Upload组件的action和处理函数：

```typescript
{
  type: 'upload',
  key: 'avatar',
  label: '头像',
  props: {
    action: '/api/upload',
    maxSize: 5 * 1024 * 1024,  // 5MB
    accept: 'image/*',
    onSuccess: (response) => {
      return response.data.url;  // 返回文件URL
    },
  },
}
```
