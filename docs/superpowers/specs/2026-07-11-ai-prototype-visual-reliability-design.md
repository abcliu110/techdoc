# AI 原型视觉可靠性规范加强设计

## 目标

加强《AI 竞品证据驱动的新产品原型统一工程方法》，使其能够系统约束 AI 原型设计和实现中的视觉错位、溢出、遮挡、裁切、层级错误、状态跳变、响应式失真、信息密度失衡及其他视觉可靠性问题。

本次只修改目标方法文档，不创建检测器、不修改项目代码，也不把尚未执行的工具认证或项目回归描述为已经完成。

## 设计原则

1. 本文继续作为单一治理与放行权威；其他 UI 规范只能补充实现细节，不得降低硬门禁。
2. 视觉问题采用“预防优先、机械事实优先、视觉模型补充、根因化修复、独立复验”的闭环。
3. 轻量流程可以减少截图和主观评审，不得跳过影响可用性的几何、内容压力和操作可达性门禁。
4. 示例代码和未认证脚本只能作为参考，不得产生权威验收结论。
5. 文档必须明确区分“方法已经定义”和“工程能力已经验证”。

## 修改范围

### 1. 视觉可靠性范围模型

新增视觉缺陷分类，至少覆盖：

- 几何：错位、越界、重叠、裁切、离屏、尺寸跳变。
- 内容：长文本、不可断词、极值数字、空数据、极限记录数。
- 层叠：Portal、弹窗、下拉框、Tooltip、fixed、sticky、z-index。
- 响应式：断点切换、缩放、语言、主题和最窄视口。
- 交互状态：加载、错误、空态、禁用、权限、选中、拖拽和恢复路径。
- 视觉语义：层级、密度、节奏、主次关系和操作可发现性。

### 2. 七层视觉正确性闭环

在原型工程和视觉验证章节中明确：

1. 生成前冻结视觉合同。
2. 生成时限制高风险布局方式。
3. 使用浏览器机械检测客观几何事实。
4. 使用隔离视觉评审检查层级、密度和专业性。
5. 执行视口、状态、内容、主题、语言和缩放压力矩阵。
6. 按最上游根因路由修复，禁止截图特调。
7. 由独立验证身份重跑受影响矩阵后放行。

### 3. 不可裁剪最低门禁

所有可运行原型，无论完整版或轻量版，最低必须满足：

- 构建和运行入口有效。
- 未解释 Console/Page Error 为零。
- 核心任务可达且结果可观察。
- 关键区域无意外溢出、裁切、遮挡、离屏和滚动陷阱。
- 最窄支持视口和最长内容边界通过。
- P0/P1 可用性视觉缺陷为零。

### 4. 检测器技术合同

强化现有几何算法定义，要求同时检查：

- 上、下、左、右四方向边界。
- 页面、实际包含块、滚动容器和视口。
- `scrollWidth/clientWidth` 与 `scrollHeight/clientHeight`。
- 合法滚动、合法覆盖和精确例外。
- Portal、fixed、sticky、absolute、transform 与层叠上下文。
- `elementsFromPoint` 命中、焦点可见和滚动末端可达。
- 字体和图像稳定后、状态切换前后的几何变化。

不得把简单父子矩形比较、全元素 O(n²) 重叠扫描或 computed style 中搜索 `var(--token)` 视为认证实现。

### 5. 内容与截断合同

统一长文本规则：不得把 `ellipsis` 或 `overflow: hidden` 当作通用修复。只有视觉合同明确允许、完整内容仍可访问、任务信息不丢失且边界夹具通过时，才能截断。

### 6. 工具认证与真实回归

增加候选规范升级条件：

- 正常夹具能够通过。
- 每种典型视觉故障都有负向夹具且能够稳定失败。
- 验证器对超时、损坏输出、篡改和环境漂移失败关闭。
- 至少完成普通页面、数据密集页面、复杂浮层页面和响应式页面的项目级回归。
- CI 能够自动执行并阻断失败。

完成前，规范状态保持候选；完成后才能依据认证报告升级为已验证执行基线。

## 非目标和边界

本次文档加强不能单独证明：后端业务逻辑、权限安全、数据一致性、金额计算、内存泄漏、栈溢出、整数溢出、数据库字段溢出或长期性能正确。

## 验证标准

文档修改完成后必须验证：

- 新增条款与现有状态机、门禁和完成定义一致。
- 不存在轻量模式跳过关键视觉可用性门禁的漏洞。
- 不再把示例脚本描述为权威验证器。
- 截断规则不存在相互矛盾。
- “候选规范”和“已验证执行基线”的升级条件可客观判定。
- UTF-8 内容无 BOM 污染、连续问号、替换字符或冲突标记。
