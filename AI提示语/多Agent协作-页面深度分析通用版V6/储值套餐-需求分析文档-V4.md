## 术语表

| 术语 | 定义 | 来源 |
|------|------|------|
| 储值套餐 | 商户创建的储值营销活动，顾客按固定金额档位充值可获赠送金额、积分、礼品等 | 📸 |
| 档位 | 套餐中的一个充值金额档次 | 📸 |
| 优惠券礼品 | 档位下赠送的单张优惠券 | 📸 |
| 券包 | 档位下赠送的券包（多张优惠券组合） | 📸 |
| 发放规则 | 礼品的发放方式：一次性发放或周期性发放 | 📸 |
| 可用周期 | 套餐在一周或一月中哪些日期可用 | 📸 |
| 排除日期 | 套餐不可使用的特定日期 | 📸 |
| 储值协议 | 商家与用户之间储值权益的法律文本 | 📸 |
| 等级延期 | 储值后将用户已有会员等级有效期延长指定天数 | 📸 |

## 一、业务概述

### 1.1 系统定位

- 系统：全来店商户中心 📸
- 模块：会员中心 > 储值活动 📸
- 页面：新增储值套餐（create 模式） 📸
- 操作者：商户管理员 🤖
- 受益者：商户（促进顾客预充值）、最终用户（获得赠送优惠） 🤖

### 1.2 表单形态

- 左侧导航：3个步骤锚点 —「基础信息」/「储值规则」/「购买规则」 📸
- 右上角按钮：「取消」「上一步」「保存」 📸
- 底部：「我已阅读并同意《法律声明》」checkbox 📸
- 保存机制：右上角「保存」按钮统一提交 📸
- 新增 vs 编辑差异：[⚠️ 假设：启用后部分字段只读]

### 1.3 业务流程概述

```
商户管理员 → 进入「储值活动」列表 → 点击「新增储值套餐」
  → 填写「基础信息」（品牌、分类、日期、周期、时段、协议等）
  → 配置「储值规则」（档位金额、赠送方案、礼品、发放规则、会员升级等）
  → 配置「购买规则」（渠道、门店、用户限制、购买次数等）
  → 勾选「我已阅读并同意《法律声明》」
  → 点击「保存」→ 创建储值套餐（草稿状态）
```

```

 ├── package_exclude_date (排除日期, 1:N) [D][OWN]

 ├── package_agreement (协议关联) [J]

[REF] member_level — 会员等级
[REF] organization — 门店

[REF] miniprogram — 小程序
```
