# All-in-One 需求分析 Prompt（通用版 V3）

## 概述

本文档将 Agent-0（信息补全者）到 Agent-4（文档整合者）的完整分析流程合并为**单个 Prompt**，实现"一个对话窗口、多轮交互"完成完整需求分析文档。

### V3 核心变化（相对 V2）

V2 的信息补全采用「一次列出所有问题，用户批量回答」模式，实际使用中存在以下问题：
- 问题太多，用户容易遗漏或敷衍
- 用户无法用截图来回答（截图与问题编号难以对应）
- 无法根据前一个回答动态调整后续问题

V3 改为**逐题追问模式（一问一答）**，并针对低智能模型做了大幅简化：

**交互方式改进：**
- 每次只问 1 个问题，降低用户认知负担
- 支持用户用**选项、文字、截图**任意方式回答
- **鼓励用截图回答**：展开下拉框截图、点击按钮后截图、切换选项截图等
- AI 可从截图中自动提取信息，智能跳过已解答的问题
- 用户可随时说"开始分析"提前结束提问

**提问质量改进：**
- **优先问隐藏交互状态**：下拉框选项、按钮点击后界面、Checkbox勾选后变化、Radio切换后变化
- **深度追踪机制**：当回答揭示新 UI 元素时，自动追加问题（如勾选「优惠券」→ 出现「+加入礼品」按钮 → 追问按钮弹窗内容）
- **不会问截图中已经能看到的信息**（如已选中的Radio值、可读的帮助文字）

**Prompt 结构改进（兼容低智能模型）：**
- 全局规则前置（6条），贯穿始终
- 去掉"内部队列"等抽象概念，改为具体的「待探索清单」
- 去掉 A-G 分类法，改为 □ 勾选检查清单，机械可执行
- 阶段编号对齐（阶段三输出，不再写"阶段二"）
- 示例对话数据一致，无自相矛盾
- 初扫从"逐一列出每个元素"简化为"2-3段概括"，节省注意力

### 核心收益

| 对比项 | 分步模式（5 个 Agent） | V2 合并模式 | V3 逐题追问模式 |
|--------|----------------------|-------------|----------------|
| 对话窗口数 | 5 个 | 1 个 | 1 个 |
| 截图传递次数 | 5 次 | 1 次 | 1 次初始 + 按需补充 |
| 信息补全方式 | 1 次列出所有问题 | 1 次列出所有问题 | 逐题追问，支持截图回答 |
| 用户体验 | 多窗口切换 | 批量回答压力大 | 轻松一问一答 |
| 信息采集质量 | 中 | 中（易遗漏） | 高（逐题确认 + 截图补充） |
| 中间产物管理 | 手动传递 | 自动流转 | 自动流转 |

### 适用场景

- 快速出需求文档，不需要逐步审核中间产物
- 截图较完整、业务背景较清晰的场景
- 想要通过截图补充交互细节的场景（如展开下拉框、切换 Tab）
- 想要减少操作步骤的用户

### 不适用场景

- 需要精细控制每一步分析结果的场景（建议用分步模式）
- 截图特别多、业务极其复杂的场景（上下文窗口可能不够）
- 需要代码生成（Agent-5）或红队审查（Agent-6）时，仍需另开对话

## 三阶段交互流程

```
用户发送截图 + 业务背景
        │
        ▼
┌─ 阶段一（AI 立即执行）─────────────────────┐
│  1. 截图初扫 — 列出可见元素、评估完整度      │
│  2. 信息缺口识别 — 6 维度扫描（内部执行）    │
│  3. 输出初扫结果 + 提出第 1 个问题           │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 阶段二（一问一答循环）────────────────────┐
│  用户回答（选项/文字/截图）                   │
│      ↓                                       │
│  AI 记录答案，提出下一个问题                  │
│  （截图回答可自动消化多个问题）               │
│      ↓                                       │
│  循环直到：问完 / 用户说"开始分析"           │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 阶段三（AI 自动执行）────────────────────┐
│  4. 构建增强输入包（内部，不单独输出）        │
│  5. 业务分析（第0-2层）                      │
│  6. 规则挖掘（第3-4层）                      │
│  7. 架构分析（第5-7层 + 工程维度）           │
│  8. 文档整合 — 去重、交叉验证、格式标准化     │
└──────────────────────────────────────────────┘
        │
        ▼
  输出完整需求分析文档
```

## 合并版 vs 分步版的精简策略

| 分步版特性 | 合并版处理 | 原因 |
|-----------|-----------|------|
| 独立的交接摘要章节 | 去掉 | 同一上下文内不需要结构化传递 |
| 独立的推理备忘录章节 | 改为 `[⚠️ 假设]` `[⚠️ 待确认]` 内联标记 | 同一 AI 可直接在正文中标注 |
| 增强输入包单独输出 | 作为内部中间状态 | 不需要用户手动传递 |
| 多 Agent 交叉验证 | 自动完成 | 同一 AI 产出的内容，自行交叉验证 |
| 假设与推断追踪表 | 保留，集中在文档末尾 | 便于审阅和对接红队审查 |
| 一次性提问清单 | 改为逐题追问 | 降低用户负担，支持截图回答 |

## 使用说明

### 第一轮：发送截图

打开一个新对话，粘贴 `prompt-all-in-one-需求分析.txt` 中的 Prompt，然后发送：

```
这是 [系统名称] 的 [页面名称] 页面。
[附上截图]
[可选：补充业务背景]
```

AI 会输出：
1. 截图初扫结果 — 确认 AI 看到的是否正确
2. 第 1 个问题 — 带选项，支持截图回答

### 第二轮起：逐题回答

每次 AI 只问 1 个问题，你可以用以下方式回答：

| 回答方式 | 示例 |
|---------|------|
| 选字母 | `A` 或 `B` |
| 文字说明 | `这个下拉框有3个选项：日结、周结、月结` |
| 发截图 | 直接发送展开下拉框/切换Tab后的截图 |
| 截图+文字 | 发截图并补充说明 |
| 跳过 | `跳过`（该问题会被标记为假设） |
| 提前结束 | `开始分析` 或 `剩余全部跳过`（立即进入分析阶段） |

AI 会在每次提问时显示进度（如 `3/15 | P0 剩余 2 个`），方便你掌握进度。

### 分析结果

所有问题回答完毕（或提前结束）后，AI 会直接输出完整的需求分析文档。

### 后续步骤

拿到需求分析文档后，如需：
- **代码生成**：另开对话，使用 `prompt-05-代码生成师.txt`，将需求文档作为输入
- **红队审查**：另开对话，使用 `prompt-06-红队审查员.txt`，将需求文档作为输入

## 与分步模式的输出兼容性

合并版的输出文档结构与 Agent-4（文档整合者）的输出模板完全一致，因此：
- 可直接作为 Agent-5（代码生成师）的输入
- 可直接作为 Agent-6（红队审查员）的输入
- 无需任何格式转换

## 注意事项

1. **上下文窗口**：逐题追问模式会消耗更多上下文（每次问答都是一轮对话），如果问题特别多（>20个），注意上下文窗口容量。
2. **截图回答的优势**：对于隐藏交互状态（下拉框选项、Tab内容、弹窗等），直接发截图比文字描述更准确高效，AI 会自动从截图中提取信息。
3. **提前结束**：不必回答所有问题。P0 问题回答完后，随时可以说"开始分析"进入分析阶段，未回答的问题会被标记为假设。
4. **中间结果不可审核**：合并版跳过了中间产物的人工审核环节。如果你需要逐步确认实体识别、规则提取等中间结果，建议使用分步模式。
5. **反馈环不适用**：合并版不支持 Agent-6 的反馈环机制。如果红队审查发现重大问题，需要在同一对话中追加修改请求。
