# 新增储值套餐 — 完整需求分析文档

---

## 术语表

| 术语 | 含义 |
|------|------|
| 储值套餐 | 商户设置的充值优惠活动，用户购买后将金额存入账户，可获得赠送金额、积分、礼品、会员升级等权益 |
| 档位 | 储值套餐中的固定金额选项，每个档位对应不同的储值金额和赠送配置，最多5个 |
| 券包 | 包含多张优惠券的组合礼品包 |
| 可用金额限制 | 储值后金额的使用限制策略（当日/当餐），超出部分冻结 |
| 会员方案 | 会员体系的分类方案（如测试会员、实体卡方案、红火会员卡、会员卡） |
| 会员等级 | 会员方案下的具体等级（如VIP1、萌新会员、常客会员等） |
| 适用组织 | 储值套餐生效的门店/区域范围 |
| 适用渠道 | 用户可购买储值套餐的入口（门店pos、微信小程序、支付宝小程序等） |
| 生效类型 | 优惠券的生效时间计算方式（按天/按小时/按天零点） |
| 计入实收 | 优惠券核销时是否将金额计入商户实收金额 |

## 一、业务概述

### 1.1 系统定位

- **所属系统**：全来店商户中心 📸 截图可见
- **所属子系统**：营销中心 > 会员中心 > 储值活动 📸 截图可见
- **操作者**：商户管理员（拥有商户中心后台权限的用户） 🤖 AI 推断
- **受益者**：商户（获得预付款现金流）和终端消费者（获得储值优惠） 🤖 AI 推断
- **页面作用**：创建一个新的储值套餐活动，配置充值金额、赠送权益、购买规则等 📸 截图可见

- **交互模式**：分步表单（步骤导航：基础信息 → 储值规则 → 购买规则） 📸 截图可见

```

     → 储值推荐规则（档位配置）

适用组织 → 指定门店/区域参与

```
