# 快速使用指南（通用版）

## 适用场景

任何 B 端业务页面：订单管理、用户管理、商品管理、审批流程、报表配置、权限设置等。
不预设业务领域，AI 根据原型图自行识别业务语境。

---

## 完整执行链路

```
原型图 + 业务背景
    │
    ▼
Agent-1（业务分析师）── 产出：实体清单、字段定义、判别字段
    │
    ▼
Agent-2（规则工程师）── 产出：联动规则、约束矩阵、不变量
    │
    ▼
Agent-3（系统架构师）── 产出：API清单、状态机、并发方案
    │
    ├──► Agent-4（文档整合者）── 产出：完整需求文档（给产品/测试/前端）
    │
    └──► Agent-5（代码生成师）── 产出：可运行的 Spring Boot 代码
```

- Agent-1 → 2 → 3 串行执行，每步只传**交接摘要**给下游
- Agent-4 和 Agent-5 可以**并行执行**，输入都是 Agent-1+2+3 的输出

## Step 1：Agent-1 业务分析师

**发送内容：**
```
[粘贴 01-Agent-业务分析师-通用.md 中的 System Prompt]

业务背景：这是 [系统名称] 的 [页面名称] 页面，操作者是 [角色]。
[附上截图]
```

**审核要点：**
- 实体识别完整（有没有漏掉从表）
- 判别字段全部识别（对照所有 Toggle/Radio/Select）
- 字段名是英文下划线命名

**传给下一步：** 复制「四、交接摘要」部分

## Step 4：Agent-4 文档整合者（需求文档）

**发送内容：**
```
[粘贴 04-Agent-文档整合者-通用.md 中的 System Prompt]

===== Agent-1 业务分析报告 =====
[粘贴 Agent-1 完整输出]

===== Agent-2 规则分析报告 =====
[粘贴 Agent-2 完整输出]

===== Agent-3 架构分析报告 =====
[粘贴 Agent-3 完整输出]
```

注意：这步发的是 3 个 Agent 的**完整输出**，不是交接摘要。

**审核要点：**
- 术语表统一全文用词
- 字段数据字典覆盖所有可见字段
- 验收标准可测试（无模糊描述）
- 未决问题已分配责任人

## 灵活组合

不是每次都需要跑完全部步骤：

| 目标 | 使用 Agent |
|------|-----------|
| 只需数据库表设计 | Agent-1 |
| 只需接口文档 | Agent-1 → Agent-3 |
| 只需校验规则 | Agent-1 → Agent-2 |
| 只要需求文档 | Agent-1 → 2 → 3 → 4 |
| 只要后端代码 | Agent-1 → 2 → 3 → 5 |
| 文档 + 代码都要 | Agent-1 → 2 → 3 → 4（并行）5 |
