第五层：外部系统集成

5-1 系统依赖地图
- 为页面涉及的每个写操作（保存/提交/审批/删除等），识别：
  - 调用的下游服务或模块
  - 调用方式：同步 / 异步（消息队列）
  - 失败处理策略：事务回滚 / 补偿重试 / 降级处理
- 格式：{操作} → {下游服务}（{同步/异步}）→ 失败策略：{策略}

5-2 读写模型分离（CQRS 视角）
- 写模型（管理端表单）：字段全、约束严、格式原始
- 读模型（展示端/列表）：字段精简、格式友好、可能聚合多表
- 建立两者间的字段映射差异表

5-3 数据级联规则
- ★ 基于 Agent-1 的实体层级关系，为每对父子实体定义级联策略
- 父实体变更/删除时，每个子实体的处理策略（操作 | 影响判断 | 策略 | 用户提示）
- 主表软删除时，所有子表（[D]/[P]/[G]）和关联表（[J]）是否需要同步处理

5-4 内联创建的 API 模式
- 识别页面中所有"+ 新增xxx"按钮，描述其 API 交互模式
- 内联创建与主表单的事务边界


第六层：风险建模

6-1 并发争用分析
- 识别所有可能存在并发争用的业务点（库存、配额、余额、唯一性等）
- 参考 Agent-2 推理备忘录中关于并发场景的不确定推断，评估是否需要额外防护
- 为每个争用点设计防护方案（争用点 | 风险 | 防护方案 | 实现细节/锁粒度/Key结构）
- 如无并发风险，明确说明原因

6-2 数据安全模型（根据业务场景选择适用项）
- 涉及金额/积分/配额：隔离存储方案、消费优先级、退款/回滚规则
- 涉及敏感数据：脱敏规则、访问日志
- 涉及文件/附件：存储策略、访问权限
- 不涉及以上场景：明确说明"本页面无特殊数据安全要求"

6-3 防重/幂等设计
- 识别需要幂等保证的操作（重复提交、网络重试场景）
- 幂等机制：唯一请求ID / 业务唯一键 / 状态机保护

6-4 版本/快照策略（如适用）
- 快照触发时机（状态变更时 / 定期 / 手动）
- 快照内容范围
- 快照用途（审计 / 回滚 / 纠纷）


第七层：生命周期

7-1 状态机
- 绘制完整的状态流转图（文本表示）
- 每个转换标注：触发条件 / 执行者（系统自动 vs 人工操作）/ 副作用 / 是否可逆
- 验证：每个非终态都必须有出路（无死锁）
- 参考 Agent-1 推理备忘录中关于状态与生命周期的异常信号
- 如无状态机（简单 CRUD），明确说明

7-2 字段级操作权限矩阵
- 按「实体状态 × 字段」建立权限矩阵
- 符号：可编辑 / 只读保护 / 仅查看 / 不显示
- 标注只读保护字段的保护理由

7-3 审计日志
- 必须审计的字段清单及理由（财务影响 / 法律影响 / 权益影响 / 安全影响）
- 审计标准字段：operator_id + timestamp + client_ip + old_value + new_value + change_reason

7-4 通知触发点
- 识别所有需要发送通知的事件及通知对象
- 如无通知需求，明确说明
