# T6 — 工程落地

> **用途：** API 设计、数据埋点、性能规划、可扩展性。开发团队落地实施使用。

---

## Prompt

```
请从工程落地角度分析页面：

1. API 设计推导
   从页面交互反推API边界：
   | 页面交互 | Endpoint | Method | 幂等要求 | 备注 |
   特别标注：涉及支付的必须强幂等，保存草稿需防重复提交

2. 财务对账（如涉及金额）
   - 本金/赠送金消费优先级
   - 多门店下的分账规则
   - 退款资产回滚顺序

3. 数据埋点
   - 用户行为漏斗：曝光→点击→支付发起→支付成功
   - 管理员操作热度：哪些配置项最频繁被修改、哪个字段最多触发校验错误

4. 可扩展性
   - 识别当前写死的枚举，预判扩展方向
   - 预留扩展接口（如赠送类型从优惠券扩展到实物礼品）
   - 建议使用策略模式/位运算等可扩展设计

5. 性能与容量
   - 热路径识别：高频读操作建议缓存策略（如活动有效性校验→本地缓存TTL 30s）
   - 增长型数据：购买记录等需分区/归档策略
   - 定时任务：周期计数器重置、活动状态自动流转

输出格式：API用RESTful表格，埋点用漏斗图描述，扩展性用"当前→未来→方案"格式。
```
