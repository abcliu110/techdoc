# T2 — 数据建模

> **用途：** 从页面字段反推实体模型、字段字典和依赖关系。后端开发必备。

---

## Prompt

```
请基于页面截图进行数据建模分析：

1. 实体识别
   - 识别主实体与从实体，标注关系（1:1 / 1:N / M:N）和数量约束
   - 区分核心实体（有独立ID）与值对象（无独立ID，与实体同生命周期）
   - 用文字描述实体关系图

2. 字段数据字典
   为每个 UI 元素建立记录，使用表格：
   | 字段名 | UI元素 | 数据类型 | 存储建议 | 必填 | 默认值 | 备注 |
   多选字段需说明存储策略选择理由（位运算/JSON数组/中间表）

3. 判别字段
   - 识别所有控制其他字段"显隐/必填/计算逻辑"的主控字段
   - 格式：判别字段 → 控制 [受控字段] 的 [逻辑]
   - 判别字段必须显式存储，不可由其他字段推导

4. 跨区依赖
   - 在A区定义、被B/C区消费的驱动字段
   - 依赖方向决定后端校验执行顺序

5. 默认值语义
   - Opt-out（默认全选）= 系统鼓励宽泛适用
   - Opt-in（默认不选）= 系统要求主动声明
   - 默认值 = 产品团队对最佳实践的编码

输出格式：实体关系用树形结构，字段字典用表格，判别字段用"→"箭头格式。
```
