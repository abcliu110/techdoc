# T7 — 整合输出

> **用途：** 当使用了多个模板分步分析后，用此模板将所有输出整合为一份完整需求文档。

---

## Prompt

```
请将以上所有分析内容整合为一份完整的需求分析文档，执行以下操作：

1. 去重合并
   - 多次分析中对同一字段/规则的重复描述，合并为单一权威描述
   - 描述有冲突的，标注 [⚠️ 待确认] 并列出两种说法

2. 交叉验证
   - 数据字典中每个字段是否都有对应的约束定义
   - 每条不变量是否有对应的服务层校验
   - API列表是否覆盖所有页面交互
   - 发现遗漏标注 [🔍 补漏] 并尝试补充

3. 术语统一
   - 建立术语表，统一全文用词
   - 实体名、字段名全文一致

4. 补充内容
   - 为每个功能模块标注优先级：P0（核心）/ P1（重要）/ P2（增强）
   - 编写验收标准：GIVEN {前置条件} WHEN {操作} THEN {预期结果}
   - 建议开发任务拆解顺序和依赖关系
   - 汇总所有未决问题，标注需要谁来回答

输出格式：
按以下结构组织：术语表 → 业务概述 → 页面结构 → 数据模型 → 业务规则 → 约束矩阵 → 系统集成 → 风险安全 → 生命周期 → 工程维度 → 验收标准 → 任务拆解 → 未决问题
```
