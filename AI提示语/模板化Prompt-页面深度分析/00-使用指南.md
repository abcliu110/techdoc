# 模板化 Prompt 拆分方案 — 页面深度分析 V1.0

## 一、方案定位

将原始 1500 字 Master Prompt 拆为 **7 个独立模板 + 1 个整合模板**，每个 ≤ 300 字。

### 与方案 B（多 Agent）的区别

| 维度 | 方案 B 多 Agent | 方案 C 模板化 |
|------|---------------|-------------|
| 使用门槛 | 需要 4 轮对话或编排平台 | 单次对话，复制粘贴即可 |
| 适合场景 | 完整需求文档产出 | 快速定向分析某几个维度 |
| 深度 | 每层专家级深度 | 每层中等深度，胜在灵活 |
| 耗时 | 20-40 分钟（4 轮） | 3-10 分钟（1-2 轮） |
| 上下文消耗 | 分散在 4 个窗口 | 集中在 1 个窗口 |
| 适合谁 | 需求分析师、架构师 | 开发人员、产品经理快速摸底 |

## 二、模板清单

```
模板化Prompt-页面深度分析/
├── 00-使用指南.md              ← 本文件
├── T0-语境定位.md              ← 第0层：谁在用、什么系统、什么表单
├── T1-空间解构.md              ← 第1层：区块分区、视觉信号、负空间
├── T2-数据建模.md              ← 第2层：实体、字段字典、判别字段
├── T3-规则挖掘.md              ← 第3层：联动规则、不变量、约束
├── T4-系统集成.md              ← 第4层：外部依赖、CQRS、级联
├── T5-风控生命周期.md           ← 第5层：并发、资金安全、状态机、权限
├── T6-工程落地.md              ← 第6层：API、埋点、性能、扩展性
└── T7-整合输出.md              ← 汇总模板：将多个模板输出合并为文档
```

## 三、组合菜单（按需点菜）

### 场景 1：产品经理快速摸底
> 只需了解页面结构和业务规则

**组合：** `T0 + T1 + T3`（约 800 字 Prompt）

### 场景 2：后端开发建模
> 需要数据模型和 API 设计

**组合：** `T0 + T2 + T6`（约 800 字 Prompt）

### 场景 3：前端开发联动逻辑
> 需要字段联动规则和条件渲染

**组合：** `T0 + T2（仅判别字段部分）+ T3`（约 700 字 Prompt）

### 场景 4：架构评审
> 需要系统集成和风控分析

**组合：** `T0 + T4 + T5`（约 800 字 Prompt）

### 场景 5：完整需求文档
> 等同于原始 Master Prompt，但分步执行

**组合：** 先 `T0 + T1 + T2`，再 `T3 + T4`，最后 `T5 + T6`，用 `T7` 整合

## 四、使用方法

### 方法 A：单次组合（推荐，适合 1-3 个模板）

```
[粘贴 T0 内容]

[粘贴 T3 内容]

---
请分析以下页面截图：
[附上截图]
背景：这是 XX 系统的 XX 配置页面。
```

### 方法 B：分步追问（适合 4 个以上模板）

```
第1轮：[T0 + T1 + T2] + 截图
第2轮："继续，请按以下维度分析：" + [T3 + T4]
第3轮："继续，请按以下维度分析：" + [T5 + T6]
第4轮：[T7] "请将以上所有分析整合为需求文档"
```

### 方法 C：System Prompt 预设（适合高频使用）

在 Claude/ChatGPT 的自定义指令中预设 T0 作为常驻上下文，每次对话只需发送具体模板 + 截图。

## 六、设计原则

1. **每个模板自包含**：不依赖其他模板也能独立使用
2. **T0 是万能前缀**：任何组合都建议带上 T0，建立分析坐标系
3. **输出格式内嵌**：每个模板自带输出格式要求，不需要额外指定
4. **≤ 300 字硬约束**：确保不稀释 AI 注意力

---

## 七、方案 B vs 方案 C 选择建议

```
需要完整需求文档 + 追求最高质量？ → 方案 B（多Agent协作）
需要快速定向分析 + 灵活组合？     → 方案 C（模板化Prompt）
两者可以混用：用方案 C 快速摸底，发现复杂度高再切方案 B 深入分析。
```
