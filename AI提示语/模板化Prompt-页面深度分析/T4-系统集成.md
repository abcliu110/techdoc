# T4 — 系统集成

> **用途：** 分析外部系统依赖、读写模型差异和数据级联规则。架构师和后端开发使用。

---

## Prompt

```
请分析页面涉及的系统集成关系：

1. 外部系统依赖地图
   为每个操作（保存/发布/购买等）标注：
   {操作} → {下游服务}（同步/异步）→ 失败策略：{事务回滚/补偿重试/降级}
   区分同步调用（影响主流程）和异步消息（不阻断主流程）

2. 读写模型分离（CQRS视角）
   - 写模型（管理员表单）：字段全、约束严、格式原始
   - 读模型（消费者展示）：字段精简、格式友好
   用表格对比：写模型字段 | 读模型展示 | 转换逻辑

3. 数据级联规则
   父实体变更/删除时，子实体如何处理：
   | 操作 | 影响判断 | 策略（禁止/软删除/级联/警告） | 用户提示 |

4. 内联创建API
   页面中"+ 新增xxx"按钮的API交互模式：
   - 请求：POST /xxx → 返回 {id, name}
   - 与主表单的事务边界：独立事务，失败不影响主表单

输出格式：依赖地图用箭头链格式，级联规则用表格，API用RESTful风格描述。
```
