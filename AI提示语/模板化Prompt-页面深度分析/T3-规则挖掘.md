# T3 — 规则挖掘

> **用途：** 提取显式/隐式业务规则、联动逻辑和约束体系。前后端开发都需要。

---

## Prompt

```
请对页面截图进行业务规则深度挖掘：

1. 隐式文本挖掘
   - Tooltip文字 → 转化为精确业务规则（格式：原文 → 规则 → 技术要点）
   - Placeholder → 格式约束和合法示例
   - 帮助/警告文字 → 区分"建议"(WARN)和"强制"(BLOCK)
   - 按钮文字："保存"≠"发布"≠"提交"，各自触发什么副作用

2. UI 条件渲染规则（穷举所有联动）
   逐条列出，格式：
   RULE-{编号}:
     WHEN {字段} = {条件}
     THEN {目标字段} → {显示/隐藏/必填/禁用/值变更}
   特别关注：Toggle联动、Radio切换、多选组合条件、级联选择器

3. 业务不变量 vs 验证规则（严格区分）
   - 不变量：任意时刻必须成立，违反=数据损坏
     INVARIANT-{编号}: {表达式}（违反后果）
   - 验证规则：提交时校验，提交前允许暂时不满足
     VALIDATION-{编号}: {表达式}（校验时机）

4. 约束分级
   为每个字段按7级定义约束：类型/范围/必填/跨字段/唯一性/不变量/并发
   标注严重性：BLOCK（阻断）/ LOGIC（逻辑错误）/ WARN（警告）

输出格式：规则必须编号，必须可执行（有明确字段和条件），不接受模糊描述。
```
