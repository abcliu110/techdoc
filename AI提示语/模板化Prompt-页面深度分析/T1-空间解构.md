# T1 — 空间解构

> **用途：** 分析页面布局结构、视觉信号和"刻意缺失"的元素。适合产品经理和 UI 设计师。

---

## Prompt

```
请对页面截图进行空间解构分析：

1. 区块分区
   - 将页面划分为独立职能区块（如"基础信息区""策略核心区""准入限制区"）
   - 每个区块的业务目标是什么
   - 区块间的信息流向：如何引导用户从"定义是什么"→"配置怎么给"→"限制给谁用"

2. 视觉层级信号
   - 缩进/分组 = 父子依赖关系（缩进在某字段下方 = 该字段开启时才生效）
   - 颜色语义：绿色选中 = 激活；灰色 = 禁用/条件未满足
   - 标签措辞："推荐"vs"指定"vs"限制"背后的产品意图

3. 负空间分析
   - 页面刻意缺失的按钮暗示什么约束？（如无"复制"= 不支持克隆）
   - 缺失的字段暗示什么系统假设？（如无"货币类型"= 单币种假设）

4. 导航结构
   - 面包屑路径 = 数据层级归属
   - "保存"/"发布"/"提交"按钮的语义差异

输出格式：每个区块用表格描述（区块名 | 位置 | 业务目标 | 包含字段），负空间用"缺失元素 → 暗示假设 → 扩展影响"格式。
```
