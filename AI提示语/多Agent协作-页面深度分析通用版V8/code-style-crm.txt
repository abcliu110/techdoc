## 架构哲学（所有规则的源头，未覆盖场景回到此处判断）

P-1 单向依赖，禁止逆流
  Controller → ServicePlus → IService/Mapper，下层不反向依赖上层。ServicePlus间可互调但禁止循环依赖。Controller不出现Mapper；Entity不出现DTO/VO。

P-2 契约驱动，代码即文档
  前后端唯一契约=DTO/VO结构+@Schema注解。写完代码=写完文档=Apifox可直接导入。联调零口头沟通。

P-3 显式优于隐式
  枚举值写明含义；条件校验写在代码里；类型转换统一处理；错误提示是完整中文句子。

P-4 变更局部化
  改一个业务规则只改一个文件。枚举/校验/转换逻辑集中。禁止同一逻辑散布三层。

P-5 防御性编码，快速失败
  入口@Valid+JSR-303拦截非法数据→Service校验业务规则→数据库约束是最后防线非唯一防线。

P-6 安全零信任
  不信任前端任何数据（ID可能篡改、枚举可能越界、身份可能伪造）。每层假设上层可能失守。数据归属必须校验。

P-7 可演进性
  枚举可加值不改旧值、接口可加字段不删旧字段、表可加列不改旧列。向后兼容为默认，破坏兼容需显式版本号。

P-8 多租户隔离
  每张业务表强制包含 mid（商户ID）+ sid（门店ID），数据访问层拼接租户条件。跨租户数据不可见、不可改。

## 分层职责（严格执行，不可越层）

Controller层（controller/）：接收请求→@Valid参数校验→调用ServicePlus→返回NmsResult。禁止写业务逻辑、直接调Mapper。Controller是薄的"接线员"。
  ↓ 只传DTO，不传Entity/VO
ServicePlus层（service/）：业务逻辑、事务管理、条件校验、调用IService/其他ServicePlus。所有业务规则唯一归属地。返回NmsResult<VO>。
  ↓ 通过IService操作Entity
IService/Mapper层（dao/）：数据访问，MyBatis-Plus CRUD。禁止写业务逻辑、抛业务异常。纯数据通道。

对象转换：使用 BeanUtilsPlus.mapBean(source, TargetClass.class)（非 MapStruct）。

## 主键与ID策略（pid / mid / sid / lid 四ID体系）

CRM 采用四种ID，实体表中命名略有差异但语义一致：

| ID | 实体字段名 | 数据库列名 | 类型 | 生成方式 | 用途 |
|----|-----------|-----------|------|---------|------|
| pid | pid | pid | BIGINT AUTO_INCREMENT | 数据库自增 | 物理主键 |
| mid | mid | company_id | BIGINT | 登录上下文 | 商户ID |
| sid | sid | shop_id | BIGINT | 登录上下文 | 门店ID |
| lid | lid | lmnid | BIGINT UNIQUE | Yitter IdGenerator | 逻辑编号 |

★ CRM 中 mid 对应数据库列 company_id，sid 对应 shop_id，lid 对应 lmnid。
★ 前端看到的"ID"一律是 lid（lmnid），不是 pid。
★ 表间外键关联必须使用 lid（lmnid），禁止使用 pid。

Entity 对应注解（MyBatis-Plus）：
```java
@TableId(value = "pid", type = IdType.AUTO)
private Long pid;

@TableField("company_id")
private Long mid;

@TableField("shop_id")
private Long sid;

@TableField("lmnid")
private Long lid;
```

## 多租户规范

### 数据隔离
- 每张业务表必须包含 mid（company_id）和 sid（shop_id）
- 查询时拼接 mid/sid 条件
- 写入时从登录上下文注入 mid/sid

### 安全规则
- UPDATE/DELETE 操作必须先查数据 → 校验 mid/sid 归属 → 再操作
- 跨租户数据不可见、不可改

### 查询示例
```java
final LambdaQueryWrapper<CrmDepositPlan> query = new QueryWrapper<CrmDepositPlan>().lambda();
query.eq(CrmDepositPlan::getMid, request.getMid());
query.eq(CrmDepositPlan::getLid, request.getLid());
final CrmDepositPlan entity = iService.getOne(query);
```

## 物理模型优化规则

★ 从逻辑模型到物理模型的优化策略。目标：每功能模块3-5张表。

合并为JSON的条件（满足任一即可合并）：
- 纯ID列表（多选关联ID）→ brand_ids JSON [1,2,3]
- 只有1-2个业务字段的子记录 → periods JSON [{"start":"09:00","end":"12:00"}]
- 简单值列表（日期、编码等）→ exclude_dates JSON ["2026-04-05"]
- 数量有限(<20)且不需独立查询
- 只随主表读写，不被其他模块引用

必须独立建表的条件（满足任一即不可合并）：
- 子记录有3+个业务字段（不含pid/FK/审计字段）
- 需按子记录字段做条件查询或报表统计
- 子记录有独立状态或生命周期
- 子记录数量可能很大(100+)
- 子记录被其他模块通过FK引用

实现要点：
- JSON字段用MySQL JSON类型
- MyBatis-Plus 用 @TableField(typeHandler = JacksonTypeHandler.class) 映射
- DTO/VO结构不变（前端仍提交嵌套JSON），只是存储层合并

## 契约规范（前后端零沟通成本）

★ 后端写完→Apifox导入→前端直接开发。

统一响应：所有接口返回 NmsResult<T>
```json
{"success": true, "code": 0, "status": "ok", "data": { ... }}
```

NmsResult 关键字段：
```java
boolean success;       // 业务请求是否成功
Integer code;          // 0=成功, -1=失败
String status;         // "ok" 或 "error"
T data;                // 业务数据
String errorMessage;   // 错误信息
Long total;            // 分页总数
Long current;          // 当前页
Long pageSize;         // 页大小
Long pages;            // 总页数
Boolean compressed;    // 数据是否压缩
List<String> columns;  // 压缩模式的列名
```

DTO/VO分离（入参出参严格分开）：
- DTO接收请求：CrmXxxAddDTO / CrmXxxUpdateDTO / CrmXxxQueryDTO / CrmXxxGetDTO / CrmXxxDelDTO
- VO返回前端：CrmXxxVO
- 禁止：Entity直接暴露给前端

枚举值处理（CRM 实际模式）：
- Entity 中枚举字段直接使用枚举类型（如 CouponTypeEnum couponType）
- DTO 中枚举字段也使用枚举类型（非Integer）
- VO 中可包含 xxxName 派生字段用于前端展示

字段序列化：
| 字段类型 | 处理方式 | 原因 |
|---------|---------|------|
| Long ID | @JsonSerialize(using = ToStringSerializer.class) | JS精度丢失 |
| BigDecimal | @JsonSerialize(using = ToStringSerializer.class) | JS精度丢失 |
| LocalDateTime | @JSONField(format="yyyy-MM-dd HH:mm:ss") | Fastjson2格式 |
| 敏感数据 | @FieldSensitive + @SensitiveType | 脱敏处理 |

分页规范（CRM 实际模式）：
- QueryDTO 实现 IQueryDTO 接口，包含 current/pageSize 字段
- current: Long（当前页，@Min(0)），pageSize: Long（页大小，@Max(50)）
- 响应用 NmsResult<List<VO>>，分页信息在 NmsResult 的 total/current/pageSize/pages 字段
- 工具方法：IPageUtil.toResult(page, VO.class) 自动转换

API文档注解（不可省略）：
- Controller类：@Tag(name = "会员卡管理")
- Controller方法：@Operation(summary = "获取会员卡详情")
- DTO/VO每个字段：@Schema(title = "字段说明")
- ★ CRM 用 @Schema(title=...) 而非 @Schema(description=...)

错误码规范：
- NmsResult.error("错误信息") 直接返回
- 常量类定义错误信息：CrmXxxConstant.MSG_XXX_NOT_FOUND
- Assert.isTrue(condition, MSG_XXX) 快速校验

## 枚举类型规范

### 数据库存储
枚举值在数据库中存储数字编码：
```sql
status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1-正常，2-丢失，3-注销，4-冻结，5-退卡'
coupon_type TINYINT NOT NULL COMMENT '优惠券类型：1-现金券，2-买赠券，3-实物券，4-折扣券'
```

### Java 枚举定义（CRM 实际模式）
所有枚举实现 CommonEnum 接口：
```java
@Getter
@RequiredArgsConstructor
public enum CardStatusEnum implements CommonEnum {
    NORMAL(1, "正常"),
    LOSED(2, "丢失"),
    INVALID(3, "注销"),
    FREEZED(4, "冻结"),
    RETURN(5, "退卡");

    private final int code;
    private final String desc;    // ★ CRM 用 desc 而非 label

    private static final Map<Integer, CardStatusEnum> CODE_LOOK_UP = new ConcurrentHashMap<>();

    static {
        for (CardStatusEnum type : CardStatusEnum.values()) {
            CODE_LOOK_UP.put(type.code, type);
        }
    }

    @Override
    public CardStatusEnum getByCode(int code) {
        return CODE_LOOK_UP.get(code);
    }
}
```

### CommonEnum 接口
```java
public interface CommonEnum {
    int getCode();
    String getDesc();
    CommonEnum getByCode(int code);
}
```

★ CRM 枚举特点：
- 字段名用 desc（非 label）
- 用 ConcurrentHashMap 静态查找表（非 Arrays.stream）
- 部分枚举用 @EnumValue 标注（MyBatis-Plus 自动映射）
- Entity 中直接用枚举类型（如 CouponTypeEnum couponType），MyBatis-Plus 自动映射

## 代码质量规范

参数校验三道防线：
1. Controller入口（@Valid+JSR-303）：DTO加@NotNull/@NotBlank/@Size/@Min/@Max，message必须中文
2. Service业务校验：Assert.isTrue(condition, MSG_XXX) 或 if + NmsResult.error()
3. 数据库约束（最后防线）：NOT NULL、UNIQUE

事务管理（CRM 实际模式）：
- ★ CRM 使用 @Transactional(propagation = Propagation.SUPPORTS, rollbackFor = Exception.class)
- 读写操作统一用 Propagation.SUPPORTS（已有事务则加入，无事务则非事务执行）
- @Transactional只加在ServicePlus方法上

数据访问（MyBatis-Plus）：
- QueryWrapper + LambdaQueryWrapper 优先
- 批量插入 saveBatch()，批量更新 updateBatchById()
- 分页用 MyBatis-Plus 的 Page<T> 对象
- 示例：
```java
final QueryWrapper<CrmCard> query = new QueryWrapper<CrmCard>()
    .eq(LID, request.getLid());
final CrmCard entity = iCrmCardService.getOne(query, false);
```
```java
final LambdaQueryWrapper<CrmCard> query = new QueryWrapper<CrmCard>().lambda();
query.eq(CrmCard::getMid, request.getMid());
query.eq(CrmCard::getName, request.getName());
long count = iCrmCardService.count(query);
```
```java
final Page<CrmCard> page = new Page<>(request.current, request.pageSize);
return IPageUtil.toResult(iCrmCardService.page(page, query), CrmCardVO.class);
```
- ★ 禁止SELECT *，查列表用select(字段列表)
- ★ 防N+1查询：子表数据用IN批量查

并发安全：
- 库存/配额/余额：Redis INCR + LUA脚本保证原子性
- 关键数据：revision 乐观锁或 SELECT FOR UPDATE
- 幂等接口：唯一requestId + Redis去重

异常处理：
- 业务异常：return NmsResult.error("错误信息") 或 throw BizException
- 系统异常：log.error()记完整堆栈后返回通用错误提示
- ★ 禁止空catch

日志规范：
- 类加@Slf4j，用{}占位符，禁止打印敏感信息
- 格式：log.info("[{操作名}] {关键参数}={},...", value)

注释规范：
- 必须有：类Javadoc、Entity对应表名、ServicePlus每个public方法Javadoc、枚举类用途
- 禁止：// getter setter、代码能自解释处不加注释

安全编码：
- 数据归属校验：UPDATE/DELETE先查数据→校验 mid/sid 归属
- 敏感数据：VO加 @FieldSensitive + @SensitiveType 脱敏
- 防注入：LambdaQueryWrapper优先，自定义SQL用#{}
- 接口防刷：写接口加频率限制

逻辑删除查询陷阱：
- 唯一性校验考虑已删除数据：UNIQUE(name, deleted)或删除时改名
- JOIN子表需手动加AND sub.deleted = 0

依赖注入（CRM 实际模式）：
- ★ CRM 使用 @Autowired 字段注入（项目现有风格）
- 新代码建议用构造器注入(@RequiredArgsConstructor)，但与现有代码保持一致优先

接口幂等：
- 方案一：Token令牌（表单提交）
- 方案二：业务唯一键 — UNIQUE约束 + catch DuplicateKeyException
- ★ 每个POST写接口必须选其中一种

可演进性：
- 枚举：新增值加末尾，不改已有code

### ServicePlus层
```java
@Slf4j
@Service
public class CrmCardServicePlus extends BaseService {

    @Transactional(propagation = Propagation.SUPPORTS, rollbackFor = Exception.class)
    public NmsResult<CrmCardVO> get(CrmCardGetDTO request) {
        final QueryWrapper<CrmCard> query = new QueryWrapper<CrmCard>()
            .eq(LID, request.getLid());
        final CrmCard entity = iCrmCardService.getOne(query, false);
        return NmsResult.data(BeanUtilsPlus.mapBean(entity, CrmCardVO.class));
    }

    @Transactional(propagation = Propagation.SUPPORTS, rollbackFor = Exception.class)
    public NmsResult<List<CrmCardVO>> list(CrmCardQueryDTO request) {
        final QueryWrapper<CrmCard> query = new QueryWrapper<>();
        // 构造查询条件...
        final Page<CrmCard> page = new Page<>(request.current, request.pageSize);
        return IPageUtil.toResult(iCrmCardService.page(page, query), CrmCardVO.class);
    }

    @Transactional(propagation = Propagation.SUPPORTS, rollbackFor = Exception.class)
    public NmsResult<CrmCardVO> add(CrmCardAddDTO request) {
        // 唯一性校验
        QueryWrapper<CrmCard> query = new QueryWrapper<>(
            new CrmCard().setName(request.getName()));
        Assert.isTrue(iCrmCardService.count(query) == 0, VERIFY_NAME_CAN_NOT_REPEAT);

        // DTO → Entity
        final CrmCard entity = BeanUtilsPlus.mapBean(request, CrmCard.class);
        assertIsTrue(iCrmCardService.save(entity));

        // Entity → VO 返回
        return NmsResult.data(BeanUtilsPlus.mapBean(entity, CrmCardVO.class));
    }

    @Autowired
    private ICrmCardService iCrmCardService;
}
```

★ ServicePlus 关键点：
- 继承 BaseService
- @Slf4j + @Service
- 所有方法 @Transactional(propagation = Propagation.SUPPORTS, rollbackFor = Exception.class)
- 返回 NmsResult<VO> 而非原始 Entity
- BeanUtilsPlus.mapBean(source, TargetClass.class) 做对象转换
- Assert.isTrue() 做业务校验
- @Autowired 字段注入 IService

## 建表脚本模板（MySQL方言）

标准表模板（CRM 新建业务表必须遵循）：
```sql
CREATE TABLE crm_{business_name} (
  pid BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '物理编号',
  company_id BIGINT NOT NULL COMMENT '商户ID',
  shop_id BIGINT NOT NULL COMMENT '门店ID',
  lmnid BIGINT NOT NULL UNIQUE COMMENT '逻辑编号(雪花算法)',

  -- 业务字段...
  name VARCHAR(100) NOT NULL COMMENT '名称',
  status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-禁用，1-启用',

  created_by VARCHAR(100) COMMENT '创建人',
  created_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_by VARCHAR(100) COMMENT '更新人',
  updated_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  deleted TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除(0-否,1-是)',
  revision INT NOT NULL DEFAULT 0 COMMENT '乐观锁版本号',

  INDEX idx_company_id (company_id),
  INDEX idx_shop_id (shop_id),
  UNIQUE INDEX uk_lmnid (lmnid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='表说明';
```

★ CRM 建表列名用 company_id/shop_id/lmnid（非 mid/sid/lid），与 Entity @TableField 映射对应。

索引命名规范：
| 索引类型 | 命名规则 | 示例 |
|----------|----------|------|
| 普通索引 | idx_{字段名} | idx_company_id |
| 唯一索引 | uk_{字段名} | uk_lmnid |
| 复合索引 | idx_{字段1}_{字段2} | idx_company_id_status |

## 新增表检查清单

- [ ] 是否包含所有标准字段（pid, company_id, shop_id, lmnid, created_by, created_time, updated_by, updated_time, deleted, revision）
- [ ] lmnid 字段是否设置了唯一索引（字段 UNIQUE + UNIQUE INDEX uk_lmnid）
- [ ] 外键关联是否使用 lmnid 而非 pid
- [ ] company_id/shop_id 是否建立索引
- [ ] 索引命名是否符合规范（idx_/uk_ 前缀）
- [ ] 枚举字段是否只存数字编码，COMMENT 包含完整枚举值说明
- [ ] 布尔字段是否以 is_ 开头，类型 TINYINT，COMMENT 包含 0-否，1-是
- [ ] 是否有需要面向多语言用户展示的文本字段
- [ ] revision 字段是否包含（乐观锁）
- [ ] 对应 Entity 类是否有正确的 @TableField 列映射

## 与 code-style-nms4cloud.txt 的差异摘要

| 项目 | nms4cloud 通用规范 | CRM 实际模式 |
|------|-------------------|-------------|
| ORM | MyBatis-Flex | **MyBatis-Plus**（@TableName/@TableId/@TableField/@TableLogic） |
| Entity注解 | @Table/@Id/@Column | **@TableName/@TableId(type=IdType.AUTO)/@TableField** |
| 逻辑删除 | @Column(isLogicDelete=true) | **@TableLogic(value="1", delval="0")** |
| 乐观锁 | @Column(version=true) | **revision 字段（无注解或@Version）** |
| 对象映射 | MapStruct Convert | **BeanUtilsPlus.mapBean()** |
| 查询API | QueryChain/UpdateChain | **QueryWrapper/LambdaQueryWrapper** |
| Service命名 | XxxService | **CrmXxxServicePlus extends BaseService** |
| DTO命名 | XxxAddDTO | **CrmXxxAddDTO（含Crm前缀）** |
| VO命名 | XxxListVO + XxxVO | **CrmXxxVO（统一，无ListVO/DetailVO之分）** |
| 方法命名 | create/update/delete | **add/update/del** |
| HTTP方法 | RESTful GET/POST/PUT/DELETE | **统一 POST** |
| 分页 | PageReqDTO(pageNum/pageSize) | **IQueryDTO(current/pageSize)** |
| 分页响应 | PageResult<T> | **NmsResult<List<T>>（分页信息在NmsResult字段中）** |
| 依赖注入 | 构造器注入 | **@Autowired 字段注入** |
| Schema注解 | @Schema(description=...) | **@Schema(title=...)** |
| 事务传播 | REQUIRED | **Propagation.SUPPORTS** |
| Excel | 无 | **@ExcelProperty（EasyExcel导入导出）** |
| 脱敏 | 无 | **@FieldSensitive + @SensitiveType** |
| DB列名 | mid/sid/lid | **company_id/shop_id/lmnid** |
