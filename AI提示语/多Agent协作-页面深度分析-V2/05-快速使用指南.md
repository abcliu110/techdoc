# 快速使用指南 — 多 Agent 协作页面深度分析

## 30 秒上手

你有一张业务系统页面截图，想产出开发级需求文档。按以下 4 步操作：

---

## Step 1：发给 Agent-1（业务分析师）

**打开一个新对话，发送：**

```
[粘贴 01-Agent-业务分析师.md 中的 System Prompt 部分]
```

然后附上截图 + 一句话背景：
> "这是 XX 系统的 XX 配置页面，请按框架分析。"

**拿到输出后：**
- 检查实体识别是否完整
- 检查判别字段是否遗漏
- 复制最后的「交接摘要」部分备用

---

## Step 2：发给 Agent-2（规则工程师）

**打开一个新对话，发送：**

```
[粘贴 02-Agent-规则工程师.md 中的 System Prompt 部分]

---
以下是上游业务分析师的交接摘要：
[粘贴 Agent-1 的交接摘要]
```

然后附上同一张截图。

**拿到输出后：**
- 检查联动规则是否穷举
- 检查不变量 vs 验证规则是否区分清楚
- 复制最后的「交接摘要」部分备用

---

## Step 3：发给 Agent-3（系统架构师）

**打开一个新对话，发送：**

```
[粘贴 03-Agent-系统架构师.md 中的 System Prompt 部分]

---
以下是上游分析师的交接摘要：

【业务分析师摘要】
[粘贴 Agent-1 的交接摘要]

【规则工程师摘要】
[粘贴 Agent-2 的交接摘要]
```

然后附上同一张截图。

**拿到输出后：**
- 检查状态机有无死锁
- 检查 API 列表是否完整
- 检查并发方案是否具体

---

## Step 4：发给 Agent-4（文档整合者）

**打开一个新对话，发送：**

```
[粘贴 04-Agent-文档整合者.md 中的 System Prompt 部分]

---
以下是 3 位分析师的完整输出，请整合为最终需求文档：

===== Agent-1 业务分析报告 =====
[粘贴 Agent-1 完整输出]

===== Agent-2 规则分析报告 =====
[粘贴 Agent-2 完整输出]

===== Agent-3 架构分析报告 =====
[粘贴 Agent-3 完整输出]
```

**注意：** 这一步发给 Agent-4 的是 3 个 Agent 的**完整输出**，不是交接摘要。

## 各平台操作差异

### Claude / ChatGPT / Gemini（手动对话）

按上述 4 步，每步开一个新对话。适合偶尔使用。

### Cursor / Claude Code（IDE 内）

可以用 Agent 子任务实现半自动化：
1. 把 4 个 Prompt 文件放在项目中
2. 用 Agent 工具串行调度 4 个子任务
3. 中间产物通过文件传递
4. 最终文档中可直接生成**代码落地映射表**与**规则落点清单**，作为代码实现依据

### Dify / Coze（低代码平台）

1. 创建 4 个 LLM 节点，分别配置 System Prompt
2. 节点间用变量传递交接摘要
3. 加入人工审核节点（可选）

### AutoGen / CrewAI / LangGraph（代码编排）

```python
# 伪代码示意
analyst = Agent(role="业务分析师", prompt=load("01-Agent-业务分析师.md"))
rule_eng = Agent(role="规则工程师", prompt=load("02-Agent-规则工程师.md"))
architect = Agent(role="系统架构师", prompt=load("03-Agent-系统架构师.md"))
integrator = Agent(role="文档整合者", prompt=load("04-Agent-文档整合者.md"))

# 串行执行
r1 = analyst.run(image=screenshot, context="储值活动配置页面")
r2 = rule_eng.run(image=screenshot, upstream=r1.handoff_summary)
r3 = architect.run(image=screenshot, upstream=[r1.handoff_summary, r2.handoff_summary])
r4 = integrator.run(upstream=[r1.full_output, r2.full_output, r3.full_output])

# 输出最终文档
save(r4.output, "需求分析文档.md")
```

## 常见问题

### Q: 一定要 4 步全做吗？

不一定。根据需要可以灵活组合：

| 需求 | 使用 Agent |
|------|-----------|
| 只需数据建模 | Agent-1 |
| 只需校验规则 | Agent-1 → Agent-2 |
| 只需 API 设计 | Agent-1 → Agent-3 |
| 完整需求文档 | Agent-1 → 2 → 3 → 4 |

### Q: 截图不清楚 / 页面太长怎么办？

- 分区截图：按区块分别截图，每个 Agent 发送完整截图集
- 辅助说明：在截图旁边用文字补充截图看不清的内容（如 Tooltip 文字）

### Q: Agent 输出质量不满意怎么办？

- 只需重跑不满意的那个 Agent，不需要全部重来
- 追问补充："请补充分析 XX 字段的联动规则" / "请增加 XX 场景的并发分析"

### Q: 能用同一个对话窗口完成吗？

可以但不推荐。原因：
- 上下文窗口会被前面的输出占满，后面 Agent 的质量下降
- 分开对话可以并行（让 Agent-2 和 Agent-3 的部分工作同时进行）
- 分开对话方便单独重跑某个 Agent
