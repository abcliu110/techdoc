# 快速使用指南 — 多 Agent 协作页面深度分析 V3.1

## 30 秒了解 V3.1

V3.1 相比 V3 的核心变化：
- Agent-0 新增**阶段五：技术资产扫描**，可接入代码/DDL/API文档等技术资产
- Agent-1/2/3 新增**隐性知识探针**，反向推导系统运行所需的未见信息
- 所有输出表格新增**四级知识来源分类**（📸截图/👤用户/📄资产/🤖推断）
- Agent-4 新增**探针汇总章节**，与技术资产交叉验证
- Agent-5 新增**探针验证维度**

V3 相比 V2 的核心变化：
- 新增 **Agent-0（信息补全者）**：在分析前通过结构化提问补全截图缺失的信息
- 新增 **Agent-5（红队审查员）**：在整合后对文档做对抗性攻击测试
- 每个 Agent 新增**推理备忘录**输出，保留不确定推断供下游参考
- 支持**反馈环**：Agent-5 发现问题可触发上游 Agent 定向重跑


**（V3.1 新增）可选：附上技术资产**

如果你手头有相关的技术资产，一并提供可以大幅提升分析质量：

| 资产类型 | 怎么提供 | 效果 |
|---------|---------|------|
| 代码片段 | 粘贴 Controller/Service/Entity 代码 | 提取已实现的校验逻辑和字段约束 |
| 数据库 DDL | 粘贴建表语句 | 提取字段类型、长度、唯一约束、外键 |
| API 文档 | 粘贴 Swagger/OpenAPI JSON | 提取接口定义和参数校验规则 |
| 需求文档/PRD | 粘贴相关章节 | 提取已有的业务规则和状态流转 |
| 配置文件 | 粘贴枚举字典、权限配置 | 提取枚举值域和角色权限 |
| 错误码表 | 粘贴错误码定义 | 提取已定义的错误处理规范 |

> 提示：不需要提供全部，任意组合都有价值。一份 DDL + Entity 代码就能让下游 `[🤖 AI 推断]` 减少 50%+。

**交互流程：**
1. Agent-0 输出「截图初扫」→ 你确认它看到的是否正确
2. Agent-0 输出「提问清单」→ 你回答 P0 问题（必须），P1/P2 尽量回答

**拿到输出后：**

## 反馈环操作指南

当 Agent-5 的报告中包含 FEEDBACK 指令时：

1. **按优先级排序**：P0 必须执行，P1 建议执行，P2 可选
2. **重跑目标 Agent**：在对应 Agent 的对话中追加 FEEDBACK 内容
   ```
   以下是红队审查员的反馈，请针对性补充分析：

   FEEDBACK-001:
     类型：补充分析
     内容：{具体内容}
   ```
3. **重跑 Agent-4**：将补充后的输出重新发给 Agent-4 整合
4. **重跑 Agent-5**：验证修复是否到位
5. **最多 2 轮**：反馈环最多执行 2 轮，防止无限循环

## 各平台操作差异

### Claude / ChatGPT / Gemini（手动对话）

按上述 6 步（Step 0-5），每步开一个新对话。适合偶尔使用。

### Cursor / Claude Code（IDE 内）

可以用 Agent 子任务实现半自动化：
1. 把 7 个 Prompt 文件放在项目中
2. 用 Agent 工具串行调度 6 个子任务
3. 中间产物（交接摘要 + 推理备忘录）通过文件传递
4. Agent-5 的 FEEDBACK 指令可自动触发重跑

### Dify / Coze（低代码平台）

1. 创建 6 个 LLM 节点，分别配置 System Prompt
2. 节点间用变量传递交接摘要 + 推理备忘录
3. Agent-0 节点后加入人工输入节点（回答问题）
4. Agent-5 节点后加入条件分支（有 🔴 → 反馈环 / 无 🔴 → 输出）

### AutoGen / CrewAI / LangGraph（代码编排）

```python
# 伪代码示意
completer  = Agent(role="信息补全者",  prompt=load("01-Agent-信息补全者.md"))
analyst    = Agent(role="业务分析师",  prompt=load("02-Agent-业务分析师.md"))
rule_eng   = Agent(role="规则工程师",  prompt=load("03-Agent-规则工程师.md"))
architect  = Agent(role="系统架构师",  prompt=load("04-Agent-系统架构师.md"))
integrator = Agent(role="文档整合者",  prompt=load("05-Agent-文档整合者.md"))
red_team   = Agent(role="红队审查员",  prompt=load("07-Agent-红队审查员.md"))

# Step 0: 信息补全
r0 = completer.run(image=screenshot, context="储值活动配置页面")
# → 人工回答问题 → r0_enhanced = completer.followup(answers)

# Step 1-3: 串行分析
r1 = analyst.run(image=screenshot, enhanced_input=r0_enhanced)
r2 = rule_eng.run(image=screenshot, enhanced_input=r0_enhanced,
                  upstream=r1.handoff_with_memo)
r3 = architect.run(image=screenshot, enhanced_input=r0_enhanced,
                   upstream=[r1.handoff_with_memo, r2.handoff_with_memo])

# Step 4: 整合
r4 = integrator.run(upstream=[r1.full_output, r2.full_output, r3.full_output])

# Step 5: 红队审查 + 反馈环
for round in range(2):  # 最多 2 轮
    r5 = red_team.run(document=r4.output, enhanced_input=r0_enhanced,
                      image=screenshot)
    if not r5.has_critical:
        break
    # 执行反馈环
    for fb in r5.feedbacks:
        target_agent = get_agent(fb.target)
        target_agent.run_feedback(fb.content)
    r4 = integrator.run(upstream=[r1.full_output, r2.full_output, r3.full_output])

save(r4.output, "需求分析文档.md")
```

## 常见问题

### Q: 一定要 6 步全做吗？

不一定。根据需要灵活组合：

| 需求 | 使用 Agent |
|------|-----------|
| 只需信息梳理 | Agent-0 |
| 只需数据建模 | Agent-0 → Agent-1 |
| 只需校验规则 | Agent-0 → Agent-1 → Agent-2 |
| 只需 API 设计 | Agent-0 → Agent-1 → Agent-3 |
| 完整需求文档 | Agent-0 → 1 → 2 → 3 → 4 |
| 完整 + 对抗验证 | Agent-0 → 1 → 2 → 3 → 4 → 5 |

### Q: Agent-0 可以跳过吗？

可以，但不推荐。跳过 Agent-0 意味着：
- 下游 Agent 会产生更多 `[⚠️ 假设]` 标记
- Agent-5 红队审查时会发现更多信息缺口
- 最终可能还是要回来补信息，不如一开始就做

如果你的截图非常完整（覆盖所有状态、所有 Tab），且你对业务非常熟悉，可以自己写一份简化版的增强输入包替代 Agent-0。

### Q: 推理备忘录太长怎么办？

推理备忘录是给下游 Agent 的"参考资料"，不是必读内容。如果上下文窗口紧张：
- 优先传递交接摘要（必须）
- 推理备忘录只传递标记为"高置信度"和"异常信号"的条目
- Agent-4 整合时会自动筛选高价值条目

### Q: 反馈环要跑几轮？

通常 1 轮就够。设计上限为 2 轮，防止无限循环。如果 2 轮后仍有 🔴 发现，说明输入信息本身不足，应该回到 Agent-0 补充信息而非继续循环。

### Q: 能用同一个对话窗口完成吗？

可以但不推荐。原因与 V2 相同：
- 上下文窗口会被前面的输出占满，后面 Agent 的质量下降
- 分开对话可以并行
- 分开对话方便单独重跑某个 Agent（特别是反馈环场景）

### Q: V3 比 V2 多了 2 个 Agent，会不会太慢？

Agent-0 是一次性投入（问完就不用再跑），Agent-5 是可选的质量保障步骤。实际体感：
- Agent-0 增加约 5-10 分钟（取决于你回答问题的速度）
- 如果提供了技术资产，Agent-0 的技术资产扫描增加约 2-3 分钟
- Agent-5 增加约 3-5 分钟
- 但因为输入质量提升，Agent-1~3 的输出质量更高，减少了人工审核和返工时间
- 总体效率通常持平或更优

### Q: 隐性知识探针问题太多怎么办？（V3.1 新增）

探针问题是"待验证问题清单"，不是必须全部回答的。处理策略：
- P0 探针：必须验证，否则可能导致架构方向性错误
- P1 探针：建议验证，不验证会增加假设
- P2 探针：可选，有则更好
- 如果你提供了技术资产，Agent-4 会自动用技术资产回答部分探针问题
- 未回答的探针问题会进入最终文档的"未决问题清单"，不会阻塞交付
