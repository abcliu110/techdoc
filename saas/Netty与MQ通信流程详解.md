## 二、消息类型与协议

### 2.1 协议格式

```
┌─────────────────────────────────────┐
│  长度(4字节) │ 类型(1字节) │ 数据(n字节) │
│  writeInt()  │ writeByte() │ writeBytes()│
└─────────────────────────────────────┘
```

- **长度**: 总消息长度 = 5 + 数据长度
- **类型**: 0=心跳, 1=业务消息, 2=订阅, 3=取消订阅
- **数据**: JSON 字符串的 UTF-8 字节

### 2.2 消息类型定义

```java
// CustomHeartbeatHandler.java
public static final byte IDLE_QUIT = 0;        // 空闲断开
public static final byte HEARTBEAT_REQUEST = 1; // 心跳请求
public static final byte HEARTBEAT_RESPONSE = 2; // 心跳响应
public static final byte CUSTOM_MSG = 3;        // 自定义消息
```

### 3.3 双跳中继流程

```
┌────────────────────────────────────────────────────────────────────────────────┐
│                           双跳中继流程                                          │
├────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   nms4cloud-mq                                      nms4cloud-netty            │
│   ┌──────────────────────────────────┐         ┌──────────────────────────┐    │
│   │ Netty Server (:9999)             │         │ Netty Server (:9999)     │    │
│   │ ┌──────────────────────────────┐ │         │ ┌──────────────────────┐ │    │
│   │ │ 接收 POS 终端消息            │ │         │ │ 接收来自 mq 的消息   │ │    │
│   │ └──────────────────────────────┘ │         │ └──────────────────────┘ │    │
│   │                                  │         │                          │    │
│   │         │                        │         │                          │    │
│   │         ▼                        │         │                          │    │
│   │ ┌──────────────────────────────┐ │         │                          │    │
│   │ │ 消息通过 ServerHandler       │ │         │                          │    │
│   │ │ 路由到 topic_to_context      │ │         │                          │    │
│   │ └──────────────────────────────┘ │         │                          │    │
│   │                                  │         │                          │    │
│   │         │                        │         │                          │    │
│   │         ▼                        │         │                          │    │
│   │ ┌──────────────────────────────┐ │   TCP ──┘                          │    │
│   │ │ Netty Client (连接到         │ │◄────────────                        │    │
│   │ │ nms4cloud-netty:9999)        │ │                                    │    │
│   │ └──────────────────────────────┘ │                                    │    │
│   └──────────────────────────────────┘                                    │    │
│                                                                                 │
│   说明:                                                                      │
│   1. POS 终端连接到 nms4cloud-mq 的 9999                                     │
│   2. nms4cloud-mq 内部有 Client 连接到 nms4cloud-netty:9999                  │
│   3. 消息可在两个 Netty Server 之间转发                                       │
│   4. 形成双跳中继架构，提供消息冗余和负载均衡                                   │
│                                                                                 │
└────────────────────────────────────────────────────────────────────────────────┘
```

## 四、核心类说明

### 4.1 服务端核心类

| 类名 | 路径 | 职责 |
|------|------|------|
| NettyServer | nms4cloud-mq/.../netty/server/ | 启动 Netty 服务器，监听 9999 端口 |
| ServerHandler | nms4cloud-mq/.../netty/server/ | 处理消息，路由到 topic_to_context |
| ClientHandler | nms4cloud-mq/.../netty/client/ | 处理与 nms4cloud-netty 的连接 |
| CustomHeartbeatHandler | nms4cloud-mq/.../netty/ | 心跳检测，空闲断开 |

### 4.2 客户端核心类

| 类名 | 路径 | 职责 |
|------|------|------|
| NettyClient | nms4cloud-pos3boot/.../netty/client/ | POS 端 Netty 客户端 |
| ClientHandler | nms4cloud-pos3boot/.../netty/client/ | 处理服务端消息 |

### 4.3 MQ 核心类

| 类名 | 路径 | 职责 |
|------|------|------|
| SendMqttMsgConsumer | nms4cloud-mq/.../consumer/ | 消费 MQTT_SEND_MSG |
| MqMqttMsgServicePlus | nms4cloud-mq/.../service/ | 消息分发服务 |

## 五、错误处理机制

### 5.1 心跳检测

```java
// IdleStateHandler 参数说明
p.addLast(new IdleStateHandler(0, 0, 5));
// 读空闲超时: 0 (不检测)
// 写空闲超时: 0 (不检测)
// 读写空闲超时: 5秒 (5秒无读写则触发)
```

### 5.2 重连机制

```java
// NettyClient.java
futureListener.channel().eventLoop().schedule(this::doConnect, 10, TimeUnit.SECONDS);
// 连接失败后，10秒后重试
```

### 5.3 日志记录

所有关键操作都有日志记录:
- 连接成功/失败
- 消息发送/接收
- 订阅/取消订阅
- 异常堆栈
