# 储值方案日期校验修改说明

## 问题描述

在创建或修改储值方案时，当开始日期和结束日期设置为同一天（例如：2026-03-13 到 2026-03-13），系统会报错：

```
结束日期必须晚于开始日期
```

这个校验过于严格，不允许创建当天有效的活动。

## 业务需求

实际业务中，经常需要创建当天有效的储值活动，例如：
- 单日促销活动（2026-03-13 00:00:00 到 2026-03-13 23:59:59）
- 限时抢购活动（当天有效）
- 节日特惠活动（仅限当天）

因此，应该允许开始日期和结束日期为同一天。

## 修改内容

### 1. 修改校验逻辑

**文件：** `CrmDepositPlanServicePlus.java`

**修改前：**
```java
assertIsTrue(endTime.isAfter(beginTime), CrmDepositPlanConstant.MSG_END_TIME_MUST_AFTER_BEGIN_TIME);
```

**修改后：**
```java
// 允许结束日期等于或晚于开始日期（支持同一天的活动）
assertIsTrue(!endTime.isBefore(beginTime), CrmDepositPlanConstant.MSG_END_TIME_MUST_AFTER_BEGIN_TIME);
```

**说明：**
- 原逻辑：`endTime.isAfter(beginTime)` - 结束日期必须严格晚于开始日期
- 新逻辑：`!endTime.isBefore(beginTime)` - 结束日期不能早于开始日期（允许相等）

### 2. 更新错误消息

**文件：** `CrmDepositPlanConstant.java`

**修改前：**
```java
public static final String MSG_END_TIME_MUST_AFTER_BEGIN_TIME = "结束日期必须晚于开始日期";
```

**修改后：**
```java
public static final String MSG_END_TIME_MUST_AFTER_BEGIN_TIME = "结束日期不能早于开始日期";
```

**说明：** 错误消息更准确地反映了新的校验逻辑。

## 校验规则对比

| 场景 | 开始日期 | 结束日期 | 修改前 | 修改后 |
|------|---------|---------|--------|--------|
| 同一天 | 2026-03-13 00:00:00 | 2026-03-13 23:59:59 | ❌ 不通过 | ✅ 通过 |
| 同一天（相同时间） | 2026-03-13 12:00:00 | 2026-03-13 12:00:00 | ❌ 不通过 | ✅ 通过 |
| 跨天 | 2026-03-13 00:00:00 | 2026-03-14 23:59:59 | ✅ 通过 | ✅ 通过 |
| 结束早于开始 | 2026-03-14 00:00:00 | 2026-03-13 23:59:59 | ❌ 不通过 | ❌ 不通过 |

## 兼容性说明

### 向后兼容

✅ 完全向后兼容

- 原本能通过校验的数据（结束日期晚于开始日期）仍然能通过
- 新增支持同一天的活动，不影响现有数据

### 数据迁移

❌ 不需要数据迁移

- 现有数据不受影响
- 新的校验规则更宽松，不会导致现有数据不合法
