## 详细关系说明

### 1. 层级关系

```
配置阶段 → 执行阶段 → 记录阶段

crm_deposit_plan (定义规则)
    ↓
会员选择 + 支付
    ↓
crm_card_balance (更新余额，使用乐观锁)
    ↓
crm_card_balance_task (记录操作)
    ↓
crm_card_record (生成流水)
    ↓
crm_member_gift_record (发放优惠券)
```

### 2. 主要联接点

#### crm_deposit_plan → crm_deposit_category
```
crm_deposit_plan.category_lid = crm_deposit_category.lid
```
用途：分类管理储值方案

#### crm_deposit_plan → crm_deposit_agreement
```
crm_deposit_plan.agreement_lids[] (JSON数组) → crm_deposit_agreement.lid
```
用途：关联储值需要确认的协议条款

#### crm_card_balance → crm_card_balance_task
```
crm_card_balance.lid = crm_card_balance_task.card_lid
```
用途：一个卡可有多条操作记录

#### crm_card_balance_task → crm_card_record
```
crm_card_balance_task.lid = crm_card_record.task_lid
```
用途：每条操作任务可对应详细的交易流水

#### crm_card_balance → crm_member_gift_record
```
crm_card_balance.lid = crm_member_gift_record.card_lid
```
用途：记录该卡领取的各种礼品

## 数据流向

### 储值流程数据流向
```
1. 用户输入
   ↓
2. 查询 crm_deposit_plan (获取可用方案)
   ├─ 过滤条件：status, date_type, cycle_rule, time_periods等
   ├─ 查询 crm_deposit_category (获取分类名称)
   └─ 查询 crm_deposit_agreement (获取协议)
   ↓
3. 用户选择档位
   ├─ 提取 tier_rule 中的 saveAmount, giftAmount
   ├─ 获取 coupons[], couponPackages[] 信息
   └─ 检查 purchase_limit_rule
   ↓
4. 支付成功回调
   ├─ 更新 crm_card_balance (使用乐观锁)
   │   ├─ principal += saveAmount
   │   ├─ gift += giftAmount
   │   └─ total = principal + gift
   ├─ 插入 crm_card_balance_task (记录操作)
   ├─ 插入 crm_card_record (生成流水)
   ├─ 发放优惠券
   │   └─ 插入 crm_member_gift_record
   ├─ 升级会员等级（如需要）
   └─ 更新 crm_deposit_plan 统计
       ├─ total_sold_count += 1
       └─ total_sold_amount += saveAmount
```

### 消费流程数据流向
```
1. 用户结账
   ↓
2. 查询 crm_card_balance
   └─ 获取当前的 principal, gift, total
   ↓
3. 扣减逻辑
   ├─ 检查 total >= 消费金额
   ├─ 计算金额限制 (amount_limit_type)
   ├─ 优先扣除 gift
   └─ 再扣除 principal
   ↓
4. 更新余额（乐观锁）
   └─ gift -= deductGift
   └─ principal -= deductPrincipal
   └─ total -= 总扣减
   ↓
5. 记录操作
   ├─ 插入 crm_card_balance_task
   │   └─ operation_model = "CONSUME"
   └─ 插入 crm_card_record
       ├─ balance_before/after
       ├─ principal_before/after
       └─ give_amount_before/after
```

## 查询优化建议

### 常见查询
```sql
-- 1. 查询会员可用的储值方案
SELECT * FROM crm_deposit_plan
WHERE mid = ? AND sid = ? AND status = 1
  AND date_type = 2 OR (begin_time <= NOW() AND end_time >= NOW())
ORDER BY sort_no;

-- 2. 查询会员卡余额
SELECT principal, gift, total FROM crm_card_balance
WHERE lid = ?;

-- 3. 查询会员最近的储值记录
SELECT * FROM crm_card_balance_task
WHERE card_lid = ? AND operation_model = 'RECHARGE'
ORDER BY created_time DESC LIMIT 10;

-- 4. 对账：查询卡的交易流水
SELECT * FROM crm_card_record
WHERE card_lid = ?
ORDER BY created_time DESC;
```

### 建议的索引
```sql
-- crm_deposit_plan
CREATE INDEX idx_mid_sid_status ON crm_deposit_plan(mid, sid, status);
CREATE INDEX idx_date_type ON crm_deposit_plan(date_type);
CREATE INDEX idx_category_lid ON crm_deposit_plan(category_lid);

-- crm_card_balance
CREATE UNIQUE INDEX uk_cno ON crm_card_balance(cno);

-- crm_card_balance_task
CREATE INDEX idx_card_lid_operation ON crm_card_balance_task(card_lid, operation_model);
CREATE INDEX idx_created_time ON crm_card_balance_task(created_time);

-- crm_card_record
CREATE INDEX idx_card_id_created ON crm_card_record(card_id, created_time);
```

## 标准值说明

### operation_model（操作类型）
- RECHARGE - 充值
- CONSUME - 消费
- REFUND - 退款
- CANCEL - 撤销
- ADJUST - 调整

### source（余额来源）
- 1 - 储值
- 2 - 返利
- 3 - 赠送
- 4 - 系统调整

### status（方案状态）
- 0 - 禁用
- 1 - 启用

### plan_type（方案类型）
- 1 - 储值分期送礼
- 2 - 智能储值
- 3 - 储值套餐
