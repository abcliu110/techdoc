# 旧会员储值系统 - 表结构速查

## 7张核心数据表一览

```
CrmSaveRule (储值规则)
  ├─ 100+ 行定义规则（本金、赠送、优惠券等）
  └─ 被 CrmDealTask 引用

CrmSaveRuleInterval (储值区间)
  └─ 阶梯式赠送规则

CrmCard (会员卡)
  ├─ balance (总可用余额)
  ├─ principal_balance (本金)
  ├─ give_balance (赠送)
  └─ 10+ 个统计字段

CrmCardBalance (余额变动)
  ├─ 单次变动的记录
  └─ 与 CrmCard 配合使用

CrmCardRecord (交易流水)
  ├─ 最完整的审计记录
  ├─ 包含 before/after 状态
  └─ 用于对账和纠纷处理

CrmDealTask (交易主任务)
  ├─ 每次储值/消费的主记录
  ├─ 支付状态和金额信息
  └─ 支持多种操作类型

CrmDealTaskItem (交易明细)
  ├─ 一个任务可有多条明细
  └─ 每条明细对应一个规则
```

## 业务字段计算公式

### 储值金额计算

```
IF 选择了多个规则：
  principalAmount = rule1.save_amount + rule2.save_amount + ...
  giveAmount = rule1.give_amount + rule2.give_amount + ...
  givePoint = rule1.give_point + rule2.give_point + ...
ELSE:
  principalAmount = 选中规则.save_amount
  giveAmount = 选中规则.give_amount
  givePoint = 选中规则.give_point

totalAmount = principalAmount + giveAmount
```

### 分期金额计算

```
IF 规则.unpaid_amount 存在：
  unpaidAmount = 规则.unpaid_amount
ELSE IF 规则.unpaid_rate 存在：
  unpaidAmount = totalAmount * 规则.unpaid_rate
ELSE:
  unpaidAmount = 0
```

### 消费金额扣减

```
remainAmount = 使用金额

IF give_balance >= remainAmount:
  give_balance -= remainAmount
  principal_balance -= 0
ELSE:
  principal_balance -= (remainAmount - give_balance)
  give_balance = 0

balance -= 使用金额
```

## 关键约束和检查

### 存储前验证

✓ CrmSaveRule:
  - name 唯一性
  - save_amount > 0
  - give_amount >= 0
  - 优惠券 ID 有效性
  - begin_time < end_time

✓ 储值提交前：
  - 卡存在且有效
  - 规则启用 (status_=1)
  - 在有效期内
  - 门店在 apply_to_store 中
  - 选择的优惠券有效（如果 optional=true）

✓ 消费前：
  - balance >= 使用金额
  - 卡未过期
  - 余额未冻结

### 并发保护

```sql
-- 使用乐观锁更新 CrmCard
UPDATE crm_card
SET balance = balance + ?,
    principal_balance = principal_balance + ?,
    revision = revision + 1
WHERE lid = ? AND revision = ?
```

如果 WHERE 条件 revision 不匹配，说明有其他线程已更新，需重试。

## 数据关系速记

```
会员卡(CrmCard)
  └─ 储值(CrmDealTask: operation=Charge)
     ├─ 关联规则(CrmSaveRule)
     ├─ 多个明细(CrmDealTaskItem)
     └─ 更新余额
        ├─ 记录到 CrmCardBalance
        └─ 审计到 CrmCardRecord

  └─ 消费(CrmDealTask: operation=Consume)
     ├─ 扣除余额
     ├─ 记录到 CrmCardBalance
     └─ 审计到 CrmCardRecord
```

## 常见查询模式

### 查询会员当前余额
```
SELECT balance, principal_balance, give_balance, points
FROM crm_card
WHERE lid = ?
```

### 查询储值历史
```
SELECT * FROM crm_card_record
WHERE card_lid = ? AND operation_model = 'Charge'
ORDER BY created_time DESC
```

### 查询消费历史
```
SELECT * FROM crm_card_record
WHERE card_lid = ? AND operation_model = 'Consume'
ORDER BY created_time DESC
```

### 查询可用规则
```
SELECT * FROM crm_save_rule
WHERE status_ = 1
  AND begin_time <= NOW()
  AND end_time >= NOW()
  AND (apply_to_store IS NULL OR JSON_CONTAINS(apply_to_store, '"门店ID"'))
ORDER BY sort_order
```

## 常见问题解答

**Q: principal 和 give 的区别？**
A: principal 是用户实际支付的钱，give 是商户赠送的钱。消费时优先扣 give。

**Q: 为什么需要 CrmCardBalance？**
A: 记录每次变动，用于审计和对账。CrmCard 只记录累计值。

**Q: balance 和 principal_balance 的关系？**
A: balance = principal_balance + give_balance，是总可用余额。

**Q: unpaid 是什么？**
A: 分期未付金额。比如储值100元，分期10%，则 unpaid=10。

**Q: 支付订单号 (out_trade_no) 的作用？**
A: 关联 WeChat 等外部支付系统的订单，用于支付回调和对账。

**Q: 为什么有 CrmDealTaskItem？**
A: 支持一次储值选择多个规则。每个规则一条 item。
