## 概述

会员储值是CRM系统中一个重要的营销和财务功能，允许商户定义灵活的储值方案，会员可以通过多种渠道和方式进行储值，并享受相应的优惠和赠送。

### 核心特性

- **多档位储值方案**：支持多个储值金额档位，每个档位可设置不同的赠送内容
- **灵活的限制规则**：支持周期、时段、日期、渠道、门店、品牌、会员等级等多维度限制
- **丰富的赠送形式**：支持赠送金额、积分、优惠券、券包、会员升级等
- **精细的权限控制**：支持按会员方案和等级进行差异化配置
- **完整的发放机制**：支持一次性发放和周期性发放

## 核心业务流程

### 1. 储值方案配置流程

```
商户创建储值方案
    ↓
定义基本信息（名称、分类、描述等）
    ↓
配置有效期规则（固定日期或永久有效）
    ↓
配置可用性规则（周期、时段、排除日期）
    ↓
配置渠道和门店规则
    ↓
配置品牌和会员等级规则
    ↓
配置购买限制规则
    ↓
定义储值档位规则
  ├─ 档位1: 储值100元，赠50元，赠送优惠券×1
  ├─ 档位2: 储值500元，赠150元，赠送优惠券×2
  └─ 档位3: 储值1000元，赠300元，赠送优惠券×3、券包×1
    ↓
提交审核
    ↓
方案启用
```

### 2. 会员储值流程

```
会员选择储值
    ↓
系统查询可用储值方案
  ├─ 基于渠道过滤（POS、H5、小程序等）
  ├─ 基于当前时间过滤（有效期、周期、时段）
  ├─ 基于会员等级过滤
  └─ 基于购买限制过滤
    ↓
会员选择储值档位
    ↓
系统计算赠送内容
  ├─ 计算赠送金额
  ├─ 计算赠送积分
  ├─ 准备发放的优惠券
  ├─ 准备发放的券包
  └─ 检查是否升级会员等级
    ↓
会员确认金额和赠送内容
    ↓
选择支付方式支付
    ↓
支付成功回调处理
  ├─ 更新会员卡余额（本金+赠送）
  ├─ 创建余额变动记录
  ├─ 发放优惠券
  ├─ 发放券包
  ├─ 发放积分
  ├─ 升级会员等级（如需要）
  └─ 更新储值方案统计数据
    ↓
储值完成
```

### 3. 会员储值余额使用流程

```
会员购物结账
    ↓
检查会员卡的储值余额
    ↓
会员选择使用储值（可选）
    ↓
系统计算可用金额
  ├─ 检查本金余额
  ├─ 检查赠送金额余额
  ├─ 检查金额限制规则（当日/当餐）
  └─ 根据优先级确定消费顺序
    ↓
会员确认使用金额
    ↓
扣除卡余额
  ├─ 优先扣除赠送金额（gift）
  └─ 再扣除本金（principal）
    ↓
更新卡余额
    ↓
创建流水记录
    ↓
交易完成
```

## 关键业务逻辑

### 1. 储值档位规则（tier_rule）详解

这是 `crm_deposit_plan` 中最复杂的 JSON 字段，定义了每个储值档位的详细配置。

#### JSON 结构

```json
{
  "tiers": [
    {
      "saveAmount": 100.00,
      "giftAmount": 20.00,
      "giftPoints": 50,
      "recommendTitle": "推荐",
      "isUpgradeMemberLevel": false,
      "upgradeMemberLevels": [
        {
          "memberPlanLid": 100001,
          "memberPlanName": "金卡方案",
          "memberLevelLid": 200001,
          "memberLevelName": "黄金会员",
          "memberLevelCode": "GOLD"
        }
      ],
      "isLevelDelay": true,
      "levelDelayDays": 30,
      "isFirstChargeOnly": false,
      "isGiftEnabled": true,
      "coupons": [
        {
          "couponLid": 300001,
          "couponCode": "COUPON001",
          "couponName": "满减券",
          "couponType": 1,
          "issueQtyPerTime": 1,
          "categoryName": "通用券",
          "termOfValidityMethod": "XD",
          "validDays": 30,
          "effectiveMethod": "TS",
          "whenToTakeEffect": 0,
          "isCountInActualAmount": true,
          "actualAmountType": "COUPON_WRITE_OFF_AMOUNT",
          "actualAmountValue": null,
          "sortNo": 1
        }
      ],
      "couponPackages": [
        {
          "packageLid": 400001,
          "packageCode": "PKG001",
          "packageName": "券包1",
          "categoryName": "通用券包",
          "packageQty": 1,
          "sortNo": 1
        }
      ],
      "issueRuleType": "ONE_TIME",
      "issueTime": null,
      "sortNo": 1
    }
  ]
}
```

#### 字段详解

**档位基本信息：**
- `saveAmount`: 储值本金金额（必填，>0）
- `giftAmount`: 赠送金额（可选，>=0）
- `giftPoints`: 赠送积分（可选，>=0）
- `recommendTitle`: 推荐标签（如"热销"、"推荐"）
- `isFirstChargeOnly`: 是否仅首次充值时赠送（限制性字段）

**会员升级配置：**
- `isUpgradeMemberLevel`: 是否自动升级会员等级（二级开关）
- `upgradeMemberLevels`: 升级配置列表（当上述开关为true时必填）
  - 支持按不同会员方案配置不同升级目标
  - 例如：金卡方案升到黄金，钻卡方案升到钻石
- `isLevelDelay`: 是否延期等级（true时等级有效期延长）
- `levelDelayDays`: 延期天数（当isLevelDelay=true时必填）

**赠送礼品配置：**
- `isGiftEnabled`: 是否启用赠送礼品（一级开关）
- `coupons`: 优惠券列表（当isGiftEnabled=true时可选）
  - 每个优惠券包含发放数量、有效期、生效时间等
  - `issueQtyPerTime`: 每次发放数量
  - `termOfValidityMethod`: 有效期类型
    - XD = 相对有效期（基于validDays）
    - GD = 固定有效期（startEffectiveTime 到 endEffectiveTime）
    - ZRY = 自然月内有效
    - YJ = 永久有效
  - `effectiveMethod`: 生效时间计算方式
    - TS = 按天计算
    - TZD = 按天零点计算
    - XS = 按小时计算
  - `whenToTakeEffect`: 何时生效
    - 0 = 立即生效
    - >0 = 延迟，单位由effectiveMethod决定
  - `isCountInActualAmount`: 是否计入实收金额
    - 用于核算商户实际收入

- `couponPackages`: 券包列表（当isGiftEnabled=true时可选）
  - 券包是多张券的组合
  - `packageQty`: 每次发放的券包数量

**发放规则配置：**
- `issueRuleType`: 发放规则类型
  - ONE_TIME = 一次性发放（储值时立即发放）
  - PERIODIC = 周期性发放（定期发放）
- 周期发放时需要配置：
  - `issueCycleType`: DAILY/WEEKLY/MONTHLY
  - `issueCycleValues`: 周期值列表
  - `issueTime`: 发放时间
  - `issueTimes`: 总发放次数
  - `firstIssueImmediate`: 首次是否立即发放

```json
    "limitMessage": "您的购买次数今日已达上限，请明日再来"
  },
  "totalSaleLimit": {
    "enabled": true,
    "limit": 1000,
    "limitMessage": "储值套餐已售罄"
  },
  "totalSalePeriodLimit": {
    "enabled": true,
    "periodType": "DAILY",
    "limit": 100,
    "limitMessage": "储值套餐今日已被抢光，请明日再来"
  }
}
```

**限制类型说明：**

1. **perUserTotalLimit** - 单个会员总购买限制
   - 该会员对该方案的总购买次数上限
   - 例如：10次，表示该会员最多购买10次

2. **perUserPeriodLimit** - 单个会员周期购买限制
   - 该会员在周期内的购买次数上限
   - periodType: DAILY/WEEKLY/MONTHLY/YEARLY
   - 例如：DAILY + 5，表示每天最多购买5次

3. **totalSaleLimit** - 总销售限制
   - 该方案的总销售次数上限
   - 例如：1000次，方案全局最多销售1000次

4. **totalSalePeriodLimit** - 总销售周期限制
   - 该方案在周期内的销售次数上限
   - 例如：DAILY + 100，表示全局每天最多销售100次

### 3. 可用性规则详解

#### cycle_rule（周期规则）

```json
{
  "type": "WEEKLY",
  "values": [1, 2, 3, 4, 5]  // 1=周一，7=周日
}
```

支持的周期类型：
- DAILY: 每天可用
- WEEKLY: 按星期几，values: [1-7]
- MONTHLY: 按月份几号，values: [1-31]

#### time_periods（时段规则）

```json
{
  "type": "ALL_DAY"  // 全天可用
}

// 或

{
  "type": "SPECIFIED",
  "periods": [
    {
      "startMin": 540,    // 9:00（540分钟）
      "endMin": 1020      // 17:00（1020分钟）
    },
    {
      "startMin": 1200,   // 20:00
      "endMin": 1320      // 22:00
    }
  ]
}
```

#### exclude_dates（排除日期）

```json
[
  "2026-01-01",
  "2026-02-10",
  "2026-05-01"
]
```

在这些日期，该方案不可用。

### 4. 金额限制规则（amount_limit_type）

当 `amount_limit_type != 0` 时，会在消费时进行限制：

- `0`: 不限制（默认）
- `1`: 储值当日限制（一天内的储值金额上限）
- `2`: 储值当餐限制（一笔订单中的储值使用金额上限）

配合 `max_amount_value` 使用：
```json
{
  "amount_limit_type": 1,
  "max_amount_value": 50.00  // 当日最多使用50元
}

// 或百分比
{
  "amount_limit_type": 2,
  "max_amount_value": 20  // 当餐最多使用订单金额的20%
}
```

### 2. 并发控制

**问题：** 多个储值请求同时到达时，余额计算可能出现错误

```sql
-- 更新时检查版本号
UPDATE crm_card_balance
SET principal = principal + ?,
    gift = gift + ?,
    total = total + ?,
    revision = revision + 1,
    updated_time = NOW()
WHERE lid = ? AND revision = ?
```


### 4. 金额消费逻辑

```java
// 会员消费时，优先扣除赠送金额，再扣除本金
public BigDecimal deductBalance(Long cardLid, BigDecimal amount) {
  CrmCardBalance balance = crmCardBalanceService.getByLid(cardLid);

  if (balance.getTotal().compareTo(amount) < 0) {
    throw new InsufficientBalanceException("余额不足");
  }

  BigDecimal deductGift = BigDecimal.ZERO;
  BigDecimal deductPrincipal = BigDecimal.ZERO;
  BigDecimal remaining = amount;

  // 第一步：优先扣除赠送金额
  if (balance.getGift().compareTo(remaining) > 0) {
    deductGift = remaining;
    remaining = BigDecimal.ZERO;
  } else {
    deductGift = balance.getGift();
    remaining = remaining.subtract(deductGift);
  }

  // 第二步：扣除本金
  if (remaining.compareTo(BigDecimal.ZERO) > 0) {
    deductPrincipal = remaining;
  }

  // 更新余额
  balance.setGift(balance.getGift().subtract(deductGift));
  balance.setPrincipal(balance.getPrincipal().subtract(deductPrincipal));
  balance.setTotal(balance.getTotal().subtract(amount));

  crmCardBalanceService.updateById(balance);

  return amount;
}
```
