# 储值方案销售统计更新 - 使用说明

## 功能说明

在订单支付成功后，需要更新储值方案的销售统计数据，包括：
- `total_sold_count`：已售总数量
- `total_sold_amount`：已售总金额

## 使用方法

### 1. 单个订单更新

```java
@Autowired
private CrmDepositPlanServicePlus depositPlanService;

// 在订单支付成功后调用
public void onOrderPaid(Long orderId, Long planLid, BigDecimal amount) {
    // 更新销售统计
    NmsResult<String> result = depositPlanService.updateSoldStatistics(planLid, amount);

    if (!result.isSuccess()) {
        log.error("更新销售统计失败 - orderId:{}, planLid:{}, error:{}",
            orderId, planLid, result.getMsg());
        // 可以选择重试或记录到失败队列
    }
}
```

### 2. 批量订单更新

```java
// 批量订单场景
public void onBatchOrdersPaid(List<Order> orders) {
    Map<Long, BigDecimal> statistics = new HashMap<>();

    for (Order order : orders) {
        Long planLid = order.getPlanLid();
        BigDecimal amount = order.getAmount();

        // 累加同一方案的金额
        statistics.merge(planLid, amount, BigDecimal::add);
    }

    // 批量更新
    NmsResult<String> result = depositPlanService.batchUpdateSoldStatistics(statistics);

    if (!result.isSuccess()) {
        log.error("批量更新销售统计失败 - error:{}", result.getMsg());
    }
}
```

## 并发处理

### 乐观锁机制

方法内部使用 MyBatis-Plus 的乐观锁机制（`@Version` 注解），防止并发更新导致的数据不一致。

**工作原理：**
```sql
-- 更新时会自动添加 revision 条件
UPDATE crm_deposit_plan
SET total_sold_count = total_sold_count + 1,
    total_sold_amount = total_sold_amount + #{amount},
    revision = revision + 1
WHERE lid = #{lid} AND revision = #{oldRevision}
```

如果 `revision` 不匹配（被其他事务修改），更新会失败。

### 重试策略

**方案1：简单重试**
```java
public void updateWithRetry(Long planLid, BigDecimal amount) {
    int maxRetries = 3;
    int retryCount = 0;

    while (retryCount < maxRetries) {
        NmsResult<String> result = depositPlanService.updateSoldStatistics(planLid, amount);

        if (result.isSuccess()) {
            return;
        }

        retryCount++;
        if (retryCount < maxRetries) {
            try {
                Thread.sleep(100 * retryCount); // 指数退避
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    log.error("更新销售统计失败，已重试{}次 - planLid:{}", maxRetries, planLid);
}
```

**方案2：使用 Spring Retry**
```java
@Retryable(
    value = {Exception.class},
    maxAttempts = 3,
    backoff = @Backoff(delay = 100, multiplier = 2)
)
public void updateWithSpringRetry(Long planLid, BigDecimal amount) {
    NmsResult<String> result = depositPlanService.updateSoldStatistics(planLid, amount);

    if (!result.isSuccess()) {
        throw new RuntimeException("更新失败: " + result.getMsg());
    }
}
```

## 事务处理

**重要：** 更新销售统计必须在订单支付事务提交之后执行，避免事务回滚导致统计数据不准确。

### 推荐方式：事务事件监听

```java
// 1. 定义订单支付成功事件
public class OrderPaidEvent {
    private Long orderId;
    private Long planLid;
    private BigDecimal amount;
    // getters and setters
}

// 2. 在订单支付成功后发布事件
@Service
public class OrderService {

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    @Transactional
    public void payOrder(Long orderId) {
        // 更新订单状态
        // ...

        // 发布事件（事务提交后触发）
        eventPublisher.publishEvent(new OrderPaidEvent(orderId, planLid, amount));
    }
}

// 3. 监听事件并更新统计
@Component
public class OrderPaidEventListener {

    @Autowired
    private CrmDepositPlanServicePlus depositPlanService;

    @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
    public void handleOrderPaid(OrderPaidEvent event) {
        depositPlanService.updateSoldStatistics(
            event.getPlanLid(),
            event.getAmount()
        );
    }
}
```

## 性能优化

### 1. 异步更新

对于高并发场景，可以异步更新统计数据：

```java
@Service
public class AsyncStatisticsService {

    @Autowired
    private CrmDepositPlanServicePlus depositPlanService;

    @Async("statisticsExecutor")
    public void updateAsync(Long planLid, BigDecimal amount) {
        depositPlanService.updateSoldStatistics(planLid, amount);
    }
}

// 配置线程池
@Configuration
public class AsyncConfig {

    @Bean("statisticsExecutor")
    public Executor statisticsExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(10);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("statistics-");
        executor.initialize();
        return executor;
    }
}
```

### 2. Redis 缓存 + 定时同步

对于超高并发场景，可以先更新 Redis，定时同步到数据库：

```java
@Service
public class RedisStatisticsService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    // 实时更新 Redis
    public void incrementInRedis(Long planLid, BigDecimal amount) {
        String countKey = "deposit:plan:sold:count:" + planLid;
        String amountKey = "deposit:plan:sold:amount:" + planLid;

        redisTemplate.opsForValue().increment(countKey, 1);
        redisTemplate.opsForValue().increment(amountKey, amount.doubleValue());
    }

    // 定时同步到数据库（每分钟执行）
    @Scheduled(cron = "0 * * * * ?")

            String amountKey = "deposit:plan:sold:amount:" + planLid;

            Double amount = Double.parseDouble(redisTemplate.opsForValue().get(amountKey));

            // 同步到数据库
            // ...

            // 清除 Redis 数据
            redisTemplate.delete(countKey);
            redisTemplate.delete(amountKey);
        }
    }
}
```

### 2. 补偿机制

更新失败时记录到补偿表，定时重试：

```sql
CREATE TABLE crm_deposit_plan_statistics_compensation (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  plan_lid BIGINT NOT NULL,
  amount DECIMAL(18,2) NOT NULL,
  retry_count INT DEFAULT 0,
  status TINYINT DEFAULT 0 COMMENT '0-待处理,1-已处理,2-失败',
  created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 注意事项

1. ✅ 必须在订单支付事务提交后调用
2. ✅ 需要处理乐观锁更新失败的情况（重试）
3. ✅ 高并发场景建议使用异步更新或 Redis 缓存
4. ✅ 定期校验数据一致性
5. ⚠️ 不要在事务内同步调用，避免长事务
6. ⚠️ 注意 BigDecimal 精度问题

## 测试建议

### 单元测试

```java
@Test
public void testUpdateSoldStatistics() {
    Long planLid = 100001L;
    BigDecimal amount = new BigDecimal("100.00");

    // 获取更新前的数据
    CrmDepositPlan before = getById(planLid);

    // 执行更新
    NmsResult<String> result = depositPlanService.updateSoldStatistics(planLid, amount);

    // 验证结果
    assertTrue(result.isSuccess());

    // 获取更新后的数据
    CrmDepositPlan after = getById(planLid);

    assertEquals(before.getTotalSoldCount() + 1, after.getTotalSoldCount());
    assertEquals(before.getTotalSoldAmount().add(amount), after.getTotalSoldAmount());
}
```

### 并发测试

```java
@Test
public void testConcurrentUpdate() throws InterruptedException {
    Long planLid = 100001L;
    BigDecimal amount = new BigDecimal("100.00");
    int threadCount = 10;

    CountDownLatch latch = new CountDownLatch(threadCount);
    ExecutorService executor = Executors.newFixedThreadPool(threadCount);

    for (int i = 0; i < threadCount; i++) {
        executor.submit(() -> {
            try {
                depositPlanService.updateSoldStatistics(planLid, amount);
            } finally {
                latch.countDown();
            }
        });
    }

    latch.await();
    executor.shutdown();

    // 验证最终数据
    CrmDepositPlan plan = getById(planLid);
    // 应该正好增加了 threadCount 次
}
```
