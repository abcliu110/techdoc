# 批量赠券和批量充值前端菜单拆分

## 需求目标

会员营销的卡券营销模块中，原有「批量赠券/批量充值」入口拆分为两个前端菜单：

- 批量赠券
- 批量充值

两个菜单进入后使用同一个页面、同一套列表、同一套后端接口，只根据入口隐藏不同字段。

## 调整边界

本次只涉及 `nms4cloud-biz-ui` 前端界面：

- 不修改 CRM 后端接口。
- 不修改数据库结构。
- 不新增权限点。
- 不改变原有列表查询、详情、保存、删除、手动执行、执行记录逻辑。
- 不迁移历史数据。

## 页面复用方案

继续复用现有页面组件：

```text
src/pages/CrmCardMarketingMrg/components/BatchCoupons
```

新增两个入口路径，组件仍指向同一个页面：

```text
/membermarketing/crmcardmarketingmrg/batchcoupons
/membermarketing/crmcardmarketingmrg/batchrecharge
```

页面根据当前路径判断模式：

```text
batchcoupons  -> 批量赠券
batchrecharge -> 批量充值
```

## 字段显示规则

公共字段两个入口都显示：

- 名称
- 筛选器
- 活动备注
- 执行类型
- 执行周期
- 执行模式
- 执行时间

批量赠券入口显示：

- 优惠券
- 数量
- 领用模式

批量赠券入口隐藏：

- 赠送积分
- 赠送余额 / 赠送金额
- 本金金额

批量充值入口显示：

- 赠送积分
- 赠送余额 / 赠送金额
- 本金金额

批量充值入口隐藏：

- 优惠券
- 数量
- 领用模式

## 兼容说明

原有「批量赠券/批量充值」页面的业务能力不删除，只拆分入口展示。

列表仍使用原接口：

```text
POST /api/scrm/crm_actuator/list
```

新增、修改、详情、删除、手动执行和执行记录仍使用原接口。前端只控制表单字段展示和提交字段，不新增接口字段。

## 验证要点

1. 卡券营销首页显示两个菜单：「批量赠券」「批量充值」。
2. 点击「批量赠券」后进入同一列表页面，新增/编辑/查看弹窗只显示赠券相关字段。
3. 点击「批量充值」后进入同一列表页面，新增/编辑/查看弹窗只显示充值相关字段。
4. 两个入口列表、删除、手动执行、执行记录功能保持一致。
5. 前端类型检查通过。
