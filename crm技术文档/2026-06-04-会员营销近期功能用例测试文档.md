# 会员营销近期功能用例测试文档

编写日期：2026-06-04

适用范围：

- 会员储值活动
- 会员消费积分
- 会员消费赠券
- 会员消费返现
- 小程序核券

本文用于测试人员按业务链路执行用例，不替代接口文档和代码设计文档。执行时优先使用测试商户、测试门店、测试会员、测试券模板，避免污染生产或演示数据。

## 1. 测试目标

1. 验证后台配置可以正确保存、查询、启停、删除或回显。
2. 验证 POS、CRM、小程序、订单、报表之间的主链路结果一致。
3. 验证会员余额、积分、优惠券、返现、核销状态等核心资产不重复、不漏发、不误撤。
4. 验证门店范围、会员类型、会员等级、有效期、金额口径、券模板状态等限制条件生效。
5. 验证反结账、退款、取消核销、重复请求等回滚和幂等场景。

## 2. 测试环境与公共前置条件

### 2.1 账号与环境

| 项目 | 要求 |
|---|---|
| CRM 后台 | 能登录测试商户，具备会员营销、会员卡、优惠券、报表相关权限 |
| POS 测试设备 | 能创建普通消费单、会员卡支付、现金/微信/支付宝支付、反结账、退款 |
| 小程序 | 能登录测试会员，能进入点菜页、购物车页、下单页 |
| 数据库 | 能查询 CRM、订单、BI 报表相关数据 |
| MQ/异步任务 | 消费正常；涉及报表或异步落库时允许轮询等待 |

### 2.2 公共测试数据

| 数据 | 建议 |
|---|---|
| 测试商户 | 使用固定测试 `mid`，不要跨商户复用用例数据 |
| 测试门店 | 至少准备门店 A、门店 B，用于适用门店和不适用门店对比 |
| 测试会员 | 已绑定手机号、openid、会员卡，会员卡有足够余额和初始积分 |
| 会员类型/等级 | 至少准备普通等级、VIP 等级，用于规则匹配和不匹配 |
| 普通券模板 | 已审核、未终止、在领取期内、有库存、无过严限领 |
| 实物/单品券模板 | `SWQ` 或系统实物券，绑定一个可售菜品和单位 |
| 菜品 | 至少准备一个普通菜品和一个实物券绑定菜品 |
| 支付方式 | 会员卡支付、非会员卡支付、混合支付均可用 |

## 4. 会员储值活动用例

### 4.1 覆盖点

- 活动新增、修改、启用、停用、列表和详情回显。
- 固定金额档位、区间金额档位。
- 赠送金额不能大于储值金额。
- 小程序可见活动必须满足门店、品牌、会员类型、会员等级、活动时间、状态等条件。
- 储值成功后本金、赠送金额、赠券、购买限制、充值记录正确。
- 备注为空的会员储值操作可以提交。
- 支付重复回调或重复提交不重复入账。

## 5. 会员消费积分用例

### 5.1 覆盖点

- 积分权益规则保存、修改、查询、POS 同步。
- 普通消费结账后按规则赠送积分。
- 规则关闭、会员类型/等级不匹配、时间不匹配时不赠积分。
- 生日、会员日、可用周期、等级倍率等特殊倍率生效。
- 储值支付、积分抵现等不应计入的金额不参与赠分。
- 反结账、退款、撤销时生成负向积分流水，不误扣其它订单积分。
- 重复请求不重复赠分或重复撤销。

| PTS-02 | POS 同步积分规则 | PTS-01 已保存 | 调用或等待 POS 同步规则 | POS 能拿到当前商户规则 | `/crm_points_rule/listSync` 返回规则 |

## 6. 会员消费赠券用例

### 6.1 覆盖点

- 消费赠券规则新增、修改、查询、删除。
- 适用门店、会员类型、会员等级、有效期、星期和时段限制。
- 金额口径：会员卡支付金额 H、非会员卡支付金额 F、账单整单金额 B。
- 同一金额口径下优先命中满额门槛更高的规则。
- 固定赠券和每满递增赠券。
- 券模板审核、终止、领取时间、库存、限领限制。
- 反结账、整单退款、部分退款时券作废或按剩余金额重发。
- 重复请求不重复发券。

### 6.2 用例矩阵

| ID | 场景 | 前置条件 | 操作步骤 | 预期结果 | 核对点 |
|---|---|---|---|---|---|
| CPN-01 | 新增固定消费赠券规则 | 有可发放普通券模板 | 后台新增规则：满 100 送 1 张，金额口径 B | 保存成功，详情回显一致 | `crm_consumption_coupon_rule` 正确 |
| CPN-02 | 账单金额 B 命中 | CPN-01 启用 | POS 账单 100，会员卡或现金支付均可 | 发 1 张券 | `crm_consume_coupon_op_record` 成功，`crm_coupon_order` 新增 1 |

| CPN-04 | 非会员卡金额 F 命中 | 存在 F 规则 | 现金/微信支付金额满足 F | 命中 F 规则并发券 | request 金额快照中 F 与规则一致 |

| CPN-07 | 多规则优先级 | B 口径有满 50 和满 100 两条规则 | POS 消费 100 | 命中满额门槛更高规则 | `rule_lid` 为满 100 规则 |

| CPN-09 | 券模板库存不足 | 券模板库存小于应发数 | POS 消费命中规则 | 发券失败或按系统错误返回，不生成不完整资产 | `error_msg` 记录原因 |

| CPN-11 | 重复请求幂等 | 同一订单同一 round 重放 | 重复调用消费赠券 | 不重复发券 | 同一幂等键只有一条成功记录 |

## 8. 小程序核券用例

### 8.1 覆盖点

- 小程序可用券列表展示系统实物券/单品券。
- 点菜页选择或扫码后进行预核销。
- 预核销成功后购物车出现对应菜品。
- 移除预核销菜品时自动取消预核销或解锁。
- 点击现在下单时由后端正式核销，前端不单独调正式核销。
- 已过期、已使用、已作废、门店不适用、绑定菜品不可用时不能核销。
- 取消核销后券状态恢复到可用或符合当前业务状态。
- 重复核销、重复取消核销具备幂等保护。

### 8.2 用例矩阵

| ID | 场景 | 前置条件 | 操作步骤 | 预期结果 | 核对点 |
|---|---|---|---|---|---|
| WOF-01 | 查询可用实物券 | 会员已有未核销实物/单品券 | 小程序打开核券入口，查询可用券 | 返回可用券及绑定菜品信息 | `/get_available_product_coupon` 返回券、菜品、可用状态 |

| WOF-04 | 移除预核销菜品 | WOF-02 成功 | 购物车移除该菜品 | 后端自动取消预核销，购物车不再显示该菜品 | 券状态可再次使用或锁定解除 |
| WOF-05 | 下单正式核销 | WOF-02 成功 | 点击现在下单 | 创建正式订单，券正式核销 | `crm_coupon_order` 状态为已核销，订单明细包含菜品 |
| WOF-06 | 免费券金额 | 券配置免费菜品 | 下单 | 菜品按免费规则计价 | 订单金额和券规则一致 |
| WOF-07 | 折扣券金额 | 券配置折扣 | 下单 | 菜品按折扣价计价 | 订单金额和折扣一致 |

| WOF-10 | 过期券不可核销 | 券已过期 | 查询或预核销 | 不可用并返回过期原因 | 状态不被改变 |

## 9. 跨功能联动用例

| ID | 场景 | 操作步骤 | 预期结果 |
|---|---|---|---|
| MIX-01 | 一笔普通消费同时触发积分、赠券、返现 | 配置三类规则均命中，POS 会员结账 | 积分增加、券新增、余额返现均正确，三类记录互不覆盖 |
| MIX-02 | 会员卡支付优先金额口径 | 会员卡支付 + B/F 规则同时存在 | 消费赠券按 H/F/B 优先级命中；返现按整单规则命中；积分按可计积分金额计算 |
| MIX-03 | 门店隔离 | 规则只适用门店 A，在门店 B 消费 | 三类权益均不应误发 |
| MIX-04 | 整单反结账全链路撤销 | 已触发积分、赠券、返现后整单反结账 | 积分扣回、券作废、返现撤销，余额和报表回到合理状态 |
| MIX-05 | 部分退款全链路处理 | 已触发三类权益后部分退款 | 按当前业务策略撤销或重算，不能重复发放，不能误撤其它订单 |
| MIX-06 | 异步延迟观察 | 结账后立即查、30 秒后查、60 秒后查 | 最终一致；中间态不作为失败，超时后记录 MQ/报表延迟 |

## 10. 建议执行顺序

1. 先执行后台配置类用例：`DEP-01~DEP-06`、`PTS-01~PTS-02`、`CPN-01`、`CBK-01`。
2. 再执行单功能正向主链路：`DEP-07`、`PTS-03`、`CPN-02`、`CBK-02`、`WOF-01~WOF-05`。
3. 再执行限制条件：门店、会员等级、有效期、库存、限领、金额上下限。
4. 再执行幂等和回滚：重复请求、反结账、退款、取消核销、支付重复回调。
5. 最后执行跨功能联动：`MIX-01~MIX-06`。

## 11. 每轮测试后必须核对的结果

| 检查项 | 标准 |
|---|---|
| 会员余额 | 本金、赠送金额、消费扣款、返现、撤销返现加减方向正确 |
| 会员积分 | 正向赠分、负向撤销流水成对可追溯 |
| 优惠券 | 发券数量正确，核销/作废/可用状态正确 |
| 返现 | 返现金额、封顶、撤销金额正确 |
| 储值 | 支付成功才入账；失败、关单、重复回调不入账 |
| 小程序核券 | 预核销、购物车、正式核销、取消核销状态一致 |
| 幂等 | 同一订单、同一 round、同一交易号不重复产生资产 |
| 报表 | 报表最终与 CRM 原始表一致；允许短时间异步延迟 |

## 13. 测试通过标准

1. 所有正向主链路用例通过。
2. 所有限制条件不会误发资产。
3. 所有撤销、退款、取消核销场景资产状态可回滚且可追溯。
4. 重复请求不会重复入账、重复赠分、重复发券、重复返现或重复核销。
5. CRM 原始表、会员卡资产、POS/订单状态、BI 报表最终一致。
6. 如果报表存在异步延迟，必须能在原始业务表中确认最终正确状态，并记录延迟时间。

## 14. 已知高风险回归点

| 风险点 | 必测用例 | 说明 |
|---|---|---|
| 消费赠券每满递增少发券 | `CPN-05` | 必须同时核对 `coupon_num` 和实际券实例数 |
| 返现余额字段误判 | `CBK-02`、`CBK-08` | 返现金额体现在会员余额和余额流水中，不应依赖不存在的 `backBalance` 字段 |
| 会员卡支付部分退款 | `CBK-12`、`MIX-05` | 如果当前业务只支持全额退款，部分退款用现金绑定会员场景验证 |
| 小程序预核销和正式核销状态不一致 | `WOF-04`、`WOF-05`、`WOF-14` | 下单失败不能让券永久处于错误状态 |
| 储值支付重复回调 | `DEP-10` | 重复回调不能重复入账、重复赠券 |
| 积分撤销早于赠分 | `PTS-11` | 撤销应可重试，不能导致后续正向赠分无法撤销 |
