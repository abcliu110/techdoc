# CRM储值方案 - 券和券包验证逻辑说明

## 概述
储值方案在新增和更新时，需要验证档位规则中配置的优惠券和券包是否存在且有效。

## 验证时机
在以下操作时触发验证：
1. 新增储值方案 (`CrmDepositPlanServicePlus.add()`)
2. 更新储值方案 (`CrmDepositPlanServicePlus.update()`)

## 验证流程

### 1. 档位规则验证入口
```java
private void validateTierRule(Long mid, Long sid, DepositTierRule tierRule)
```

**验证内容：**
- 档位列表不能为空
- 每个档位的储值金额必须大于0
- 收集所有券ID和券包ID
- 验证券和券包是否存在

### 2. 券验证方法
```java
private void validateCouponsExist(Long mid, Long sid, Set<Long> couponIds)
```

**验证逻辑：**
```sql
SELECT COUNT(*) FROM crm_coupon 
WHERE company_id = ? 
  AND shop_id = ? 
  AND lmnid IN (?)
  AND (is_pkg = false OR is_pkg IS NULL)
  AND status_ = 1
```

**验证规则：**
- 查询条件：商户ID + 门店ID + 券ID列表
- 必须是优惠券（`is_pkg = false` 或 `is_pkg IS NULL`，兼容旧数据）
- 必须是启用状态（`status = 1`）
- 查询结果数量必须等于传入的券ID数量
- 如果数量不匹配，说明有券不存在、不是有效优惠券或已禁用

**错误消息：**
```java
CrmDepositPlanConstant.MSG_COUPON_NOT_EXIST_OR_INVALID
// "优惠券不存在或不是有效优惠券，请检查优惠券ID"
```

### 3. 券包验证方法
```java
private void validateCouponPackagesExist(Long mid, Long sid, Set<Long> packageIds)
```

**验证逻辑：**
```sql
SELECT COUNT(*) FROM crm_coupon 
WHERE company_id = ? 
  AND shop_id = ? 
  AND lmnid IN (?)
  AND is_pkg = true
  AND status_ = 1
```

**验证规则：**
- 查询条件：商户ID + 门店ID + 券包ID列表
- 必须是券包（`is_pkg = true`）
- 必须是启用状态（`status = 1`）
- 查询结果数量必须等于传入的券包ID数量
- 如果数量不匹配，说明有券包不存在、不是有效券包或已禁用

**错误消息：**
```java
CrmDepositPlanConstant.MSG_PACKAGE_NOT_EXIST_OR_INVALID
// "券包不存在或不是有效券包，请检查券包ID"
```

## 数据收集逻辑

### 档位中的券收集
```java
Set<Long> allCouponIds = new HashSet<>();
Set<Long> allPackageIds = new HashSet<>();

for (TierConfig tier : tierRule.getTiers()) {
    // 1. 收集档位直接配置的券
    if (tier.getCoupons() != null) {
        for (CouponItem coupon : tier.getCoupons()) {
            allCouponIds.add(coupon.getCouponLid());
        }
    }
    
    // 2. 收集券包及券包内的券
    if (tier.getCouponPackages() != null) {
        for (CouponPackageItem pkg : tier.getCouponPackages()) {
            allPackageIds.add(pkg.getPackageLid());
            
            // 券包内的券也要验证
            for (CouponItem coupon : pkg.getCoupons()) {
                allCouponIds.add(coupon.getCouponLid());
            }
        }
    }
}
```

## 重复性验证

### 档位内券不能重复
```java
Set<Long> couponIds = new HashSet<>();
for (CouponItem coupon : tier.getCoupons()) {
    assertIsTrue(
        couponIds.add(coupon.getCouponLid()),
        CrmDepositPlanConstant.MSG_DUPLICATE_COUPON_ID_IN_TIER
    );
}
```

**错误消息：** "同一档位内不能配置重复的优惠券"

### 档位内券包不能重复
```java
Set<Long> packageIds = new HashSet<>();
for (CouponPackageItem pkg : tier.getCouponPackages()) {
    assertIsTrue(
        packageIds.add(pkg.getPackageLid()),
        CrmDepositPlanConstant.MSG_DUPLICATE_PACKAGE_ID_IN_TIER
    );
}
```

**错误消息：** "同一档位内不能配置重复的券包"

### 券包内券不能重复
```java
Set<Long> packageCouponIds = new HashSet<>();
for (CouponItem coupon : pkg.getCoupons()) {
    assertIsTrue(
        packageCouponIds.add(coupon.getCouponLid()),
        CrmDepositPlanConstant.MSG_DUPLICATE_COUPON_ID_IN_PACKAGE
    );
}
```

**错误消息：** "同一券包内不能配置重复的优惠券"

## 券的字段验证

### 必填字段
```java
private void validateTierCouponItem(CouponItem coupon) {
    // 1. 每人每次发放张数
    assertNotNull(coupon.getIssueQtyPerTime(), 
        CrmDepositPlanConstant.MSG_COUPON_ISSUE_QTY_REQUIRED);
    assertIsTrue(coupon.getIssueQtyPerTime() > 0, 
        CrmDepositPlanConstant.MSG_COUPON_ISSUE_QTY_MUST_POSITIVE);
    
    // 2. 有效期类型
    assertNotNull(coupon.getTermOfValidityMethod(), 
        CrmDepositPlanConstant.MSG_COUPON_VALIDITY_TYPE_REQUIRED);
    
    // 3. 生效类型
    assertNotNull(coupon.getEffectiveMethod(), 
        CrmDepositPlanConstant.MSG_COUPON_EFFECTIVE_METHOD_REQUIRED);
    
    // 4. 何时生效
    assertNotNull(coupon.getWhenToTakeEffect(), 
        CrmDepositPlanConstant.MSG_COUPON_WHEN_TO_TAKE_EFFECT_REQUIRED);
    
    // 5. 是否计入实收
    assertNotNull(coupon.getIsCountInActualAmount(), 
        CrmDepositPlanConstant.MSG_COUPON_IS_COUNT_IN_ACTUAL_AMOUNT_REQUIRED);
}
```

### 有效期验证
根据`termOfValidityMethod`的不同值，验证不同的字段：

#### 相对有效期（XD）
```java
if (termOfValidityMethod == TermOfValidityMethodEnum.XD) {
    assertNotNull(validDays, MSG_COUPON_VALID_DAYS_REQUIRED);
    assertIsTrue(validDays > 0, MSG_COUPON_VALID_DAYS_MUST_POSITIVE);
}
```

#### 固定有效期（GD）
```java
if (termOfValidityMethod == TermOfValidityMethodEnum.GD) {
    assertNotNull(startEffectiveTime, MSG_COUPON_START_EFFECTIVE_TIME_REQUIRED);
    assertNotNull(endEffectiveTime, MSG_COUPON_END_EFFECTIVE_TIME_REQUIRED);
    assertIsTrue(endEffectiveTime.isAfter(startEffectiveTime), 
        MSG_COUPON_END_EFFECTIVE_TIME_INVALID);
}
```

### 生效时间验证
根据`effectiveMethod`和`whenToTakeEffect`的组合验证：

#### 按小时生效（XS）
```java
if (effectiveMethod == EffectiveMethodEnum.XS) {
    if (whenToTakeEffect == WhenToTakeEffectType.CUSTOM) {
        assertNotNull(effectiveTime, MSG_COUPON_EFFECTIVE_HOURS_REQUIRED);
        assertIsTrue(effectiveTime > 0, MSG_COUPON_EFFECTIVE_HOURS_MUST_POSITIVE);
    }
    // 按小时时，不能有effectiveDays
    assertIsNull(effectiveDays, MSG_COUPON_EFFECTIVE_DAYS_MUST_EMPTY_FOR_HOURLY);
}
```

#### 按天生效（TS/TZD）
```java
if (effectiveMethod == EffectiveMethodEnum.TS || 
    effectiveMethod == EffectiveMethodEnum.TZD) {
    if (whenToTakeEffect == WhenToTakeEffectType.CUSTOM) {
        assertNotNull(effectiveDays, MSG_COUPON_EFFECTIVE_DAYS_REQUIRED);
        assertIsTrue(effectiveDays > 0, MSG_COUPON_EFFECTIVE_DAYS_MUST_POSITIVE);
    }
    // 按天时，不能有effectiveTime
    assertIsNull(effectiveTime, MSG_COUPON_EFFECTIVE_HOURS_MUST_EMPTY_FOR_NON_HOURLY);
}
```

### 实收金额验证
根据`isCountInActualAmount`的值验证：

#### 不计入实收
```java
if (!isCountInActualAmount) {
    assertIsNull(actualAmountType, 
        MSG_COUPON_ACTUAL_AMOUNT_FIELDS_MUST_EMPTY_WHEN_NOT_COUNTED);
    assertIsNull(actualAmountValue, 
        MSG_COUPON_ACTUAL_AMOUNT_FIELDS_MUST_EMPTY_WHEN_NOT_COUNTED);
}
```

#### 计入实收
```java
if (isCountInActualAmount) {
    assertNotNull(actualAmountType, MSG_COUPON_ACTUAL_AMOUNT_TYPE_REQUIRED);
    
    if (actualAmountType == ActualAmountType.CUSTOM_AMOUNT) {
        assertNotNull(actualAmountValue, MSG_COUPON_ACTUAL_AMOUNT_VALUE_REQUIRED);
        assertIsTrue(actualAmountValue > 0, MSG_COUPON_ACTUAL_AMOUNT_VALUE_MUST_POSITIVE);
    } else if (actualAmountType == ActualAmountType.WRITE_OFF_AMOUNT_RATIO) {
        assertNotNull(actualAmountValue, MSG_COUPON_ACTUAL_AMOUNT_VALUE_REQUIRED);
        assertIsTrue(actualAmountValue > 0 && actualAmountValue <= 100, 
            MSG_COUPON_ACTUAL_AMOUNT_RATIO_MUST_IN_RANGE);
    }
}
```

    company_id BIGINT NOT NULL COMMENT '商户ID',

);

## 测试建议

### 正常场景
1. 配置存在的券ID
2. 配置存在的券包ID
3. 券包内配置存在的券ID

### 异常场景
1. 配置不存在的券ID → 应返回错误
2. 配置不存在的券包ID → 应返回错误
3. 配置券ID但实际是券包 → 应返回错误
4. 配置券包ID但实际是券 → 应返回错误
5. 同一档位内配置重复的券 → 应返回错误
6. 同一档位内配置重复的券包 → 应返回错误
7. 券包内配置重复的券 → 应返回错误
8. 券字段验证失败 → 应返回对应错误

### 边界场景
1. 券ID列表为空 → 跳过验证
2. 券包ID列表为空 → 跳过验证
3. 跨商户查询券 → 应查询不到
4. 跨门店查询券 → 应查询不到

## 性能优化

### 批量查询
使用`IN`查询一次性验证所有券和券包，避免N+1问题：
```java
query.in("lmnid", couponIds);  // 一次查询所有券
```

### 去重
使用`Set`收集ID，自动去重：
```java
Set<Long> allCouponIds = new HashSet<>();
Set<Long> allPackageIds = new HashSet<>();
```

## 注意事项

1. **券和券包的区分**
   - 券：`is_pkg = false`
   - 券包：`is_pkg = true`
   - 必须严格区分，不能混用

2. **商户和门店隔离**
   - 验证时必须传入商户ID和门店ID
   - 确保数据隔离性

3. **错误消息统一**
   - 所有错误消息必须使用常量
   - 不允许硬编码字符串

4. **验证顺序**
   - 先验证字段完整性
   - 再验证重复性
   - 最后验证存在性

5. **券包内券的验证**
   - 券包本身要验证存在性
   - 券包内的券也要验证存在性
   - 两者都要验证
