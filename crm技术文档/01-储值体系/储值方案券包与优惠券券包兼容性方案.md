# 储值方案券包与优惠券券包兼容性方案

## 📅 分析日期
2026-03-10

---

## 🎯 问题描述

项目中存在两种"券包"概念：

### 1. 优惠券系统的券包（crm_coupon）
- **表**: `crm_coupon` (is_pkg=true) + `crm_coupon_map`
- **用途**: 用户可以直接领取的券包，领取后获得多张子券
- **特点**: 
  - 券包本身是一个独立的优惠券实体
  - 通过 `is_pkg` 字段标识
  - 通过 `crm_coupon_map` 表关联子券
  - 用户领取券包 = 领取多张子券

### 2. 储值方案的券包（tier_rule.couponPackages）
- **表**: `crm_deposit_plan` (tier_rule JSON字段)
- **用途**: 储值方案档位赠送的券包
- **特点**:
  - 券包是储值档位的赠品之一
  - 存储在 JSON 字段中
  - 只记录券包ID和数量
  - 用户充值后获得券包

**核心问题**: 储值方案中的 `packageLid` 应该引用哪个券包？如何保证数据一致性？

## ✅ 兼容性方案

### 方案一：直接引用优惠券系统的券包（推荐）

**设计思路**：
- 储值方案的 `packageLid` 直接引用 `crm_coupon` 表中 `is_pkg=true` 的券包
- 用户充值后，系统根据 `packageLid` 查询券包，然后发放券包内的所有子券

**优点**：
- ✅ 复用现有的券包系统
- ✅ 数据一致性好
- ✅ 券包管理统一
- ✅ 券包内容变更自动生效

**缺点**：
- ❌ 储值方案依赖优惠券系统
- ❌ 删除券包需要检查储值方案引用

**实现步骤**：

#### 1. 验证券包存在性

在储值方案保存时，验证 `packageLid` 引用的券包是否存在：

```java
/**
 * 验证储值档位规则
 */
private void validateTierRule(DepositTierRule tierRule) {
    if (tierRule == null || CollUtil.isEmpty(tierRule.getTiers())) {
        throw new BizException(CrmDepositPlanConstant.MSG_TIER_LIST_REQUIRED);
    }
    
    for (DepositTierRule.TierConfig tier : tierRule.getTiers()) {
        // 验证优惠券
        if (CollUtil.isNotEmpty(tier.getCoupons())) {
            validateCoupons(tier.getCoupons());
        }
        
        // 验证券包
        if (CollUtil.isNotEmpty(tier.getCouponPackages())) {
            validateCouponPackages(tier.getCouponPackages());
        }
    }
}

/**
 * 验证券包列表
 */
private void validateCouponPackages(List<DepositTierRule.CouponPackageItem> packages) {
    if (CollUtil.isEmpty(packages)) {
        return;
    }
    
    // 提取所有券包ID
    List<Long> packageLids = packages.stream()
        .map(DepositTierRule.CouponPackageItem::getPackageLid)
        .filter(Objects::nonNull)
        .collect(Collectors.toList());
    
    if (CollUtil.isEmpty(packageLids)) {
        throw new BizException("券包ID不能为空");
    }
    
    // 批量查询券包（避免N+1问题）
    LambdaQueryWrapper<CrmCoupon> query = new LambdaQueryWrapper<>();
    query.select(CrmCoupon::getLid, CrmCoupon::getName, CrmCoupon::getIsPkg);
    query.eq(CrmCoupon::getMid, getCurrentMid());
    query.in(CrmCoupon::getLid, packageLids);
    query.eq(CrmCoupon::getIsPkg, true);  // 必须是券包
    query.eq(CrmCoupon::getStatus, 1);    // 必须有效
    
    List<CrmCoupon> couponPackages = iCrmCouponService.list(query);
    
    // 验证所有券包都存在
    if (couponPackages.size() != packageLids.size()) {
        Set<Long> foundIds = couponPackages.stream()
            .map(CrmCoupon::getLid)
            .collect(Collectors.toSet());
        
        List<Long> notFoundIds = packageLids.stream()
            .filter(id -> !foundIds.contains(id))
            .collect(Collectors.toList());
        
        throw new BizException(
            String.format("以下券包不存在或不是有效的券包：%s", notFoundIds)
        );
    }
    
    // 验证券包数量
    for (DepositTierRule.CouponPackageItem pkg : packages) {
        if (pkg.getPackageQty() == null || pkg.getPackageQty() <= 0) {
            throw new BizException(
                String.format("券包【%s】的数量必须大于0", pkg.getPackageName())
            );
        }
    }
}
```

#### 2. 发放券包

用户充值后，根据档位配置发放券包：

```java
/**
 * 用户充值后发放赠品
 */
public void grantDepositRewards(Long userId, Long depositPlanId, Long tierId) {
    // 1. 查询储值方案
    CrmDepositPlan plan = getDepositPlan(depositPlanId);
    DepositTierRule tierRule = plan.getTierRule();
    
    // 2. 找到对应的档位
    DepositTierRule.TierConfig tier = tierRule.getTiers().stream()
        .filter(t -> t.getSortNo().equals(tierId))
        .findFirst()
        .orElseThrow(() -> new BizException("档位不存在"));
    
    // 3. 发放优惠券
    if (CollUtil.isNotEmpty(tier.getCoupons())) {
        grantCoupons(userId, tier.getCoupons());
    }
    
    // 4. 发放券包
    if (CollUtil.isNotEmpty(tier.getCouponPackages())) {
        grantCouponPackages(userId, tier.getCouponPackages());
    }
}

/**
 * 发放券包
 */
private void grantCouponPackages(
    Long userId, 
    List<DepositTierRule.CouponPackageItem> packages
) {
    for (DepositTierRule.CouponPackageItem pkg : packages) {
        // 查询券包信息
        CrmCoupon couponPackage = getCouponPackage(pkg.getPackageLid());
        
        // 查询券包包含的子券
        List<CrmCouponMap> subCoupons = getSubCoupons(pkg.getPackageLid());
        
        // 根据券包数量发放
        for (int i = 0; i < pkg.getPackageQty(); i++) {
            // 发放券包内的所有子券
            for (CrmCouponMap subCoupon : subCoupons) {
                Long subCouponLid = Long.valueOf(subCoupon.getDeputyCode());
                Integer quantity = subCoupon.getNumber();
                
                // 发放子券
                issueCoupon(userId, subCouponLid, quantity);
            }
        }
        
        // 记录券包发放日志
        logPackageGrant(userId, pkg.getPackageLid(), pkg.getPackageQty());
    }
}

/**
 * 查询券包包含的子券
 */
private List<CrmCouponMap> getSubCoupons(Long packageLid) {
    LambdaQueryWrapper<CrmCouponMap> query = new LambdaQueryWrapper<>();
    query.eq(CrmCouponMap::getMid, getCurrentMid());
    query.eq(CrmCouponMap::getMainCode, packageLid.toString());
    query.eq(CrmCouponMap::getStatus, 1);
    
    return iCrmCouponMapService.list(query);
}
```

#### 3. 删除券包前检查引用

删除券包时，需要检查是否被储值方案引用：

```java
/**
 * 删除券包前检查
 */
public NmsResult<String> deleteCouponPackage(Long packageLid) {
    // 1. 检查是否被储值方案引用
    List<CrmDepositPlan> referencedPlans = findPlansReferencingPackage(packageLid);
    
    if (CollUtil.isNotEmpty(referencedPlans)) {
        List<String> planNames = referencedPlans.stream()
            .map(CrmDepositPlan::getName)
            .collect(Collectors.toList());
        
        return NmsResult.error(
            String.format("该券包被以下储值方案引用，无法删除：【%s】", 
                String.join("】、【", planNames))
        );
    }
    
    // 2. 删除券包
    // ... 删除逻辑
    
    return NmsResult.ok();
}

/**
 * 查找引用了指定券包的储值方案
 */
private List<CrmDepositPlan> findPlansReferencingPackage(Long packageLid) {
    // 查询所有储值方案
    LambdaQueryWrapper<CrmDepositPlan> query = new LambdaQueryWrapper<>();
    query.eq(CrmDepositPlan::getMid, getCurrentMid());
    query.isNotNull(CrmDepositPlan::getTierRule);
    
    List<CrmDepositPlan> allPlans = iCrmDepositPlanService.list(query);
    
    // 过滤出引用了该券包的方案
    return allPlans.stream()
        .filter(plan -> {
            DepositTierRule tierRule = plan.getTierRule();
            if (tierRule == null || CollUtil.isEmpty(tierRule.getTiers())) {
                return false;
            }
            
            // 检查每个档位的券包列表
            return tierRule.getTiers().stream()
                .anyMatch(tier -> {
                    if (CollUtil.isEmpty(tier.getCouponPackages())) {
                        return false;
                    }
                    return tier.getCouponPackages().stream()
                        .anyMatch(pkg -> packageLid.equals(pkg.getPackageLid()));
                });
        })
        .collect(Collectors.toList());
}
```

### 方案二：券包快照（备选方案）

**设计思路**：
- 储值方案保存时，将券包的完整信息（包括子券列表）快照到 JSON 中
- 用户充值时，直接使用快照数据发放券包

**优点**：
- ✅ 储值方案独立，不依赖优惠券系统
- ✅ 券包内容固定，不受后续修改影响
- ✅ 删除券包不影响已有储值方案

**缺点**：
- ❌ 数据冗余
- ❌ 券包内容变更不会自动同步
- ❌ 维护成本高

**数据结构**：

## ⚠️ 注意事项

### 1. 数据类型一致性

**问题**：
- `crm_coupon.lmnid` 是 BIGINT
- `crm_coupon_map.main_code` 是 VARCHAR

**解决**：
- 在查询时进行类型转换
- 或者统一修改为 BIGINT 类型

### 2. 券包内容变更

**问题**：
- 券包内容变更后，已有储值方案的赠品会自动变更

**解决**：
- 方案一：接受这个行为（推荐）
- 方案二：使用快照方案

### 3. 性能优化

**问题**：
- 批量发放券包时可能有性能问题

**解决**：
- 使用批量查询避免 N+1 问题
- 使用异步任务发放券包
- 添加缓存

### 4. 事务处理

**问题**：
- 发放券包失败时如何回滚

**解决**：
- 使用事务保证原子性
- 记录发放日志便于追溯
- 提供补偿机制
