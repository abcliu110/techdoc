# 券和券包验证逻辑改进说明

## 改进日期
2026-03-12

## 改进概述
优化了CRM储值方案中券和券包的验证逻辑，提高了代码质量和验证准确性。

## 改进内容

### 1. 修复硬编码字符串问题

**问题：**
```java
// 原代码
assertIsTrue(count == couponIds.size(), "优惠券不存在或不是有效优惠券，请检查优惠券ID");
```

**改进：**
```java
// 新代码
assertIsTrue(count == couponIds.size(), CrmDepositPlanConstant.MSG_COUPON_NOT_EXIST_OR_INVALID);
```

**新增常量：**
```java
// CrmDepositPlanConstant.java
public static final String MSG_COUPON_NOT_EXIST_OR_INVALID = "优惠券不存在或不是有效优惠券，请检查优惠券ID";
```

**改进理由：**
- 遵循代码规范，所有错误消息必须使用常量
- 便于统一管理和维护错误消息
- 便于国际化支持

### 2. 优化查询条件

**问题：**
```java
// 原代码 - 使用IFNULL函数
query.eq("IFNULL(is_pkg,0)", false);
```

**改进：**
```java
// 新代码 - 使用MyBatis-Plus的条件构造器
query.and(wrapper -> wrapper.eq("is_pkg", false).or().isNull("is_pkg"));
```

**改进理由：**
- `IFNULL`是SQL函数，在MyBatis-Plus的QueryWrapper中使用不规范
- 使用条件构造器更符合MyBatis-Plus的使用方式
- 代码更清晰，更易维护
- 兼容旧数据（is_pkg为NULL的情况）

### 3. 添加状态验证

**问题：**
原代码只验证券/券包是否存在，没有验证状态

**改进：**
```java
// 券验证
query.eq("status_", 1);  // 必须是启用状态

// 券包验证
query.eq("status_", 1);  // 必须是启用状态
```

**改进理由：**
- 禁用的券/券包不应该被使用
- 防止配置已禁用的券/券包
- 提高数据准确性

### 4. 完善注释文档

**改进：**
- 添加了详细的方法注释
- 说明了业务逻辑
- 列出了验证规则
- 添加了参数说明

**示例：**
```java
/**
 * 验证优惠券是否存在且为有效优惠券
 * 
 * 业务说明：
 * 1. 查询条件：商户ID + 门店ID + 优惠券ID列表
 * 2. 必须是优惠券（is_pkg = false 或 is_pkg IS NULL）
 * 3. 必须是启用状态（status = 1）
 * 4. 查询结果数量必须等于传入的优惠券ID数量
 * 
 * @param mid 商户ID
 * @param sid 门店ID
 * @param couponIds 优惠券ID集合
 */
private void validateCouponsExist(Long mid, Long sid, Set<Long> couponIds)
```

## 验证逻辑对比

### 券验证

#### 改进前
```sql
SELECT COUNT(*) FROM crm_coupon 
WHERE company_id = ? 
  AND shop_id = ? 
  AND lmnid IN (?)
  AND IFNULL(is_pkg, 0) = false
```

#### 改进后
```sql
SELECT COUNT(*) FROM crm_coupon 
WHERE company_id = ? 
  AND shop_id = ? 
  AND lmnid IN (?)
  AND (is_pkg = false OR is_pkg IS NULL)
  AND status_ = 1
```

**改进点：**
1. 去除IFNULL函数，使用标准SQL
2. 添加status状态验证
3. 明确处理NULL值情况

### 券包验证

#### 改进前
```sql
SELECT COUNT(*) FROM crm_coupon 
WHERE company_id = ? 
  AND shop_id = ? 
  AND lmnid IN (?)
  AND IFNULL(is_pkg, 0) = true
```

#### 改进后
```sql
SELECT COUNT(*) FROM crm_coupon 
WHERE company_id = ? 
  AND shop_id = ? 
  AND lmnid IN (?)
  AND is_pkg = true
  AND status_ = 1
```

**改进点：**
1. 去除IFNULL函数
2. 添加status状态验证
3. 券包必须明确标记为is_pkg=true

## 测试建议

### 新增测试场景

1. **禁用券测试**
   - 配置已禁用的券（status=0）
   - 预期：验证失败，返回错误消息

2. **禁用券包测试**
   - 配置已禁用的券包（status=0）
   - 预期：验证失败，返回错误消息

3. **NULL值兼容性测试**
   - 配置is_pkg为NULL的券
   - 预期：作为普通券处理，验证通过

4. **混合状态测试**
   - 同时配置启用和禁用的券
   - 预期：只有启用的券通过验证

### 回归测试

确保以下场景仍然正常工作：
1. 配置存在且启用的券 → 成功
2. 配置存在且启用的券包 → 成功
3. 配置不存在的券 → 失败
4. 配置不存在的券包 → 失败
5. 配置券ID但实际是券包 → 失败
6. 配置券包ID但实际是券 → 失败

- 添加status条件后，可以利用索引
- 建议添加复合索引：`(company_id, shop_id, lmnid, is_pkg, status_)`

### 建议索引
```sql
-- 券查询索引
CREATE INDEX idx_coupon_validation 
ON crm_coupon(company_id, shop_id, lmnid, is_pkg, status_);

-- 或分别创建
CREATE INDEX idx_company_shop_lmnid ON crm_coupon(company_id, shop_id, lmnid);
CREATE INDEX idx_is_pkg_status ON crm_coupon(is_pkg, status_);
```
