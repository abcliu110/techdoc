# 优惠券与券包系统设计文档

**日期**: 2026-05-13
**范围**: nms4cloud CRM / POS / Order 优惠券全链路

---

## 一、系统概述

优惠券系统是 nms4cloud CRM 的核心营销能力，覆盖了券的**定义 → 发放 → 使用（核销）→ 反核销 → 消费赠券**的完整生命周期。

### 1.1 核心概念

| 概念 | 表 | 说明 |
|------|------|------|
| **券模板** | `crm_coupon` | 商家创建的券规则（面值、门槛、有效期规则等），可发放多次 |
| **券实例** | `crm_coupon_order` | 实际发放给会员的一张券，有唯一 `lid`，绑定手机号，有状态 |
| **券包** | `crm_coupon` + `crm_coupon_map` | 一种特殊的券模板，`is_pkg=true`，包含多个子券 |
| **消费权益任务** | `crm_consume_benefit_task` | 消费赠券和反结账回收的幂等任务台账 |

### 1.2 券类型（`CouponTypeEnum`）

| 类型 | code | 说明 |
|------|------|------|
| XJQ | 现金券 | 抵扣现金金额 |
| WP | 物品券 | 兑换指定菜品 |
| SWQ | 实物券 | 兑换实物商品 |
| ZJ | 智简券 | 智简优惠券，有专用处理器 |

### 1.3 券状态（`CouponStatusEnum`）

| 状态 | 含义 | 可核销 |
|------|------|--------|
| WHX | 未核销 | ✅ |
| YHX | 已核销 | ❌ |
| YGQ | 已过期 | ❌ |
| YZF | 已作废 | ❌ |
| KGQ | 快过期 | — |
| WSX | 未生效 | ❌ |
| YSX | 已失效 | ❌ |

## 三、券发放流程

### 3.1 发放入口（`CrmCouponOpServicePlus`）

券发放通过 `couponOpAdd()` 方法实现，由多个渠道共同调用：

```
消费赠券    → OrderConsumeBenefitOutboxService    → CRM_CONSUME_BENEFIT_EVENT_QUEUE
储值赠券    → CrmCardOpServicePlus.cardCharge()   → couponOpAdd()
活动赠券    → CrmCardOpServicePlus.grantCoupon()  → couponOpAdd()
会员自领    → CardOpForCustomerController.takeCoupon() → couponOpAdd()
手动发放    → CardOpForCustomerController.issuanceCoupon() → couponOpAdd()
```

### 3.2 发放核心逻辑（`CrmCouponOpServicePlus.couponOpAdd()`）

```java
@Transactional
public CouponOpVO couponOpAdd(CouponOpAddDTO request) {
    // 1. 参数校验
    // 2. 库存扣减（received_number - couponNum）
    // 3. 计算有效期
    if (isPkg) {
        // 券包发放：查询 crm_coupon_map，展开子券
        issuancePkg(coupon, card, couponOpAddDTO, couponNum);
    } else {
        // 普通券发放
        issuanceNormal(coupon, card, couponOpAddDTO, couponNum);
    }
    // 4. 批量写入 crm_coupon_order
    // 5. 发送微信模板消息通知
}
```

### 3.3 券包发放特殊逻辑

领取券包时（`is_pkg=true`）：
1. 查询 `crm_coupon_map` 获取所有子券和数量
2. 对每个子券按 `number` 展开生成多张券实例
3. 每张券的 `pkgCoupon` / `pkgCouponCode` 字段记录所属券包信息
4. 所有子券使用相同的有效期计算规则

### 4.2 加载券（`loadCoupons`）

**场景**：收银员输入券码，POS 从云端加载券信息到本地。

```
前端 → DwdBillOpsServiceImpl.loadCoupons()
     → CouponHandler.load()
        → Nms4CloudCrmService.getCrmCouponOrder()   [HTTP POST /crm_coupon_order/getSign]
           参数：mid + lid（券实例的雪花ID）
           返回：CrmCouponOrderVO（券实例 + 券模板合并信息）
        → DwdCouponMapper.insertBatch()   写入本地 dwd_coupon 表
```

**加载时填充的字段**（`NmsCouponHandler.load()`）：

| POS 本地字段 | 来源 | 说明 |
|-------------|------|------|
| `faceAmount` | `CrmCouponOrder.faceValue` | 券面值 |
| `thresholdAmount` | `CrmCouponOrder.fullMoney` | 满减门槛 |
| `couponType` | `CrmCouponOrder.type` | WP=物品券 / WV=现金券 / ZJ=智简券 |
| `productNo` | `CrmCouponVO.dishCode` | 指定菜品ID（物品券才有） |
| `beginAt` / `endAt` | 有效期 | 券有效期 |
| `phone` | `CrmCouponOrder.unionid` | 会员手机号 |
| `memberName` | `CrmCouponOrder.member` | 会员名称 |

### 4.3 核销券（`addCoupons`）

**场景**：结账确认时，POS 将券从"未核销"变为"已核销"。

```
结账确认 → DwdBillOpsServiceImpl.addCoupons()
        → 预校验（券是否已被其他账单使用，防重复）
        → 按券类型分组（WP/WV/ZJ）
        → 分布式锁 "userLid:devId:source_coupon"
        → CouponHandler.writeOff()
           → Nms4CloudCrmService.writeOffSign()   [HTTP POST /crm_coupon_order/writeOffSign]
              参数：couponNo, orderId, amount, tblTypeLid, operator
           → CRM: CrmCouponOrderServicePlus.writeOffOne()
              UPDATE crm_coupon_order
                SET coupon_status = 'YHX',
                    write_off_time = NOW(),
                    write_off_staff = '收银员',
                    used_shop_code = sid,
                    order_bill_id = 订单号
                WHERE lid = couponNo AND coupon_status = 'WHX'
           → CRM: 发送微信模板消息通知会员
        → 写 dwd_coupon 表（writeOff=true, writeOffAt, writeOffBy）
        → 物品券/套餐券：自动加菜到账单（dwd_food 表）
        → 重新计算账单金额（couponAmount 累加）
```

**物品券特殊处理**：核销物品券时，自动将指定菜品作为赠菜加到账单，无需会员手动选择。

### 4.4 反核销（`rmvCoupons`）

**场景**：结账前撤销已加载的券。

```
前端 → DwdBillOpsServiceImpl.rmvCoupons()
     → CouponHandler.cancelOff()
        → Nms4CloudCrmService.cancelWriteOffBatchCoupon()  [HTTP POST /crm_card_op/customer/cancelWriteOffBatchCouponSign]
           → CRM: CrmCouponOrderServicePlus.cancelWriteOffBatchCoupon()
              UPDATE crm_coupon_order
                SET coupon_status = 'WHX',
                    write_off_time = NULL,
                    write_off_staff = NULL,
                    used_time = NULL,
                    used_shop_code = NULL
                WHERE lid IN (...) AND coupon_status = 'YHX'
        → 清空 dwd_coupon 表的核销标记
        → 清空 dwd_food 表关联的券信息
```

### 4.5 折扣券特殊处理

结账方法中有一支特殊逻辑（`DwdBillOpsServiceImpl` 第 464–492 行）：折扣券（`request.getCouponLid()` 非空）**直接核销但不抵扣账单金额**：

```java
// 使用了折扣券，要先核销
CrmCouponWriteOffOneDTO writeOffOneDTO = new CrmCouponWriteOffOneDTO();
writeOffOneDTO.setAmount(BigDecimal.ZERO);  // amount=0，不抵扣账单金额
nms4CloudCrmService.writeOffSign(writeOffOneDTO);
```

### 4.6 退款反核销（`refundBill`）

反结账/退款时，`PayWayEnum.FJ` 分支处理优惠券撤销（`DwdBillOpsServiceImpl` 第 3597–3640 行）：
1. 查询已核销的券
2. 调用 `cancelOff()` 取消核销
3. 将 `dwd_coupon` 表的 `writeOff` 标记回 `false`

## 五、线上订单用券链路

> 小程序/H5 自助结账的用券流程。

### 5.1 核销流程（`PayOrderServiceImpl.pay()`）

```
结账确认 → checkCouponPay()      校验券可用性（调用 CRM）
        → cardOpForCustomerFeign.writeOffCouponInner()   批量核销
        → CrmCouponOrderServicePlus.writeOffInner()
           UPDATE crm_coupon_order SET coupon_status = 'YHX' ...
           → 发送微信模板消息
           → 创建延时队列（5分钟）自动检查
        → 失败 → unlockCouponInner()  自动解锁券
```

### 5.2 智简优惠券特殊处理

线上结账时判断商户是否为智简会员（`zjCrmCardOpService.isValid(mid)`）：

```java
if (zjCrmCardOpService.isValid(mid)) {
    // 走智简优惠券接口
    zjCrmCardOpService.writeOffCoupon(...);
} else {
    // 走标准券核销接口
    crmCouponOrderServicePlus.writeOffInner(...);
}
```

### 6.1 整体流程

```
┌─────────────────────────────────────────────────────────────┐

┌─────────────────────────────────────────────────────────────┐
│                    CRM 侧（nms4cloud-crm）                  │
│                                                             │
│  CrmConsumeBenefitEventConsumer.process()                   │
│     ↓                                                       │
│  [幂等检查] 按 businessKey 查询 crm_consume_benefit_task    │
│     ↓                                                       │
│  [发券任务] 创建 PENDING 任务 → PROCESSING                  │
│     ↓                                                       │
│  CrmConsumerCouponQueueConsumer.consume()                    │
│     ↓                                                       │
│  [规则匹配] ConsumeCouponRuleMatcher                        │
│     ↓                                                       │
│  [执行发券] couponOpAdd()                                    │
│     ↓                                                       │
│  [记录明细] crm_consume_coupon_grant_detail（若有）          │

└─────────────────────────────────────────────────────────────┘
```

### 6.2 幂等保证

消费赠券通过 `crm_consume_benefit_task` 表保证幂等：

```java
// 业务唯一键（不含主键，覆盖所有影响幂等的维度）
businessKey = mid | sid | ORDER | lifecycleId | GRANT | orderId | cardLid

// MQ 重投 → 同一业务键 → 已存在任务 → 直接返回
// 唯一约束：uk(mid, origin_source, origin_lifecycle_id, event_type, card_lid, out_order_id)
```

### 6.3 配置开关

消费赠券通过 `g_enableConsumerCoupon` 配置控制：

| 配置值 | 含义 |
|--------|------|
| N | 不启用 |
| F | 不启用 |
| ALL | 线上、线下都启用 |
| ONLINE_ONLY | 仅线上订单启用 |
| OFFLINE_ONLY | 仅线下/POS 清台启用 |

## 七、反结账回收券

### 7.1 整体流程

```
┌─────────────────────────────────────────────────────────────┐
│                  订单侧（nms4cloud-order）                  │
│                                                             │
│  反结账 → OrderConsumeBenefitOutboxService.enqueueRevoke() │
│        → 发送 MQ: CRM_CONSUME_BENEFIT_EVENT_QUEUE (REVOKE)  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    CRM 侧（nms4cloud-crm）                  │
│                                                             │
│  CrmConsumeBenefitEventConsumer.processRevoke()            │
│     ↓                                                       │
│  [查原任务] 按 businessKey(GANT) 找到发券任务               │
│     ↓                                                       │
│  [创建回收任务] REVOKING 状态                               │
│     ↓                                                       │
│  [执行回收] couponOpRevoke()                                │
│     ↓                                                       │
│  CrmCouponOpServicePlus.couponOpRevoke()                   │
│     UPDATE crm_coupon_order                                 │
│       SET coupon_status = 'YHX',                           │
│           abandon_staff = operator                          │
│       WHERE mid = ?                                        │
│         AND order_bill_id = 订单号                         │
│         AND coupon_status IN ('WHX', 'YGQ')                │
│     ↓                                                       │
│  任务状态：REVOKING → REVOKED                              │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 回收策略

| 券状态 | 处理方式 |
|--------|----------|
| WHX（未核销） | 直接回收，`coupon_status = WHX → YHX` |
| YGQ（已过期但未核销） | 仍应回收或标记无效 |
| YHX（已核销） | 不能简单删除，需走企业级异常策略 |
