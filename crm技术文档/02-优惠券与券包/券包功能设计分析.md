# 券包（Coupon Package）功能设计分析

## 📅 分析日期
2026-03-10

---

## 📦 一、功能概述

券包是一种特殊的优惠券类型，允许将多个普通优惠券打包成一个券包，用户领取券包后可以获得包内的所有子券。

**典型场景**：
- 新人大礼包：包含多张不同面额的代金券
- 节日礼包：包含多种类型的优惠券
- 会员升级礼包：包含专属优惠券组合

## 🗄️ 二、数据库设计

### 2.1 主表：crm_coupon（优惠券表）

```sql
CREATE TABLE crm_coupon (
    pid BIGINT AUTO_INCREMENT PRIMARY KEY,
    lmnid BIGINT NOT NULL COMMENT '券ID',
    company_id BIGINT NOT NULL COMMENT '商户ID',
    shop_id BIGINT COMMENT '门店ID',
    name VARCHAR(255) COMMENT '券名称',
    is_pkg TINYINT(1) COMMENT '是否为券包：1=券包，0=普通券',
    coupon_type ENUM(...) COMMENT '券类型',
    status_ INT COMMENT '状态',
    -- ... 其他券的属性字段
    INDEX idx_mid_ispkg (company_id, is_pkg),
    INDEX idx_lid (lmnid)
) COMMENT='优惠券表（包含普通券和券包）';
```

**关键字段**：
- `is_pkg`: 区分券包和普通券的标识字段
  - `true/1`: 这是一个券包
  - `false/0/null`: 这是一个普通券

### 2.2 关联表：crm_coupon_map（券包映射表）

```sql
CREATE TABLE crm_coupon_map (
    pid BIGINT AUTO_INCREMENT PRIMARY KEY,
    lmnid BIGINT NOT NULL COMMENT '映射记录ID',
    company_id BIGINT NOT NULL COMMENT '商户ID',
    shop_id BIGINT COMMENT '门店ID',
    name VARCHAR(255) COMMENT '券包设置名称',
    main_code VARCHAR(50) COMMENT '主券ID（券包的lid）',
    deputy_code VARCHAR(50) COMMENT '副券ID（子券的lid）',
    main VARCHAR(255) COMMENT '主券名称（券包名称）',
    deputy VARCHAR(255) COMMENT '副券名称（子券名称）',
    number INT COMMENT '数量（该子券在券包中的数量）',
    status_ INT COMMENT '状态',
    INDEX idx_main_code (main_code),
    INDEX idx_deputy_code (deputy_code)
) COMMENT='券包映射表（券包与子券的关联关系）';
```

**关键字段**：
- `main_code`: 券包ID（字符串类型）
- `deputy_code`: 子券ID（字符串类型）
- `number`: 该子券在券包中的数量

### 4.1 查询券包（GET /crm_coupon/get）

**Service方法**: `CrmCouponServicePlus.get()`

```java
public NmsResult<CrmCouponVO> get(CrmCouponGetDTO request) {
    // 1. 查询券基本信息
    CrmCoupon entity = iCrmCouponService.getOne(query);
    CrmCouponVO couponVO = BeanUtilsPlus.mapBean(entity, CrmCouponVO.class);
    
    // 2. 如果是券包，查询子券列表
    if (NullSafeUtils.nullSafe(entity.getIsPkg())) {
        QueryWrapper<CrmCouponMap> couponMapQuery = 
            new QueryWrapper<>().eq("main_code", entity.getLid());
        List<CrmCouponMap> subCoupons = iCrmCouponMapService.list(couponMapQuery);
        couponVO.setSubCouponList(
            BeanUtilsPlus.mapList(subCoupons, CrmCouponMapVO.class)
        );
    } else {
        couponVO.setSubCouponList(Collections.emptyList());
    }
    
    return NmsResult.data(couponVO);
}
```

### 4.3 保存券包（POST /crm_coupon/add 或 /update）

**Service方法**: `CrmCouponServicePlus.saveCouponMap()`

```java
private void saveCouponMap(
    List<CrmCouponMapAddDTO> subCouponList, 
    CrmCoupon crmCoupon
) {
    // 1. 删除旧的关联关系
    QueryWrapper<CrmCouponMap> query = query(crmCoupon, "main_code");
    iCrmCouponMapService.remove(query);
    
    // 2. 验证是否为券包
    if (!NullSafeUtils.nullSafe(crmCoupon.getIsPkg())) {
        return; // 不是券包，不处理
    }
    
    // 3. 验证子券列表不能为空
    if (CollUtil.isEmpty(subCouponList)) {
        throw new BizException("请设置券包子券");
    }
    
    // 4. 提取子券ID列表
    List<Long> subIds = subCouponList.stream()
        .map(CrmCouponMapAddDTO::getDeputyCode)
        .filter(StrUtil::isNotBlank)
        .map(Long::valueOf)
        .collect(Collectors.toList());
    
    // 5. 验证子券必须是普通券（is_pkg=false）
    QueryWrapper<CrmCoupon> subCouponQuery = new QueryWrapper<>()
        .eq("IFNULL(is_pkg,0)", false)  // 只能选择普通券
        .lambda()
        .select(CrmCoupon::getLid, CrmCoupon::getName)
        .eq(CrmCoupon::getMid, crmCoupon.getMid())
        .eq(CrmCoupon::getSid, crmCoupon.getSid())
        .eq(CrmCoupon::getStatus, 1)
        .in(CrmCoupon::getLid, subIds);
    
    Map<Long, CrmCoupon> subCouponMap = 
        iCrmCouponService.list(subCouponQuery).stream()
        .collect(Collectors.toMap(CrmCoupon::getLid, Function.identity()));
    
    // 6. 创建券包映射记录
    List<CrmCouponMap> couponMaps = new ArrayList<>();
    for (CrmCouponMapAddDTO mapAddDTO : subCouponList) {
        // 验证数量
        Assert.isTrue(
            NullSafeUtils.nullSafe(mapAddDTO.getNumber()) > 0, 
            "券包里的单品券数量不能为空或者小于1"
        );
        
        // 验证子券存在
        Long deputyCode = Long.valueOf(mapAddDTO.getDeputyCode());
        CrmCoupon subCoupon = subCouponMap.get(deputyCode);
        Assert.isTrue(
            subCoupon != null,
            String.format("子券【%s:%s】不存在！", 
                mapAddDTO.getDeputy(), mapAddDTO.getDeputyCode())
        );
        
        // 创建映射记录
        CrmCouponMap couponMap = new CrmCouponMap();
        couponMap.setMid(crmCoupon.getMid());
        couponMap.setSid(crmCoupon.getSid());
        couponMap.setLid(IdWorkerPlus.getId());
        couponMap.setStatus(1);
        couponMap.setMainCode(crmCoupon.getLid().toString());
        couponMap.setDeputyCode(deputyCode.toString());
        couponMap.setMain(crmCoupon.getName());
        couponMap.setDeputy(subCoupon.getName());
        couponMap.setNumber(mapAddDTO.getNumber());
        
        couponMaps.add(couponMap);
    }
    
    // 7. 批量保存
    iCrmCouponMapService.saveBatch(couponMaps);
}
```

## ✅ 五、业务规则

### 5.1 券包创建规则

1. **券包只能包含普通券**
   - 券包（is_pkg=true）不能包含另一个券包
   - 子券必须是 is_pkg=false 的普通券
   - 防止券包嵌套，避免复杂度

2. **数量限制**
   - 每个子券在券包中的数量必须 > 0
   - 通过 `number` 字段控制
   - 验证：`Assert.isTrue(number > 0, "券包里的单品券数量不能为空或者小于1")`

3. **状态验证**
   - 子券必须是有效状态（status=1）
   - 子券必须存在且未被删除
   - 子券必须属于同一商户和门店

4. **子券列表不能为空**
   - 券包必须至少包含1个子券
   - 验证：`if (CollUtil.isEmpty(subCouponList)) throw new BizException("请设置券包子券")`

### 5.2 券包更新规则

1. **先删除后创建**
   - 更新券包时，先删除所有旧的关联关系
   - 然后重新创建新的关联关系
   - 保证数据一致性

2. **子券可重复使用**
   - 同一个子券可以被多个券包引用
   - 删除券包不影响子券本身

3. **券包名称同步**
   - 更新券包名称时，自动更新 crm_coupon_map 中的 main 字段
   - 保持数据一致性

### 5.3 券包删除规则

1. **级联删除**
   - 删除券包时，需要同时删除 crm_coupon_map 中的关联记录
   - 但不删除子券本身

2. **引用检查**
   - 删除子券前，需要检查是否被券包引用
   - 如果被引用，应该提示用户或禁止删除

## 🔄 六、数据流程

### 6.1 创建券包流程

```
1. 用户创建券包
   ↓
2. 设置券包基本信息
   - name: "新人大礼包"
   - is_pkg: true
   ↓
3. 保存券包记录到 crm_coupon
   - INSERT INTO crm_coupon (lid, name, is_pkg, ...) VALUES (1001, '新人大礼包', 1, ...)
   ↓
4. 选择子券
   - 从普通券列表中选择（is_pkg=false）
   - 设置每个子券的数量
   ↓
5. 保存关联关系到 crm_coupon_map
   - INSERT INTO crm_coupon_map (main_code, deputy_code, number, ...) VALUES ('1001', '2001', 2, ...)
   - INSERT INTO crm_coupon_map (main_code, deputy_code, number, ...) VALUES ('1001', '2002', 3, ...)
   - INSERT INTO crm_coupon_map (main_code, deputy_code, number, ...) VALUES ('1001', '2003', 1, ...)
   ↓
6. 完成
```

### 6.2 用户领取券包流程

```
1. 用户领取券包
   ↓
2. 查询券包信息
   - SELECT * FROM crm_coupon WHERE lid = 1001 AND is_pkg = 1
   ↓
3. 查询券包包含的子券
   - SELECT * FROM crm_coupon_map WHERE main_code = '1001'
   ↓
4. 根据 number 字段发放对应数量的子券
   - 发放 2 张 "10元代金券"
   - 发放 3 张 "5元代金券"
   - 发放 1 张 "免费菜品券"
   ↓
5. 记录领取记录
   ↓
6. 完成
```

### 6.3 更新券包流程

```
1. 用户更新券包
   ↓
2. 更新券包基本信息
   - UPDATE crm_coupon SET name = '新人超级大礼包' WHERE lid = 1001
   ↓
3. 删除旧的关联关系
   - DELETE FROM crm_coupon_map WHERE main_code = '1001'
   ↓
4. 重新创建关联关系
   - INSERT INTO crm_coupon_map (main_code, deputy_code, number, ...) VALUES (...)
   ↓
5. 完成
```

## ⚠️ 八、注意事项

### 8.1 数据类型不一致

**问题**：
- `crm_coupon.lmnid` 是 BIGINT 类型
- `crm_coupon_map.main_code` 和 `deputy_code` 是 VARCHAR 类型

**影响**：
- 需要在查询时进行类型转换
- 可能影响查询性能

**建议**：
- 统一使用 BIGINT 类型
- 或者在应用层统一处理类型转换

### 8.2 级联删除

**问题**：
- 删除券包时，需要手动删除 crm_coupon_map 中的关联记录
- 删除子券时，需要检查是否被券包引用

**建议**：
- 添加外键约束（ON DELETE CASCADE）
- 或者在 Service 层实现级联删除逻辑

### 8.3 券包嵌套

**问题**：
- 当前设计不支持券包嵌套（券包包含券包）
- 通过 `is_pkg=false` 的验证防止嵌套

**建议**：
- 如果未来需要支持嵌套，需要重新设计数据结构
- 可以考虑使用树形结构或递归查询

### 8.4 性能优化

**问题**：
- 查询券包时需要关联查询 crm_coupon_map
- 如果券包数量多，可能影响性能

**建议**：
- 添加索引：`idx_main_code`, `idx_deputy_code`
- 使用缓存：将券包信息缓存到 Redis
- 批量查询：避免 N+1 查询问题

## 🎯 九、优化建议

### 9.1 数据库优化

1. **统一数据类型**
   ```sql
   ALTER TABLE crm_coupon_map 
   MODIFY COLUMN main_code BIGINT COMMENT '主券ID（券包的lid）',
   MODIFY COLUMN deputy_code BIGINT COMMENT '副券ID（子券的lid）';
   ```

2. **添加外键约束**
   ```sql
   ALTER TABLE crm_coupon_map
   ADD CONSTRAINT fk_main_code 
   FOREIGN KEY (main_code) REFERENCES crm_coupon(lmnid) ON DELETE CASCADE,
   ADD CONSTRAINT fk_deputy_code 
   FOREIGN KEY (deputy_code) REFERENCES crm_coupon(lmnid) ON DELETE RESTRICT;
   ```

3. **添加索引**
   ```sql
   CREATE INDEX idx_main_code ON crm_coupon_map(main_code);
   CREATE INDEX idx_deputy_code ON crm_coupon_map(deputy_code);
   CREATE INDEX idx_mid_ispkg ON crm_coupon(company_id, is_pkg);
   ```

### 9.2 代码优化

1. **批量查询避免 N+1**
   ```java
   // 不好的做法
   for (CrmCoupon package : packages) {
       List<CrmCouponMap> subCoupons = getSubCoupons(package.getLid());
   }
   
   // 好的做法
   List<Long> packageIds = packages.stream()
       .map(CrmCoupon::getLid)
       .collect(Collectors.toList());
   Map<Long, List<CrmCouponMap>> subCouponsMap = 
       getSubCouponsByPackageIds(packageIds);
   ```

2. **使用缓存**
   ```java
   @Cacheable(value = "coupon:package", key = "#packageId")
   public CrmCouponVO getPackageWithSubCoupons(Long packageId) {
       // 查询券包和子券
   }
   ```

3. **添加常量类**
   ```java
   public class CouponConstant {

       public static final Boolean IS_NORMAL_COUPON = false;
       public static final String MSG_PACKAGE_CANNOT_CONTAIN_PACKAGE = 
           "券包不能包含另一个券包";
       public static final String MSG_SUB_COUPON_QUANTITY_INVALID = 
           "券包里的单品券数量不能为空或者小于1";
   }
   ```

## 📝 十、总结

### 10.1 设计特点

✅ **简单清晰**：通过 `is_pkg` 字段区分券包和普通券
✅ **灵活可配**：每个子券的数量可以独立配置
✅ **易于维护**：通过关联表管理，易于增删改
✅ **防止嵌套**：券包不能包含券包，避免复杂度

### 10.2 核心表结构

- **crm_coupon**: 存储券包和普通券的基本信息
- **crm_coupon_map**: 存储券包与子券的关联关系

### 10.3 关键业务逻辑

1. 查询券包时，根据 `is_pkg=true` 过滤
2. 查询子券时，通过 `main_code` 关联查询 crm_coupon_map
