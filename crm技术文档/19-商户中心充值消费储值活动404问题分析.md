# 商户中心充值、消费-储值活动 404 问题分析

## 1. 问题现象

商户中心进入 `会员营销 -> 充值、消费 -> 储值活动` 时，页面偶发或反复显示前端 404：

```text
404
页面不存在
```

截图中的异常地址形态为：

```text
/membermarketing/crmmarketingmgr/topupcampaign/topupcampaign
```

当前前端代码注册的正确路由为：

```text
/membermarketing/crmmarketingmrg/topupcampaign
```

异常地址同时存在两个问题：

1. `crmmarketingmgr` 与正确路由 `crmmarketingmrg` 不一致。
2. `/topupcampaign` 被重复追加了一次。

因此该问题首先属于前端路由匹配失败，不是后端接口 404。

## 2. 相关代码证据

### 2.1 静态路由注册

文件：

```text
D:\mywork\nms4cloud-biz-ui\config\routes.ts
```

路由只注册了以下路径：

```ts
{
  path: '/membermarketing/crmmarketingmrg/topupcampaign',
  name: '储值活动',
  icon: 'BookOutlined',
  component: './CrmMarketingMrg/components/TopUpCampaignMrg',
}
```

没有注册：

```text
/membermarketing/crmmarketingmgr/topupcampaign
/membermarketing/crmmarketingmrg/topupcampaign/topupcampaign
/membermarketing/crmmarketingmgr/topupcampaign/topupcampaign
```

### 2.2 充值、消费桌面卡片配置

文件：

```text
D:\mywork\nms4cloud-biz-ui\src\pages\CrmMarketingMrg\index.tsx
```

储值活动卡片配置为相对路径：

```ts
{
  icon: crm_save_rule,
  title: '储值活动',
  pathname: '/topupcampaign',
  children: () => <TopUpCampaignMrg />,
}
```

### 2.3 Desktop 组件拼接路径

文件：

```text
D:\mywork\nms4cloud-biz-ui\src\components\Desktop\index.tsx
```

点击卡片时，当前实现使用当前浏览器路径直接拼接卡片路径：

```ts
const {
  location: { pathname },
} = window;

const path = pathname + i.pathname;
history.replace(path);
```

这意味着：

```text
当前路径 /membermarketing/crmmarketingmrg
点击 /topupcampaign
=> /membermarketing/crmmarketingmrg/topupcampaign
```

这是正常情况。

但如果当前已经处于子路由：

```text
当前路径 /membermarketing/crmmarketingmrg/topupcampaign
点击 /topupcampaign
=> /membermarketing/crmmarketingmrg/topupcampaign/topupcampaign
```

这就是重复追加问题。

如果当前路径本身已经被污染为：

```text
/membermarketing/crmmarketingmgr/topupcampaign
```

再次点击储值活动后会变成：

```text
/membermarketing/crmmarketingmgr/topupcampaign/topupcampaign
```

这与截图现象一致。

## 3. 为什么会反复出现

该问题反复出现的关键，不只是一次路由拼错，而是错误路径可能被运行时状态保存下来。

### 3.1 顶部标签页保存完整 pathname

文件：

```text
D:\mywork\nms4cloud-biz-ui\src\app.tsx
```

自定义标签页使用完整 `pathname` 作为标签 key：

```ts
items={Object.entries(keepElements.current).map(([pathname, value]: any) => ({
  key: pathname,
  ...
}))}
```

如果错误路径已经进入标签页缓存，后续点击该标签仍会回到错误路径。

### 3.2 收藏功能保存完整 path

文件：

```text
D:\mywork\nms4cloud-biz-ui\src\hooks\useFavorites.ts
```

收藏数据存储在 localStorage：

```ts
const STORAGE_KEY = 'user_favorite_pages';
```

添加收藏时直接保存传入 path：

```ts
items.push({ path, name, addedAt: Date.now() });
```

如果用户曾在错误路径上添加收藏，或者错误路径被当作当前标签页路径收藏，那么后续从收藏进入会持续复现 404。

### 3.3 浏览器历史和外部入口也可能保留错误路径

浏览器历史、书签、第三方跳转、聊天工具链接、后台菜单旧数据等都可能携带错误路径。

当前本地仓库搜索结果显示：

```text
nms4cloud-biz-ui 当前源码未找到 crmmarketingmgr
nms4cloud 后端 permission2packages.json 中也只找到 crmmarketingmrg
```

因此 `crmmarketingmgr` 更可能来自以下来源之一：

1. 线上数据库中的旧菜单/权限路径。
2. 浏览器收藏、系统收藏或顶部标签缓存。
3. 旧版本前端构建产物或 CDN 缓存。
4. 外部入口复制了错误 URL。
5. 用户手动访问过错误 URL 后被标签页缓存保留。

## 4. 根因判断

### 4.1 直接根因

`Desktop` 组件使用当前 `window.location.pathname` 作为基础路径，再拼接卡片的相对 `pathname`。

该写法依赖一个隐含前提：

```text
当前 pathname 必须永远是模块首页路径
```

例如：

```text
/membermarketing/crmmarketingmrg
```

但实际运行时，当前路径可能是子路由：

```text
/membermarketing/crmmarketingmrg/topupcampaign
```

也可能是历史错误路径：

```text
/membermarketing/crmmarketingmgr/topupcampaign
```

一旦这个前提不成立，路径就会被重复拼接，导致 404。

### 4.2 反复复现根因

错误 URL 进入运行时状态后，没有被校验或纠正：

1. 顶部标签页直接使用完整 pathname。
2. 收藏直接保存完整 path。
3. `Desktop` 跳转不校验目标路由是否存在。
4. 没有历史错误路径兼容重定向。

因此一次错误路径可能被保存，并在后续操作中反复触发。

## 5. 排查方法

### 5.1 浏览器控制台检查收藏路径

在出现问题的浏览器控制台执行：

```js
localStorage.getItem('user_favorite_pages')
```

重点检查是否存在：

```text
crmmarketingmgr
topupcampaign/topupcampaign
```

如果存在，说明收藏数据已经污染。

### 5.2 检查 sessionStorage 权限路径

执行：

```js
sessionStorage.getItem('permissionPaths')
sessionStorage.getItem('permissionPathKeys')
```

重点确认运行时权限路径是否仍指向旧路径。

### 5.3 检查线上菜单或权限数据库

在后台菜单、权限、功能包、角色权限数据中搜索：

```text
crmmarketingmgr
topupcampaign/topupcampaign
```

如果存在，需要修正为：

```text
crmmarketingmrg
```

### 5.4 检查线上构建产物

确认当前线上静态资源是否已经是最新版本：

1. 检查浏览器 Network 中加载的 JS 文件 hash。
2. 确认 CDN 或 nginx 静态目录是否仍存在旧构建。
3. 强制刷新页面或清理缓存后再测试。

## 6. 临时处理方式

### 6.1 用户侧临时恢复

1. 关闭当前错误标签页。
2. 清理浏览器里该系统的错误收藏。
3. 清理 localStorage 中 `user_favorite_pages` 里的错误路径。
4. 直接访问正确路径：

```text
/membermarketing/crmmarketingmrg/topupcampaign
```

### 6.2 运维或管理员侧临时恢复

1. 检查后台菜单/权限路径，确认没有 `crmmarketingmgr`。
2. 检查是否有旧前端静态资源缓存。
3. 通知用户清理错误收藏或重新登录。

## 7. 修复建议

### 7.1 优先修复 Desktop 路径拼接

不要使用当前页面的完整 pathname 直接拼接子路径。

更稳妥的方式是：

1. `Desktop` 支持传入固定 `basePath`。
2. 每个模块首页显式传入自己的 base path。
3. 卡片点击时使用 `basePath + i.pathname`。

例如充值、消费模块固定为：

```text
/membermarketing/crmmarketingmrg
```

这样无论当前是否在子页面，点击储值活动都只会跳到：

```text
/membermarketing/crmmarketingmrg/topupcampaign
```

### 7.2 增加历史错误路径兼容

为了兼容已经保存到收藏、浏览器历史或外部链接的旧路径，可以增加一次性重定向：

```text
/membermarketing/crmmarketingmgr/*
=> /membermarketing/crmmarketingmrg/*
```

并处理重复片段：

```text
/membermarketing/crmmarketingmrg/topupcampaign/topupcampaign
=> /membermarketing/crmmarketingmrg/topupcampaign
```

该方案属于兼容修复，不应删除现有正确路由。

### 7.3 收藏和标签页增加合法性校验

收藏或标签页跳转前可检查目标路由是否存在。

如果不存在，可以：

1. 跳转到模块首页。
2. 删除无效收藏。
3. 提示用户该收藏已失效。

## 9. 结论

该问题不是储值活动业务页面本身缺失，也不是后端接口不存在。

核心问题是：

```text
前端运行时路径被错误入口污染后，Desktop 组件继续基于当前 pathname 追加相对路径，导致 URL 越拼越错。
```

反复出现的原因是：

```text
错误 URL 可能被顶部标签页、收藏、浏览器历史、旧菜单或外部链接保存下来，而当前系统没有对这些历史错误路径做校验、清理或兼容重定向。
```

推荐修复优先级：

1. 修复 `Desktop` 路径拼接，使用固定模块 base path。
2. 增加历史错误路径兼容重定向。
3. 清理线上菜单/权限/收藏中的 `crmmarketingmgr` 和重复 `topupcampaign`。
4. 对收藏和标签页跳转增加路由合法性校验。
