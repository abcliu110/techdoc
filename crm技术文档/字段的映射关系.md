## 结论

- `mycat` 别名在不同服务中都指向 `gylregdb`，但 CRM 服务的主机是 `172.16.0.54:3306`，多数其他服务是 `172.16.0.12:3306`。
- `biz` -> `a_biz`；`crm` -> `a_crm`；`order` -> `a_order`；`pay` -> `a_payment`；`platform` -> `a_platform`；`pos` -> `a_pos`；`pt` -> `a_product`；`wechat` -> `a_weachat`；`wms` -> `a_wms`。
- 如果某个数据源别名后面还有 MyCat/代理层二次路由，本目录的 Nacos YAML 只能证明应用连接到的 JDBC database，不能继续证明代理后的最终物理库。
