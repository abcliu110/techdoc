# 消费赠券配置字段说明（用户文档）

更新时间：2026-05-26

适用页面：CRM / 会员营销 / 消费赠券 / 新增或修改消费赠券

## 1. 这是什么功能

消费赠券用于配置“会员消费结账成功后，系统自动给会员发优惠券”的规则。

简单理解：

- 顾客是指定会员类型、会员等级。
- 顾客在指定门店消费。
- 消费时间在规则有效期内。
- 账单金额或指定付款金额达到门槛。
- 系统给这个会员发指定数量的优惠券。

注意：消费赠券只针对普通消费结账。会员存款单、会员充值单、回款单不属于普通消费，不应该触发消费赠券。

## 2. 发券触发条件

POS 普通账单结账成功后才会触发消费赠券。必须同时满足下面条件：

1. 账单绑定了会员。
2. 门店开启了消费权益开关。
3. 账单不是回款单。
4. 账单不是会员存款单或会员充值单。
5. POS 能正常调用 CRM 服务。
6. CRM 中存在匹配的消费赠券规则。
7. 要发放的券模板本身允许发券，例如已审核、未终止、在领取时间内、有库存、未超过会员领取限制。

如果其中任何一项不满足，本次消费不会发券。

## 3. 字段说明

### 消费赠券规则名称

用于区分不同消费赠券规则的名称。

示例：`518会员类型`

建议命名时写清楚规则用途，例如：

- `VIP会员满100送菜品券`
- `东莞店5月消费赠券`
- `测试518会员消费赠券`

该名称只用于后台识别，不影响顾客看到的券名称。

### 优惠券

选择本规则要发放的优惠券模板。

示例：`开胃酸菜焖鹅血01`

注意事项：

- 这里选的是“券模板”，不是已经发给会员的券。
- 真正发券时，会按这个模板生成会员名下的新券。
- 券模板必须可发放。如果券模板已终止、未审核、过期、库存不足，规则命中了也可能发券失败。
- 消费赠券会受券模板的“每人每日领取限制”和“每人总领取限制”影响。如果券模板设置每人每天最多领 1 张，同一个会员当天第二次消费可能不会再发同一张券。

### 赠券数量

每次规则命中时，基础赠送的券数量。

示例：`1`

含义：

- 如果不勾选“赠送规则”，命中后固定送这里填写的数量。
- 如果勾选“赠送规则”，这里会作为倍数基础参与计算。

示例：

- 赠券数量 = `1`
- 满赠金额 = `100`
- 金额计算方式 = `账单金额`
- 消费账单金额 = `100`
- 结果：送 1 张券。

### 满赠金额

消费金额达到这个门槛后，规则才可能命中。

示例：`100`

含义：

- 金额达到 `100`，可以命中。
- 金额低于 `100`，不会命中。

这里用哪个金额判断，取决于“金额计算方式”。

注意：满赠金额只负责判断规则是否命中，不直接决定最终发几张券。最终发券数量还要看“赠券数量”和“赠送规则”。

### 会员类型

限制哪些会员类型可以参与本规则。

示例：`测试518`

含义：

- 选择了会员类型：只有该会员类型的会员可以命中。
- 不选择会员类型：通常表示不限制会员类型，所有会员类型都可以继续参与后续判断。

实际发券时，CRM 会以会员档案中的会员类型为准，不以 POS 本地传来的名称为准。

### 会员等级

限制哪些会员等级可以参与本规则。

示例：`VIP`

含义：

- 选择了会员等级：只有该等级会员可以命中。
- 不选择会员等级：通常表示不限制会员等级。

建议：如果规则只面向某个会员类型下的 VIP，一定要同时确认“会员类型”和“会员等级”都配置正确。

### 赠送规则

用于配置“每满多少元按赠券数量翻倍赠送”。

页面显示类似：

`每满 100 元依次按赠券数量翻倍赠送，最多可赠送 1 张`

这个配置由三个字段组成：

- 是否勾选“每满”：是否启用递增赠送。
- 每满金额：每达到多少元增加一档。
- 最多可赠送：单次最多发多少张券。

如果不勾选：

- 命中规则后，固定按“赠券数量”发券。

如果勾选：

- 系统先判断本次命中的金额是否达到“满赠金额”。
- 如果没有达到满赠金额，不发券。
- 如果达到满赠金额，再用本次命中的金额除以“每满金额”，向下取整。
- 再乘以“赠券数量”。
- 最后不超过“最多可赠送”。

重要：当前系统不是“超过满赠金额后，再按每满金额递增”。系统不会先扣掉满赠金额再计算递增数量，而是命中满赠金额后，用整笔命中金额直接除以每满金额。

公式：

```text
命中金额 < 满赠金额：
  发券数量 = 0

命中金额 >= 满赠金额 且勾选每满：
  发券数量 = floor(命中金额 / 每满金额) * 赠券数量
  如果最多可赠送 > 0，再取 min(发券数量, 最多可赠送)
```

说明：

- 系统用本次命中的金额除以“每满金额”，向下取整。
- 再乘以“赠券数量”。
- 最后不超过“最多可赠送”。

示例 1：不勾选赠送规则

- 赠券数量 = `1`
- 满赠金额 = `100`
- 消费金额 = `300`
- 结果：仍然送 1 张。

示例 2：勾选赠送规则

- 赠券数量 = `1`
- 满赠金额 = `100`
- 每满金额 = `100`
- 最多可赠送 = `3`
- 消费金额 = `250`
- 计算：`250 / 100 = 2`，向下取整为 2
- 结果：送 2 张。

示例 3：勾选赠送规则并限制最多可赠送

- 赠券数量 = `1`
- 每满金额 = `100`
- 最多可赠送 = `3`
- 消费金额 = `500`
- 理论可送 5 张，但最多可赠送 3 张
- 结果：送 3 张。

示例 4：满赠金额和每满金额不同

- 赠券数量 = `1`
- 满赠金额 = `500`
- 每满金额 = `100`
- 最多可赠送 = `6`
- 金额计算方式 = `账单金额`

计算结果：

| 账单金额 | 是否达到满赠 500 | 每满 100 计算 | 最多 6 张后 | 实际发券 |
|---:|---|---:|---:|---:|
| 499 | 否 | 不计算 | 0 | 0 |
| 500 | 是 | 5 张 | 5 张 | 5 |
| 599 | 是 | 5 张 | 5 张 | 5 |
| 600 | 是 | 6 张 | 6 张 | 6 |
| 699 | 是 | 6 张 | 6 张 | 6 |
| 700 | 是 | 7 张 | 6 张 | 6 |

这个配置表示：先要求账单金额至少达到 `500`，达到后再按整笔账单金额每 `100` 元送 `1` 张，最多送 `6` 张。

它不是：满 `500` 送 `1` 张、满 `600` 送 `2` 张、满 `700` 送 `3` 张。

注意：如果“最多可赠送”填 `0` 或不填，按当前系统逻辑通常表示不限制上限。

### 有效期限

控制这条消费赠券规则什么时候有效。

页面常见选项：

- 固定期限
- 相对期限

#### 固定期限

使用“生效时间”和“失效时间”控制。

示例：

- 生效时间：`2026-05-01`
- 失效时间：`2026-05-31`

含义：

- 在这个时间范围内结账，规则才可能命中。
- 不在这个时间范围内结账，规则不会命中。

注意：判断时间使用结账时间，不是创建规则时间，也不是会员注册时间。

#### 相对期限

相对期限通常按规则创建时间再加上配置的小时数或天数计算有效区间。

如果业务人员只是配置月度活动，建议优先使用“固定期限”，更直观，也更方便核对。

### 生效时间

固定期限下，规则开始生效的时间。

示例：`2026-05-01`

结账时间早于生效时间，不发券。

### 失效时间

固定期限下，规则结束生效的时间。

示例：`2026-05-31`

结账时间超过失效时间，不发券。

注意：页面选择日期时，需要确认系统实际保存的是当天开始还是当天结束。如果发现 5 月 31 日当天部分时间不发券，需要让技术人员确认保存的具体时间点。

### 金额计算方式

金额计算方式决定系统用哪一个金额和“满赠金额”比较。

当前有三种方式：

| 页面选项 | 系统含义 | 适用场景 |
| --- | --- | --- |
| 账单金额 | 使用整张账单金额 | 满整单金额送券 |
| 会员卡金额 | 使用会员储值卡支付金额 | 只想按会员卡扣款金额送券 |
| 非会员卡扣款金额 | 使用非会员卡支付金额 | 只想按现金、微信、支付宝等非会员卡金额送券 |

#### 账单金额

用整张账单金额判断是否满赠。

示例：

- 规则：满赠金额 = `100`
- 金额计算方式 = `账单金额`
- 账单总金额 = `100`
- 其中会员卡支付 = `8`
- 现金支付 = `92`
- 结果：按 `100` 判断，满足满赠。

适合场景：只关心顾客本次消费总额，不关心用什么方式付款。

#### 会员卡金额

只用会员卡支付金额判断是否满赠。

示例：

- 规则：满赠金额 = `100`
- 金额计算方式 = `会员卡金额`
- 账单总金额 = `100`
- 会员卡支付 = `8`
- 现金支付 = `92`
- 结果：按 `8` 判断，不满足满赠。

适合场景：鼓励顾客使用会员储值余额消费。

#### 非会员卡扣款金额

只用非会员卡支付金额判断是否满赠。

示例：

- 规则：满赠金额 = `100`
- 金额计算方式 = `非会员卡扣款金额`
- 账单总金额 = `100`
- 会员卡支付 = `8`
- 现金支付 = `92`
- 结果：按 `92` 判断，不满足满赠。

适合场景：只对现金、微信、支付宝等非会员卡付款部分送券。

页面红字提示说明：如果使用“非会员卡扣款金额”，系统还会受线下支付方式里的“参与消费赠送”属性控制。也就是说，某个线下支付方式如果没有设置参与消费赠送，即使用这种支付方式付款，也可能不计入非会员卡扣款金额。

### 星期

限制规则在哪些星期几生效。

示例：

- 勾选一、二、三：只有周一、周二、周三结账才可能命中。
- 不勾选：通常表示不按星期限制。

如果同时配置了星期和时间段，需要两个限制都满足。

### 时间段

通过页面中间的加号添加时间段限制。

用途：限制每天哪个时间段可以发券。

示例：

- 时间段：`11:00 - 14:00`
- 只有午市时间结账才可能命中。

注意：

- 如果配置了时间段，结账时间必须落在时间段内。
- 如果配置多个时间段，命中其中一个时间段即可。
- 如果没有配置时间段，通常表示不按时间段限制。

### 适用门店

控制哪些门店可以使用这条规则。

选项：

- 所有门店
- 选择门店

#### 所有门店

所有门店的消费账单都可以参与本规则。

#### 选择门店

只有指定门店的消费账单才可以参与本规则。

示例：`东莞店`

含义：只有东莞店结账的普通消费账单才可能触发本规则。其他门店即使会员、金额、时间都满足，也不会发券。

### 选择门店

当“适用门店”选择“选择门店”时，需要在这里选择具体门店。

可选择一个或多个门店。

建议：跨门店活动选择“所有门店”；单店活动选择指定门店，避免其他门店误发券。

## 4. 系统匹配规则的顺序

实际发券时，系统大致按下面顺序判断：

1. POS 普通消费结账成功。
2. 判断账单是否绑定会员。
3. 排除回款单、会员存款单、会员充值单。
4. 根据结账时间判断规则有效期、星期、时间段。
5. 根据门店判断是否在适用门店范围内。
6. 根据会员类型、会员等级筛选规则。
7. 根据金额计算方式和满赠金额匹配规则。
8. 计算本次应赠券数量。
9. 调用优惠券模板发放校验。
10. 生成会员名下的券。

## 5. 多条规则同时满足时怎么选

如果同一个会员、同一笔账单同时满足多条规则，系统会按金额计算方式分别找符合条件的规则，并选择该金额口径下满赠金额最高且不超过本次金额的规则。

当前 POS 新链路中的金额口径匹配优先级是：

1. 会员卡金额
2. 非会员卡扣款金额
3. 账单金额

也就是说，混合支付时，如果同时存在“会员卡金额”规则和“账单金额”规则，并且会员卡金额规则已经命中，系统会优先使用会员卡金额规则。

如果你希望截图里的“账单金额满 100 送券”稳定命中，建议避免同时配置同一会员、同一门店、同一时间范围内更优先的会员卡金额规则。

## 6. 为什么消费了但没有送券

常见原因如下：

### 账单不是普通消费

以下账单不会触发消费赠券：

- 回款单
- 会员存款单
- 会员充值单
- 储值方案充值单

原因：这些账单有自己的业务链路。尤其是储值方案可能已经按储值规则送券，不能再走消费赠券，否则同一笔储值可能重复送券。

### 会员不匹配

可能情况：

- 账单没有绑定会员。
- 会员类型不是规则配置的会员类型。
- 会员等级不是规则配置的会员等级。

### 门店不匹配

规则选择了指定门店，但实际消费发生在其他门店。

### 时间不匹配

可能情况：

- 结账时间早于生效时间。
- 结账时间晚于失效时间。
- 星期不在规则勾选范围内。
- 时间段不在规则配置范围内。

### 金额不满足

重点看“金额计算方式”。

例如：

- 账单总金额 100
- 会员卡支付 8
- 现金支付 92

如果金额计算方式是：

- 账单金额：按 100 判断，满足满 100。
- 会员卡金额：按 8 判断，不满足满 100。
- 非会员卡扣款金额：按 92 判断，不满足满 100。

### 券模板限制

规则命中了，但最终仍可能因为券模板限制不发券。

常见限制：

- 券模板未审核。
- 券模板已终止。
- 券模板不在领取时间内。
- 券库存不足。
- 会员已达到每天最大领取次数。
- 会员已达到总领取次数。

特别说明：消费赠券会受券模板“每人每日领取限制”和“每人总领取限制”影响。储值方案赠券可以跳过部分会员领取次数限制，但消费赠券不跳过。

## 7. 为什么第一次消费发券，第二次消费不发券

最常见原因是券模板设置了领取限制。

例如券模板设置：

- 每人每天最多领取 1 张

那么同一个会员当天第一次消费满足规则后会发券；第二次消费即使仍满足消费赠券规则，也可能因为券模板每日领取上限不再发券。

这种情况不是消费赠券规则没有命中，而是券模板发放校验拦截。

## 8. 截图配置的含义示例

截图中的配置可以解释为：

- 规则名称：`518会员类型`
- 发放券：`开胃酸菜焖鹅血01`
- 赠券数量：`1`
- 满赠金额：`100`
- 会员类型：`测试518`
- 会员等级：`VIP`
- 金额计算方式：`账单金额`
- 有效期：`2026-05-01` 到 `2026-05-31`
- 适用门店：`东莞店`
- 最多可赠送：`1` 张

所以，东莞店的 `测试518` 类型、`VIP` 等级会员，在有效期内发生普通消费，账单金额达到 100，理论上会送 1 张 `开胃酸菜焖鹅血01`。

如果没有送券，应优先检查：

1. 这笔是不是普通消费账单，不是会员存款/充值单。
2. 账单是否确实绑定该会员。
3. 结账门店是否是东莞店。
4. 账单金额是否达到 100。
5. 券模板是否有库存、是否已审核、是否超过会员领取限制。

## 9. 和会员存款、会员充值的关系

会员存款/充值不应触发消费赠券。

原因：

- 会员存款/充值不是“消费商品或服务”的普通消费账单。
- 储值方案可能配置自己的赠券规则。
- 如果充值单同时触发消费赠券，会造成同一笔储值既拿储值赠券，又拿消费赠券。

因此系统应区分：

| 场景 | 是否触发消费赠券 | 说明 |
| --- | --- | --- |
| 普通消费结账 | 是 | 满足规则后发券 |
| 混合支付普通消费 | 是 | 按金额计算方式判断 |
| 会员存款单 | 否 | 不属于普通消费 |
| 会员充值单 | 否 | 不属于普通消费 |
| 储值方案充值单 | 否 | 走储值自己的权益规则 |
| 回款单 | 否 | 不属于普通消费 |

## 10. 核对和排查建议

业务人员排查时，先按页面条件核对：

1. 会员是否正确。
2. 会员类型、等级是否正确。
3. 门店是否正确。
4. 结账时间是否在有效期、星期、时间段内。
5. 金额计算方式对应的金额是否达标。
6. 券模板是否还能发。

技术人员排查时，建议优先看：

- POS 本地消费赠券 round 是否生成。
- CRM 消费赠券幂等记录是否生成。
- CRM 券发行记录是否生成。
- 会员券发行明细报表是否增加。

常见排查表：

| 表或报表 | 用途 |
| --- | --- |
| POS 本地 `crm_consume_coupon_round` | 看 POS 是否登记了本次消费赠券意图 |
| CRM `crm_consume_coupon_op_record` | 看 CRM 是否收到请求、是否命中规则、是否发券成功 |
| CRM `crm_coupon_order` | 看会员名下是否实际生成了券 |
| 会员券发行明细报表 | 看报表是否能查到本次发券 |
| 会员券发行汇总报表 | 看券发行数量是否汇总增加 |

## 11. 技术口径说明

本节给技术人员核对用。

页面主要字段落到消费赠券规则表：

| 页面字段 | 规则字段 |
| --- | --- |
| 消费赠券规则名称 | `name` |
| 优惠券 | `coupon_lid` / `coupon_name` |
| 赠券数量 | `gift_quantity` |
| 满赠金额 | `full_amount` |
| 会员类型 | `card_type_lid` |
| 会员等级 | `card_type_level_lid` |
| 赠送规则是否启用 | `gift_by_increase` |
| 每满金额 | `gift_eve_amount` |
| 最多可赠送 | `gift_max_quantity` |
| 有效期限 | `term_of_validity_method` |
| 生效时间 | `start_effective_time` |
| 失效时间 | `end_effective_time` |
| 金额计算方式 | `amount_method` |
| 适用所有门店 | `is_all_store` |
| 指定门店 | `store_sids` |

金额计算方式枚举：

| 枚举 | 含义 |
| --- | --- |
| `B` | 账单金额 |
| `H` | 会员卡金额 |
| `F` | 非会员卡金额 |

POS 触发入口：

- 普通 POS 结账：`CrmConsumeCouponLocalService.grantConsumeCouponForCheckout`
- 快捷支付结账：`CashPayHandler` 结账后调用同一入口

CRM 发券接口：

- `POST /crm_consume_coupon/grantSign`

CRM 主要处理：

- 先按 `roundKey` 幂等。
- 根据会员卡查 CRM 会员档案。
- 匹配消费赠券规则。
- 计算应赠券数量。
- 调用通用发券服务生成券。

重要边界：

- 消费赠券当前已排除回款单和会员存款/充值单。
- 消费赠券不跳过券模板的会员每日领取限制和总领取限制。
- 储值方案赠券与消费赠券是不同权益链路，不应在同一笔储值上叠加。
