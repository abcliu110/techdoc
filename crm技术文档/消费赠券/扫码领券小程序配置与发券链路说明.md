# 扫码领券小程序配置与发券链路说明

日期：2026-05-17

范围：CRM 券/红包管理生成的券二维码，用户扫码进入小程序后领取或购买优惠券的链路。

## 1. 结论

CRM 券二维码不是直接发券入口，而是一个普通 HTTPS 链接。微信小程序后台需要配置“普通链接二维码打开小程序”规则，用户扫码后微信才会把这个链接交给小程序入口页解析。

当前源码里，领券二维码应统一进入小程序入口页：

```text
pagesIndex/pageIndex/index
```

入口页解析参数后，如果发现：

```text
flag=coupon
```

就跳转到真正的领券页：

```text
pagePop/couponcollection/index?coupon_lid={券模板lid}
```

用户在领券页点击“立即领取”后，才会调用后端发券接口并写入 `crm_coupon_order`。

## 2. 小程序后台如何配置

进入微信公众平台小程序后台：

```text
开发管理 -> 开发设置 -> 扫普通链接二维码打开小程序
```

新增或修改规则时，重点配置下面几项。

### 2.1 私版小程序券码配置

券二维码示例：

```text
https://wx.lmnsaas.com/yszx?code=2055914152532905986&flag=coupon
```

后台规则建议填写：

```text
二维码地址：https://wx.lmnsaas.com/yszx
小程序路径：pagesIndex/pageIndex/index
测试范围：先选开发版验证，验证通过后发布
```

说明：

- `https://wx.lmnsaas.com/yszx` 是规则匹配前缀。
- `code=2055914152532905986` 是具体券模板 `crm_coupon.lmnid`，每张券不同，不要固定写进规则。
- `flag=coupon` 表示这是优惠券扫码场景。
- 微信命中规则后，会把完整原始链接放到小程序入口参数 `q` 中。

### 2.2 公版小程序券码配置

公版小程序需要在 URL 参数中携带商户标识，示例：

```text
https://wx.lmnsaas.com/nmscommon?mid=yszx&code=2055914152532905986&flag=coupon
```

后台规则建议填写：

```text
二维码地址：https://wx.lmnsaas.com/nmscommon
小程序路径：pagesIndex/pageIndex/index
测试范围：先选开发版验证，验证通过后发布
```

说明：

- `nmscommon` 表示公版小程序壳入口。
- `mid=yszx` 用来定位具体商户。
- 公版小程序缺少 `mid` 时，登录和商户初始化可能无法定位到正确商户。

### 2.3 不建议直接配置领券页

不要把小程序路径直接配置为：

```text
pagePop/couponcollection/index
```

原因是微信传给小程序的是完整普通链接参数 `q`，当前真正解析 `q` 的代码在：

```text
pagesIndex/pageIndex/index
```

`pagePop/couponcollection/index` 只接收已经转换好的：

```text
coupon_lid={券模板lid}
```

它不负责解析微信传入的完整普通链接。

## 3. 二维码 URL 参数含义

以私版链接为例：

```text
https://wx.lmnsaas.com/yszx?code=2055914152532905986&flag=coupon
```

参数含义：

| 参数 | 含义 | 来源 |
|---|---|---|
| `yszx` | 商户路径前缀，通常对应商户标识或私版小程序入口 | CRM 后端拼接 `bizAdmin.getMerchantId()` |
| `code` | 券模板逻辑编号，即 `crm_coupon.lmnid` | CRM 券新增/编辑时生成 |
| `flag` | 业务场景标识，`coupon` 表示优惠券扫码入口 | CRM 券二维码固定参数 |

## 4. 前端如何解析参数

源码位置：

```text
D:\mywork\taro-mall\src\pagesIndex\pageIndex\index.tsx
```

核心逻辑：

```ts
const {params: {q = {}} = {}} = curRoute?.router
const param = decodeURIComponent(q)
const codeParams: any = Tips.codeUrlToParams(param)
```

`Tips.codeUrlToParams()` 会解析 `?` 后面的参数：

```text
code=2055914152532905986
flag=coupon
```

当 `flag=coupon` 时，入口页跳转到领券页：

```ts
Taro.navigateTo({
  url: `/pagePop/couponcollection/index?coupon_lid=${codeParams?.code}`,
})
```

所以扫码后的前端跳转结果是：

```text
pagePop/couponcollection/index?coupon_lid=2055914152532905986
```

## 5. 领券页如何查询券详情

源码位置：

```text
D:\mywork\taro-mall\src\pagePop\couponcollection\index.tsx
```

领券页读取路由参数：

```ts
let pageParams = Tips.getRouteParams()
```

然后用 `coupon_lid` 查询券详情：

```ts
get_purchase_detail({
  lid: pageParams.coupon_lid
})
```

请求接口：

```text
POST /scrm/crm_coupon/get_purchase_detail
```

后端入口：

```text
D:\mywork\nms4cloud\nms4cloud-app\2_business\nms4cloud-crm\nms4cloud-crm-app\src\main\java\com\nms4cloud\crm\app\controller\CrmCouponController.java
```

方法：

```java
@PostMapping(value = "/get_purchase_detail")
public NmsResult<CrmCouponVO> getPurchaseDetail(@Valid @RequestBody CrmCouponGetDTO request) {
  CUserVO user = getUser();
  request.setMid(user.getMid());
  return crmCouponServicePlus.get(request);
}
```

这里会从当前登录用户中取 `mid`，再按 `lid` 查询 `crm_coupon`。

## 6. `enableDirectTake` 控制领取还是购买

`enableDirectTake` 读取的是券模板表：

```text
crm_coupon.enable_direct_take
```

对应后端实体字段：

```java
CrmCoupon.enableDirectTake
```

对应后台表单字段：

```text
是否可以直接领用
```

领券页根据它决定按钮文案和动作：

```ts
enableDirectTake ? '立即领取' : '立即购买'
```

含义：

- `enableDirectTake=true`：点击按钮后直接走发券接口。
- `enableDirectTake=false`：点击按钮后走购买券/支付流程，支付成功后再发券。

## 7. 直接领取如何发券

当前用户点击“立即领取”后，前端调用：

```ts
takeCoupon({ couponLid })
```

请求接口：

```text
POST /scrm/crm_card_op/customer/take_coupon
```

后端入口：

```text
D:\mywork\nms4cloud\nms4cloud-app\2_business\nms4cloud-crm\nms4cloud-crm-app\src\main\java\com\nms4cloud\crm\app\controller\card\CardOpForCustomerController.java
```

核心代码：

```java
crmCouponOpServicePlus.couponOpAdd(
    new CouponOpAddDTO()
        .setMid(mid)
        .setCouponList(
            List.of(
                new CouponIssuanceDTO()
                    .setCouponCode(request.getCouponLid())
                    .setCouponNum(1)))
        .setLid(cardLid)
        .setOrderId(IdWorkerPlus.getIdStr())
        .setOperator("微餐厅领取")
        .setCouponChannel(CouponChannelEnum.SelfTake));
```

关键点：

- `couponLid` 来自二维码里的 `code`。
- `cardLid` 来自当前登录会员。
- 每次领取数量固定为 `1`。
- 发券渠道是 `CouponChannelEnum.SelfTake`。

## 8. 后端发券校验

统一发券服务：

```text
D:\mywork\nms4cloud\nms4cloud-app\2_business\nms4cloud-crm\nms4cloud-crm-service\src\main\java\com\nms4cloud\crm\service\card\CrmCouponOpServicePlus.java
```

`couponOpAdd()` 会做以下校验：

| 校验项 | 说明 |
|---|---|
| `mid` 不能为空 | 必须有商户上下文 |
| `cardLid` 不能为空 | 必须有会员卡 |
| 会员手机号不能为空 | 用于记录领券人 |
| 券必须存在 | 按 `mid + couponLid` 查 `crm_coupon` |
| `enableDirectTake=true` | 自助领取场景必须允许直接领用 |
| `isReview=true` | 券必须已审核 |
| `isTerminator=false/null` | 券不能被停用 |
| 当前时间在领取时间内 | 校验 `beginReceptionTime`、`endReceptionTime` |
| 库存足够 | 校验并扣减 `received_number` |
| 未超过领取限制 | 校验每日/总领取次数 |

如果任一校验失败，接口返回错误，前端只弹提示，不会写入券记录。

## 9. 发券落库

发券时会先扣减券模板库存：

```sql
received_number = ifNull(received_number, limit_number) - 1
```

然后插入会员券记录表：

```text
crm_coupon_order
```

普通券会生成一条 `CrmCouponOrder`，关键字段包括：

| 字段 | 值 |
|---|---|
| `company_id` | 券模板 `mid` |
| `shop_id` | 券模板 `sid` |
| `lmnid` | 新生成的会员券记录 ID |
| `coupon` | 券名称 |
| `coupon_code` | 券模板 `crm_coupon.lmnid` |
| `get_date` | 领取时间 |
| `valid_date` | 生效时间 |
| `deadline` | 失效时间 |
| `coupon_status` | `WHX`，待核销 |
| `get_channel` | `SelfTake` |
| `order_bill_id` | 本次领取生成的业务单号 |
| `openid` | 会员微信 openid |
| `unionid` | 当前代码中写入会员手机号 |
| `member_code` | 会员编号 |

如果领取的是券包，则会按券包明细展开生成子券记录，并记录 `pkg_coupon`、`pkg_coupon_code`。

## 10. 购买券分支

如果 `enableDirectTake=false`，前端按钮显示为“立即购买”，调用：

```text
POST /scrm/crm_coupon/purchase
```

购买链路会创建购买记录和支付单。支付成功后，后端再调用同一套 `couponOpAdd()` 发券，只是渠道变成购买相关渠道。

## 11. 常见配置错误

### 11.1 扫码后没有打开小程序

优先检查微信后台规则：

```text
二维码地址是否匹配实际链接前缀
小程序路径是否存在
规则是否已发布
测试范围是否包含当前版本
```

### 11.2 打开小程序但没有进入领券页

优先检查：

```text
URL 是否带 flag=coupon
URL 是否带 code
小程序路径是否进入 pagesIndex/pageIndex/index
```

如果配置到了 `pagePop/couponcollection/index`，当前代码不会解析 `q`，通常不能正确进入券详情。

### 11.3 公版小程序无法定位商户

公版链接必须携带 `mid`：

```text
https://wx.lmnsaas.com/nmscommon?mid=yszx&code=券lid&flag=coupon
```

### 11.4 进入领券页但领取失败

检查券模板配置：

```text
是否可以直接领用 = 是
审核状态 = 已审核
启用状态 = 已启用
当前时间在领取时间范围内
剩余库存 > 0
未超过单人每日/总领取限制
用户已登录并有会员卡
会员手机号不为空
```

## 12. 端到端链路摘要

```text
微信后台配置普通链接二维码规则
  -> 用户扫码 https://wx.lmnsaas.com/yszx?code=券lid&flag=coupon
  -> 微信命中规则，拉起小程序 pagesIndex/pageIndex/index
  -> 小程序入口从 q 中解析 code、flag
  -> flag=coupon，跳转 pagePop/couponcollection/index?coupon_lid=券lid
  -> 领券页查询 /scrm/crm_coupon/get_purchase_detail
  -> 用户点击立即领取
  -> 前端调用 /scrm/crm_card_op/customer/take_coupon
  -> 后端 couponOpAdd 校验券、会员、库存、领取限制
  -> 扣减 crm_coupon.received_number
  -> 插入 crm_coupon_order
  -> 会员券状态为 WHX，等待核销
```
