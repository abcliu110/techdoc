# 消费返现配置字段说明

日期：2026-05-27  
范围：CRM 消费返现配置页，包括新增、修改消费返现规则时页面字段的业务含义。  
目标：说明页面上每个字段的用途、启用条件和配置注意点，避免配置人员把规则范围、返现计算方式或适用门店理解错误。

## 1. 结论

“消费返现”配置页用于维护某类会员在指定时间、指定门店、指定消费金额区间内可以获得的返现规则。

一条返现规则通常由以下几类条件共同决定：

1. 活动有效期：活动开始时间、活动结束时间。
2. 会员范围：会员类型、会员等级。
3. 消费门槛：最小消费金额、最大消费金额。
4. 返现算法：返现计算类型、返现金额、返现金额比例、最大返现金额。
5. 使用限制：延期使用天数、适用门店、选择门店。

只有订单或消费事实同时满足这些条件时，才应命中该返现规则。

## 2. 字段说明

| 字段 | 是否必填 | 含义 | 配置注意点 |
|---|---|---|---|
| 会员卡等级和返现 | 是 | 返现规则名称或规则标识，用于区分不同会员类型、会员等级下的返现配置。 | 建议使用能体现业务范围的名称，例如“VIP满100返10”或“测试会员消费返现”。 |
| 活动开始时间 | 是 | 返现规则开始生效的日期。 | 消费时间早于该日期时，不应命中规则。 |
| 活动结束时间 | 是 | 返现规则结束生效的日期。 | 消费时间晚于该日期时，不应命中规则；结束时间应大于等于开始时间。 |
| 会员类型 | 是 | 规则适用的会员分类。 | 只有该会员类型下的会员才参与返现。 |
| 会员等级 | 是 | 规则适用的会员等级，例如 VIP、普通会员等。 | 需要和会员当前等级匹配；不同等级应配置不同规则。 |
| 最小消费金额 | 否 | 参与返现的最低消费金额门槛。 | 消费金额低于该值时不返现；如果为空或为 0，需要以后端实际校验逻辑为准。 |
| 最大消费金额 | 否 | 参与返现的最高消费金额限制。 | 消费金额高于该值时不返现；应不小于最小消费金额。 |
| 返现计算类型 | 否 | 决定返现按固定金额还是按消费比例计算。 | 常见值包括“按金额”和“按比例”。不同类型启用的金额字段不同。 |
| 返现金额 | 否 | 固定返现金额。 | 当返现计算类型为“按金额”时使用；例如消费满足条件后固定返 10 元。 |
| 返现金额比例 | 否 | 按消费金额计算返现的百分比。 | 当返现计算类型为“按比例”时使用；页面单位为 `%`。 |
| 最大返现金额 | 否 | 按比例返现时的封顶金额。 | 用于限制最高返现金额，避免大额消费导致返现金额无限放大。 |
| 延期使用天数 | 否 | 返现到账后延迟多少天可以使用，或返现权益延迟多少天生效。 | 配置为 0 通常表示立即可用；具体生效时间以后端入账和余额可用规则为准。 |
| 适用门店 | 是 | 控制返现规则适用于所有门店还是指定门店。 | 选择“所有门店”时不需要再指定门店；选择“选择门店”时必须配置门店列表。 |
| 选择门店 | 条件必填 | 指定参与返现活动的门店。 | 仅当“适用门店”为“选择门店”时生效；未选中的门店不应命中该规则。 |

## 3. 返现计算类型

### 3.1 按金额

按金额表示消费满足规则条件后，直接返固定金额。

示例：

- 最小消费金额：100
- 最大消费金额：9999
- 返现计算类型：按金额
- 返现金额：10

含义：会员在活动期内消费金额达到 100 且不超过 9999 时，固定返现 10。

配置注意点：

1. `返现金额` 应大于 0。
2. `返现金额比例` 通常不参与计算。
3. `最大返现金额` 对固定金额返现通常没有实际意义，除非后端另有兼容逻辑。

### 3.2 按比例

按比例表示根据消费金额乘以返现比例计算返现金额。

示例：

- 消费金额：200
- 返现金额比例：5%
- 最大返现金额：20

计算结果：200 * 5% = 10，未超过最大返现金额，因此返现 10。

如果消费金额为 1000，按 5% 计算为 50，但最大返现金额为 20，则最终返现应封顶为 20。

配置注意点：

1. `返现金额比例` 应大于 0。
2. `最大返现金额` 建议配置为大于 0 的封顶值。
3. `返现金额` 通常不参与比例计算。

## 4. 门店适用范围

### 4.1 所有门店

当“适用门店”选择“所有门店”时，规则对全部门店生效。

适用场景：

1. 全国或全部直营网点统一活动。
2. 不区分门店的会员返现策略。
3. 门店范围不需要单独控制的测试规则。

### 4.2 选择门店

当“适用门店”选择“选择门店”时，规则只对已选择的门店生效。

适用场景：

1. 单店活动。
2. 区域门店活动。
3. 灰度试点活动。

配置注意点：

1. 必须至少选择一个门店。
2. 后续新增门店不会自动加入该规则，除非业务另有同步机制。
3. 如果消费发生门店不在选择列表中，不应命中该返现规则。

## 6. 配置检查清单

保存返现规则前建议检查：

1. 活动开始时间和结束时间是否正确，结束时间是否早于开始时间。
2. 会员类型、会员等级是否是本次活动真正要覆盖的人群。
3. 最小消费金额是否小于等于最大消费金额。
4. 返现计算类型是否和填写的返现金额字段一致。
5. 按金额返现时，返现金额是否大于 0。
6. 按比例返现时，返现金额比例和最大返现金额是否合理。
7. 选择门店模式下，门店列表是否完整。
8. 延期使用天数是否符合活动规则和财务口径。

## 7. 风险提示

1. 最小消费金额和最大消费金额配置成相同值时，只有刚好等于该金额的消费才可能命中。
2. 返现金额、返现金额比例、最大返现金额都配置为 0 时，规则可能只是“可命中但无实际返现”。
3. 活动时间、会员范围、门店范围任何一个条件不匹配，都会导致规则不生效。
4. 如果系统存在多条重叠规则，需要确认后端是取第一条、取最大优惠、叠加返现，还是按优先级处理。
5. 页面字段含义以配置表单为准，实际是否生效仍取决于后端规则匹配、返现任务生成和余额入账链路。
