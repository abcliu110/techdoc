# 会员身份录入与落库位置

## 1. 结论

POS 账单只要录入了会员，会员身份的主记录位置是 `dwd_bill`。

会员打折和会员储值消费的区别是：

- 会员打折：会员身份主要看 `dwd_bill.card_no`，折扣信息看 `dwd_bill.discount_*` 和 `dwd_food.discount_*`。
- 会员储值消费：会员身份同时记录在 `dwd_bill` 和 `dwd_pay`，其中 `dwd_pay.type_ = HYK` 表示会员卡支付。

积分逻辑判断“此账单是否有会员”，当前主要使用 `memberCheckVO`；如果结账链路没有传入 `memberCheckVO`，则从 `dwd_bill.card_no` 兜底加载会员。

## 2. 会员打折时的会员身份记录

会员打折不是靠会员卡支付判断会员身份，而是靠账单是否已经录入会员。

主要落库表：`dwd_bill`

关键字段：

| 表 | 字段 | 含义 |
| --- | --- | --- |
| `dwd_bill` | `card_no` | 会员卡号，积分逻辑兜底加载会员的核心字段 |
| `dwd_bill` | `card_lid` | 会员卡 LID |
| `dwd_bill` | `card_type_lid` | 会员类型/会员方案 LID |
| `dwd_bill` | `card_type` | 会员类型名称 |
| `dwd_bill` | `card_level` | 会员等级名称 |
| `dwd_bill` | `member_name` | 会员姓名 |
| `dwd_bill` | `member_type_name` | 会员类型名称 |
| `dwd_bill` | `phone` | 会员手机号 |

会员打折相关字段：

| 表 | 字段 | 含义 |
| --- | --- | --- |
| `dwd_bill` | `is_vip_price` | 是否使用会员价 |
| `dwd_bill` | `discount_amount` | 账单折扣金额汇总 |
| `dwd_bill` | `discount_rate` | 账单折扣率 |
| `dwd_bill` | `discount_range` | 折扣名称/折扣范围 |
| `dwd_bill` | `discount_by` | 折扣来源/操作说明 |
| `dwd_bill` | `discount_for` | 折扣原因 |
| `dwd_bill` | `discount_by_man` | 是否手动打折 |
| `dwd_bill` | `discount_at` | 打折时间 |
| `dwd_bill` | `discount_lid` | 折扣规则 LID |

菜品级折扣记录在 `dwd_food`：

| 表 | 字段 | 含义 |
| --- | --- | --- |
| `dwd_food` | `discount_amount` | 菜品折扣金额 |
| `dwd_food` | `discount_rate` | 菜品折扣率 |
| `dwd_food` | `food_discount_rate` | 菜品会员价/折扣率口径 |
| `dwd_food` | `discount_range` | 折扣名称/范围 |
| `dwd_food` | `discount_by` | 折扣来源 |
| `dwd_food` | `discount_for` | 折扣原因 |
| `dwd_food` | `discount_at` | 打折时间 |
| `dwd_food` | `discount_by_man` | 是否手动打折 |
| `dwd_food` | `discount_by_auto` | 是否自动打折 |
| `dwd_food` | `paid_amount` | 折扣后的菜品实付金额 |

判断一笔账单是否是会员打折，可以优先看：

```sql
dwd_bill.card_no IS NOT NULL
AND (
  dwd_bill.is_vip_price = 1
  OR dwd_bill.discount_by = '智简会员权益折扣'
  OR dwd_bill.discount_for LIKE '%会员权益%'
  OR dwd_bill.discount_rate IS NOT NULL
  OR dwd_bill.discount_amount > 0
)
```

判断一笔账单是否有会员储值消费，可以看：

```sql
EXISTS (
  SELECT 1
  FROM dwd_pay p
  WHERE p.mid = dwd_bill.mid
    AND p.saas_order_no = dwd_bill.lid
    AND p.type_ = 'HYK'
    AND p.card_no IS NOT NULL
)
```

## 4. 积分逻辑如何识别会员

POS 本地积分逻辑在结账后会先尝试拿 `memberCheckVO`。

如果调用方已经传入 `memberCheckVO`，直接认为当前账单有关联会员。

如果没有传入，则读取：

```text
dwd_bill.card_no
```

然后通过会员查询接口重新加载会员信息。

因此，会员打折但未使用储值卡消费时，只要 `dwd_bill.card_no` 已经落库，当前账单仍然可以被积分逻辑识别为会员账单。

如果会员打折没有写入 `dwd_bill.card_no`，积分逻辑无法仅凭折扣字段识别会员身份。
