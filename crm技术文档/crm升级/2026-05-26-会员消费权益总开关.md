# 会员消费权益总开关升级说明

## 背景

POS 近期新增了三类会员消费权益：

- 消费积分
- 消费赠券
- 消费返现

这些权益都会在门店结账、退款撤销、退款重算和自动补偿链路中触发。为了避免升级后门店在未确认业务策略时自动发放权益，本次新增一个统一配置开关，由门店配置决定是否执行上述功能。

## 配置项

| 项目 | 内容 |
| --- | --- |
| 配置 key | `g_crmConsumeBenefitEnabled` |
| 中文名 | 启用会员消费权益 |
| 参数定义表 | `sys_config_type` |
| 门店配置值表 | `sys_config_data` |
| 作用范围 | 单商户、单门店 |
| 值字段 | `boolean_val` |
| 默认值 | `false` |

建议后台展示名称使用“启用会员消费权益”。如果需要更直观，也可以展示为“启用会员消费积分/赠券/返现”。

## 生效规则

该开关只认显式 `true`：

- `boolean_val = 1`：执行消费积分、消费赠券、消费返现。
- `boolean_val = 0`：不执行上述三类权益。
- 配置不存在：按 `false` 处理，不执行。
- 配置值为空：按 `false` 处理，不执行。
- 定时补偿任务拿不到门店上下文：按 `false` 处理，不执行。

## POS 使用方式

POS 通过 `SycConfigDataService` 读取门店配置：

```java
sycConfigDataService.get(mid, sid, PosConfigKey.g_crmConsumeBenefitEnabled, false)
```

第四个参数 `false` 是缺省值，保证配置缺失时不会自动开启。

开关判断放在三个本地权益服务内部：

- `CrmPointsEarnLocalService`：消费积分
- `CrmConsumeCouponLocalService`：消费赠券
- `CrmConsumeCashLocalService`：消费返现

这样可以覆盖结账、退款撤销、退款重算和自动补偿入口，调用方不需要重复判断。

## 后台系统参数选项

后台路径“商户中心 -> 基础资料 -> 运营资料 -> 系统参数”展示的是 `sys_config_type` 里的参数定义。POS 运行时读取的是 `sys_config_data` 里的门店参数值。

因此，仅新增 POS 代码常量还不够。如果页面搜索 `g_crmConsumeBenefitEnabled` 没有数据，需要先执行参数定义脚本：

```text
D:\mywork\techdoc\crm技术文档\crm升级\脚本修改\nms4cloud\V20260526__crm_consume_benefit_sys_config_type.sql
```

脚本行为：

- 在 `sys_config_type` 注册参数定义，中文名为“启用会员消费权益”。
- 参数类型为布尔值，`type = 1`。
- 默认值为 `defVal = 'false'`。
- 所属模块从 `CONFIG_TYPE_MODULE` 字典下的“会员模块”动态读取。
- 不会写入任何门店的 `sys_config_data` 启用值。

执行后，在会员模块下搜索 `g_crmConsumeBenefitEnabled` 应能看到该选项。门店是否启用仍需要在系统参数设置中显式保存。

## 门店启用脚本

不需要内置默认关闭的门店值脚本。原因是代码已经把配置缺失视为 `false`，升级后未配置门店不会自动执行会员消费权益。

如果某个门店需要启用，需要插入或更新配置：

```sql
INSERT INTO sys_config_data (
  company_id,
  shop_id,
  name,
  lmnid,
  boolean_val
) VALUES (
  /* 商户ID */ 0,
  /* 门店ID */ 0,
  'g_crmConsumeBenefitEnabled',
  0,
  1
);
```

如果门店已经存在该配置，直接更新：

```sql
UPDATE sys_config_data
SET boolean_val = 1
WHERE company_id = /* 商户ID */
  AND shop_id = /* 门店ID */
  AND name = 'g_crmConsumeBenefitEnabled';
```

关闭配置：

```sql
UPDATE sys_config_data
SET boolean_val = 0
WHERE company_id = /* 商户ID */
  AND shop_id = /* 门店ID */
  AND name = 'g_crmConsumeBenefitEnabled';
```

## 上线验证

1. 未配置 `g_crmConsumeBenefitEnabled` 的门店，结账后不应触发消费积分、消费赠券、消费返现。
2. 配置 `boolean_val = 0` 的门店，结账和退款链路都不应触发上述权益。
3. 配置 `boolean_val = 1` 的门店，按原消费权益逻辑执行。
4. 退款撤销、退款重算、自动补偿任务均受同一个开关控制。

## 注意事项

- 这是会员消费权益总开关，不控制储值、会员价、会员折扣、已有会员支付等其他能力。
- 不建议上线时批量默认开启。应按门店确认后显式配置 `boolean_val = 1`。
- 如果后续后台需要配置界面，字段类型应使用布尔开关，默认值显示为关闭。
