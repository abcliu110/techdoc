## 4. 报表逐项说明

## 4.1 会员概览

### 报表名称

- `会员概览`
- 接口：`/boss/member_analyse`

### 取数方式

1. 从 `crm_member` 查询时间区间内新增会员。
2. 从 `crm_card_record` 查询这些新会员在区间内的消费流水。
3. 从 `crm_card_record` 查询全部会员在区间内的充值和消费流水。
4. 从 `crm_card` 汇总当前余额。

### 涉及表

- `crm_member`
- `crm_card_record`
- `crm_card`

### 展示字段与计算逻辑

| 字段 | 计算逻辑 |
| --- | --- |
| `newMemberNum` | `crm_member` 中 `join_time between beginTime and endTime` 的会员数量 |
| `newConsumeTimes` | 新会员在 `crm_card_record` 中 `operation_model in ('Consume','Refund','CancelOrder')` 的消费次数汇总 |
| `newConsumeAmount` | 新会员消费流水 `sum_of_consume` |
| `newConsumeGiveAmount` | 新会员消费赠送金额汇总 |
| `newConsumePrincipalAmount` | 新会员消费本金汇总 |
| `allChargeTimes` | 全部会员 `Charge/ChargeCancel` 流水次数汇总 |
| `allCharge` | 全部会员充值金额汇总 |
| `allGiveCharge` | 全部会员充值赠送金额汇总 |
| `allPrincipalCharge` | 全部会员充值本金汇总 |
| `allConsumeTimes` | 全部会员 `Consume/Refund/CancelOrder` 次数汇总 |
| `allConsumeAmount` | 全部会员消费金额汇总 |
| `allConsumeGiveAmount` | 全部会员消费赠送金额汇总 |
| `allConsumePrincipalAmount` | 全部会员消费本金汇总 |
| `newRegisterNum` | 直接等于 `newMemberNum` |
| `memberNumSum` | `crm_card` 账户数量 `COUNT(1)` |
| `surplusBalance` | `SUM(balance)` |
| `surplusGiveBalance` | `SUM(give_balance)` |
| `surplusPrincipalBalance` | `SUM(principal_balance)` |

### 过滤条件

- `company_id = mid`
- 可选 `shop_id in sidList`
- 会员新增统计使用 `join_time between beginTime and endTime`
- 流水使用 `if_deal_success = 1`
- 消费流水类型：
  - `Consume`
  - `Refund`
  - `CancelOrder`
- 充值流水类型：
  - `Charge`
  - `ChargeCancel`

## 4.6 会员排行

### 报表名称

- `会员排行`
- 接口：`/boss/member_rank`

### 取数方式

按不同排行类型分别从 `crm_card_record` 或 `crm_card` 聚合。

### 涉及表

- `crm_card_record`
- `crm_card`

### 展示字段

| 字段 | 含义 |
| --- | --- |
| `memberName` | 会员姓名 |
| `cardId` | 会员卡号 |
| `phone` | 手机号 |
| `amount` | 排行值 |

### 计算逻辑

#### 按消费金额排行

- 来源：`crm_card_record`
- 分组：`member_name, card_id_alias, phone`
- 计算：`SUM(amount)` 或按既有逻辑取反求和排序

#### 按充值金额排行

- 来源：`crm_card_record`
- 分组：`member_name, card_id_alias, phone`
- 计算：`SUM(principal_amount)`

#### 按余额排行

- 来源：`crm_card`
- 计算：直接按 `balance` 排序

#### 按消费次数排行

- 来源：`crm_card_record`
- 计算：`COUNT(IF(amount>=0,1.0,-1.0))`

### 过滤条件

- 公司、门店、时间范围
- 消费类类型：
  - `Consume`
  - `Refund`
  - `CancelOrder`
- 充值类类型：
  - `Charge`
  - `ChargeCancel`

## 5. 说明与注意事项

### 5.1 需要重点确认的实现细节

1. `会员赠券记录` 当前将 `id` 直接作为“赠券数量”展示，代码中未见真正数量聚合逻辑，业务语义需要人工确认。
2. `会员消费次数排行` 使用 `COUNT(IF(amount>=0,1.0,-1.0))`，其定义更接近“满足条件的流水笔数”，不是严格意义上的业务消费单数。
3. `会员消费统计` 中 `newCardCount` 当前与 `newMemberCount` 逻辑一致，若业务上“开卡”和“入会”不是同义，需要后续核对。
4. 余额类报表的“期初”大多为“期末快照 + 区间流水反推”结果，不是单独查询一张期初表。

### 5.2 建议后续补充

若需要进一步落地为测试、排错、字段核对文档，建议继续补充：

- 每个报表的代码文件路径
- 每个报表对应 SQL 片段
- 每个字段的前端列名与后端字段名映射
- 每个 `operation_model` 的完整业务含义说明
