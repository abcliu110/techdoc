# crm_coupon_order 时间字段含义说明

日期：2026-05-14

## 1. 文档范围

本文档只说明 `crm_coupon_order` 表中的时间字段含义，依据 CRM 代码中的实体注释、发券逻辑、核销逻辑、校验逻辑和报表展示字段整理。

## 2. 字段总览

| 字段名 | 中文含义 | 业务用途 |
| --- | --- | --- |
| `time_of_appointment` | 预约时间 | 预约自提场景下的预约时间 |
| `store_of_appointment` | 预约门店 | 设计元数据如此命名，但代码类型写成了 `LocalDateTime`，存在字段建模异常 |
| `deadline` | 有效期 / 过期时间 | 判断优惠券是否过期 |
| `get_date` | 领用时间 | 发券时间，也用于按天统计领取次数 |
| `used_time` | 使用卷时间 | 核销时写入，表示实际使用时间 |
| `write_off_time` | 核销时间 | 核销动作发生时间 |
| `abandon_time` | 作废时间 | 撤销或作废时写入 |
| `valid_date` | 开始生效日期 | 优惠券开始可用的日期 |
| `start_valid_time` | 券包子券首次使用时间 | 券包子券的首次可用时间，来自券模板 |

## 3. 逐字段说明

### 3.1 `time_of_appointment`

- 中文含义：预约时间。
- 代码依据：实体注释和 VO / DTO 均标为“预约时间”。
- 业务含义：用于可预约自提的券，在预约场景下记录用户预约的具体时间点。

### 3.2 `store_of_appointment`

- 中文含义：预约门店。
- 代码依据：设计元数据里 `defName` 是“预约门店”。
- 说明：该字段在 Java 实体里声明成了 `LocalDateTime`，和“门店”语义不一致。
- 结论：从命名和设计意图看，它应表达门店信息；但就当前代码形态，不应把它当作正常时间字段理解。

### 3.3 `deadline`

- 中文含义：有效期。
- 业务含义：这张券的失效时间点。
- 写入逻辑：发券时由券模板的 `endUseTime` 计算后写入。
- 使用逻辑：列表、统计、校验都会用它判断是否过期。

### 3.4 `get_date`

- 中文含义：领用时间。
- 业务含义：优惠券实际发放到用户名下的时间。
- 写入逻辑：发券时直接写当前时间。
- 额外用途：代码里用它做“每日领取次数”统计。

### 3.5 `used_time`

- 中文含义：使用卷时间。
- 业务含义：优惠券真正被使用的时间。
- 写入逻辑：核销时与 `write_off_time` 一起写入当前时间。
- 额外用途：用于统计某券在指定周期内的使用次数。

### 3.6 `write_off_time`

- 中文含义：核销时间。
- 业务含义：核销动作发生的时间。
- 写入逻辑：核销接口直接写当前时间。
- 展示逻辑：报表列表列名就是“核销时间”。

### 3.7 `abandon_time`

- 中文含义：作废时间。
- 业务含义：优惠券被撤销、作废的时间。
- 写入逻辑：撤销优惠券时写入当前时间，同时状态改为作废态。

### 3.8 `valid_date`

- 中文含义：领取的优惠券开始使用日期。
- 业务含义：优惠券从哪一天开始生效。
- 写入逻辑：发券时按券模板开始使用日期取当天零点写入。
- 校验逻辑：如果当前时间早于它，就提示“还没到开始生效日期”。

### 3.9 `start_valid_time`

- 中文含义：券包子券首次使用时间。
- 业务含义：券包内子券的首次可用时间。
- 写入逻辑：发券时从券模板复制。
- 校验逻辑：校验优惠券是否到达使用日期时会读取它。

## 4. 代码层结论

`crm_coupon_order` 里真正参与生命周期判断的核心时间字段是：

- `get_date`
- `valid_date`
- `deadline`
- `used_time`
- `write_off_time`
- `abandon_time`

其中：

- `get_date` 表示“什么时候发到用户手里”
- `valid_date` / `start_valid_time` 表示“什么时候开始能用”
- `deadline` 表示“什么时候过期”
- `used_time` / `write_off_time` 表示“什么时候核销”
- `abandon_time` 表示“什么时候作废”

## 5. 注意点

1. `store_of_appointment` 的字段类型与字段语义不一致，属于明显异常项。
2. `used_time` 和 `write_off_time` 在当前代码中通常写成同一时刻。
3. `valid_date` 和 `start_valid_time` 都和“开始可用”相关，但前者偏订单实例，后者偏券包子券首次使用控制。
