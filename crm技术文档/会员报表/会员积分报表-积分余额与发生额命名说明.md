# 会员积分报表-积分余额与发生额命名说明

## 1. 背景

在会员积分、财务报表、积分流水和会员账户展示中，常见的“上次积分、本次积分、总积分”容易产生歧义，不建议作为正式字段名或报表列名直接使用。

主要问题：

- “上次积分”不明确是上一次交易后的积分、当前交易前的积分，还是上一统计周期的积分。
- “本次积分”不明确是本次新增积分、本次扣减积分，还是本次净变动积分。
- “总积分”不明确是当前可用积分、历史累计获得积分，还是累计获得扣除累计消耗后的净值。

积分在财务领域更适合按“账户余额”和“发生额”表达：

- 余额：某个时间点账户中剩余的积分。
- 发生额：某次交易或某个统计周期内发生的积分增减。
- 流水：记录每一次积分变动的明细。

## 2. 推荐核心概念

| 原始说法 | 推荐概念 | 适用场景 | 说明 |
|---|---|---|---|
| 上次积分 | 变动前余额 | 单笔积分流水 | 本次积分变动发生前，会员积分账户中的余额。 |
| 本次积分 | 本次发生额 | 单笔积分流水 | 本次交易导致的积分增加或减少，可正可负。 |
| 总积分 | 变动后余额 | 单笔积分流水 | 本次积分变动完成后，会员积分账户中的余额。 |
| 上次积分 | 期初余额 | 期间报表 | 查询周期开始时的积分余额。 |
| 本次积分 | 本期发生额 | 期间报表 | 查询周期内积分增加和减少的汇总。 |
| 总积分 | 期末余额 | 期间报表 | 查询周期结束时的积分余额。 |

## 3. 单笔积分流水命名

单笔流水用于记录订单赠送积分、积分抵扣、人工调整、活动赠送、积分过期、冲正等一次性变动。

推荐列名：

| 报表列名 | 字段建议 | 含义 |
|---|---|---|
| 变动前余额 | `beforePointBalance` | 本次操作前的积分余额。 |
| 本次发生额 | `pointChange` | 本次增加或减少的积分，增加为正数，扣减为负数。 |
| 变动后余额 | `afterPointBalance` | 本次操作完成后的积分余额。 |

推荐展示示例：

| 业务类型 | 变动前余额 | 本次发生额 | 变动后余额 |
|---|---:|---:|---:|
| 消费赠送积分 | 100 | +20 | 120 |
| 积分抵扣 | 120 | -30 | 90 |
| 人工调整 | 90 | +10 | 100 |

说明：

- “本次发生额”必须带正负方向，避免把增加和扣减混在一起。
- 如果前端或报表不适合展示负数，可以拆成“本次增加积分”和“本次减少积分”两列。
- 单笔流水不建议使用“期初、期末”，因为“期”更适合统计周期，不适合单次交易。

## 4. 期间积分报表命名

期间报表用于按日、月、发生门店、会员、等级、渠道等维度汇总积分。

推荐列名：

| 报表列名 | 字段建议 | 含义 |
|---|---|---|
| 期初积分余额 | `openingPointBalance` | 查询周期开始时的积分余额。 |
| 本期获得积分 | `earnedPoints` | 查询周期内因消费、活动、人工赠送等获得的积分。 |
| 本期消耗积分 | `usedPoints` | 查询周期内因抵扣、兑换等消耗的积分。 |
| 本期调整积分 | `adjustedPoints` | 查询周期内人工调整、冲正、补偿等非正常业务积分。 |
| 本期过期积分 | `expiredPoints` | 查询周期内过期失效的积分。 |
| 期末积分余额 | `closingPointBalance` | 查询周期结束时的积分余额。 |

推荐平衡关系：

```text
期末积分余额 = 期初积分余额 + 本期获得积分 - 本期消耗积分 + 本期调整积分 - 本期过期积分
```

如果系统中“调整积分”可能为正也可能为负，应在字段说明中明确：

```text
本期调整积分为净调整值，正数表示调增，负数表示调减。
```

如果报表更偏财务对账，可以使用“本期增加积分”和“本期减少积分”：

| 报表列名 | 含义 |
|---|---|
| 期初积分余额 | 周期开始时余额。 |
| 本期增加积分 | 周期内所有增加积分的合计。 |
| 本期减少积分 | 周期内所有减少积分的合计。 |
| 期末积分余额 | 周期结束时余额。 |

对应关系：

```text
期末积分余额 = 期初积分余额 + 本期增加积分 - 本期减少积分
```

## 5. “总积分”的拆分建议

“总积分”不建议作为字段名直接使用。需要根据业务含义拆分为以下概念：

| 推荐概念 | 字段建议 | 含义 |
|---|---|---|
| 当前积分余额 | `currentPointBalance` | 当前会员积分账户余额。 |
| 可用积分余额 | `availablePointBalance` | 当前可用于抵扣、兑换的积分。 |
| 冻结积分余额 | `frozenPointBalance` | 已锁定但暂不可用的积分。 |
| 累计获得积分 | `totalEarnedPoints` | 历史累计获得的积分总数。 |
| 累计消耗积分 | `totalUsedPoints` | 历史累计消耗的积分总数。 |
| 累计过期积分 | `totalExpiredPoints` | 历史累计过期失效的积分总数。 |

展示给运营或财务时，优先使用：

```text
当前积分余额
可用积分余额
累计获得积分
累计消耗积分
```

展示给会员时，优先使用：

```text
可用积分
即将过期积分
累计获得积分
```

## 6. 推荐命名方案

### 6.1 单笔流水

推荐使用：

```text
变动前余额
本次发生额
变动后余额
```

对应字段：

```text
beforePointBalance
pointChange
afterPointBalance
```

适合场景：

- 积分流水明细。
- 订单积分明细。
- 人工调整记录。
- 积分冲正记录。

### 6.2 期间报表

推荐使用：

```text
期初积分余额
本期获得积分
本期消耗积分
本期调整积分
本期过期积分
期末积分余额
```

对应字段：

```text
openingPointBalance
earnedPoints
usedPoints
adjustedPoints
expiredPoints
closingPointBalance
```

适合场景：

- 会员积分日报。
- 会员积分月报。
- 门店积分汇总报表。
- 财务积分对账报表。

## 7. 落地规则

1. 报表字段不要使用“上次积分、本次积分、总积分”作为最终命名。
2. 单笔流水必须表达“变动前、变动值、变动后”。
3. 期间报表必须表达“期初、本期发生、期末”。
4. “总积分”必须按业务语义拆成“当前余额、可用余额、累计获得、累计消耗”等明确字段。
5. 涉及积分扣减、抵扣、过期、冲正时，字段说明必须明确正负方向。
6. 如果同一张报表同时提供“余额”和“累计值”，列名必须显式区分，避免把当前余额误解为历史累计。

## 8. 最终建议

CRM 会员积分报表中建议统一采用以下概念：

单笔积分流水：

```text
变动前余额 / 本次发生额 / 变动后余额
```

期间积分报表：

```text
期初余额 / 本期获得积分 / 本期消耗积分 / 本期调整积分 / 本期过期积分 / 期末余额
```

会员账户展示：

```text
当前余额 / 可用余额 / 冻结余额 / 累计获得积分 / 累计消耗积分
```

## 9. 需要调整的全部内容汇总

如果当前页面或报表要按财务口径和业务口径一起修正，建议一次性调整为下面这组内容：

### 9.1 需要修改的现有字段

- `积分类型` 改为 `记账方向` + `业务类型`
  - `记账方向` 只表示积分是入账还是出账，通常由积分正负、账户增减方向或记账规则判断。
  - `业务类型` 对应 `operation_model`，表示这笔积分因什么业务发生，例如充值、消费、退款、开卡、返现、兑换、转入、转出、退卡、提现、回款、反回款、蓝补、红冲、期初。
  - `operation_model` 不是方向字段，同一方向下可以包含多个业务类型。
- `收入 / 支出` 改为 `入账 / 出账`
- `门店名称` 改为 `发生门店`
- `交易日期` 改为 `记账时间`
- `上次积分` 改为 `变动前余额` 或 `期初余额`
- `本次积分` 改为 `本次发生额` 或 `本期发生额`
- `总积分` 改为 `变动后余额`、`期末余额`、`当前余额`，按场景拆开

### 9.2 需要新增的字段

#### 9.2.1 业务字段

- 业务单号
- 关联单号
- 来源渠道
- 操作人 / 审核人
- 变动原因
- 状态：待生效 / 生效 / 冲正 / 失效
- 积分规则版本或活动编号

#### 9.2.2 财务字段

- 期初积分余额
- 本期增加积分
- 本期减少积分
- 本期调整积分
- 本期过期积分
- 期末积分余额
- 冻结积分余额
- 可用积分余额
- 余额校验差额

### 9.3 报表结构拆分

- 积分流水明细报表：看每笔怎么变
- 积分余额报表：看当前账户有多少积分
- 积分汇总报表：看发生门店、日期、活动、会员等级等维度的统计
- 积分对账报表：看期初 + 发生 - 期末 是否平衡

### 9.4 记账表达统一

- 单笔流水必须表达：`变动前余额 / 本次发生额 / 变动后余额`
- 期间报表必须表达：`期初余额 / 本期发生额 / 期末余额`
- 方向和原因必须拆开，不要混成一个“积分类型”
- “总积分”不作为最终字段名，必须拆成明确业务含义

### 9.5 最终建议的一句话版本

如果只保留一版标准口径，建议统一成：

```text
单笔流水：变动前余额 / 本次发生额 / 变动后余额
期间报表：期初余额 / 本期获得积分 / 本期消耗积分 / 本期调整积分 / 本期过期积分 / 期末余额
账户展示：当前余额 / 可用余额 / 冻结余额 / 累计获得积分 / 累计消耗积分
```
