## 进制

在 JavaScript 中，**原生支持**的进制表示法主要有 4 种（二进制、八进制、十进制、十六进制）。此外，通过内置函数可以处理 **2 到 36 进制** 之间的转换。

以下是详细说明：

### 1. 字面量表示法（直接写在代码中）

在 ES6 (ECMAScript 2015) 之后，JavaScript 规范化了各种进制的写法，通常通过**前缀**来区分：

| 进制 | 英文 | 前缀 (大小写均可) | 示例 | 对应十进制值 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **二进制** | Binary | `0b` 或 `0B` | `0b101` | 5 | ES6 新增 |
| **八进制** | Octal | `0o` 或 `0O` | `0o755` | 493 | ES6 新增 (推荐) |
| **十进制** | Decimal | 无 | `123` | 123 | 默认 |
| **十六进制** | Hexadecimal | `0x` 或 `0X` | `0xFF` | 255 | 常用于颜色、字节 |

**⚠️ 关于八进制的特别说明：**
*   **旧式写法（Legacy）：** 在旧版本 JS 中，以 `0` 开头的数字会被视为八进制（例如 `010` 等于 8）。
*   **严格模式（Strict Mode）：** 在严格模式（`'use strict'`）下，禁止使用 `0` 开头的旧式八进制写法，必须使用 `0o` 前缀，否则会报错。

---

### 2. 进制转换方法

虽然字面量只支持上述 4 种，但 JS 提供了方法来处理任意 **2 到 36** 进制的数据。

#### A. 任意进制 转 十进制 (`parseInt`)
使用 `parseInt(string, radix)`，其中 `radix` 范围是 2-36。

```javascript
parseInt("101", 2);   // 5  (二进制转十进制)
parseInt("17", 8);    // 15 (八进制转十进制)
parseInt("F", 16);    // 15 (十六进制转十进制)
parseInt("Z", 36);    // 35 (三十六进制转十进制)
```

#### B. 十进制 转 任意进制 (`toString`)
使用 `number.toString(radix)`，其中 `radix` 范围是 2-36。

```javascript
let num = 255;

num.toString(2);   // "11111111" (转二进制)
num.toString(8);   // "377"      (转八进制)
num.toString(16);  // "ff"       (转十六进制)
num.toString(36);  // "73"       (转三十六进制)
```

### 3. 为什么最高支持到 36 进制？
因为数字（0-9）只有 10 个，字母（a-z）有 26 个。
`10 + 26 = 36`。
所以在 36 进制中，`0-9` 代表数值 0-9，`a` 代表 10，一直到 `z` 代表 35。

### 总结
1.  **写代码时**：原生支持 **2、8、10、16** 进制（使用 `0b`, `0o`, `0x` 前缀）。
2.  **运算/转换时**：支持 **2 到 36** 之间任意进制的转换。
