# 领域定义：企业业务低代码平台

采集日期：2026-07-06

## 领域名称

企业业务低代码平台。

## 目标用户

- 企业 IT 部门、数字化部门、实施顾问、二开团队。
- 需要快速构建业务应用、流程应用、数据后台、集成应用的业务技术团队。

## 核心业务场景

- 业务对象建模：主数据、单据、业务档案、关系对象。
- 页面与视图生成：列表、表单、详情、工作台、移动端页面。
- 权限与组织：角色、组织、数据范围、字段级权限、租户/空间隔离。
- 流程与自动化：审批流、事件触发、状态流转、任务分派、运行监控。
- 集成与扩展：API、连接器、插件、脚本、二开扩展点。
- 发布与治理：版本、迁移、模型资产、合规、审计、可观测。

## 典型业务对象

```text
Tenant / Workspace / App
BusinessObject / Field / Relation
Page / View / Component
Role / Permission / Policy / Organization
Workflow / State / Action / Rule
DataSource / Connector / Plugin / Version
```

## 核心痛点

- 只从 UI Builder 起步容易缺失业务对象生命周期、权限、流程和版本治理。
- 只学商业闭源竞品会缺少源码机制证据。
- 只学开源项目 README 会把宣传能力误写成成熟能力。
- 企业级业务低代码需要同时处理“灵活配置”和“模型治理”的张力。

## 主要竞品

- 企业级业务低代码：金蝶 AI 苍穹 / 云苍穹。
- ERP 型元数据框架：Frappe。
- 数据模型驱动低代码：NocoBase、Directus、NocoDB。
- 内部工具低代码：Appsmith、ToolJet、Budibase、Lowcoder。

## 研究范围

- 领域模型、页面模型、权限模型、流程模型、集成模型、扩展模型、发布治理、非功能性设计。
- 金蝶等闭源竞品只采集合法公开资料，不推断源码结构为事实。
- GitHub 开源项目必须逐步从 README/官网推进到源码级证据。

## 不研究范围

- 不反编译、不绕过登录、不采集非公开资料。
- 不复制竞品 UI 表达、代码实现、品牌资产。
- 不把单个竞品的营销表述直接转成产品需求。

## 预期用于生成什么产品

用于支撑自研企业业务低代码平台的产品蓝图、模块 PRD、领域模型、权限/流程/页面/集成设计、ADR、测试规格和实施路线图。
