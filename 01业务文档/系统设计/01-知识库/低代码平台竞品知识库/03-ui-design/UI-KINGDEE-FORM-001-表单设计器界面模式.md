## 与通用表单设计器的差异

金蝶表单设计器更接近“业务对象设计器 + 表单/单据设计器 + 发布部署工具链”的组合，而不是 Jotform/Typeform 那类独立表单工具。它的 UI 设计中心不是填写者体验，而是企业业务对象、单据状态、权限、转换、流程和部署的一致性。

## 对自研平台的启发

1. 表单设计器要围绕业务对象入口，而不是从空白画布开始。
2. 字段拖放后必须立刻进入属性、权限、状态、编码规则和校验体系。
3. 表单设计器要内置元数据校验输出区，避免错误直到运行时才暴露。
4. 同一表单多个页面布局应作为一等能力，支持 PC、移动端、查看/编辑/审批等不同视图。
5. 表单发布必须和菜单、权限、流程、部署包或应用版本关联。
