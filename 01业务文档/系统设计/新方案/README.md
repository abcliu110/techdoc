# 新方案目录说明

本目录按"知识库 → 产品方案 → 归档"组织。跨领域方法论与流程已迁移到同级 `../00-方法论与流程/`。

> 全局索引见 `../README.md`（系统设计文档体系全景，含方法论与工程规范的桥梁关系和全局术语对照）。
> 工程规范见 `../工程规范/README.md`（含 AI Agent 加载协议、8 条规范治理原则和 AI 执行检查清单）。

## 目录结构

```text
01-知识库/
02-产品方案/
99-归档/
```

## 目录职责

| 目录 | 职责 |
|---|---|
| `01-知识库/` | 按领域沉淀竞品证据、截图拆解、源码分析、能力矩阵、缺口清单 |
| `02-产品方案/` | 具体产品的 PRD、架构、详细设计、规范、任务卡、源码仓库 |
| `99-归档/` | 已废弃、历史版本或暂不使用的材料 |

## 使用规则

1. 新建领域知识库，放入 `01-知识库/`。
2. 新建产品方案，放入 `02-产品方案/`。
3. 方法论和通用流程放入同级 `../00-方法论与流程/`，不要混入具体产品目录。
4. 知识库是证据资产，产品方案是设计和交付资产；两者通过引用关联，不互相混放。
5. 移动或重命名目录后，应同步更新相关文档中的路径引用。
