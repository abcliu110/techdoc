## 1. 追溯口径

本表用于满足《02-流程-知识库驱动新产品创建与迭代.md》v0.5 的 PRD 门禁：

```text
蓝图目标 -> 用户故事 -> 功能/非功能需求
```

状态口径：

| 状态 | 含义 |
|---|---|
| traced | 已能从蓝图追溯到用户故事，再追溯到 PRD 需求 |
| partial | 有追溯关系，但下游任务/测试仍需在 CapabilityTraceMatrix 中细化 |
| planned | 后续里程碑能力，不作为 M0/M1 must |
| review-needed | 需要人工或 ADR 终审 |

## 4. 孤儿需求与悬空故事检查

| 检查项 | 结果 | 说明 |
|---|---|---|
| P0/P1 功能需求是否均有 storyId | 通过 | M0/M1 主线需求已映射到 US-MODEL、US-RUNTIME、US-PERM、US-WORKFLOW、US-RULE |
| 非功能需求是否均有上游故事 | 通过 | 每条 NFR 均映射到至少一个用户故事 |
| 是否存在悬空用户故事 | 未发现 | 每个用户故事至少映射一组 PRD 需求 |
| 是否存在孤儿 P0/P1 需求 | 未发现 | P0/P1 主线需求已有上游故事；M3/M4 planned 能力保留 planned 状态 |
| 是否可判定 covered | 否 | covered 需继续由 `../06-任务与测试/CapabilityTraceMatrix.md` 精确核验设计、任务和测试引用 |

## 5. PRD 门禁结论

```text
gate: pass-for-review
reason:
  用户需求文档已存在；
  P0/P1 功能需求可追溯到用户故事；
  用户故事可追溯到产品蓝图；
  非功能需求有上游故事和验收方向；
  未验证竞品能力仍保留为 review-needed / planned，不直接作为不可逆实现依据。
remaining:
  CapabilityTraceMatrix 仍需继续精确核验 covered/partial/missing；
  decisionOwner 未指定，不能替代人工完成最终范围和风险接受确认。
```
