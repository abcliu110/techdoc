# 低代码平台 · 用户需求：用例与用户故事

## 1. 用户角色清单

| roleId | 用户角色 | 目标 | 关联蓝图字段 |
|---|---|---|---|
| ROLE-ADMIN | 平台管理员 | 管理租户、版本、插件、审计和运维安全 | roles、permissionBoundary |
| ROLE-TENANT-ADMIN | 租户管理员 | 管理本租户应用、成员、角色和发布 | roles、mainProcesses |
| ROLE-DESIGNER | 应用设计者 | 通过建模和配置交付业务应用 | roles、coreObjects、mainProcesses |
| ROLE-RUNTIME-USER | 业务用户 | 在已发布应用中处理业务数据和流程 | roles、mainProcesses |
| ROLE-PLUGIN-DEV | 插件开发者 | 扩展字段、动作、校验、连接器和页面块 | coreObjects、nonGoals |
| ROLE-OPS-AUDITOR | 运维/审计人员 | 排障、审计、监控、恢复和合规检查 | metrics、permissionBoundary |

## 2. 用户故事

| storyId | 角色 | 场景 | 用户故事 | 前置条件 | 主流程 | 异常流程 | 验收准则 | 蓝图追溯 |
|---|---|---|---|---|---|---|---|---|
| US-MODEL-001 | 应用设计者 | 创建业务对象 | 作为应用设计者，我要创建客户、订单等业务对象并定义字段、关系和状态，以便不用写代码就能表达业务模型。 | 租户和应用已创建 | 新建对象 -> 添加字段 -> 配置关系/状态 -> 保存草稿 | 编码重复、字段类型非法、关系目标不存在时阻断保存或发布 | 可创建客户和订单对象；非法字段被拒绝；对象草稿可被后续发布流程读取 | coreObjects、mainProcesses |
| US-MODEL-002 | 应用设计者 | 发布对象模型 | 作为应用设计者，我要把已校验的对象模型发布到运行态，以便生成动态表和运行时元数据。 | 对象草稿校验通过 | 触发布 -> 生成 DDL Plan -> 执行发布 -> 生成 Version snapshot -> 加载 MetaGraph | 危险 DDL、引用缺失、执行失败时进入可恢复状态 | 新建对象能生成物理表；新增字段能生成物理列；重复发布幂等；失败可解释 | stateModel、metrics |
| US-RUNTIME-001 | 业务用户 | 使用运行态数据 | 作为业务用户，我要在已发布应用中新增、查询、编辑客户和订单，以便完成日常业务处理。 | 应用已发布，用户有权限 | 打开列表 -> 新增记录 -> 编辑字段 -> 保存 -> 查询记录 | 非法输入、无权限字段、跨租户访问被拒绝 | CRUD 走 `/api/data/*`；字段类型校验一致；跨租户数据不可见 | mainProcesses、permissionBoundary |
| US-PERM-001 | 租户管理员 | 配置角色和数据权限 | 作为租户管理员，我要配置对象权限、字段权限、数据范围和动作权限，以便业务用户只能访问被授权的数据。 | 应用和角色已存在 | 创建角色 -> 配置对象权限 -> 配置字段权限 -> 配置数据范围 -> 授权用户 | 用户禁用、组织调整、字段脱敏、无权限导出时按规则处理 | 字段权限在 meta/list/get/update/export/audit 中语义一致；双租户隔离测试通过 | permissionBoundary |
| US-WORKFLOW-001 | 应用设计者 | 配置状态和动作 | 作为应用设计者，我要为订单配置草稿、提交、审批、驳回等状态和动作，以便业务流程可被平台统一执行和审计。 | 对象启用状态机 | 定义状态 -> 定义 transition -> 设置角色和条件 -> 发布 -> 运行态触发动作 | 普通 update 直接改状态、重复提交、并发审批时被阻断或幂等处理 | 启用状态机的对象只能通过 transition 改状态；重复请求不重复执行副作用 | coreObjects、stateModel |
| US-RULE-001 | 应用设计者 | 配置规则和表达式 | 作为应用设计者，我要配置必填、带出、校验、规则动作和表达式，以便减少代码开发并保持业务约束一致。 | 对象和字段已定义 | 定义规则 -> 绑定触发点 -> 配置条件/动作 -> 发布 -> 运行时执行 | 表达式循环、超时、越权读取、规则循环时阻断 | 规则链有最大执行轮数；表达式有版本、依赖图、超时和权限泄漏防护 | coreObjects、mainProcesses |
| US-UI-001 | 应用设计者 | 生成和调整页面 | 作为应用设计者，我要自动生成列表、表单、详情页面，并在权限边界内调整布局，以便快速交付业务应用界面。 | 对象已发布或草稿可预览 | 生成默认页面 -> 调整 schema -> 预览 -> 发布 | 页面引用不存在字段、前端隐藏字段试图放权、XSS 内容时阻断 | 页面 schema 只能控制展示布局，不能放宽后端权限和校验；XSS 测试通过 | coreObjects、mainProcesses |
| US-EXT-001 | 插件开发者 | 扩展平台能力 | 作为插件开发者，我要通过受控 SPI 扩展字段类型、动作、通知和 Connector，以便不修改核心代码也能扩展平台。 | 插件机制和扩展点可用 | 注册扩展 -> 校验兼容 -> 安装启用 -> 运行时调用 | 插件禁用、版本不兼容、越权访问动态表时阻断 | 插件不得绕过租户隔离、权限、SQL 白名单、表达式沙箱和出网治理 | nonGoals、permissionBoundary |
| US-OPS-001 | 运维/审计人员 | 监控、审计和恢复 | 作为运维/审计人员，我要查看发布、权限拒绝、运行时错误、审计和恢复记录，以便保障平台稳定运行和合规。 | 监控和审计事件已配置 | 查看指标 -> 定位 traceId -> 执行 Runbook -> 复核审计 | 发布失败、outbox 积压、权限异常、租户注销时按 Runbook 处理 | REST、异步任务、规则、DDL、插件、前端错误可用 traceId 串联 | metrics、permissionBoundary |
| US-PACKAGE-001 | 租户管理员 | 导入导出和迁移应用 | 作为租户管理员，我要通过导出包迁移应用配置，以便在环境之间安全迁移应用。 | 应用已发布，有导出权限 | 生成导出包 -> 安全扫描 -> 导入目标环境 -> 兼容校验 | 包含密钥、跨租户引用、恶意包、版本不兼容时阻断 | 导出包不得含密钥明文和业务数据；导入恶意包被阻断并给出影响报告 | mainProcesses、permissionBoundary |
| US-COMMERCIAL-001 | 平台管理员 | 管理商业化能力 | 作为平台管理员，我要管理 License、功能包、容量额度和私有化授权，以便平台能支持商用交付。 | 商业化模块进入 M3 | 配置功能包 -> 绑定额度 -> 校验授权 -> 记录审计 | 离线授权过期、超限、应急授权、私有化关闭指标上报时按策略处理 | License 表达功能包、容量、离线授权、宽限期、应急授权和审计 | metrics、nonGoals |
