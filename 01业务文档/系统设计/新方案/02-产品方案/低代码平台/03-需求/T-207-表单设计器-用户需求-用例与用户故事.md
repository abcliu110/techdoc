## 2. 复杂度等级判定

| 项 | 判定 |
|---|---|
| complexityLevel | L3 复杂模块 |
| localCriticality | 局部 L4：元模型、发布、权限、插件边界属于平台核心能力 |
| qualityLevel | L3-strict |
| 启用门槛 | 阶段 1 L3 需求矩阵；阶段 2 交互/规则/权限/状态矩阵；阶段 5 测试矩阵 |

触发条件：

| 条件 | 是否触发 | 说明 |
|---|---|---|
| 涉及 3 个以上用户角色 | 是 | 应用设计者、业务建模者、租户管理员、业务用户、插件开发者、运维审计、UI 设计师。 |
| 涉及多运行模式 | 是 | 设计态、预览态、运行态、发布态、权限模拟态。 |
| 涉及权限、审计、发布、回滚、插件、表达式 | 是 | 均为核心链路。 |
| 涉及复杂 UI 工作台 | 是 | 五区高密度设计器、拖拽、属性、规则、大纲、路径。 |
| 涉及跨 3 个以上既有模块 | 是 | BO、Renderer、权限、表达式、发布、插件、AI、审计。 |
| 需要端到端测试 | 是 | 销售订单样例必须贯穿设计、发布、运行。 |

因此，本文不允许只写“支持配置”“支持拖拽”“测试通过”这类空泛需求。每个 P0 需求必须绑定角色、场景、输入、输出、权限/状态影响和验收断言。

## 4. 核心业务对象与术语表

| 术语 | 定义 | 不允许混用 |
|---|---|---|
| BusinessObject / BO | 业务对象，定义字段、子表、关系和业务约束 | 不能等同于页面 schema。 |
| FormDefinition | 动态表单基础定义，绑定 BO、页面类型、规则、权限、插件、版本 | 不能等同于单个页面布局。 |
| PageSchema | 某个视图的布局树和控件引用 | 不能直接修改 BO 实体结构。 |
| PageType | 页面类型，如动态表单、单据、基础资料、列表、详情、过滤框 | 不能只作为前端路由类型。 |
| ViewType | 同一对象下的视图，如表单、列表、详情、移动表单 | 不能替代 PageType。 |
| FieldNode | 绑定 BO 字段的页面节点 | 不能绑定不存在或无权限字段。 |
| ComponentNode | 非字段型展示/容器/动作组件 | 不能绕过组件注册表。 |
| RuleSet | UI 规则和业务规则集合 | 不能只在前端运行。 |
| PermissionOverlay | 页面侧权限覆盖或模拟结果 | 不能放宽后端权限。 |
| PublishSnapshot | 发布版本快照 | 不能在发布后被直接修改。 |
| PluginRegistration | 插件注册声明 | 不能等同于任意脚本。 |

## 6. P0 业务闭环样例：销售订单表单

| 环节 | 输入 | 操作 | 输出/断言 |
|---|---|---|---|
| 业务对象 | 销售订单、客户、订单明细、数量、单价、金额、总金额、审批意见 | 选择销售订单 BO | 表单草稿绑定 bo_sales_order。 |
| 设计布局 | BO 字段、组件注册表 | 拖入客户、日期、明细子表、总金额、审批意见 | PageSchema 只引用 BO 字段，不新增实体字段。 |
| 配属性 | 字段节点 | 客户必填；总金额只读；审批意见条件必填 | 属性进入保存校验。 |
| 配规则 | 数量、单价、金额、总金额 | 配 `数量 * 单价 = 金额`、`SUM(金额) = 总金额` | 规则依赖无环，服务端可执行。 |
| 配权限 | 销售、经理、文员 | 销售编辑自己订单；经理审批；文员金额只读/脱敏 | 权限不能放宽后端字段权限。 |
| 预览 | 草稿 schema、模拟角色 | 切换销售/经理/文员角色 | 隐藏、只读、脱敏原因可见。 |
| 发布 | 草稿、旧版本 | 执行发布校验和影响分析 | P0 blocker 阻断；通过后生成快照。 |
| 运行 | 发布快照、业务数据 | 录入、保存、提交 | 运行态与预览态一致，规则和权限由服务端判定。 |
| 审计 | traceId、versionId | 查询发布和运行日志 | 可定位操作者、版本、规则、插件和错误。 |
