# M1 测试规格

> 版本：v0.1
> 范围：T-101~T-108
> 目标：验证运行时引擎、权限、表达式、状态机、规则、性能、事件内核、复合唯一和身份 SPI 基线。
> ADR 门禁：PRD 的租户隔离、服务端统一判权、可靠副作用、幂等和可恢复边界始终必测；PERM、OUTBOX、EXPR 等候选 ADR 的精确类型、字段、状态和算法，只有在对应 ADR/等价决策记录 accepted 后才进入阻断性验收。未终审项不得用于声明 M1 完成。

---

## 1. 测试分层

| 层级 | 范围 |
|---|---|
| 单元测试 | 表达式、权限合并、字段转换、状态机判定 |
| 集成测试 | 动态 CRUD、MySQL SQL 生成、审计、outbox、Redis metaHash |
| 安全测试 | SQL 注入、表达式逃逸、tenant 绕过、字段越权 |
| 契约测试 | API 请求响应、错误码、字段类型、权限矩阵 |
| 性能测试 | MetaGraph、list、AccessView、transition |

## 2. T-101 表达式测试

| 测试类 | 必测 |
|---|---|
| `ExpressionCompileTest` | 语法正确/错误、函数白名单、变量白名单 |
| `ExpressionSandboxTest` | 反射、类加载、IO、网络、环境变量全部阻断 |
| `ExpressionTimeoutTest` | 死循环或大计算超时 |
| `ExpressionDependencyGraphTest` | 字段依赖、link 一层依赖、循环依赖 |
| `ExpressionPermissionLeakTest` | 无读权限字段不能进入上下文 |
| `ExpressionCompatibilityTest` | expr-v1 回放资产 |

## 3. T-102 动态数据 API 测试

| 测试类 | 必测 |
|---|---|
| `DynamicCrudSmokeTest` | customer/order add/list/get/update/del |
| `TenantIsolationRuntimeTest` | 双租户互不可见 |
| `DynamicSqlWhitelistTest` | field/op/sort/table 注入被拒 |
| `FieldTypeRuntimeConversionTest` | 22 种字段运行时输入转换和非法值拒绝 |
| `RequestMetaHashTest` | 过旧 metaHash 写入被拒 |
| `DataWriteServiceBoundaryTest` | 写路径只能经过 DataWriteService |
| `IdempotencyRuntimeTest` | add/action/importCommit 重复请求幂等 |
| `AuditLogRuntimeTest` | 写审计含 traceId/metaHash/permVersion |

## 7. T-107 业务事件中心测试

| 测试类 | 必测 |
|---|---|
| `PlatformEventPublishTest` | record.created/updated/deleted、state.transitioned、version.published 发布事件 |
| `EventPayloadContractTest` | payload 含 tenantId/appId/objectCode/recordLid/metaHash/traceId 且敏感字段脱敏 |
| `BeforeSubscriptionRollbackTest` | before 同步订阅可阻断事务，阻断时无业务写入 |
| `AfterSubscriptionOutboxTest` | after 异步订阅进入 outbox，重复投递按 eventId 幂等 |
| `RuleEventConsumerTest` | rule 通过 EventSubscription 消费事件，不再直连对象 trigger |

## 8. T-108 复合唯一与 IdentityProvider SPI 测试

| 测试类 | 必测 |
|---|---|
| `CompositeUniqueSchemaTest` | unique_constraints 生成含 tenant_id/app_id/delete_token 的复合唯一索引 |
| `CompositeUniqueRuntimeTest` | 未删除记录重复组合拒绝，软删除后同组合可重建 |
| `CompositeUniqueErrorTest` | 违反唯一约束返回 constraintCode、fieldCodes 和人话提示 |
| `IdentityProviderSpiSerializationTest` | provider 配置、mappingPolicy、syncPolicy DTO 可序列化且密钥字段加密 |
| `IdentityProviderBoundaryTest` | M1 只允许 SPI 预留，不启用三方登录运行时 |

## 9. 贯穿样例

M1 必须通过“客户-订单-审批”的后端运行态验收：

```text
双租户创建客户和订单
销售只能看自己的订单
文员金额只读
经理审批
金额大于 10000 必填审批意见
重复审批幂等
恶意 filters 不能短路 data_scope
```
