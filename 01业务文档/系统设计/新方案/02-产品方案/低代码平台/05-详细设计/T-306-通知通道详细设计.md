# T-306 通知通道详细设计

> 版本：v0.1
> 里程碑：M3
> 适用任务：T-306
> 依据：PRD REQ-042、REQ-083；ADR-LOWCODE-OUTBOX-001
> 决策门禁：通知权限、模板最小化、可靠投递、幂等和可恢复是强制边界；`NotificationOutbox`、投递状态和本地 outbox 结构是 proposed ADR 的候选实现，M3 开工前必须终审 ADR 或形成等价已批准版本化契约。

---

## 1. 目标

实现通知模板、收件人解析、字段白名单、outbox 投递、重试、死信和人工重放。

## 2. 通知对象

```text
NotificationTemplate
NotificationChannel
NotificationRecipientRule
NotificationOutbox
NotificationDeliveryLog
```

## 3. 模板安全

1. 模板只能引用字段白名单。
2. 敏感字段默认不可引用。
3. 收件人必须有对应记录的最小读权限。
4. 模板渲染不得执行任意表达式，只允许变量替换和白名单函数。

## 4. 投递

通知只通过 outbox 异步投递：

```text
PENDING
SENDING
SUCCESS
FAILED_RETRYABLE
DEAD
REPLAYED
```

重试使用指数退避；死信可人工重放。

## 5. 验收

1. 无权限收件人不发送。
2. 敏感字段模板被拒绝。
3. 通道失败进入重试和死信。
4. 死信可人工重放且幂等。
