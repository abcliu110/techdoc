# T-307~T-310 商用业务能力详细设计

> 版本：v0.1
> 里程碑：M3
> 适用任务：T-307、T-308、T-309、T-310
> 依据：PRD REQ-070~078、REQ-079~090；ADR-LOWCODE-OBJECT-EXT-001、ADR-LOWCODE-CONVERSION-001、ADR-LOWCODE-FLEXORG-001、ADR-LOWCODE-APP-PACKAGE-001
> 决策门禁：上述 ADR 当前均为 proposed。PRD 已确认的能力范围和安全/兼容不变量继续有效；本文所有核心对象名、字段、合并/反写算法、存储模型、状态机和 License 模型都是候选，相关任务开工前必须终审对应 ADR 或形成等价已批准版本化决策记录。

---

## 1. 目标

补齐企业业务低代码区别于内部工具构建器的商用能力：标准对象扩展、行业模板、单据转换、反写、引用追踪、弹性域、多组织、复杂编码、报表、打印、菜单、门户、移动端、多语言、应用包、License。

## 2. 标准对象与客户扩展

核心对象：

```text
StandardObject
IndustryTemplate
CustomerExtension
ExtensionMergeReport
UpgradeReplayReport
```

规则：

1. 标准对象、行业模板、客户扩展、插件扩展必须记录来源。
2. 平台升级不能覆盖客户扩展。
3. 合并冲突必须出报告并阻断。
4. 扩展字段、规则、页面必须进入发布流程。

## 3. 单据转换、反写、引用追踪

核心对象：

```text
LinkConfig
Conversion
WriteBack
LinkTrace
OverLimitPolicy
ClosePolicy
```

能力：

```text
源单 -> 目标单
字段映射
分组策略
选单条件
反写数量/金额/状态
超额控制
关闭控制
上查/下查
```

删除、回滚、导入导出、权限裁剪必须保留链路语义。

## 4. 弹性域、多组织、复杂编码

核心对象：

```text
FlexDimension
FlexAttribute
FlexCombination
OrgFunction
OrgRelation
CodeRule
```

规则：

1. 弹性域不得落普通 JSON 大字段。
2. 组织不仅是 org_id，还包含职能和业务关系。
3. 编码规则支持组织、日期、流水、条件段。
4. 报表和权限必须能识别弹性域与组织关系。

## 5. 报表、打印、菜单、门户、移动端、多语言

核心对象：

```text
ReportDef
MetricDef
PrintTemplate
Menu
PortalEntry
MobileView
I18nResource
```

边界：

1. 报表不是列表导出，必须有指标口径、维度、权限裁剪。
2. 打印模板必须有版本归档。
3. 菜单权限独立于 PageSchema。
4. 移动端视图不能直接复用桌面布局。
5. i18n 文案不写死在 schema。

## 6. 应用包、市场与 License

核心对象：

```text
PackageManifest
PackageInstall
MarketplaceEntry
LicensePolicy
LicenseGrant
UsageQuota
UsageMeter
```

License 表达：

```text
功能包
容量额度
有效期
离线授权
宽限期
应急授权
超限策略
审计要求
```

## 7. 验收

1. 标准对象升级不覆盖客户字段、规则和页面。
2. 订单转出库、出库转收款能追踪来源并反写数量/金额。
3. 超额转换被阻断。
4. 物料规格/批次维度不以普通 JSON 大字段运行。
5. 多组织关系能参与权限与编码规则。
6. 报表按权限裁剪，打印模板有版本归档。
7. 菜单、门户、移动端入口按角色裁剪。
8. License 超限有可解释处理和审计。
