问题定位：从对比中看到的差距需要回改原型
维度 金蝶 / 商业软件做法 当前 T-207 问题 修正方向
左侧宽度 金蝶左控件栏约 265-290px；Visual Studio 可折叠 / 固定 / 隐藏；DevExpress 多 Dock 面板按需切换。 T-207 调整为 52px + 304px，总宽约 356px。三列保证中文名可读，并减少对画布的挤压。 保留三列，必须支持折叠到仅 Activity Bar；默认宽度建议约 304px，且不能再扩大。
组件项形态 金蝶是图标在上、名称在下的高密度网格；Visual Studio Toolbox 更像列表/小图标工具箱。 T-207 卡片调整为图标上、中文名下、抓手角标；三列时中文名称更稳定。 改成“图标居中 + 中文名两行内 + 抓手右上角”的紧凑工具箱块，减少横向挤压。
分组展开策略 金蝶分类标题清晰，可折叠；IDE 工具箱按上下文和分类逐步展开。 T-207 一次展开七组，首屏信息过载，用户找不到重点。 默认展开“布局容器、基础表单、表格明细”三组，其余折叠；搜索时展开命中分组。
Activity Bar Power Apps / Retool 左侧入口是工作模式，不是同一列表筛选；通常图标优先，文字通过 tooltip 或面板标题说明。 T-207 用文字按钮直接占用 52px 宽度，信息明确但不够专业；视觉更像导航按钮，不像 IDE 活动栏。 Activity Bar 改为图标 + tooltip；面板标题显示中文工作模式，如“组件工具箱”。
字段入口 DevExpress Field List 明确独立于 Toolbox；Power Apps Data / Tree view 也与 Insert 分离。 T-207 文档定义正确，但 PT-001 主图只显示组件，用户无法在首屏理解字段入口切换后是什么。 PT-001 增加 Activity 切换说明区或小型状态预览；PT-025 继续展示完整六入口切换内容。
右侧属性 金蝶按业务 / 样式 / 布局分层；DevExpress Properties 支持搜索、分类、说明、表达式。 T-207 PT-001 右侧 Inspector 信息太少，看不出专业属性面板能力。 主图右侧至少展示“业务、布局、样式、规则、权限、事件”页签和属性搜索。
测试抓手 商业软件不一定暴露测试抓手，但我们的自动化平台必须可稳定模拟用户。 T-207 已有 data-testid 和拖拽抓手，这是超越点；但视觉抓手占用横向空间。 抓手视觉缩小为角标，DOM 抓手保持稳定，不牺牲卡片名称可读性。
