## 1. 评审结论

结论：画布工作区区域合格，可以进入下一个设计器区域。

限定：当前不能声明超过金蝶、DevExpress 或 Visual Studio。当前只能声明“PT-027 画布工作区达到企业级 PC Web 表单/单据设计器画布区域的本轮可交付门槛，并具备可自动化测试的抓手与状态合同”。

## 2. P0 检查

| 检查项 | 结论 | 证据 |
|---|---|---|
| 画布不是静态截图 | 通过 | 缩放、断点、开关、投放、抓手均有 DOM 状态 |
| 画布主体可见 | 通过 | PT-027 进入后画布、工具条、右侧 Inspector 均可见 |
| 支持复杂 Web 布局能力 | 通过 | Grid、Tabs、Split、Sticky、EntryTable 同屏表达 |
| 选中态不是纯视觉 | 通过 | 节点 `data-selected` 与 Inspector 当前节点同步 |
| 抓手设计可测试 | 通过 | 移动抓手、8 个 resize 抓手、Split 抓手均有唯一 `data-testid` |
| 合法投放可回放 | 通过 | 字段投放到 Grid 单元为 legal |
| 非法投放可拒绝 | 通过 | 布局组件投放到表格列为 illegal，`data-schema-mutated=false` |
| 命令条不溢出 | 通过 | 修复后缩放、断点、抓手均在可视区域 |
| 长画布核心命令可达 | 通过 | 底部状态条提供缩放和断点镜像入口，复用同一状态机 |
| 说明文字不挤占画布 | 通过 | 画布以控件和状态为主，无长说明堆叠 |
| 企业风格克制 | 通过 | 灰白 IDE 风格、中文业务名称、密度适中 |

P0：无。

## 3. 本轮发现并修复的问题

| 问题 | 严重级别 | 修复 |
|---|---|---|
| PT-027 初版页面宽度过大，右侧 resize 抓手跑出可视区域 | P0 | 收敛侧栏宽度和画布缩尺宽度，确保抓手可见可点 |
| 命令条横向溢出，缩放按钮点击不稳定 | P0 | 命令条改成可换行紧凑布局 |
| 非法投放后事件冒泡，拒绝结果被节点选中结果覆盖 | P0 | drop target 点击、drop、keydown 阶段阻止冒泡 |
| 深层投放回放后顶部命令条离开视口 | P0 | 底部状态条增加镜像命令并接入同一状态机 |
