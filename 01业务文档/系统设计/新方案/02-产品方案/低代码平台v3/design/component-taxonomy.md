# 企业级表单设计器组件目录规范

## 1. 目标

组件面板不仅用于展示图标，还应是组件注册表的可视入口。每个条目必须能够追溯到组件协议、合法父子关系、数据绑定、状态权限、设备能力和运行时渲染器。

本目录吸收本地竞品知识库中已经确认的金蝶云·苍穹企业业务控件，以及 Mendix、OutSystems、Power Apps、Retool、Appian、DevExpress 的页面结构、响应式容器、层级浏览和高密度数据控件经验。分类与命名为本产品独立设计，不复制任何竞品界面。

## 2. 面板信息架构

组件面板固定提供以下入口：

1. 搜索：按名称、别名、协议类型和适用对象检索。
2. 常用：最近使用、收藏和当前页面推荐。
3. 布局：只改变页面结构，不生成业务字段。
4. 输入字段：绑定标量字段并参与校验和提交。
5. 业务引用：绑定组织、人员、基础资料等业务语义。
6. 数据容器：承载集合、分录和层级实体。
7. 展示与媒体：不直接写入业务实体的内容展示。
8. 导航与流程：导航、动作、状态和审批协作。
9. 分析与报表：图表、指标和嵌入式分析。
10. 扩展：受治理的自定义组件和嵌入内容。

页面类型必须参与过滤。基础资料、单据、动态表单和报表共享底层协议，但显示不同的推荐集和禁用原因。

## 3. 组件清单

### 3.1 布局容器

| 组件 | 主要用途 | 关键约束 |
|---|---|---|
| 字段布局 | 企业表单字段的响应式多列排列 | 仅接收字段和字段组合；PC 与移动布局可覆盖 |
| 字段组合 | 将相关字段组织为可复用字段组 | 只能放入字段布局或兼容字段容器 |
| Section | 带标题、帮助、折叠和摘要的语义分区 | 可接收字段、Columns、数据组件 |
| Columns | 约束式多列布局 | 子项按断点定义列宽，不使用自由像素定位 |
| Flex | 横向或纵向弹性排列 | 显式声明方向、换行、间距、对齐和溢出 |
| 高级面板 | 标题、摘要、工具栏、内容区分离的复合容器 | 各插槽必须显式建模，不能把子节点扁平化 |
| 页签 | 平级内容分区 | 子节点必须为页签项；支持独立移动布局策略 |
| 向导 | 有顺序和完成状态的步骤流程 | 子节点必须为步骤；不能与普通页签共用状态协议 |
| 分割容器 | 可调整尺寸的主从区域 | PC 优先；需最小尺寸、折叠和位置记忆 |
| 栅格看板 | 卡片式仪表盘编排 | 与字段布局分离；使用网格坐标和碰撞规则 |
| 轮播容器 | 依次展示媒体或内容页 | 不用于高频业务录入和关键审批步骤 |
| 侧边栏/抽屉 | 辅助详情和上下文操作 | 必须定义打开来源、焦点返回和关闭策略 |

### 3.2 输入字段

| 组件族 | 组件 |
|---|---|
| 文本 | 单行文本、多行文本、多语言文本、掩码文本、密码 |
| 数值 | 整数、小数、金额、百分比、计量单位、步进器 |
| 日期时间 | 日期、时间、日期时间、日期范围、时间范围 |
| 选择 | 下拉、单选组、复选组、开关、级联选择、标签选择 |
| 内容输入 | 富文本、Markdown、颜色、签名 |
| 文件 | 附件、图片上传、文件上传 |

字段组件必须实现统一的 `value / onChange / error / disabled / readonly / required` 协议，并允许字段类型进一步收紧可选渲染器。

### 3.3 业务引用

| 组件 | 业务语义 |
|---|---|
| 基础资料选择 | 引用稳定业务档案，支持编码和名称显示映射 |
| 树形基础资料选择 | 在层级档案中选择业务值，选择与展开状态分离 |
| 上级基础资料 | 维护层级父节点并阻止循环关系 |
| 分组选择 | 引用分类/分组体系 |
| 组织选择 | 受组织权限和业务组织范围约束 |
| 人员选择 | 受组织、岗位和数据权限约束 |
| 部门选择 | 引用组织结构中的部门节点 |
| 地址选择 | 结构化行政区划与详细地址协作 |
| 主数据内码 | 跨系统引用与映射，只读或受控编辑 |
| 状态/枚举 | 引用受治理的业务枚举和状态机值 |

业务引用不能降级为普通文本或通用下拉。组件协议必须携带引用对象、显示字段、过滤条件、权限上下文和失效值处理策略。

### 3.4 数据与单据

| 组件 | 数据形态 | 依赖与启用条件 |
|---|---|---|
| DataGrid | 二维集合 | 绑定集合实体或查询结果 |
| TreeGrid | 层级集合 + 二维列 | 需要稳定键及父键/嵌套数据适配 |
| 单据体 | 单据父实体的一对多分录 | 仅单据/兼容业务对象；绑定分录实体 |
| 子单据体 | 父分录行的一对多孙级实体 | 必须先存在父单据体并绑定父实体关系 |
| 树形单据体 | 带父子层级的分录实体 | 需要层级键、拖拽规则和循环校验 |
| 树形子单据体 | 标准或树形父分录的层级子实体 | 必须选择合法父单据体和关系 |
| 卡片分录 | 每行以卡片呈现的分录 | 绑定分录实体和卡片模板 |
| 子卡片分录 | 随父卡片当前行联动的孙级实体 | 必须绑定父卡片分录 |
| 只读表格 | 查询结果展示 | 不提供实体写入和编辑事务 |
| Spread | 电子表格式计算和录入 | 授权、公式、剪贴板和导出能力需独立治理 |
| 查询过滤器 | 查询参数和方案 | 绑定查询协议，不绑定实体字段提交 |
| 分页器 | 集合翻页 | 只能附着于支持分页的数据组件 |

`单据体` 不是普通 DataGrid 的样式变体。它还拥有行状态、父子实体事务、批量录入、汇总、单据规则和权限语义。

### 3.5 层级浏览

| 组件 | 核心任务 | 不可替代原因 |
|---|---|---|
| Tree | 展开并浏览节点 | 不承担业务值输入协议 |
| Tree Picker | 从层级中选择一个或多个业务值 | 选择值、焦点、展开和勾选必须分离 |
| Tree Browser | 持续浏览层级并驱动详情区 | 主从联动，不是弹出选择器 |
| Column Browser | 逐级分栏浏览深层层级 | 与树的缩进和展开交互不同 |
| TreeGrid | 层级数据的多列查看与编辑 | 同时拥有列、编辑、过滤和行状态 |

禁止用单个“树控件”的模式开关覆盖上述全部协议。

### 3.6 展示、媒体和反馈

文本、标题、分隔线、标签、徽标、提示、空状态、骨架屏、进度、统计指标、详情列表、时间线、图片、图片预览、音频、视频、地图、地理位置、二维码、富文本展示、Markdown 展示、WebOffice（受兼容性治理）。

### 3.7 导航、动作和流程

操作栏、按钮组、上下文菜单、导航菜单、树形菜单、浮动菜单、面包屑、步骤状态、状态条、审批记录、流程轨迹、操作日志、评论/协作、通知提示。

动作组件必须绑定命令协议，并经过操作权限、状态机和幂等策略校验，不能只保存一段前端点击脚本。

### 3.8 分析与报表

指标卡、基础图表、组合图表、透视表、报表列表、报表树、筛选器、计算字段、轻分析入口、嵌入式分析工作区。

轻分析属于独立 BI 工作区，不作为普通图表的一个属性开关。

### 3.9 扩展

IFrame、HTML、安全脚本组件、自定义组件、组件组合、页面片段、业务模板。

扩展组件必须声明输入、输出、事件、权限、网络访问、资源版本和隔离方式；未通过准入测试时不能进入生产发布清单。

## 4. 首批可交互范围

第一阶段面板不以“数量最多”为目标，而以覆盖复杂基础资料和单据的完整闭环为门槛。以下组件必须真实支持从面板拖入、合法落点预览、设计树写入、属性编辑、撤销重做、序列化和反显：

- 布局：字段布局、字段组合、Section、Columns、Flex、高级面板、页签、向导、分割容器。
- 字段：文本、数值、金额、日期、日期时间、下拉、单选、复选、开关、附件。
- 业务引用：基础资料、树形基础资料、组织、人员、上级基础资料。
- 数据：DataGrid、TreeGrid、单据体、子单据体、树形单据体、卡片分录、查询过滤器。
- 流程：操作栏、状态条、审批记录、操作日志。
- 展示：标题、文本、提示、图片、富文本、Markdown。

尚未接入运行时的组件仍可在完整目录中出现，但必须明确标为“规划中”，且不可进入拖放模式。禁用提示应说明缺失的运行时、页面类型、父组件或数据绑定条件。

## 5. 注册表最小协议

```ts
interface ComponentManifest {
  type: string
  title: string
  aliases: string[]
  category: string
  icon: string
  status: 'ready' | 'preview' | 'planned' // 目录启用状态，不代表生产成熟度
  supportedPageTypes: Array<'masterData' | 'bill' | 'dynamicForm' | 'report'>
  supportedDevices: Array<'desktop' | 'tablet' | 'mobile'>
  dataBinding: 'none' | 'field' | 'reference' | 'entity' | 'query'
  accepts: string[]
  requiresParent?: string[]
  requiresCapabilities?: string[]
  defaultSchema: Record<string, unknown>
  inspectorPanels: string[]
  runtimeRenderer: string
}
```

面板显示、搜索、拖放可用性、属性页和运行时映射必须读取同一个注册表，禁止在 HTML、拖放逻辑和渲染器中分别维护互相漂移的组件清单。

## 6. 验收标准

1. 完整目录中的组件都有唯一类型、分类、状态和搜索别名。
2. 面板能区分“可拖入”“条件未满足”“规划中”，并显示原因。
3. 子单据体在没有父单据体时禁用；父关系建立后自动可用。
4. Tree Picker、Tree Browser、Column Browser、TreeGrid 是四个独立条目和协议。
5. 单据体、子单据体、树形单据体和卡片分录拥有独立元数据，不能映射成同一个表格组件。
6. 页面类型切换不会丢失完整目录，但会调整推荐、可用状态和解释。
7. 新增组件只需注册 Manifest 和对应渲染器，不需要修改面板模板。

## 7. 证据边界

已确认事实来自本地知识库中金蝶复杂字段与布局控件研究、商业表单设计器综合证据、Appian/DevExpress 控件研究，以及 Power Apps/Retool 深化证据。本文给出的 V3 分类、命名、首批范围和注册表协议属于基于证据的产品设计推导，不代表任何竞品的原始组件清单。
