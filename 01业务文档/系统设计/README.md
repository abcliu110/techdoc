> 版本：v1.0

## 体系全景

本目录包含两套独立但互补的文档体系，外加领域知识和参考资料。

```text
系统设计/
├── 00-方法论与流程/           ← 跨领域方法论、工作流、研发流程
├── 新方案/                    ← 知识库与产品交付体系（道+法）
│   ├── 01-知识库/             ← 按领域沉淀的竞品证据和知识卡片
│   ├── 02-产品方案/           ← 具体产品的 PRD、架构、详细设计、源码
│   └── 99-归档/               ← 已废弃或历史版本
│
├── 工程规范/                  ← 工程规范体系（法+术+器）
│   ├── 通用规范/              ← 跨项目工程底线（MUST/SHOULD/条件规则）
│   ├── 低代码平台规范/        ← 低代码平台特有契约
│   ├── 参考资料/              ← AI提示与业务建模指南（非强制）
│   └── README.md              ← 工程规范入口（含AI Agent加载协议+8条治理原则）
│
├── 评审工作流/                ← 评审流程与检查清单
├── 模型/                      ← 数据模型与领域模型设计
├── 界面模型/                  ← UI界面模型与设计规范
├── 界面设计/                  ← 具体界面设计稿
├── 小程序相关/                ← 小程序专项设计
├── erp知识/                   ← ERP领域知识沉淀
└── *.md                       ← 顶层方法论参考文档
```

```text

桥梁1：知识库卡片 → 工程规范的输入
```

## 全局术语对照

两套体系使用不同术语，以下是对照表：

| 方法论术语 | 工程规范术语 | 含义 |
|---|---|---|
| 证据强度/证据等级 | — | 知识条目可信度分级（直接事实/高可信推断/假设/已验证） |
| 知识库成熟度 L0-L3 | 门禁状态（要求/已实现/已验证/人工评审） | 产物可信度阶段 |
| ADR（架构决策记录） | accepted ADR / proposed ADR | 已批准/候选技术决策 |
| DoD（完成定义） | 验收清单 / 验证矩阵 | 产物通过条件 |
| 失败模式候选卡 | MUST 条款 | 不可违反的工程约束来源 |
| P0/P1/P2 分级 | MUST/SHOULD/条件规则 | 阻断级别 |
| H0/H1/H2 人类审批门禁 | 人工评审 | 需要人类确认的检查点 |

## 治理原则

工程规范的 8 条治理原则（P1-P8，见 `工程规范/README.md` §规范治理原则）同样适用于方法论体系：

- **P1 规范经济性**：方法论文档总量有预算，新增需退役等量旧文档
- **P2 机制优先于纪律**：能被 CI/GitHub Actions 自动校验的不靠人工记忆
- **P3 可证伪性**：每条方法论原则能回答"什么情况证明它过时了"
- **P4 生命周期**：方法论文档有复查周期，技术栈变更时触发复查
- **P5 反馈闭环**：方法论的有效性需度量（知识库成熟度分布、产品交付返工率）
- **P6 最小合规路径**：任何角色有最短阅读路径，不让从 30+ 文件中自行筛选
- **P7 成本-风险对称**：方法论的实施成本与不实施风险对称
- **P8 双轨制**：核心产品走完整流程，实验功能走创新轨道（隔离+限时+退出条件）
