# 整理版：小程序移动端设计规范

## 1. 文档定位

本文专门整理微信小程序、移动端 H5、Taro、uni-app 等移动端业务页面的设计规范。

它不讨论 PC 后台的大屏结构，例如左树右表、宽表格、矩阵表、三栏工作台。这些属于 PC 后台专用规范。

适用场景：

```text
微信小程序。
移动端业务办理。
扫码点餐。
移动审批。
移动支付。
移动订单详情。
门店移动作业。
会员端页面。
```

## 2. 小程序和 PC 后台的差异

| 维度 | 小程序 / 移动端 |
|---|---|
| 屏幕 | 小屏，纵向滚动 |
| 输入 | 触摸、扫码、拍照、定位 |
| 操作 | 单对象、短流程、轻量任务 |
| 信息密度 | 低到中 |
| 页面结构 | 卡片、分组、Tab、步骤、底部操作栏 |
| 平台限制 | 页面栈、tabBar、分包、授权、包体、兼容 |
| 风险 | 网络不稳定、重复提交、授权失败、页面返回路径 |

## 3. 小程序专用原则

### 3.1 小屏不承载复杂后台结构

不要把 PC 后台结构硬搬到小程序。

不适合直接搬：

```text
左树右表。
宽表格。
三栏结构。
复杂矩阵表。
超长筛选表单。
密集批量操作。
```

应改造为：

```text
范围选择器 + 列表。
筛选抽屉 + 卡片列表。
列表页 + 详情页。
分组详情。
步骤向导。
底部固定操作栏。
```

### 3.2 页面栈和返回路径必须设计

小程序有页面栈限制。

必须定义：

```text
从哪里进入。
返回到哪里。
是否是 tabBar 页面。
是否需要 redirect。
是否需要 reLaunch。
是否允许重复打开同一详情页。
```

### 3.3 tabBar 页面是一级入口

tabBar 页面适合：

```text
首页。
分类。
购物车。
订单。
我的。
工作台。
```

不适合：

```text
临时业务详情。
支付结果中间页。
授权页。
审批详情页。
复杂表单步骤页。
```

### 3.4 分包和包体

小程序必须控制包体。

原则：

```text
主包只放首屏和高频入口。
低频业务进入分包。
图片和大资源外置。
分包预加载要按业务路径设计。
```

### 3.5 授权和隐私

小程序涉及微信授权。

必须处理：

```text
授权前提示。
拒绝授权后的降级。
再次授权路径。
手机号、定位、相机、相册等敏感权限说明。
```

### 3.6 网络和弱环境

移动端网络不稳定。

必须处理：

```text
loading。
retry。
offline。
pending。
重复提交保护。
支付状态不确定。
接口超时。
页面恢复后刷新。
```

## 4. 小程序常用结构

### 4.1 卡片列表

适用：

```text
订单列表。
会员列表。
优惠券列表。
门店列表。
待办列表。
```

原则：

```text
每张卡片只放关键对象身份、状态、金额、时间、主操作。
不要模拟 PC 表格。
```

### 4.2 详情分组

适用：

```text
订单详情。
支付详情。
退款详情。
会员详情。
审批详情。
```

原则：

```text
顶部展示对象身份和状态。
关键摘要前置。
主信息、明细、流程、附件分组。
底部固定主操作。
```

### 4.3 筛选抽屉

适用：

```text
移动端列表筛选。
订单筛选。
日期筛选。
状态筛选。
```

原则：

```text
常用筛选外露。
复杂筛选放抽屉。
筛选结果要显示当前条件。
重置规则明确。
```

### 4.4 分步流程

适用：

```text
下单。
支付。
申请退款。
提交审批。
绑定门店。
资料认证。
```

原则：

```text
每一步只做一件事。
每一步可校验。
返回不丢数据。
最终提交前确认。
```

## 5. 小程序配色

小程序配色也要遵守语义原则，但可比 PC 后台更强调触达和识别。

原则：

```text
品牌色用于主按钮和核心入口。
状态色用于订单、支付、退款、审批状态。
风险色用于失败、退款异常、支付失败、删除。
大面积背景仍要克制。
按钮对比度要足够，适合户外和移动场景。
```

不要：

```text
把页面做成营销海报。
用太多彩色卡片承载业务数据。
状态只靠颜色，不写文字。
```

## 6. 小程序测试重点

小程序测试必须覆盖：

```text
页面栈。
tabBar 切换。
路由参数。
分包加载。
授权拒绝和授权成功。
扫码。
支付状态。
网络失败。
重复提交。
下拉刷新。
上拉加载。
返回路径。
```

## 7. 小程序 A 级标准

```text
页面对象清楚。
入口和返回路径清楚。
状态机清楚。
底部主操作明确。
移动端结构不照搬 PC。
授权和网络异常可恢复。
支付、扫码、定位等平台能力有失败分支。
测试能等待状态而不是固定 sleep。
```
